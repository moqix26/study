# 章节扩充标准（零基础 → 精通）

## 目标读者假设

- 会基本电脑操作，**不一定**会编程
- 编程章节假设：能复制粘贴、能看懂「变量=盒子」级别比喻
- Java 章节假设：按 01→04 顺序，不跳章

## 必加章节结构（插入到「本章与上一章的关系」之前或之后）

```markdown
## 0. 读前导读（零基础也能跟上）

### 0.1 用一句话弄懂本章
### 0.2 你需要提前知道什么（真不会就先跳到哪一章）
### 0.3 本章知识地图（学完后应能勾选全部 ☐→☑）
### 0.4 建议学习时长与节奏
### 0.5 学完本章你能做什么（可验证的具体动作）
```

## 每个技术术语首次出现格式

```markdown
**术语（English）**：一句话定义。
**生活类比**：……
**为什么重要**：……
**本章用到的地方**：§x.x
```

## 手把手步骤格式

```markdown
| 步骤 | 你的动作 | 预期看到什么 | 若不对 |
|------|----------|--------------|--------|
| 1 | … | … | 见报错表 §x |
```

## 代码/JSON 必须配「逐行读」

对 >10 行的示例，加表格：

| 行号/字段 | 含义 | 改错会怎样 |

## 闭卷自测（章末）

- 10 道：6 概念 + 2 动手 + 2 综合
- 答案放在 `### 自测参考答案` 小节

## 费曼检验

「请在不看资料的情况下，用 3 分钟向没学过编程的朋友解释本章核心。」
提供 3 条要点提纲作为对照。

## 篇幅目标（扩充后）

| 类型 | 原篇幅 | 目标 |
|------|--------|------|
| 路线图 00 | 300～400 行 | 500+ |
| 核心章 | 500～900 行 | 1200～2000 |
| 索引章 12/15 | 400～600 行 | 800+ |
