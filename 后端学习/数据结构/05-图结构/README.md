# 图结构模块导航

图用于表达任意对象之间的关系。学习图算法时，先确认四件事：**有向还是无向、是否带权、是否可能有环、规模是稀疏还是稠密**。

| 顺序 | 文档 | 解决的问题 |
|---:|---|---|
| 1 | [图的存储与遍历](01-图的存储与遍历.md) | 邻接矩阵/表、BFS、DFS、连通性 |
| 2 | [拓扑排序](02-拓扑排序.md) | DAG 的依赖顺序与有向环检测 |
| 3 | [最小生成树](03-最小生成树.md) | Prim、Kruskal 连接所有顶点的最小成本 |
| 4 | [最短路径](04-最短路径.md) | Dijkstra 单源非负权、Floyd 多源最短路 |

## 快速选型

| 场景 | 算法/存储 |
|---|---|
| 稀疏图 | 邻接表 |
| 稠密小图、频繁查边 | 邻接矩阵 |
| 无权最短步数 | BFS |
| 枚举连通分量、路径搜索 | DFS/BFS |
| 任务依赖排序 | 拓扑排序 |
| 无向网络最低建设成本 | Prim/Kruskal |
| 非负权单源最短路 | Dijkstra |
| 小规模任意两点最短路 | Floyd |

任何最短路或生成树算法都必须先核对前提；“模板能跑”不等于“问题模型正确”。
