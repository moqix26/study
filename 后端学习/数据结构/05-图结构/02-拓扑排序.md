# 拓扑排序

## 学习目标

- 理解拓扑序是对有向无环图（DAG）中先后约束的一种线性展开。
- 掌握 Kahn 入度算法，并能利用结果长度检测有向环。
- 理解 DFS 后序法的思想、正确性直觉以及两种方法的差异。
- 能判断拓扑排序是否存在、是否唯一，以及它不能解决哪些问题。

## 核心原理

### 1. 拓扑序表达的是偏序约束

对有向图 `G=(V,E)`，如果一个顶点序列满足：对每条边 `u -> v`，`u` 都出现在 `v` 之前，那么该序列就是一个拓扑序。

边的含义应解释为“前置约束”，而不一定是物理路径。例如：

- `编译源码 -> 链接程序` 表示链接前必须完成编译。
- `数据结构 -> 算法设计` 表示课程先修关系。
- `下载依赖 -> 启动服务` 表示工作流依赖。

拓扑序只是把部分先后关系排成一条线。若图中没有边约束 A 和 B 的相对位置，那么 A 在 B 前、B 在 A 前都可能合法。因此拓扑排序不是普通的“按大小排序”，也不保证输出中相邻两个顶点之间存在边。

只有有向无环图（DAG）才存在完整拓扑序：

- 若有环 `A -> B -> C -> A`，约束同时要求 `A < B < C < A`，不可能满足。
- 若图是 DAG，则当前图中一定至少有一个入度为 0 的顶点。否则从任一点不断沿入边向前走，因为顶点有限，最终必然重复顶点并形成环，与无环矛盾。

无向图不使用通常意义上的拓扑排序。一条无向边若硬解释为双向依赖，就同时得到 `u -> v` 与 `v -> u`，立即构成环。

### 2. 入度在算法中的准确含义

顶点 `v` 的原始入度是所有指向 `v` 的边数。但 Kahn 算法运行过程中，`indegree[v]` 更准确的含义是：**还没有被输出的前置顶点中，仍有多少条边指向 v**。

每输出一个顶点 `u`，就把 `u` 的所有出边视为删除，于是相应邻居的剩余入度减 1。当 `indegree[v]` 变成 0 时，说明 `v` 的全部前置依赖都已出现在结果中，此时 `v` 才能进入待处理队列。

因此 Kahn 算法维护两个关键不变量：

1. 结果序列中的顶点已经处理完，它们发出的边已从剩余图中删除。
2. 队列中的每个顶点，在当前剩余图中的入度都为 0，可以安全成为结果的下一个顶点。

注意，队列表示的是“现在可以做什么”，不是“原图中入度为 0 的顶点”。一个顶点原始入度可以大于 0，只要前置顶点都处理完，它也会在后续变为可选。

### 3. Kahn 算法逐步执行

算法步骤如下：

1. 扫描所有边，统计每个顶点的原始入度。
2. 按顶点编号扫描，把所有入度为 0 的顶点加入队列。
3. 取出队首 `u` 并追加到结果。
4. 枚举 `u` 的每条出边 `u -> v`，令 `indegree[v]--`；若恰好减为 0，把 `v` 入队。
5. 队列为空后，比较结果长度与顶点数。

三种语言示例中的 DAG 有 6 个顶点，边为：

`5->0、5->2、4->0、4->1、2->3、3->1`

初始入度为：

| 顶点 | 0 | 1 | 2 | 3 | 4 | 5 |
|---:|---:|---:|---:|---:|---:|---:|
| 初始入度 | 2 | 2 | 1 | 1 | 0 | 0 |

按编号把 4、5 加入普通队列后，完整变化如下：

| 轮次 | 取出的顶点 | 被减少的入度 | 操作后的队列 | 当前结果 |
|---:|---:|---|---|---|
| 初始 | 无 | 无 | `[4,5]` | 空 |
| 1 | 4 | `0: 2->1`，`1: 2->1` | `[5]` | `4` |
| 2 | 5 | `0: 1->0`，`2: 1->0` | `[0,2]` | `4,5` |
| 3 | 0 | 无出边 | `[2]` | `4,5,0` |
| 4 | 2 | `3: 1->0` | `[3]` | `4,5,0,2` |
| 5 | 3 | `1: 1->0` | `[1]` | `4,5,0,2,3` |
| 6 | 1 | 无出边 | `[]` | `4,5,0,2,3,1` |

可以逐边验证结果：5 在 0、2 前；4 在 0、1 前；2 在 3 前；3 在 1 前。因此该序列满足全部约束。

### 4. 为什么选择零入度顶点是安全的

设当前剩余图中的 `u` 入度为 0。把 `u` 放到当前结果末尾，不会违反任何尚未满足的边，因为剩余图中根本没有边要求某个未输出顶点先于 `u`。至于 `u` 指向的后继，它们仍留在图中，之后才会输出，方向同样正确。

删除 `u` 和它的出边后，剩余图仍是 DAG：删除顶点和边不可能凭空制造环。因此可以对更小的图重复同一论证。这个过程直到所有顶点删除，构造出的序列必然合法。

这个证明也说明“随便选一个零入度顶点”是允许的。不同选择可能产生不同拓扑序，但每次选择都满足安全条件。

### 5. 为什么结果长度可以检测环

若队列为空但结果长度小于 `V`，说明仍有顶点未输出，而且这些剩余顶点的入度都大于 0。任选一个剩余顶点，沿着一条来自剩余顶点的入边不断向前走。因为剩余顶点数有限，某个顶点最终会再次出现，从而得到一个有向环。

以示例中的循环图 `0->1、1->2、2->0` 为例，三个顶点的初始入度都为 1：

- 初始队列就是空的。
- 算法一个顶点也无法输出。
- `order.size()` 为 0，小于 3，因此报告环。

更一般地，图可以同时包含一个可处理的无环部分和一个环。Kahn 会先输出无环部分，最后停在环及其受环阻塞的后继上，所以“输出了部分顶点”仍不能算成功，必须要求结果长度恰好等于 `V`。

### 6. 多个合法序列与唯一性

示例初始队列为 `[4,5]`，第一步既可选 4，也可选 5。若先选 5，可能得到 `5,4,2,3,0,1` 等另一合法序列。因此文档示例的输出不是唯一标准答案。

Kahn 过程中：

- 若某一步队列中至少有两个顶点，就存在至少两种合法的下一步选择，完整拓扑序不唯一。
- 若从开始到结束每一步队列大小都恰为 1，并且最终输出所有顶点，则拓扑序唯一。
- 普通队列只保证返回某个合法序列，结果受初始扫描和邻接表顺序影响。
- 若题目要求字典序最小拓扑序，应把队列换成最小堆，每次取当前编号最小的零入度顶点。

“同时可选”不表示这两个任务必须并行执行，只表示依赖关系没有规定它们的相对顺序。实际调度还可能受到机器数量、耗时、资源冲突等额外约束，那已经超出单纯拓扑排序。

### 7. DFS 后序法及三色状态

另一种做法是 DFS。对边 `u -> v`，必须先完整处理 `v` 的所有后继，才能把 `u` 标记完成。于是 `v` 的完成时间早于 `u`；把所有顶点按完成顺序记录，再整体反转，就能让 `u` 出现在 `v` 之前。

DFS 检测有向环时必须区分三种状态：

- **白色/未访问**：还没有进入 DFS。
- **灰色/访问中**：已进入，但后继尚未全部处理；它仍在当前递归调用链上。
- **黑色/已完成**：该顶点及其所有后继均处理完，已退出递归。

检查边 `u -> v` 时：

- `v` 为白色：继续递归。
- `v` 为灰色：这条边回到当前递归路径上的祖先，形成有向环。
- `v` 为黑色：它只是被其他分支提前处理，不构成当前路径上的环。

只用一个 `visited` 布尔值会把灰色和黑色混在一起，无法区分“指向祖先的回边”和“指向已完成分支的交叉边”，因此不足以可靠检测有向环。

仍用前面的 6 个顶点 DAG，假设外层按 `0..5` 启动 DFS，邻居按加边顺序检查，完成轨迹为：

| 启动或深入 | 关键动作 | 当前完成序列 |
|---|---|---|
| 启动 0 | 0 无后继，染黑并记录 | `0` |
| 启动 1 | 1 无后继，染黑并记录 | `0,1` |
| 启动 2，深入 3 | 3 看到后继 1 已黑；先完成 3，再完成 2 | `0,1,3,2` |
| 启动 4 | 4 的后继 0、1 都已黑，记录 4 | `0,1,3,2,4` |
| 启动 5 | 5 的后继 0、2 都已黑，记录 5 | `0,1,3,2,4,5` |

反转完成序列得到 `5,4,2,3,1,0`，可逐边验证所有前置顶点都在后继之前。对循环图 `0->1、1->2、2->0`，搜索 2 的后继时会看到 0 仍为灰色，立即确认回边和有向环。下面三种语言示例选择实现 Kahn 法；DFS 版作为末尾练习。

### 8. 数据结构、复杂度与适用边界

邻接表中，统计入度要检查每条边一次，初始化队列要扫描每个顶点一次，处理阶段每个顶点出队一次、每条边再检查一次，总时间为 `O(V+E)`，额外的入度、队列和结果数组均为 `O(V)`。

若使用邻接矩阵，枚举每个已输出顶点的出边都要扫描一整行，总时间为 `O(V²)`。图很稠密或输入本身就是矩阵时可以接受；依赖关系通常稀疏，更常使用邻接表。

拓扑排序不要求图在无向意义下连通。多个互不相关的 DAG 分量会共同进入可选集合，它们之间可以交错排列。它也不计算任务最早完成时间、关键路径或最短路径；这些问题可以在得到拓扑序后，结合边权和额外状态继续处理。

## 复杂度表

| 方法 | 邻接表时间 | 邻接矩阵时间 | 额外空间 | 环检测方式 |
|---|---:|---:|---:|---|
| Kahn 入度法 | `O(V+E)` | `O(V²)` | `O(V)` | 输出顶点数是否小于 `V` |
| DFS 后序法 | `O(V+E)` | `O(V²)` | `O(V)` | 三色状态发现回边 |
| Kahn + 最小堆 | `O((V+E) log V)` | 通常不采用 | `O(V)` | 同 Kahn |

## 适用场景

- 课程先修关系、构建任务、包依赖、工作流调度中的合法执行顺序。
- 判断有向依赖是否存在循环。
- DAG 动态规划：先拓扑排序，再按序转移状态。
- 关键路径、最长路径等只在 DAG 上易于求解的问题。
- 需要实时处理先决条件时，Kahn 算法比 DFS 后序更直观；需要结合 DFS 做其他图分析时，可复用三色 DFS。

## C++17 完整示例

```cpp
#include <iostream>
#include <queue>
#include <utility>
#include <vector>
using namespace std;

class Graph {
private:
    int n;
    vector<vector<int>> adjacency;

public:
    explicit Graph(int vertexCount) : n(vertexCount), adjacency(n) {}

    void addEdge(int from, int to) {
        adjacency[from].push_back(to);
    }

    pair<bool, vector<int>> topologicalSort() const {
        vector<int> indegree(n, 0);
        for (int u = 0; u < n; ++u) {
            for (int v : adjacency[u]) ++indegree[v];
        }

        queue<int> ready;
        for (int vertex = 0; vertex < n; ++vertex) {
            if (indegree[vertex] == 0) ready.push(vertex);
        }

        vector<int> order;
        while (!ready.empty()) {
            int u = ready.front();
            ready.pop();
            order.push_back(u);
            for (int v : adjacency[u]) {
                if (--indegree[v] == 0) ready.push(v);
            }
        }
        return {static_cast<int>(order.size()) == n, order};
    }
};

void printResult(const string& name, const Graph& graph) {
    auto [acyclic, order] = graph.topologicalSort();
    cout << name << ": ";
    if (!acyclic) {
        cout << "cycle detected\n";
        return;
    }
    for (int i = 0; i < static_cast<int>(order.size()); ++i) {
        if (i > 0) cout << ' ';
        cout << order[i];
    }
    cout << '\n';
}

int main() {
    Graph dag(6);
    dag.addEdge(5, 0);
    dag.addEdge(5, 2);
    dag.addEdge(4, 0);
    dag.addEdge(4, 1);
    dag.addEdge(2, 3);
    dag.addEdge(3, 1);
    printResult("DAG topological order", dag);

    Graph cyclic(3);
    cyclic.addEdge(0, 1);
    cyclic.addEdge(1, 2);
    cyclic.addEdge(2, 0);
    printResult("Cyclic graph", cyclic);
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.Deque;
import java.util.List;

public class Main {
    static class TopologicalResult {
        final boolean acyclic;
        final List<Integer> order;

        TopologicalResult(boolean acyclic, List<Integer> order) {
            this.acyclic = acyclic;
            this.order = order;
        }
    }

    static class Graph {
        private final int n;
        private final List<List<Integer>> adjacency;

        Graph(int vertexCount) {
            n = vertexCount;
            adjacency = new ArrayList<>();
            for (int i = 0; i < n; i++) adjacency.add(new ArrayList<>());
        }

        void addEdge(int from, int to) {
            adjacency.get(from).add(to);
        }

        TopologicalResult topologicalSort() {
            int[] indegree = new int[n];
            for (int u = 0; u < n; u++) {
                for (int v : adjacency.get(u)) indegree[v]++;
            }

            Deque<Integer> ready = new ArrayDeque<>();
            for (int vertex = 0; vertex < n; vertex++) {
                if (indegree[vertex] == 0) ready.offer(vertex);
            }

            List<Integer> order = new ArrayList<>();
            while (!ready.isEmpty()) {
                int u = ready.poll();
                order.add(u);
                for (int v : adjacency.get(u)) {
                    if (--indegree[v] == 0) ready.offer(v);
                }
            }
            return new TopologicalResult(order.size() == n, order);
        }
    }

    static void printResult(String name, Graph graph) {
        TopologicalResult result = graph.topologicalSort();
        System.out.print(name + ": ");
        if (!result.acyclic) {
            System.out.println("cycle detected");
            return;
        }
        for (int i = 0; i < result.order.size(); i++) {
            if (i > 0) System.out.print(" ");
            System.out.print(result.order.get(i));
        }
        System.out.println();
    }

    public static void main(String[] args) {
        Graph dag = new Graph(6);
        dag.addEdge(5, 0);
        dag.addEdge(5, 2);
        dag.addEdge(4, 0);
        dag.addEdge(4, 1);
        dag.addEdge(2, 3);
        dag.addEdge(3, 1);
        printResult("DAG topological order", dag);

        Graph cyclic = new Graph(3);
        cyclic.addEdge(0, 1);
        cyclic.addEdge(1, 2);
        cyclic.addEdge(2, 0);
        printResult("Cyclic graph", cyclic);
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

type Graph struct {
	n         int
	adjacency [][]int
}

type TopologicalResult struct {
	acyclic bool
	order   []int
}

func NewGraph(n int) *Graph {
	return &Graph{n: n, adjacency: make([][]int, n)}
}

func (g *Graph) AddEdge(from, to int) {
	g.adjacency[from] = append(g.adjacency[from], to)
}

func (g *Graph) TopologicalSort() TopologicalResult {
	indegree := make([]int, g.n)
	for _, neighbors := range g.adjacency {
		for _, v := range neighbors {
			indegree[v]++
		}
	}

	ready := make([]int, 0, g.n)
	for vertex := 0; vertex < g.n; vertex++ {
		if indegree[vertex] == 0 {
			ready = append(ready, vertex)
		}
	}

	order := make([]int, 0, g.n)
	for head := 0; head < len(ready); head++ {
		u := ready[head]
		order = append(order, u)
		for _, v := range g.adjacency[u] {
			indegree[v]--
			if indegree[v] == 0 {
				ready = append(ready, v)
			}
		}
	}
	return TopologicalResult{acyclic: len(order) == g.n, order: order}
}

func printResult(name string, graph *Graph) {
	result := graph.TopologicalSort()
	fmt.Printf("%s: ", name)
	if !result.acyclic {
		fmt.Println("cycle detected")
		return
	}
	for i, vertex := range result.order {
		if i > 0 {
			fmt.Print(" ")
		}
		fmt.Print(vertex)
	}
	fmt.Println()
}

func main() {
	dag := NewGraph(6)
	dag.AddEdge(5, 0)
	dag.AddEdge(5, 2)
	dag.AddEdge(4, 0)
	dag.AddEdge(4, 1)
	dag.AddEdge(2, 3)
	dag.AddEdge(3, 1)
	printResult("DAG topological order", dag)

	cyclic := NewGraph(3)
	cyclic.AddEdge(0, 1)
	cyclic.AddEdge(1, 2)
	cyclic.AddEdge(2, 0)
	printResult("Cyclic graph", cyclic)
}
```

## 易错点与要点总结

- 拓扑排序只针对有向图，并且只有 DAG 才存在完整拓扑序。
- `order.size() < V` 说明有环；不要返回残缺序列并把它当作成功结果。
- 同一 DAG 可能有多个正确答案，判题时应检查边约束，而不是只对比某个固定序列。
- 重复边会重复增加入度；若业务语义不允许重复依赖，建图时应去重。
- Kahn 算法会修改入度数组，因此多次调用时应重新统计或复制，不能复用已被消耗的入度。
- DFS 法检测环必须区分“访问中”和“已完成”，遇到已完成顶点不是环。
- 拓扑排序不等同于最短路径，也不保证相邻输出顶点之间存在边。
- 若要字典序最小结果，应明确使用最小堆；普通队列不能提供这一保证。
