# 最小生成树：Prim 与 Kruskal

## 学习目标

- 理解生成树、最小生成树（MST）的定义与存在前提。
- 掌握切分性质与环性质，建立 Prim、Kruskal 贪心正确性的直觉。
- 能用邻接表和优先队列实现 Prim，用边集与并查集实现 Kruskal。
- 能根据图的稀疏/稠密程度、边集形式和连通性选择算法。

## 核心原理

### 1. 生成树到底在优化什么

给定一个连通无向带权图，**生成树**必须同时满足四个条件：

1. 包含原图全部 `V` 个顶点。
2. 只选择原图中已有的边。
3. 所有顶点彼此连通。
4. 不含环。

任何含 `V` 个顶点的树都恰有 `V-1` 条边。这个结论可以双向用于检查结果：连通无环图一定有 `V-1` 条边；若算法只选到少于 `V-1` 条边且再也无法连接新顶点，原图就不连通。

**最小生成树**（MST）是在所有生成树中，使所选边权总和最小的一棵。它优化的是“建设整个连通网络的总成本”，不是某个起点到其他点的路径长度。比如 MST 中从 0 到 5 的路线可能绕远，但整棵树的边权总和仍最小。

标准 MST 的前提和边界如下：

- 只针对无向图；有向图要研究最小树形图，不能直接套 Prim 或 Kruskal。
- 原图不连通时不存在覆盖全部顶点的生成树，只能求每个连通分量的最小生成树，合起来称最小生成森林。
- 负权边完全允许。只要不成环，负权边通常更应优先选择；这里不存在通过重复绕环让路径无限减小的问题。
- 所有边权互不相同时，MST 一定唯一；存在相同权值时，MST 可能不唯一，但也可能仍然唯一，不能只凭“有相等权”直接断定多解。

三种语言示例使用 6 个顶点和以下无向边：

| 边 | 权值 | 边 | 权值 | 边 | 权值 |
|---|---:|---|---:|---|---:|
| `0-1` | 4 | `0-2` | 3 | `1-2` | 1 |
| `1-3` | 2 | `2-3` | 4 | `3-4` | 2 |
| `3-5` | 3 | `2-5` | 5 | `4-5` | 6 |

后面会用这张图逐边推演 Prim 和 Kruskal。

### 2. 切分性质：安全选边的依据

把全部顶点分成两个非空集合 `S` 与 `V-S`，称为一个**切分**。一个端点在 `S`、另一个端点在 `V-S` 的边称为横切边。横切边是连接两侧所必需的，因为任何覆盖全部顶点的生成树都至少要跨过这个切分一次。

切分性质的常用表述是：**在不违背已选安全边的切分上，权值最小的横切边是一条安全边，即它可以加入某棵 MST。**

交换论证可以说明原因。设最轻横切边为 `e`，任选一棵符合当前已选边的 MST `T`：

1. 若 `T` 已包含 `e`，结论直接成立。
2. 若不包含，把 `e` 加入树 `T`，树中会形成唯一一个环。
3. 这个环从切分一侧经过 `e` 到另一侧，之后必须再通过另一条横切边 `f` 回来。
4. 因为 `e` 是最轻横切边，所以 `w(e) <= w(f)`。
5. 删除 `f` 后重新得到一棵生成树，权值不比 `T` 大；由于 `T` 已最优，新树也是 MST，并且包含 `e`。

这不是说“图中全局最小边之后每次都随便找一条小边”，而是每一步都要确认所选边跨越当前合法切分。Prim 和 Kruskal维护的切分不同，但安全性都来自这个性质。

### 3. Prim：让一棵树不断向外生长

Prim 始终维护一个已经连通的树内顶点集合 `S`。初始任选起点放入 `S`；每一步选择一条从 `S` 到 `V-S` 的最轻边，把一个新顶点接入树。

核心不变量有三个：

1. `S` 中的顶点由已选边连成一棵树。
2. 已选边数始终为 `|S|-1`，因此没有环。
3. 堆中保存从已访问顶点发出的候选边，其中既可能有真正横切边，也可能有两端后来都进入 `S` 的过期边。

邻接表加最小堆的执行流程是：

1. 标记起点，并把起点发出的边加入堆。
2. 弹出当前最轻边 `(from,to,w)`。
3. 若 `to` 已访问，该边不再跨越当前切分，直接跳过。
4. 否则选择这条边，标记 `to`，再把 `to` 发出的边加入堆。
5. 选到 `V-1` 条边后结束。

从顶点 0 开始，对示例图执行 Prim：

| 轮次 | 树内顶点 `S`（选择前） | 最轻有效横切边 | 选择后新增顶点 | 累计权值 |
|---:|---|---|---:|---:|
| 初始 | `{0}` | 候选为 `0-2(3)`、`0-1(4)` | 无 | 0 |
| 1 | `{0}` | `0-2(3)` | 2 | 3 |
| 2 | `{0,2}` | `2-1(1)` | 1 | 4 |
| 3 | `{0,1,2}` | `1-3(2)` | 3 | 6 |
| 4 | `{0,1,2,3}` | `3-4(2)` | 4 | 8 |
| 5 | `{0,1,2,3,4}` | `3-5(3)` | 5 | 11 |

逐轮看安全性：第一轮切分为 `{0}` 与其余顶点，`0-2(3)` 是最轻横切边；第二轮切分变成 `{0,2}` 与其余顶点，此时 `2-1(1)` 是最轻横切边。每次都在新的“树内/树外”切分上应用切分性质。

堆中并不只有上表列出的有效边。例如 2 加入后会出现指回 0 的边；1 加入后，堆里还可能保留 `0-1(4)`。等这些边弹出时，终点已经访问，代码把它们当作过期项跳过。懒删除避免了在堆中定位并删除旧候选，使实现更简单。

若使用邻接矩阵，可以为每个树外顶点维护“连接到当前树的最小边权 `key[v]`”，每轮线性扫描找最小 `key`。总共 `V` 轮、每轮 `O(V)`，时间 `O(V²)`，特别适合稠密图。

### 4. Kruskal：让多个连通块逐渐合并

Kruskal 不从某个起点长出一棵树，而是初始把每个顶点视为独立分量。它按全局边权从小到大检查边，只要一条边连接两个不同分量，就选择并合并分量；若两端已经连通，加入该边必然形成环，因此跳过。

算法不变量是：

1. 已选边构成一片森林，每个连通分量都是一棵树。
2. 并查集准确表示这片森林中的连通关系。
3. 当前考察边之前的所有更轻边都已经做过“选择或因成环跳过”的决定。

示例边排序后及处理过程如下：

| 顺序 | 边 | 处理前端点关系 | 决策 | 处理后的主要分量 |
|---:|---|---|---|---|
| 1 | `1-2(1)` | 1、2 分属不同集合 | 选择 | `{1,2}` |
| 2 | `1-3(2)` | 3 不在 `{1,2}` | 选择 | `{1,2,3}` |
| 3 | `3-4(2)` | 4 独立 | 选择 | `{1,2,3,4}` |
| 4 | `0-2(3)` | 0 独立 | 选择 | `{0,1,2,3,4}` |
| 5 | `3-5(3)` | 5 独立 | 选择 | `{0,1,2,3,4,5}` |

此时已经选择 5 条边，即 `V-1` 条，可以结束，总权值为 `1+2+2+3+3=11`。若继续观察后续边：

- `0-1(4)` 的两端已由 `0-2-1` 连通，加入会形成环。
- `2-3(4)` 的两端已在同一分量，也会形成环。
- 更重的 `2-5(5)`、`4-5(6)` 同理无需选择。

为什么“连接不同分量的当前最轻边”安全？取其中一个分量作为切分的一侧。当前边是按权值排序后第一条能连接两个不同分量的边；任何尚未处理的横切边都不会更轻，因此可以应用切分性质。

### 5. 并查集在 Kruskal 中承担的职责

不能只检查 `u` 和 `v` 之间是否已有直接边，因为成环取决于是否已经存在一条任意长度的路径。例如已选 `1-2` 和 `2-3` 后，再选 `1-3` 会成环，即使此前没有直接选择过 `1-3`。

并查集提供两个操作：

- `find(x)`：找到 `x` 所属集合的代表元。
- `union(a,b)`：若代表元不同，合并两个集合。

路径压缩让查找经过的节点直接指向代表元，按秩或按大小合并让较矮的树挂到较高的树下。两种优化同时使用时，单次操作的均摊复杂度为 `O(α(V))`，反阿克曼函数增长极慢，在实际规模下近似常数。

### 6. 环性质及与切分性质的配合

环性质表述为：在任意环中，如果某条边严格重于环内其他所有边，那么它不可能出现在任何 MST 中。因为删除这条最重边仍保持连通，并且用环中另一条边连接不会更贵。

Kruskal 遇到两端已连通的边时，已选森林中已经存在一条由不重于当前边的边组成的路径。当前边加入后形成环，它没有必要保留，所以安全跳过。若环中有多条并列最重边，其中某一条仍可能出现在某棵 MST，这也是相同权值可能产生多棵 MST 的来源。

### 7. Prim 与 Kruskal 如何选

| 图或输入特征 | 更自然的选择 | 原因 |
|---|---|---|
| 邻接表、稀疏图 | 堆优化 Prim 或 Kruskal | 二者都能接近 `O(E log V)` |
| 已直接给出无向边列表 | Kruskal | 排序后直接扫描，无需重建完整邻接结构 |
| 完整距离矩阵、稠密图 | 邻接矩阵 Prim | `O(V²)`，避免存储并排序接近 `V²` 条边 |
| 网络从某个已建站点向外扩展 | Prim | 中间过程始终是一棵连通树，语义直观 |
| 需要形成若干簇或最小生成森林 | Kruskal | 停止合并或处理完边集即可得到多个分量 |

邻接表加“边堆”的 Prim 中，一条无向边可能以两个方向进入候选堆，堆操作上界常写 `O(E log E)`；简单图满足 `log E=O(log V)`，所以也常写 `O(E log V)`。Kruskal 的主要成本是排序 `E` 条边，即 `O(E log E)`，并查集总成本通常低于排序。

### 8. 不连通、重边与其他边界

- Prim 从一个起点只能覆盖其连通分量。若最终选择少于 `V-1` 条边，应明确报告原图不连通；若目标是森林，则从每个未访问顶点重新启动。
- Kruskal 处理完所有边后若仍少于 `V-1` 条边，也说明不连通；当前已选边正是各分量的最小生成森林。
- 重边可以保留，算法会自然优先考虑更轻的一条；若构图前去重，应保留同一对端点之间的最小权值。
- 自环永远不能帮助连接两个不同顶点。Kruskal 中自环两端代表元相同会被跳过，Prim 中也不应把它当作扩展树的有效边。
- 总权值可能超过 32 位整数范围，累加时应依据输入上限使用 64 位类型。

## 复杂度表

| 算法/实现 | 时间复杂度 | 额外空间 | 更适合 |
|---|---:|---:|---|
| Prim：邻接矩阵 | `O(V²)` | `O(V)`（矩阵本身 `O(V²)`） | 稠密图、小规模完整距离矩阵 |
| Prim：邻接表 + 懒删除边堆 | `O(E log E)`；简单图可写成 `O(E log V)` | `O(V+E)` | 稀疏图 |
| Kruskal：排序 + 并查集 | `O(E log E)` | `O(V+E)` | 边集输入、稀疏图 |
| 并查集单次操作 | 均摊 `O(α(V))` | `O(V)` | 动态连通性判断 |

## 适用场景

- 铺设电缆、道路、管网时，以最低总成本连通所有站点。
- 聚类中的单链接层次聚类，以及删去 MST 中大边形成若干簇。
- 近似算法和网络骨架提取。
- 注意：MST 最小化的是**全部选中边的总和**，不保证任意两点之间路径最短，也不等同于最短路径树。

## C++17 完整示例

```cpp
#include <algorithm>
#include <iostream>
#include <queue>
#include <stdexcept>
#include <tuple>
#include <vector>
using namespace std;

struct Edge {
    int u;
    int v;
    int weight;
};

class DSU {
private:
    vector<int> parent;
    vector<int> rankValue;

public:
    explicit DSU(int n) : parent(n), rankValue(n, 0) {
        for (int i = 0; i < n; ++i) parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }

    bool unite(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);
        if (rootA == rootB) return false;
        if (rankValue[rootA] < rankValue[rootB]) swap(rootA, rootB);
        parent[rootB] = rootA;
        if (rankValue[rootA] == rankValue[rootB]) ++rankValue[rootA];
        return true;
    }
};

pair<vector<Edge>, int> prim(
    int n, const vector<vector<pair<int, int>>>& adjacency) {
    // 堆元素依次是：权值、起点、终点。
    priority_queue<tuple<int, int, int>,
                   vector<tuple<int, int, int>>,
                   greater<tuple<int, int, int>>> candidates;
    vector<bool> visited(n, false);
    vector<Edge> tree;
    int total = 0;

    visited[0] = true;
    for (auto [v, weight] : adjacency[0]) {
        candidates.emplace(weight, 0, v);
    }

    while (!candidates.empty() && static_cast<int>(tree.size()) < n - 1) {
        auto [weight, u, v] = candidates.top();
        candidates.pop();
        if (visited[v]) continue;

        visited[v] = true;
        tree.push_back({u, v, weight});
        total += weight;
        for (auto [next, nextWeight] : adjacency[v]) {
            if (!visited[next]) candidates.emplace(nextWeight, v, next);
        }
    }

    if (static_cast<int>(tree.size()) != n - 1) {
        throw runtime_error("graph is disconnected");
    }
    return {tree, total};
}

pair<vector<Edge>, int> kruskal(int n, vector<Edge> edges) {
    sort(edges.begin(), edges.end(), [](const Edge& a, const Edge& b) {
        return tie(a.weight, a.u, a.v) < tie(b.weight, b.u, b.v);
    });

    DSU dsu(n);
    vector<Edge> tree;
    int total = 0;
    for (const Edge& edge : edges) {
        if (dsu.unite(edge.u, edge.v)) {
            tree.push_back(edge);
            total += edge.weight;
            if (static_cast<int>(tree.size()) == n - 1) break;
        }
    }
    if (static_cast<int>(tree.size()) != n - 1) {
        throw runtime_error("graph is disconnected");
    }
    return {tree, total};
}

void printTree(const string& name, const pair<vector<Edge>, int>& result) {
    cout << name << ":\n";
    for (const Edge& edge : result.first) {
        cout << edge.u << " - " << edge.v << " : " << edge.weight << '\n';
    }
    cout << "Total weight: " << result.second << '\n';
}

int main() {
    const int n = 6;
    vector<Edge> edges = {
        {0, 1, 4}, {0, 2, 3}, {1, 2, 1},
        {1, 3, 2}, {2, 3, 4}, {3, 4, 2},
        {3, 5, 3}, {2, 5, 5}, {4, 5, 6}
    };
    vector<vector<pair<int, int>>> adjacency(n);
    for (const Edge& edge : edges) {
        adjacency[edge.u].push_back({edge.v, edge.weight});
        adjacency[edge.v].push_back({edge.u, edge.weight});
    }

    printTree("Prim", prim(n, adjacency));
    printTree("Kruskal", kruskal(n, edges));
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.Comparator;
import java.util.List;
import java.util.PriorityQueue;

public class Main {
    static class Edge {
        final int u;
        final int v;
        final int weight;

        Edge(int u, int v, int weight) {
            this.u = u;
            this.v = v;
            this.weight = weight;
        }
    }

    static class TreeResult {
        final List<Edge> edges;
        final int total;

        TreeResult(List<Edge> edges, int total) {
            this.edges = edges;
            this.total = total;
        }
    }

    static class DSU {
        private final int[] parent;
        private final int[] rank;

        DSU(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }

        int find(int x) {
            if (parent[x] != x) parent[x] = find(parent[x]);
            return parent[x];
        }

        boolean unite(int a, int b) {
            int rootA = find(a);
            int rootB = find(b);
            if (rootA == rootB) return false;
            if (rank[rootA] < rank[rootB]) {
                int temp = rootA;
                rootA = rootB;
                rootB = temp;
            }
            parent[rootB] = rootA;
            if (rank[rootA] == rank[rootB]) rank[rootA]++;
            return true;
        }
    }

    static TreeResult prim(int n, List<List<Edge>> adjacency) {
        PriorityQueue<Edge> candidates = new PriorityQueue<>(
            Comparator.comparingInt((Edge e) -> e.weight)
                      .thenComparingInt(e -> e.u)
                      .thenComparingInt(e -> e.v));
        boolean[] visited = new boolean[n];
        List<Edge> tree = new ArrayList<>();
        int total = 0;

        visited[0] = true;
        candidates.addAll(adjacency.get(0));
        while (!candidates.isEmpty() && tree.size() < n - 1) {
            Edge edge = candidates.poll();
            if (visited[edge.v]) continue;
            visited[edge.v] = true;
            tree.add(edge);
            total += edge.weight;
            for (Edge next : adjacency.get(edge.v)) {
                if (!visited[next.v]) candidates.offer(next);
            }
        }
        if (tree.size() != n - 1) {
            throw new IllegalArgumentException("graph is disconnected");
        }
        return new TreeResult(tree, total);
    }

    static TreeResult kruskal(int n, List<Edge> inputEdges) {
        List<Edge> edges = new ArrayList<>(inputEdges);
        edges.sort(Comparator.comparingInt((Edge e) -> e.weight)
                             .thenComparingInt(e -> e.u)
                             .thenComparingInt(e -> e.v));
        DSU dsu = new DSU(n);
        List<Edge> tree = new ArrayList<>();
        int total = 0;
        for (Edge edge : edges) {
            if (dsu.unite(edge.u, edge.v)) {
                tree.add(edge);
                total += edge.weight;
                if (tree.size() == n - 1) break;
            }
        }
        if (tree.size() != n - 1) {
            throw new IllegalArgumentException("graph is disconnected");
        }
        return new TreeResult(tree, total);
    }

    static void printTree(String name, TreeResult result) {
        System.out.println(name + ":");
        for (Edge edge : result.edges) {
            System.out.println(edge.u + " - " + edge.v + " : " + edge.weight);
        }
        System.out.println("Total weight: " + result.total);
    }

    public static void main(String[] args) {
        int n = 6;
        List<Edge> edges = Arrays.asList(
            new Edge(0, 1, 4), new Edge(0, 2, 3), new Edge(1, 2, 1),
            new Edge(1, 3, 2), new Edge(2, 3, 4), new Edge(3, 4, 2),
            new Edge(3, 5, 3), new Edge(2, 5, 5), new Edge(4, 5, 6)
        );

        List<List<Edge>> adjacency = new ArrayList<>();
        for (int i = 0; i < n; i++) adjacency.add(new ArrayList<>());
        for (Edge edge : edges) {
            adjacency.get(edge.u).add(new Edge(edge.u, edge.v, edge.weight));
            adjacency.get(edge.v).add(new Edge(edge.v, edge.u, edge.weight));
        }

        printTree("Prim", prim(n, adjacency));
        printTree("Kruskal", kruskal(n, edges));
    }
}
```

## Go 完整示例

```go
package main

import (
	"container/heap"
	"fmt"
	"sort"
)

type Edge struct {
	u, v, weight int
}

type TreeResult struct {
	edges []Edge
	total int
}

type EdgeHeap []Edge

func (h EdgeHeap) Len() int { return len(h) }
func (h EdgeHeap) Less(i, j int) bool {
	if h[i].weight != h[j].weight {
		return h[i].weight < h[j].weight
	}
	if h[i].u != h[j].u {
		return h[i].u < h[j].u
	}
	return h[i].v < h[j].v
}
func (h EdgeHeap) Swap(i, j int) { h[i], h[j] = h[j], h[i] }
func (h *EdgeHeap) Push(x any)   { *h = append(*h, x.(Edge)) }
func (h *EdgeHeap) Pop() any {
	old := *h
	last := old[len(old)-1]
	*h = old[:len(old)-1]
	return last
}

type DSU struct {
	parent []int
	rank   []int
}

func NewDSU(n int) *DSU {
	parent := make([]int, n)
	for i := range parent {
		parent[i] = i
	}
	return &DSU{parent: parent, rank: make([]int, n)}
}

func (d *DSU) Find(x int) int {
	if d.parent[x] != x {
		d.parent[x] = d.Find(d.parent[x])
	}
	return d.parent[x]
}

func (d *DSU) Unite(a, b int) bool {
	rootA, rootB := d.Find(a), d.Find(b)
	if rootA == rootB {
		return false
	}
	if d.rank[rootA] < d.rank[rootB] {
		rootA, rootB = rootB, rootA
	}
	d.parent[rootB] = rootA
	if d.rank[rootA] == d.rank[rootB] {
		d.rank[rootA]++
	}
	return true
}

func prim(n int, adjacency [][]Edge) (TreeResult, error) {
	visited := make([]bool, n)
	tree := make([]Edge, 0, n-1)
	total := 0
	candidates := &EdgeHeap{}
	heap.Init(candidates)

	visited[0] = true
	for _, edge := range adjacency[0] {
		heap.Push(candidates, edge)
	}
	for candidates.Len() > 0 && len(tree) < n-1 {
		edge := heap.Pop(candidates).(Edge)
		if visited[edge.v] {
			continue
		}
		visited[edge.v] = true
		tree = append(tree, edge)
		total += edge.weight
		for _, next := range adjacency[edge.v] {
			if !visited[next.v] {
				heap.Push(candidates, next)
			}
		}
	}
	if len(tree) != n-1 {
		return TreeResult{}, fmt.Errorf("graph is disconnected")
	}
	return TreeResult{edges: tree, total: total}, nil
}

func kruskal(n int, inputEdges []Edge) (TreeResult, error) {
	edges := append([]Edge(nil), inputEdges...)
	sort.Slice(edges, func(i, j int) bool {
		if edges[i].weight != edges[j].weight {
			return edges[i].weight < edges[j].weight
		}
		if edges[i].u != edges[j].u {
			return edges[i].u < edges[j].u
		}
		return edges[i].v < edges[j].v
	})
	dsu := NewDSU(n)
	tree := make([]Edge, 0, n-1)
	total := 0
	for _, edge := range edges {
		if dsu.Unite(edge.u, edge.v) {
			tree = append(tree, edge)
			total += edge.weight
			if len(tree) == n-1 {
				break
			}
		}
	}
	if len(tree) != n-1 {
		return TreeResult{}, fmt.Errorf("graph is disconnected")
	}
	return TreeResult{edges: tree, total: total}, nil
}

func printTree(name string, result TreeResult) {
	fmt.Printf("%s:\n", name)
	for _, edge := range result.edges {
		fmt.Printf("%d - %d : %d\n", edge.u, edge.v, edge.weight)
	}
	fmt.Printf("Total weight: %d\n", result.total)
}

func main() {
	n := 6
	edges := []Edge{
		{0, 1, 4}, {0, 2, 3}, {1, 2, 1},
		{1, 3, 2}, {2, 3, 4}, {3, 4, 2},
		{3, 5, 3}, {2, 5, 5}, {4, 5, 6},
	}
	adjacency := make([][]Edge, n)
	for _, edge := range edges {
		adjacency[edge.u] = append(adjacency[edge.u], edge)
		adjacency[edge.v] = append(adjacency[edge.v], Edge{edge.v, edge.u, edge.weight})
	}

	primResult, err := prim(n, adjacency)
	if err != nil {
		panic(err)
	}
	kruskalResult, err := kruskal(n, edges)
	if err != nil {
		panic(err)
	}
	printTree("Prim", primResult)
	printTree("Kruskal", kruskalResult)
}
```

## 易错点与要点总结

- Prim/Kruskal 的标准输入是无向图；邻接表中要为 Prim 添加两个方向，但 Kruskal 的边集只保留一份无向边。
- 结果必须恰有 `V-1` 条边。少于该数量通常表示原图不连通，而不是算法已经得到完整 MST。
- Prim 的堆中会有指向已访问顶点的过期边，弹出后应跳过。
- Kruskal 必须按边权排序，并用并查集判断是否成环；只判断端点是否直接相邻是不够的。
- 相同权值下输出边集可能不同，只要覆盖全部顶点、无环且总权最小即可。
- MST 允许负权边；“Dijkstra 不允许负权”的限制不能误套到 MST。
- 总权值可能超出 32 位整数，生产代码应按输入范围改用 `long long`、`long` 或 Go 的 `int64`。
- 最小生成树不保证源点到各点距离最短；它与最短路径树优化目标不同。
