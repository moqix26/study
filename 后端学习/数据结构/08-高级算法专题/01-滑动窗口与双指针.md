# 滑动窗口与双指针

## 学习目标

- 理解滑动窗口是对连续区间的增量维护，双指针是对搜索空间的单调排除。
- 掌握固定长度窗口、可变长度窗口和相向双指针的基本模板。
- 能识别指针为何可以只向一个方向移动，以及何时单调性不成立。
- 使用代表性问题实现“和至少为目标值的最短子数组”与“有序数组两数之和”。

## 核心原理

### 1. 双指针不是模板，而是单调排除

暴力枚举两个位置通常要检查 `O(n²)` 个下标对。双指针能降到线性时间，不是因为变量恰好有两个，而是因为问题具有某种单调性：根据当前状态，可以证明一批候选永远不可能成为答案，于是某个指针只向一个方向移动，不必回退。

常见形式包括：

- **相向指针**：`left` 从左端向右，`right` 从右端向左。常见于有序数组配对、回文判断和接雨水。
- **同向指针**：两个指针都向右，通常分别表示一个连续区间的左右边界。滑动窗口属于这一类。
- **快慢指针**：快指针负责扫描，慢指针负责维护有效结果区间；或二者以不同速度移动，用于链表判环。

判断一道题能否使用双指针，应先回答三个问题：

1. 两个指针分别表示什么状态或边界？
2. 当前条件不满足时，为什么只移动某一个指针就足够？
3. 被越过的候选为什么不可能在未来重新变成答案？

如果第三个问题无法证明，指针可能需要回退，线性复杂度通常也就不成立。

### 2. 滑动窗口维护的是连续区间

滑动窗口一般表示闭区间 `[left,right]`。算法不重新计算整个区间，而是在右端加入元素、左端移出元素时增量更新窗口状态，例如：

- 数值之和 `windowSum`。
- 各字符出现次数 `frequency`。
- 窗口内不同元素个数。
- 借助单调队列维护的最大值或最小值。

窗口的核心不变量必须写清楚。例如求窗口和时，不变量是：**每轮处理完边界变化后，`windowSum` 恰好等于下标 `left..right` 内元素之和**。右端加入 `numbers[right]`，左端移出 `numbers[left]`，两次更新都必须与指针移动严格对应，否则窗口变量会与实际区间错位。

滑动窗口只直接覆盖连续子数组或连续子串。若题目求任意子序列，元素可以跳选，`[left,right]` 不能完整表示候选状态，通常要换成动态规划、贪心或搜索。

### 3. 固定长度窗口

固定窗口用于“所有长度恰为 `k` 的连续区间”问题。处理顺序通常是：

1. 右端加入新元素。
2. 若窗口长度超过 `k`，移出左端元素并令 `left++`。
3. 当窗口长度恰为 `k` 时，用当前状态更新答案。

以数组 `[2,3,1,2,4,3]`、`k=3` 为例：

| 窗口 | 从上一窗口的变化 | 窗口和 |
|---|---|---:|
| `[2,3,1]` | 首次累加三个元素 | 6 |
| `[3,1,2]` | 加 2，减 2 | 6 |
| `[1,2,4]` | 加 4，减 3 | 7 |
| `[2,4,3]` | 加 3，减 1 | 9 |

每次移动只做常数次加减，所以总时间是 `O(n)`，而不是为每个窗口重新求和的 `O(nk)`。如果 `k<=0` 或 `k>n`，必须先规定返回值或错误行为；算法本身不能凭空产生合法窗口。

### 4. 可变窗口的扩张与收缩

可变窗口的长度由约束决定。通常让 `right` 负责扩张，用 `left` 负责恢复条件或压缩答案，但“何时更新答案”取决于目标：

- 求**最短可行窗口**：右端扩张直到可行；在可行期间反复收缩左端，并在每次收缩前更新最短长度。
- 求**最长合法窗口**：右端扩张；一旦不合法，收缩到重新合法；在合法状态下更新最长长度。

不能只背“先加右边、再减左边”的代码外形。真正需要保持的是题目对应的不变量。例如“至多包含 K 种字符的最长子串”在每轮结束时要保证窗口合法；“和至少为 target 的最短子数组”则允许扩张阶段暂时不可行，并在可行时尽量收缩。

### 5. 正数数组最短子数组逐步轨迹

示例问题是：数组 `[2,3,1,2,4,3]` 全为正数，求和至少为 7 的最短连续子数组。

正数带来单调性：

- `right` 右移并加入元素，窗口和只会增大。
- `left` 右移并删除元素，窗口和只会减小。

因此，当窗口和不足 7 时，移动左端只会让情况更差，唯一有希望的动作是继续扩张右端；当窗口和达到 7 时，固定当前右端，继续收缩左端可以枚举所有更短的可行后缀。

完整过程如下：

| `right` | 加入值 | 扩张后窗口与和 | 收缩过程 | 当前最短长度 |
|---:|---:|---|---|---:|
| 0 | 2 | `[0,0]`，和 2 | 不满足，不收缩 | 未找到 |
| 1 | 3 | `[0,1]`，和 5 | 不满足，不收缩 | 未找到 |
| 2 | 1 | `[0,2]`，和 6 | 不满足，不收缩 | 未找到 |
| 3 | 2 | `[0,3]`，和 8 | 记录长度 4；移出 2 后和 6 | 4 |
| 4 | 4 | `[1,4]`，和 10 | 记录 4，移出 3 得 7；记录 3，移出 1 得 6 | 3 |
| 5 | 3 | `[3,5]`，和 9 | 记录 3，移出 2 得 7；记录 2，移出 4 得 3 | 2 |

最终最短窗口是下标 `[4,5]`，元素 `[4,3]`，长度为 2。

收缩必须使用 `while`，不能只用一次 `if`。在 `right=4` 时，窗口 `[1,4]` 收缩一次后 `[2,4]` 仍满足和至少 7，必须继续收缩才能发现长度 3；同理最后才能发现长度 2。

### 6. 为什么简单窗口在负数下失效

上述策略依赖“移出左端不会让和增大”。如果数组含负数，这个性质不再成立。例如窗口左端是 `-5` 时，删除它会让窗口和增加 5。此时“和不足就绝不能移动 left”不再安全，某个当前不足的窗口可能通过删除负数变成可行。

同样，加入一个负数也可能让窗口从可行变成不可行，所以右端扩张不再单调改善条件。简单的同向窗口可能漏解。处理“含负数、和至少为目标的最短子数组”通常要使用前缀和加单调队列；若只查询区间和，则前缀和本身也很常用。

非负数（允许 0）仍不会破坏和的单调方向，但 0 可能使多次边界变化后和不变。实现与证明要允许这种相等情况，不能依赖“每移动一次和都会严格变化”。

### 7. 有序数组两数之和的排除过程

对升序数组 `[1,2,4,6,10]` 寻找和为 8 的两个数，初始化 `left=0`、`right=4`：

| 步骤 | 下标对 | 数值与和 | 决策 | 被安全排除的候选 |
|---:|---|---|---|---|
| 1 | `(0,4)` | `1+10=11` | 和过大，`right--` | 所有与 10 配对的更大左值只会得到更大的和 |
| 2 | `(0,3)` | `1+6=7` | 和过小，`left++` | 1 与下标 `0..3` 中任意数配对都不可能达到 8 |
| 3 | `(1,3)` | `2+6=8` | 找到答案 | 返回下标 1、3 |

当当前和小于目标时，固定 `left`，任何比 `numbers[right]` 更小的右端只会让和更小，所以当前左端不可能参与答案，可以整体丢弃。当前和大于目标时，固定 `right`，任何更大的左端只会让和更大，所以当前右端不可能参与答案，可以丢弃。

每一步至少移动一个指针，两个指针总共最多移动 `n-1` 次，时间 `O(n)`、额外空间 `O(1)`。前提是数组已经有序。无序数组若先排序，会改变原下标；需要返回原下标时，应保存“值与原下标”的配对，或直接使用哈希表。

### 8. 正确性与复杂度的统一理解

正数最短窗口不会漏解：对每个固定的右端，算法在窗口可行时连续右移左端，恰好检查所有以该右端结尾、在变为不可行前的候选窗口；一旦删除某个左端后不再可行，由于数值非负，再继续删除只会更小，只能等待下一个右端扩张。

相向双指针不会漏解：每次移动前都证明当前整行或整列下标组合不可能含答案，因此被跳过的组合无需重新检查。

二者的线性时间也来自同一事实：指针只前进不后退。即使可变窗口内层有 `while`，`left` 在整个算法中最多从 0 移到 n，内层所有迭代加起来仍是 `O(n)`，不是每轮右端都执行 `O(n)`。

### 9. 适用前的边界检查

- 空数组没有非空窗口；返回 0、空下标还是错误，应按接口约定处理。
- 最短和问题若 `target<=0`，空窗口是否允许会改变答案；本文代码按正目标、非空子数组设计。
- 两数之和必须要求两个不同下标，因此循环条件是 `left < right`，不能让同一元素使用两次。
- 数值和可能超过 32 位范围，窗口和及两数之和应根据数据范围使用更宽整数类型。
- 频率窗口在移动左端时必须同步减少计数，并在计数降为 0 时更新“不同元素数量”等派生状态。

## 复杂度表

| 问题/方法 | 时间复杂度 | 额外空间 | 前提 |
|---|---:|---:|---|
| 暴力枚举连续子数组 | `O(n²)`，直接求和可达 `O(n³)` | `O(1)` | 无特殊前提 |
| 正数数组可变滑动窗口 | `O(n)` | `O(1)` | 连续区间、正数带来单调性 |
| 有序数组两数之和 | `O(n)` | `O(1)` | 数组已升序 |
| 哈希表两数之和 | 平均 `O(n)` | `O(n)` | 无需有序，可保留原下标 |

## 适用场景

- 连续子数组/子串，且窗口增删时状态可快速更新。
- “最长/最短且满足某条件”的区间问题，条件随边界变化具有单调性。
- 有序序列中的配对、去重、区间收缩。
- 若问题不要求连续、指针需要频繁回退、或条件不单调，应重新考虑前缀和、哈希、二分或动态规划。

## C++17 完整示例

```cpp
#include <iostream>
#include <utility>
#include <vector>
using namespace std;

int minSubarrayLength(int target, const vector<int>& numbers) {
    int left = 0;
    long long windowSum = 0;
    int best = static_cast<int>(numbers.size()) + 1;

    for (int right = 0; right < static_cast<int>(numbers.size()); ++right) {
        windowSum += numbers[right];
        while (windowSum >= target) {
            int length = right - left + 1;
            if (length < best) best = length;
            windowSum -= numbers[left++];
        }
    }
    return best == static_cast<int>(numbers.size()) + 1 ? 0 : best;
}

pair<int, int> twoSumSorted(const vector<int>& numbers, int target) {
    int left = 0;
    int right = static_cast<int>(numbers.size()) - 1;
    while (left < right) {
        int sum = numbers[left] + numbers[right];
        if (sum == target) return {left, right};
        if (sum < target) ++left;
        else --right;
    }
    return {-1, -1};
}

int main() {
    vector<int> windowInput = {2, 3, 1, 2, 4, 3};
    cout << "Minimum length (sum >= 7): "
         << minSubarrayLength(7, windowInput) << '\n';

    vector<int> sortedInput = {1, 2, 4, 6, 10};
    auto [left, right] = twoSumSorted(sortedInput, 8);
    cout << "Two-sum indices: " << left << ' ' << right
         << ", values: " << sortedInput[left] << " + "
         << sortedInput[right] << " = 8\n";
    return 0;
}
```

## Java 17 完整示例

```java
public class Main {
    static int minSubarrayLength(int target, int[] numbers) {
        int left = 0;
        long windowSum = 0;
        int best = numbers.length + 1;

        for (int right = 0; right < numbers.length; right++) {
            windowSum += numbers[right];
            while (windowSum >= target) {
                best = Math.min(best, right - left + 1);
                windowSum -= numbers[left++];
            }
        }
        return best == numbers.length + 1 ? 0 : best;
    }

    static int[] twoSumSorted(int[] numbers, int target) {
        int left = 0;
        int right = numbers.length - 1;
        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) return new int[]{left, right};
            if (sum < target) left++;
            else right--;
        }
        return new int[]{-1, -1};
    }

    public static void main(String[] args) {
        int[] windowInput = {2, 3, 1, 2, 4, 3};
        System.out.println("Minimum length (sum >= 7): "
            + minSubarrayLength(7, windowInput));

        int[] sortedInput = {1, 2, 4, 6, 10};
        int[] pair = twoSumSorted(sortedInput, 8);
        System.out.println("Two-sum indices: " + pair[0] + " " + pair[1]
            + ", values: " + sortedInput[pair[0]] + " + "
            + sortedInput[pair[1]] + " = 8");
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

func minSubarrayLength(target int, numbers []int) int {
	left := 0
	windowSum := 0
	best := len(numbers) + 1
	for right, value := range numbers {
		windowSum += value
		for windowSum >= target {
			length := right - left + 1
			if length < best {
				best = length
			}
			windowSum -= numbers[left]
			left++
		}
	}
	if best == len(numbers)+1 {
		return 0
	}
	return best
}

func twoSumSorted(numbers []int, target int) (int, int) {
	left, right := 0, len(numbers)-1
	for left < right {
		sum := numbers[left] + numbers[right]
		if sum == target {
			return left, right
		}
		if sum < target {
			left++
		} else {
			right--
		}
	}
	return -1, -1
}

func main() {
	windowInput := []int{2, 3, 1, 2, 4, 3}
	fmt.Printf("Minimum length (sum >= 7): %d\n",
		minSubarrayLength(7, windowInput))

	sortedInput := []int{1, 2, 4, 6, 10}
	left, right := twoSumSorted(sortedInput, 8)
	fmt.Printf("Two-sum indices: %d %d, values: %d + %d = 8\n",
		left, right, sortedInput[left], sortedInput[right])
}
```

## 易错点与要点总结

- 滑动窗口通常只处理连续区间；子序列问题不能直接套用。
- 先扩大还是先收缩、何时更新答案，取决于题目要最长还是最短，不能死背单一模板。
- 最短可行窗口应在 `while` 收缩过程中更新；只用一次 `if` 会错过继续收缩的机会。
- 正数和条件提供单调性；含负数时本例算法会失效。
- 两数之和的相向移动依赖数组有序；无序输入不能根据和的大小决定丢弃哪一端。
- 计算和时要考虑整数溢出，输入范围大时使用 64 位整数。
- 无解时先检查返回的 `-1`，生产代码不要像固定有解的演示一样直接访问下标。

## 练习

1. 求不含重复字符的最长子串，并用频率表维护可变窗口。

3. 在升序数组中找三个数之和为目标值，将排序、枚举与相向双指针结合。
4. 为含负数数组研究“和至少为 K 的最短子数组”，比较前缀和 + 单调队列方案。
