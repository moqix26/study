# 回溯算法

## 学习目标

- 理解回溯是带撤销操作的深度优先搜索，而不是无条件暴力枚举。
- 掌握“选择—递归—撤销”的通用骨架，以及可行性剪枝的放置位置。
- 能区分路径、选择列表、终止条件和答案集合。
- 使用 N 皇后问题完整实现冲突检测、状态恢复与解的收集。

## 核心原理

### 1. 回溯是在决策树上系统搜索

回溯把一个解拆成一连串选择。每做一次选择，就从决策树的一层走到下一层；当部分选择不可能扩展成合法答案时立即返回；一个分支处理完后撤销最后一次选择，再尝试同层的其他分支。

例如生成 `{1,2,3}` 的排列：第一层选择第一个位置放谁，第二层选择剩余元素中的一个，第三层放最后一个元素。根到叶子的路径 `2 -> 1 -> 3` 对应排列 `[2,1,3]`。回溯不是随机试错，而是按明确顺序遍历整棵可行决策树。

一个回溯函数通常要明确五类信息：

- **路径**：已经做出的选择，如已放置的皇后列号。
- **当前层/位置**：下一步要决定什么，如当前处理第几行。
- **选择列表**：这一层可以尝试哪些候选，如所有列 `0..n-1`。
- **约束状态**：快速判断候选是否合法，如已占用列和对角线。
- **终止条件**：何时形成完整解，如 `row==n`。

如果这五项没有定义清楚，代码很容易出现多选、漏选、重复答案或撤销不完整。

### 2. “选择—递归—撤销”的状态不变量

对某个候选 `choice`，一次分支处理包含：

1. 检查 `choice` 是否满足当前约束。
2. 把它加入路径，并同步修改所有约束状态。
3. 递归处理下一层。
4. 递归返回后，按与第 2 步相反的方式恢复状态。

关键不变量是：**每次进入某一层递归函数时，共享状态必须准确表示当前路径，并且与进入该层前定义的层数一致。** 对 N 皇后而言，进入 `backtrack(row)` 时，棋盘第 `0..row-1` 行各有一个互不攻击的皇后，第 `row..n-1` 行为空，占用数组与棋盘完全一致。

撤销不是可选的清理工作，而是算法正确性的一部分。假设在第 0 行尝试第 1 列后没有清除 `usedColumn[1]`，再尝试第 0 行第 2 列时，后续层会误以为第 1 列仍被占用，从而剪掉本应合法的分支。

局部不可变值通常不需要撤销，例如递归参数 `row+1` 会随函数返回自然恢复；对共享数组、路径容器、棋盘和计数器的原地修改则必须显式恢复。

### 3. 剪枝必须建立在必要条件上

若当前部分路径已经违反合法解的必要条件，那么无论后续如何选择都无法修复，可以立即停止该分支。这就是可行性剪枝。

常见剪枝依据包括：

- **冲突剪枝**：棋盘位置与已有棋子冲突。
- **容量剪枝**：当前重量已超过背包容量，后续再加入物品只会更重。
- **重复剪枝**：同一层对相同值只选择一次，避免生成内容相同的排列或组合。
- **上下界剪枝**：即使把剩余最有利选择全部加入，也不可能超过当前最优答案。
- **前缀约束**：字符串前缀不在 Trie 中，后续不可能组成目标单词。

剪枝要求安全：被剪分支必须确定没有所需答案。仅仅“看起来不太好”不能作为剪枝条件。比如最优化问题中，当前值比已知最优差，并不一定能直接剪掉；还要证明后续最大可能收益也追不上。

剪枝越靠近决策树上层，省掉的后代节点越多。但为了剪枝维护复杂状态也有成本，需要比较节省的搜索量与额外判断开销。

### 4. N 皇后的状态建模

N 皇后要求在 `n × n` 棋盘放置 n 个皇后，任意两个皇后不能同行、同列或同对角线。本文按行递归：第 `row` 层决定第 `row` 行的皇后放在哪一列。

这样建模后：

- 每层只放一个皇后，天然保证不同行。
- `usedColumn[col]` 表示某一列是否已有皇后。
- 从左上到右下的主对角线上，`row-col` 相同。其范围是 `-(n-1)..(n-1)`，加 `n-1` 后映射到 `0..2n-2`。
- 从右上到左下的副对角线上，`row+col` 相同，范围同样是 `0..2n-2`。

因此位置 `(row,col)` 的三个冲突索引为：

- 列：`col`
- 主对角线：`row-col+n-1`
- 副对角线：`row+col`

三个布尔数组都未占用时才可放置，判断成本为 `O(1)`。若每次都向上扫描棋盘检查列和对角线，单次判断为 `O(n)`，整个搜索会增加明显常数。

放置一个皇后时必须同步修改四处状态：棋盘字符、列占用、主对角线占用、副对角线占用；回退时四处都要恢复。只恢复棋盘而忘记占用数组，或反过来，都会使状态不一致。

### 5. 4 皇后的具体递归轨迹

用 `c0,c1,...` 表示每一行选择的列。算法按列从 0 到 3 尝试，下面列出第一组解出现前的主要分支：

| 当前路径 | 下一步尝试与结果 | 后续动作 |
|---|---|---|
| `[]` | 第 0 行选列 0，得到 `[0]` | 进入第 1 行 |
| `[0]` | 列 0 冲突，列 1 对角线冲突；选列 2 | 得到 `[0,2]` |
| `[0,2]` | 第 2 行四列均与已有皇后冲突 | 回退，撤销第 1 行列 2 |
| `[0]` | 第 1 行继续选列 3 | 得到 `[0,3]` |
| `[0,3]` | 第 2 行只有列 1 可行 | 得到 `[0,3,1]` |
| `[0,3,1]` | 第 3 行无可行列 | 连续回退到第 0 行 |
| `[]` | 第 0 行改选列 1 | 得到 `[1]` |
| `[1]` | 第 1 行只有列 3 可行 | 得到 `[1,3]` |
| `[1,3]` | 第 2 行选列 0 | 得到 `[1,3,0]` |
| `[1,3,0]` | 第 3 行选列 2 | 得到完整解 `[1,3,0,2]` |

路径 `[1,3,0,2]` 对应棋盘：

| 行 | 皇后列 | 棋盘行 |
|---:|---:|---|
| 0 | 1 | `.Q..` |
| 1 | 3 | `...Q` |
| 2 | 0 | `Q...` |
| 3 | 2 | `..Q.` |

收集该解后不能直接结束，因为题目要求全部方案。算法撤销第 3 行选择，继续检查剩余分支，最终还会找到对称解 `[2,0,3,1]`。示例输出中的解数量因此为 2。

这段轨迹体现了回溯的两个层次：某个候选因即时冲突在进入递归前就被剪掉；某些当前合法的候选会在更深层发现无路可走，然后通过撤销返回。

### 6. 终止条件决定收集什么

N 皇后在 `row==n` 时收集答案，因为这表示 n 行都已完成选择。此时不能再继续枚举列，也不能在 `row==n-1` 刚放置后忘记复制棋盘。

不同问题的终止方式不同：

- 排列：路径长度等于元素数时收集。
- 子集：每个递归节点都可能是一个合法子集，通常进入函数就收集，而不是只在叶子收集。
- 组合：选择数量达到 `k` 时收集并返回。
- 寻找任意一个解：找到后可以用布尔返回值向上传播并提前终止。
- 枚举全部解：收集后仍要正常撤销，继续其他分支。

所以不能把某道题的 `if` 位置机械复制到另一道题，必须根据“一个完整答案由什么组成”确定。

### 7. 正确性：不收错、不漏解、不重复

可以从三个方面说明 N 皇后算法正确：

**可靠性**：只有列、主对角线和副对角线均未占用的位置才会放置。按行递归又保证每行一个皇后。因此到达 `row==n` 时，收集的棋盘一定合法。

**完备性**：任取一个合法棋盘。它在第 0 行有唯一皇后列，该列会被循环尝试，且因棋盘合法不会被冲突剪枝；进入第 1 行后同理。逐行归纳，这个合法棋盘对应的整条路径一定会到达叶子并被收集。

**无重复**：每一层固定代表一行，同一层每列只尝试一次。一组列序列唯一对应一个棋盘，因此同一解不会由两条不同路径生成。

### 8. 搜索规模与复杂度

如果完全不考虑冲突，第 0 行有 n 种选择，第 1 行仍有 n 种，叶子会有 `n^n` 个。加入“列不能重复”后，候选数依次至多为 `n,n-1,...,1`，常用 `O(n!)` 表示搜索时间上界；对角线剪枝会进一步减少实际访问节点，但不会把最坏情况变成多项式。

冲突判断为 `O(1)`，递归深度为 n，占用数组和当前棋盘需要 `O(n²)` 空间，其中棋盘本身占主要部分。若只用列数组表示当前路径，工作空间可降到 `O(n)`，输出时再构造棋盘。

保存答案的成本必须单独计算。若有 `S` 个解，每次收集都要复制含 `n²` 个字符的棋盘，因此答案构造的时间和结果空间都是 `O(S·n²)`。示例的总时间应理解为搜索 `O(n!)` 再加输出构造 `O(S·n²)`；任何枚举全部解的算法都无法避开与输出规模相称的成本。

### 9. 与 DFS、分治和动态规划的区别

- DFS 是一种“优先深入”的遍历顺序；回溯通常用 DFS 实现，但额外强调构造候选、约束剪枝以及递归后的状态撤销。
- 分治把问题拆成若干相对独立、通常都必须求解的子问题，再合并结果；回溯是在多个候选分支中搜索，并可能大面积剪掉分支。
- 动态规划会把相同状态的答案缓存并复用。若不同回溯路径经常到达完全相同、且未来只由该状态决定的局面，可加入记忆化；但若输出要求保留每条不同路径，不能简单合并所有路径。
- 分支限界也是搜索，但更强调利用上下界寻找最优解，常配合优先队列；普通回溯多按深度优先顺序枚举。

### 10. 常见建模边界

- 输入含重复元素时，必须明确答案按“元素下标”还是按“元素值”去重。常见做法是先排序，并在同一递归层跳过相同值。
- 组合问题通常需要 `start` 下标限制后续只能向右选择，避免 `[1,2]` 与 `[2,1]` 被当成两个组合。
- 允许重复使用候选时，递归传入当前位置；每个候选最多一次时，传入下一位置。这个下标变化决定问题模型。
- 若递归深度可能非常大，要考虑调用栈限制；不过典型回溯本身搜索规模很快增长，通常输入不会大到形成极深递归。

## 复杂度表

| 项目 | 复杂度 | 说明 |
|---|---:|---|
| N 皇后搜索时间 | 上界常写 `O(n!)` | 列不重复后每层候选递减，剪枝会进一步减少实际节点 |
| 收集所有棋盘的时间 | `O(S·n²)` | 每个完整解需要复制 `n²` 个字符 |
| 冲突判断 | `O(1)` | 三个占用数组 |
| 递归与当前状态空间 | `O(n²)` | 示例保留 `n×n` 棋盘；若只存每行的皇后列号可降为 `O(n)` |
| 保存所有棋盘 | `O(S·n²)` | `S` 为解的数量 |

## 适用场景

- 排列、组合、子集、棋盘放置、数独、单词搜索等需要构造具体方案的问题。
- 约束可以在部分解阶段检查，能有效剪枝的搜索问题。
- 输入规模通常较小；若状态重复严重或规模很大，应考虑记忆化、动态规划、启发式搜索或专用算法。

## C++17 完整示例

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

class NQueensSolver {
private:
    int n;
    vector<string> board;
    vector<bool> usedColumn;
    vector<bool> usedMainDiagonal;
    vector<bool> usedAntiDiagonal;
    vector<vector<string>> solutions;

    void backtrack(int row) {
        if (row == n) {
            solutions.push_back(board);
            return;
        }

        for (int col = 0; col < n; ++col) {
            int mainDiagonal = row - col + n - 1;
            int antiDiagonal = row + col;
            if (usedColumn[col] || usedMainDiagonal[mainDiagonal]
                || usedAntiDiagonal[antiDiagonal]) {
                continue;
            }

            board[row][col] = 'Q';
            usedColumn[col] = true;
            usedMainDiagonal[mainDiagonal] = true;
            usedAntiDiagonal[antiDiagonal] = true;

            backtrack(row + 1);

            board[row][col] = '.';
            usedColumn[col] = false;
            usedMainDiagonal[mainDiagonal] = false;
            usedAntiDiagonal[antiDiagonal] = false;
        }
    }

public:
    vector<vector<string>> solve(int size) {
        n = size;
        board.assign(n, string(n, '.'));
        usedColumn.assign(n, false);
        usedMainDiagonal.assign(2 * n - 1, false);
        usedAntiDiagonal.assign(2 * n - 1, false);
        solutions.clear();
        backtrack(0);
        return solutions;
    }
};

int main() {
    NQueensSolver solver;
    vector<vector<string>> solutions = solver.solve(4);
    cout << "Number of solutions: " << solutions.size() << '\n';
    cout << "First solution:\n";
    for (const string& row : solutions.front()) cout << row << '\n';
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {
    static class NQueensSolver {
        private int n;
        private char[][] board;
        private boolean[] usedColumn;
        private boolean[] usedMainDiagonal;
        private boolean[] usedAntiDiagonal;
        private final List<List<String>> solutions = new ArrayList<>();

        List<List<String>> solve(int size) {
            n = size;
            board = new char[n][n];
            for (char[] row : board) Arrays.fill(row, '.');
            usedColumn = new boolean[n];
            usedMainDiagonal = new boolean[2 * n - 1];
            usedAntiDiagonal = new boolean[2 * n - 1];
            solutions.clear();
            backtrack(0);
            return new ArrayList<>(solutions);
        }

        private void backtrack(int row) {
            if (row == n) {
                List<String> snapshot = new ArrayList<>();
                for (char[] line : board) snapshot.add(new String(line));
                solutions.add(snapshot);
                return;
            }

            for (int col = 0; col < n; col++) {
                int mainDiagonal = row - col + n - 1;
                int antiDiagonal = row + col;
                if (usedColumn[col] || usedMainDiagonal[mainDiagonal]
                    || usedAntiDiagonal[antiDiagonal]) {
                    continue;
                }

                board[row][col] = 'Q';
                usedColumn[col] = true;
                usedMainDiagonal[mainDiagonal] = true;
                usedAntiDiagonal[antiDiagonal] = true;

                backtrack(row + 1);

                board[row][col] = '.';
                usedColumn[col] = false;
                usedMainDiagonal[mainDiagonal] = false;
                usedAntiDiagonal[antiDiagonal] = false;
            }
        }
    }

    public static void main(String[] args) {
        NQueensSolver solver = new NQueensSolver();
        List<List<String>> solutions = solver.solve(4);
        System.out.println("Number of solutions: " + solutions.size());
        System.out.println("First solution:");
        for (String row : solutions.get(0)) System.out.println(row);
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

func solveNQueens(n int) [][]string {
	board := make([][]byte, n)
	for row := range board {
		board[row] = make([]byte, n)
		for col := range board[row] {
			board[row][col] = '.'
		}
	}
	usedColumn := make([]bool, n)
	usedMainDiagonal := make([]bool, 2*n-1)
	usedAntiDiagonal := make([]bool, 2*n-1)
	solutions := make([][]string, 0)

	var backtrack func(int)
	backtrack = func(row int) {
		if row == n {
			snapshot := make([]string, n)
			for i := range board {
				snapshot[i] = string(board[i])
			}
			solutions = append(solutions, snapshot)
			return
		}

		for col := 0; col < n; col++ {
			mainDiagonal := row - col + n - 1
			antiDiagonal := row + col
			if usedColumn[col] || usedMainDiagonal[mainDiagonal] ||
				usedAntiDiagonal[antiDiagonal] {
				continue
			}

			board[row][col] = 'Q'
			usedColumn[col] = true
			usedMainDiagonal[mainDiagonal] = true
			usedAntiDiagonal[antiDiagonal] = true

			backtrack(row + 1)

			board[row][col] = '.'
			usedColumn[col] = false
			usedMainDiagonal[mainDiagonal] = false
			usedAntiDiagonal[antiDiagonal] = false
		}
	}

	backtrack(0)
	return solutions
}

func main() {
	solutions := solveNQueens(4)
	fmt.Printf("Number of solutions: %d\n", len(solutions))
	fmt.Println("First solution:")
	for _, row := range solutions[0] {
		fmt.Println(row)
	}
}
```

## 易错点与要点总结

- 修改共享状态后必须成对撤销；漏撤销会污染兄弟分支，过早撤销则会破坏当前递归路径。
- 保存答案时要复制当前路径或棋盘，不能只保存指向同一可变对象的引用。
- 终止条件要放在枚举下一层之前；完整解应立即收集并返回。
- 主对角线与副对角线编号公式容易混淆，数组大小均为 `2n-1`。
- 回溯复杂度通常是指数级；剪枝改善实际运行量，但不能随意宣称变成多项式。
- 存在重复输入元素的排列/组合题，通常要先排序，并在同一树层跳过重复选择。
- 如果只需解的数量，无需复制完整棋盘，可显著减少结果空间。

## 练习

1. 输出 N 皇后的解数量而不保存棋盘，并比较空间消耗。
2. 实现全排列；输入含重复元素时，增加排序和同层去重。
3. 实现组合总和，分析“元素可重复使用”和“每个元素只能使用一次”时递归起点的差异。
4. 解一个 9×9 数独，使用行、列、宫占用集合剪枝。
5. 用位运算替代三个布尔数组优化 N 皇后，并测试更大的 n。
