# 分治算法

## 学习目标

- 理解分治的“分解—解决—合并”结构及其适用条件。
- 能写出递归边界、组合逻辑和复杂度递推式。
- 区分分治、普通递归、回溯与动态规划。
- 使用分治完整求解最大子数组，并恢复最优区间。

## 核心原理

### 1. 分治的结构与适用条件

分治把一个规模较大的问题拆成若干规模更小、形式相同的子问题，分别求解后再组合成原问题答案。完整过程包含：

1. **分（Divide）**：确定如何划分输入，例如按中点拆成左右区间。
2. **治（Conquer）**：递归解决子问题；规模小到某个阈值时直接求解。
3. **合（Combine）**：把子问题结果转换成当前问题结果。

“递归调用了两次”并不足以说明是好的分治。有效的分治通常满足：

- 子问题规模明显缩小，递归能到达边界。
- 子问题结构与原问题一致，可以复用同一求解逻辑。
- 子问题之间相对独立，不会大量重复计算同一状态。
- 合并成本可控，不能比直接解决原问题还昂贵很多。

归并排序在合并阶段线性合并两个有序数组；二分查找每次只递归一个半区间，几乎不需要合并；快速排序围绕枢轴划分后，子数组原地就位；快速幂把指数减半，再根据奇偶组合平方结果。它们的具体代码不同，但都符合“规模缩小 + 结果组合”。

### 2. 递归边界必须直接给出完整答案

分治函数一般处理一个明确区间 `[left,right]`。当区间只含一个元素时，不能继续按中点拆分，否则会重复进入相同区间。边界返回值必须包含上层合并需要的全部信息。

本文最大子数组要求答案是**非空**连续区间，所以单元素 `[i,i]` 的唯一合法答案就是：

- 最大和为 `numbers[i]`
- 左端点为 `i`
- 右端点为 `i`

这一定义可以正确处理全负数组。若错误地把空数组和 0 当作候选，则输入 `[-5,-2,-8]` 会返回 0 和空区间，而题目要求的非空答案应是 `[-2]`。边界语义必须与题目保持一致。

### 3. 最大子数组为什么恰好分成三类

对当前区间 `[left,right]`，令 `mid` 为中点，左半区为 `[left,mid]`，右半区为 `[mid+1,right]`。任意一个连续子数组相对分界线只有三种可能：

1. 右端点不超过 `mid`，完全位于左半区。
2. 左端点大于 `mid`，完全位于右半区。
3. 左端点不超过 `mid` 且右端点大于 `mid`，跨越分界线。

三类互斥且覆盖所有连续区间，没有第四种位置关系。因此：

- 左类最优解由递归 `solve(left,mid)` 得到。
- 右类最优解由递归 `solve(mid+1,right)` 得到。
- 跨越类由合并步骤单独计算。

当前答案就是三类结果中和最大的一个。若和相等而题目要求最短区间、最左区间或字典序规则，还需在比较函数中加入平局处理；本文代码只保证返回某个最大和区间。

### 4. 跨中点结果如何在线性时间得到

任何跨越分界线的连续子数组都必须同时包含 `mid` 和 `mid+1`。它可以唯一拆成：

- 左半区中一个“以 `mid` 结尾”的后缀。
- 右半区中一个“以 `mid+1` 开始”的前缀。

两部分彼此独立，因此分别找最大后缀和、最大前缀和，再相加即可。左侧从 `mid` 向 `left` 累加，记录遇到的最大和及起点；右侧从 `mid+1` 向 `right` 累加，记录最大和及终点。

示例数组为：

`[-2,1,-3,4,-1,2,1,-5,4]`

最顶层区间 `[0,8]` 的 `mid=4`。左侧最大后缀扫描：

| 扫描到下标 | 新加入值 | 从该下标到 4 的和 | 当前最佳左后缀 |
|---:|---:|---:|---|
| 4 | -1 | -1 | `[4,4]`，和 -1 |
| 3 | 4 | 3 | `[3,4]`，和 3 |
| 2 | -3 | 0 | 保持 `[3,4]` |
| 1 | 1 | 1 | 保持 `[3,4]` |
| 0 | -2 | -1 | 保持 `[3,4]` |

右侧最大前缀扫描：

| 扫描到下标 | 新加入值 | 从 5 到该下标的和 | 当前最佳右前缀 |
|---:|---:|---:|---|
| 5 | 2 | 2 | `[5,5]`，和 2 |
| 6 | 1 | 3 | `[5,6]`，和 3 |
| 7 | -5 | -2 | 保持 `[5,6]` |
| 8 | 4 | 2 | 保持 `[5,6]` |

所以最优跨中点区间为 `[3,4] + [5,6] = [3,6]`，元素 `[4,-1,2,1]`，和为 6。

### 5. 顶层递归如何选出最终答案

顶层不是只计算跨中点结果，还要与左右递归结果比较：

| 候选类别 | 区间 | 最大和 | 来源 |
|---|---|---:|---|
| 完全在左半区 `[0,4]` | `[3,3]` | 4 | 左递归 |
| 完全在右半区 `[5,8]` | `[8,8]` | 4 | 右递归 |
| 跨越顶层中点 | `[3,6]` | 6 | 左最大后缀 + 右最大前缀 |

三者最大的是 `[3,6]`，因此示例最终输出最大和 6、范围 `[3,6]`。

继续向下看，每个递归区间都执行完全相同的三类比较。例如区间 `[3,4]` 会比较单元素 `[3,3]` 的和 4、单元素 `[4,4]` 的和 -1，以及跨中点 `[3,4]` 的和 3，最终返回 `[3,3]`。这就是“子问题与原问题同构”。

### 6. 正确性证明的结构

可以对区间长度做归纳。

**基础情况**：长度为 1 时，唯一非空连续子数组就是该元素，返回结果显然最优。

**归纳假设**：假设所有长度小于当前区间的子问题都能返回正确最大子数组。

**归纳步骤**：当前区间的任意最优子数组必属于左、右、跨中点三类之一。根据归纳假设，两个递归调用分别正确覆盖左类和右类；线性扫描分别求出最大左后缀与最大右前缀，它们的组合是跨中点类的最优解。取三类最大值，得到当前区间全局最优解。

这个证明依赖“分类完备”和“每类都求到最优”。设计其他分治算法时，也应明确合并阶段是否漏掉了跨子问题边界的情况。

### 7. 递归树与 `O(n log n)` 的来源

当前区间规模为 n 时，产生两个规模约为 `n/2` 的递归子问题；计算跨中点结果要扫描当前区间一次，成本 `O(n)`：

`T(n) = 2T(n/2) + O(n)`

从递归树看：

| 层数 | 子问题数量 | 每个子问题规模 | 本层合并总量 |
|---:|---:|---:|---:|
| 0 | 1 | n | `O(n)` |
| 1 | 2 | n/2 | `2·O(n/2)=O(n)` |
| 2 | 4 | n/4 | `4·O(n/4)=O(n)` |
| ... | ... | ... | `O(n)` |
| `log n` 附近 | n | 1 | `O(n)` |

递归深度为 `O(log n)`，每层总合并成本为 `O(n)`，因此总时间为 `O(n log n)`。在任意时刻只沿一条递归链保留栈帧，递归栈空间为 `O(log n)`，不是 `O(n log n)`。

当 n 不是 2 的幂时，左右规模可能相差 1，但仍保持常数比例划分，渐进复杂度不变。

### 8. 划分质量会改变复杂度

分治快不快取决于子问题缩小速度。归并排序固定从中点划分，所以深度稳定为 `O(log n)`。快速排序若每次枢轴都把数组分成 0 个和 `n-1` 个元素，递推变成：

`T(n)=T(n-1)+O(n)`

总时间退化为 `O(n²)`，递归栈也可能达到 `O(n)`。随机选择枢轴、三数取中和限制递归深度，都是为了降低极端不平衡风险。

二分查找每次只继续一个半区间，递推为 `T(n)=T(n/2)+O(1)`，所以是 `O(log n)`。不能看到“每次减半”就统一写 `O(n log n)`，还要看递归分支数量与合并成本。

### 9. 与 Kadane 和可合并区间信息的关系

单次求最大子数组时，Kadane 动态规划只需维护“以当前位置结尾的最大子数组和”，时间 `O(n)`、空间 `O(1)`，通常优于本文分治实现。

分治版本的价值在于展示区间答案如何合并。如果每个区间同时维护四项信息：

- 区间总和。
- 最大前缀和。
- 最大后缀和。
- 最大子数组和。

那么两个相邻区间可以在 `O(1)` 时间合并。这正是线段树维护区间最大子段和的基础：建树 `O(n)`，区间查询约 `O(log n)`。因此某个问题有更快的一次性算法，并不表示它的分治结构没有用途。

### 10. 分治、回溯与动态规划的区别

- **分治**：子问题通常相对独立，每个子问题都要求解，最后合并。例如归并排序左右区间没有重叠。
- **回溯**：递归分支代表不同选择，目的是搜索候选解；不合法分支可以剪掉，并常伴随状态撤销。
- **动态规划**：不同计算路径会反复遇到同一状态，需要缓存并复用；重点是状态、转移和计算顺序。
- **普通递归**：只是实现方式。递归求阶乘没有把问题拆成多个子问题并合并，通常不称为分治。

如果分治产生大量重叠状态，例如朴素递归斐波那契同时反复计算 `fib(k)`，就应使用记忆化或自底向上 DP。如果子问题独立但合并代价过高，也要重新设计返回信息，避免每层重复扫描过多内容。

### 11. 工程上的边界与取舍

- 空数组没有非空最大子数组，本文代码直接报错；若业务允许空答案，应重新定义返回值。
- 求和可能溢出 32 位整数，示例结果结构使用更宽整数类型保存累计和。
- 递归对很小的子问题有函数调用开销，排序实现常在小区间切换到插入排序。
- 独立的左右子问题理论上可以并行，但任务过小时线程调度成本可能超过收益，应设置并行阈值。
- 合并结果若携带下标，比较时要统一端点闭开规则；本文全部使用闭区间 `[left,right]`。

## 复杂度表

| 最大子数组方法 | 时间复杂度 | 额外空间 | 特点 |
|---|---:|---:|---|
| 枚举起止点并求和 | `O(n³)` | `O(1)` | 最直观但重复求和 |
| 前缀和 + 枚举区间 | `O(n²)` | `O(n)` | 区间和 `O(1)` |
| 分治 | `O(n log n)` | `O(log n)` | 结构清晰、可扩展区间合并 |
| Kadane 动态规划 | `O(n)` | `O(1)` | 单次求解通常最优选择 |

## 适用场景

- 问题可按输入范围拆分，子问题相互独立，答案能高效合并。
- 排序、搜索、计算几何、区间统计及可并行计算。
- 若划分极不均衡，如快速排序每次只减少一个元素，复杂度可能退化；需要随机化、平衡划分或其他保护策略。

## C++17 完整示例

```cpp
#include <iostream>
#include <stdexcept>
#include <vector>
using namespace std;

struct Result {
    long long sum;
    int left;
    int right;
};

Result crossingResult(const vector<int>& numbers, int left, int mid, int right) {
    long long sum = 0;
    long long bestLeftSum = numbers[mid];
    int bestLeft = mid;
    for (int i = mid; i >= left; --i) {
        sum += numbers[i];
        if (sum > bestLeftSum) {
            bestLeftSum = sum;
            bestLeft = i;
        }
    }

    sum = 0;
    long long bestRightSum = numbers[mid + 1];
    int bestRight = mid + 1;
    for (int i = mid + 1; i <= right; ++i) {
        sum += numbers[i];
        if (sum > bestRightSum) {
            bestRightSum = sum;
            bestRight = i;
        }
    }
    return {bestLeftSum + bestRightSum, bestLeft, bestRight};
}

Result solve(const vector<int>& numbers, int left, int right) {
    if (left == right) return {numbers[left], left, right};
    int mid = left + (right - left) / 2;
    Result leftResult = solve(numbers, left, mid);
    Result rightResult = solve(numbers, mid + 1, right);
    Result crossResult = crossingResult(numbers, left, mid, right);

    Result best = leftResult;
    if (rightResult.sum > best.sum) best = rightResult;
    if (crossResult.sum > best.sum) best = crossResult;
    return best;
}

Result maximumSubarray(const vector<int>& numbers) {
    if (numbers.empty()) throw invalid_argument("array must not be empty");
    return solve(numbers, 0, static_cast<int>(numbers.size()) - 1);
}

int main() {
    vector<int> numbers = {-2, 1, -3, 4, -1, 2, 1, -5, 4};
    Result result = maximumSubarray(numbers);
    cout << "Maximum subarray sum: " << result.sum << '\n';
    cout << "Range: [" << result.left << ", " << result.right << "]\n";
    cout << "Elements:";
    for (int i = result.left; i <= result.right; ++i) cout << ' ' << numbers[i];
    cout << '\n';
    return 0;
}
```

## Go 完整示例

```go
package main

import "fmt"

type Result struct {
	sum         int64
	left, right int
}

func crossingResult(numbers []int, left, mid, right int) Result {
	var sum int64
	bestLeftSum := int64(numbers[mid])
	bestLeft := mid
	for i := mid; i >= left; i-- {
		sum += int64(numbers[i])
		if sum > bestLeftSum {
			bestLeftSum = sum
			bestLeft = i
		}
	}

	sum = 0
	bestRightSum := int64(numbers[mid+1])
	bestRight := mid + 1
	for i := mid + 1; i <= right; i++ {
		sum += int64(numbers[i])
		if sum > bestRightSum {
			bestRightSum = sum
			bestRight = i
		}
	}
	return Result{sum: bestLeftSum + bestRightSum, left: bestLeft, right: bestRight}
}

func solve(numbers []int, left, right int) Result {
	if left == right {
		return Result{sum: int64(numbers[left]), left: left, right: right}
	}
	mid := left + (right-left)/2
	leftResult := solve(numbers, left, mid)
	rightResult := solve(numbers, mid+1, right)
	crossResult := crossingResult(numbers, left, mid, right)

	best := leftResult
	if rightResult.sum > best.sum {
		best = rightResult
	}
	if crossResult.sum > best.sum {
		best = crossResult
	}
	return best
}

func maximumSubarray(numbers []int) Result {
	if len(numbers) == 0 {
		panic("array must not be empty")
	}
	return solve(numbers, 0, len(numbers)-1)
}

func main() {
	numbers := []int{-2, 1, -3, 4, -1, 2, 1, -5, 4}
	result := maximumSubarray(numbers)
	fmt.Printf("Maximum subarray sum: %d\n", result.sum)
	fmt.Printf("Range: [%d, %d]\n", result.left, result.right)
	fmt.Print("Elements:")
	for i := result.left; i <= result.right; i++ {
		fmt.Printf(" %d", numbers[i])
	}
	fmt.Println()
}
```

## 易错点与要点总结

- 最大子数组必须明确是否允许空数组。本例要求非空，因此全负数组应返回最大的单个元素，而不是 0。
- 跨中点方案必须同时包含 `mid` 和 `mid+1`，不能分别调用任意左/右子数组结果后拼接。
- 中点写成 `left + (right-left)/2` 可避免大整数下的加法溢出。
- 递归边界和区间闭开规则要统一；本例全部使用闭区间 `[left,right]`。
- 分治复杂度不仅由递归层数决定，还要计算每层合并成本。
- 若多个区间和相同，应提前约定平局规则，例如最短、最左；示例保留先遇到的结果。
- 单次最大子数组优先考虑 `O(n)` Kadane；学习分治时仍应知道更优专用解法。
