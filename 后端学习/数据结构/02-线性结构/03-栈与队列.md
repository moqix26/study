# 栈与队列：普通队列、双端队列和循环队列

## 学习目标

- 理解栈的后进先出（LIFO）与队列的先进先出（FIFO）。
- 掌握普通队列、双端队列和循环队列的基本操作。
- 认识数组实现中的空间复用问题，并能正确判断循环队列的空与满。

## 问题模型与直觉

栈和队列都不是在规定元素如何存储，而是在规定**元素以什么顺序进入和离开**。同一组数据可以用数组或链表保存，但只要对外暴露的操作受约束，就形成相应的抽象数据类型。

- **栈（Stack）**只操作同一端，最后进入的元素最先离开，即 LIFO。可以想成叠盘子，只能取最上面的一只。
- **队列（Queue）**从尾部加入、从头部移除，最先进入的元素最先离开，即 FIFO。可以想成排队办理业务。
- **双端队列（Deque）**允许在头尾两端插入和删除，是比栈和普通队列更一般的结构。
- **循环队列（Circular Queue）**是普通队列的一种定长数组实现。它让下标到达数组末尾后绕回开头，从而反复复用已出队的槽位。

这几种结构的关键不在元素值，而在“哪个位置允许操作”和“空、满时怎样处理”。接口通常还要明确：空结构弹出是抛异常、返回布尔值，还是返回 `(value, ok)`；固定容量队列满时是拒绝写入还是覆盖最旧元素。

## 状态不变量

### 栈

若以数组前缀保存元素，`size` 表示元素数，则有效元素位于 `[0, size)`，栈顶是 `size - 1`。`push` 只在末尾添加，`pop` 只移除末尾，因此其余元素顺序不会改变。

### 普通队列

队列中从 `front` 到 `rear` 的逻辑顺序必须与入队先后顺序一致。链式队列通常满足：

- 空队列：`head == null` 且 `tail == null`。
- 非空队列：`head` 指向下一次出队元素，`tail` 指向最后入队元素，且 `tail.next == null`。

数组队列若维护不断增长的头下标，则有效区间是 `[head, tail)`；当头部废弃空间过多时可以压缩或使用循环队列。

### 双端队列

逻辑序列的两端都必须可在 O(1) 时间定位。常见实现是分块数组或双向链表。无论底层如何，`pushFront` 后新元素成为首元素，`pushBack` 后新元素成为尾元素；从一端删除不能打乱剩余元素顺序。

### 循环队列

本文示例维护数组 `data`、队头下标 `front`、下一次写入位置 `rear` 和当前元素数 `size`，容量为 `capacity`：

1. 始终满足 `0 <= front, rear < capacity` 和 `0 <= size <= capacity`。
2. 队头元素位于 `data[front]`。
3. 新元素写入 `data[rear]`。
4. 下标前进一步使用 `(index + 1) % capacity`。
5. `size == 0` 表示空，`size == capacity` 表示满。
6. 在这一约定下，`rear == front` 既可能是空也可能是满，必须结合 `size` 判断。

另一种设计不保存 `size`，而是永远空出一个槽位：`front == rear` 表示空，`(rear + 1) % capacity == front` 表示满。它的可用容量只有数组长度减一。两种方案都正确，但不能混用判定公式。

## 核心操作逐步过程

### 栈的入栈与出栈

数组栈的 `push(x)` 把 `x` 写到下标 `size`，再令 `size++`；`pop()` 先检查非空，保存下标 `size - 1` 的值，令 `size--` 后返回。若元素是对象引用，可清空原槽位，避免无意义地延长对象生命周期。

栈为什么能用于括号匹配：扫描左括号时入栈；遇到右括号时，它必须与“最近尚未匹配的左括号”配对，而这个左括号恰好位于栈顶。若类型不匹配、栈提前为空或扫描结束后栈非空，表达式就不合法。

### 普通队列的入队与出队

链式队列入队节点 `x`：

1. 空队列时同时令 `head = tail = x`。
2. 非空时先令 `tail.next = x`，再令 `tail = x`。

出队：

1. 保存 `head.value`。
2. 令 `head = head.next`。
3. 若新 `head == null`，说明删掉的是唯一节点，还要令 `tail = null`。

若使用动态数组，不能每次通过删除下标 0 来出队，因为后续 n-1 个元素都要左移。维护头下标即可让出队 O(1)；但长期只增加头下标可能保留大块已废弃空间，需要定期压缩。

### 双端队列的四种操作

- `pushFront(x)`：在逻辑头之前接入 `x`。
- `pushBack(x)`：在逻辑尾之后接入 `x`。
- `popFront()`：移除并返回逻辑头。
- `popBack()`：移除并返回逻辑尾。

它可模拟栈：统一在同一端 `push`/`pop`；也可模拟队列：尾部 `pushBack`、头部 `popFront`。单调队列则在尾部删除不可能成为未来答案的元素，同时从头部移除已经离开窗口的下标。

### 循环队列入队

1. 若 `size == capacity`，按接口约定报告“满”。
2. 将元素写入 `data[rear]`。
3. 令 `rear = (rear + 1) % capacity`。
4. 令 `size++`。

注意 `rear` 指向的是**下一空槽**，不是当前队尾元素。若要读取最后一个元素，其下标是 `(rear - 1 + capacity) % capacity`。

### 循环队列出队

1. 若 `size == 0`，报告“空”。
2. 保存 `data[front]`。
3. 可清空该槽位。
4. 令 `front = (front + 1) % capacity`。
5. 令 `size--`。

## 手工推演：容量为 4 的循环队列

初始 `front = 0`、`rear = 0`、`size = 0`，数组为 `[_, _, _, _]`：

| 操作 | 写入/读取下标 | 数组内容 | `front` | `rear` | `size` |
| --- | ---: | --- | ---: | ---: | ---: |
| 入队 7 | 写 0 | `[7, _, _, _]` | 0 | 1 | 1 |
| 入队 8 | 写 1 | `[7, 8, _, _]` | 0 | 2 | 2 |
| 入队 9 | 写 2 | `[7, 8, 9, _]` | 0 | 3 | 3 |
| 出队 7 | 读 0 | `[_, 8, 9, _]` | 1 | 3 | 2 |
| 入队 10 | 写 3 | `[_, 8, 9, 10]` | 1 | 0 | 3 |
| 入队 11 | 写 0 | `[11, 8, 9, 10]` | 1 | 1 | 4 |

最后虽然 `front == rear == 1`，队列却是满的；逻辑出队顺序是 `8, 9, 10, 11`。这说明物理数组的下标顺序不等于队列的逻辑顺序，必须从 `front` 开始并按容量取模。

## 为什么这些操作正确

- 栈始终只改变末端，因此最后加入的元素位于栈顶并最先被删除，满足 LIFO。
- 队列只从尾部追加、从头部删除；任何两个仍在队列中的元素，其先后关系不会颠倒，所以满足 FIFO。
- 双端队列每次只修改指定端点，未删除元素的相对顺序保持不变。
- 循环队列中的 `front` 始终指向最早尚未出队的元素，`rear` 始终指向下一空槽；取模只改变物理位置，不改变逻辑先后。`size` 精确记录两者之间当前跨越的有效元素数，因此空满判定无歧义。

## 复杂度

| 结构 | 入栈/入队 | 出栈/出队 | 查看端点 | 空间 |
| --- | --- | --- | --- | --- |
| 栈 | 均摊 O(1) | O(1) | O(1) | O(n) |
| 普通队列 | O(1) | O(1) | O(1) | O(n) |
| 双端队列 | 两端均 O(1) | 两端均 O(1) | O(1) | O(n) |
| 循环队列 | O(1) | O(1) | O(1) | O(capacity) |

### 复杂度如何推导

- 数组栈只读写末端一个槽位；链式队列只更新常数个指针；循环队列只做常数次读写和取模，因此核心操作都是 O(1)。
- 动态数组栈偶尔扩容为 O(n)，连续操作的均摊入栈仍为 O(1)。
- 双端队列能否保证两端 O(1) 取决于底层实现；双向链表和标准库分块 deque 可以，普通连续数组直接头插则不能。
- 所有结构保存 n 个元素都需要 O(n) 空间；固定循环队列预先占用 O(capacity)，即使当前为空也不会缩小。

## 适用场景

- 栈：撤销、浏览历史、括号匹配、递归模拟、DFS。
- 普通队列：BFS、生产者消费者、请求排队、层序遍历。
- 双端队列：滑动窗口、0-1 BFS、同时需要两端操作的缓存。
- 循环队列：容量固定的日志缓冲、音视频帧缓冲、嵌入式任务队列。

## 不适用场景与实现边界

- 需要按任意下标频繁访问时，不应只提供栈或队列接口；应直接评估数组等结构。
- 固定循环队列必须事先决定溢出策略。拒绝写入适合不可丢数据的任务；覆盖最旧数据适合只保留最新样本的监控缓冲。
- 多线程生产者消费者队列不仅要维护结构不变量，还要处理同步、内存可见性、阻塞和关闭语义；普通容器不能自动保证线程安全。
- `capacity == 0` 时无法取模，构造循环队列时应拒绝非正容量。
- 递归算法受调用栈容量限制；数据深度不可控时，显式栈更容易控制内存和错误处理。

## 三种语言的实现差异

- **C++**：`std::stack` 和 `std::queue` 是容器适配器，默认常以 `deque` 为底层；`std::deque` 支持两端常数时间操作，但元素不保证整体连续。对空容器调用 `top/front/pop` 前必须自行检查。
- **Java**：通常使用 `ArrayDeque` 同时充当栈、队列和双端队列，优先于旧的 `Stack` 类；它不允许存 `null`。`offer/poll/peek` 用特殊返回值表达失败，`add/remove/element` 则可能抛异常。
- **Go**：标准库没有通用泛型 deque；栈常用切片尾部，队列可用切片加头下标或自行实现环形数组。切片截去前缀后，底层数组仍可能因剩余切片而存活，长生命周期队列应定期搬移有效区间或使用循环缓冲。

## C++17 完整示例

```cpp
#include <deque>
#include <iostream>
#include <queue>
#include <stdexcept>
#include <vector>

class CircularQueue {
    std::vector<int> data;
    int head = 0;
    int tail = 0;
    int count = 0;

public:
    explicit CircularQueue(int capacity) {
        if (capacity <= 0) throw std::invalid_argument("capacity must be positive");
        data.resize(static_cast<std::size_t>(capacity));
    }

    bool enqueue(int value) {
        if (count == static_cast<int>(data.size())) return false;
        data[tail] = value;
        tail = (tail + 1) % data.size();
        ++count;
        return true;
    }

    bool dequeue(int& value) {
        if (count == 0) return false;
        value = data[head];
        head = (head + 1) % data.size();
        --count;
        return true;
    }

    int size() const { return count; }
};

int main() {
    std::vector<int> stack;
    stack.push_back(1);
    stack.push_back(2);
    stack.push_back(3);
    std::cout << "栈弹出: " << stack.back() << "\n";
    stack.pop_back();

    std::queue<int> queue;
    queue.push(10);
    queue.push(20);
    std::cout << "队列出队: " << queue.front() << "\n";
    queue.pop();

    std::deque<int> deque;
    deque.push_back(5);
    deque.push_front(4);
    deque.push_back(6);
    std::cout << "双端队列: [" << deque.front() << ", "
              << deque[1] << ", " << deque.back() << "]\n";

    CircularQueue circular(3);
    circular.enqueue(7);
    circular.enqueue(8);
    circular.enqueue(9);
    int removed = 0;
    circular.dequeue(removed);
    circular.enqueue(10); // 复用刚释放的数组位置
    std::cout << "循环队列出队顺序: [";
    std::cout << removed;
    while (circular.dequeue(removed)) std::cout << ", " << removed;
    std::cout << "]\n";
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.ArrayDeque;
import java.util.Deque;
import java.util.NoSuchElementException;

public class Main {
    private static class CircularQueue {
        private final int[] data;
        private int head;
        private int tail;
        private int size;

        CircularQueue(int capacity) {
            if (capacity <= 0) throw new IllegalArgumentException("capacity must be positive");
            data = new int[capacity];
        }

        boolean enqueue(int value) {
            if (size == data.length) return false;
            data[tail] = value;
            tail = (tail + 1) % data.length;
            size++;
            return true;
        }

        int dequeue() {
            if (size == 0) throw new NoSuchElementException("queue is empty");
            int value = data[head];
            head = (head + 1) % data.length;
            size--;
            return value;
        }

        boolean isEmpty() { return size == 0; }
    }

    public static void main(String[] args) {
        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(1);
        stack.push(2);
        stack.push(3);
        System.out.println("栈弹出: " + stack.pop());

        Deque<Integer> queue = new ArrayDeque<>();
        queue.offerLast(10);
        queue.offerLast(20);
        System.out.println("队列出队: " + queue.removeFirst());

        Deque<Integer> deque = new ArrayDeque<>();
        deque.offerLast(5);
        deque.offerFirst(4);
        deque.offerLast(6);
        System.out.println("双端队列: " + deque);

        CircularQueue circular = new CircularQueue(3);
        circular.enqueue(7);
        circular.enqueue(8);
        circular.enqueue(9);
        int removed = circular.dequeue();
        circular.enqueue(10);
        StringBuilder order = new StringBuilder("[").append(removed);
        while (!circular.isEmpty()) order.append(", ").append(circular.dequeue());
        order.append("]");
        System.out.println("循环队列出队顺序: " + order);
    }
}
```

## Go 完整示例

```go
package main

import (
	"container/list"
	"fmt"
)

type circularQueue struct {
	data       []int
	head, tail int
	size       int
}

func newCircularQueue(capacity int) *circularQueue {
	if capacity <= 0 {
		panic("capacity must be positive")
	}
	return &circularQueue{data: make([]int, capacity)}
}

func (q *circularQueue) enqueue(value int) bool {
	if q.size == len(q.data) {
		return false
	}
	q.data[q.tail] = value
	q.tail = (q.tail + 1) % len(q.data)
	q.size++
	return true
}

func (q *circularQueue) dequeue() (int, bool) {
	if q.size == 0 {
		return 0, false
	}
	value := q.data[q.head]
	q.head = (q.head + 1) % len(q.data)
	q.size--
	return value, true
}

func main() {
	stack := make([]int, 0)
	stack = append(stack, 1, 2, 3)
	top := stack[len(stack)-1]
	stack = stack[:len(stack)-1]
	fmt.Println("栈弹出:", top)

	queue := []int{10, 20}
	front := queue[0]
	queue = queue[1:]
	fmt.Println("队列出队:", front)

	deque := list.New()
	deque.PushBack(5)
	deque.PushFront(4)
	deque.PushBack(6)
	fmt.Printf("双端队列: [%d, %d, %d]\n",
		deque.Front().Value, deque.Front().Next().Value, deque.Back().Value)

	circular := newCircularQueue(3)
	circular.enqueue(7)
	circular.enqueue(8)
	circular.enqueue(9)
	removed, _ := circular.dequeue()
	circular.enqueue(10)
	order := []int{removed}
	for {
		value, ok := circular.dequeue()
		if !ok {
			break
		}
		order = append(order, value)
	}
	fmt.Println("循环队列出队顺序:", order)
}
```

## 易错点

- 对空栈或空队列执行弹出、查看操作前应检查状态，或明确异常/布尔返回约定。
- Java 的 `ArrayDeque` 不允许 `null`；通常优先于旧的 `Stack` 类。
- Go 用切片做长期队列时，简单执行 `queue = queue[1:]` 可能让底层大数组迟迟不能回收；可维护头下标并定期压缩。
- 循环队列中头尾下标都必须按容量取模，且必须明确如何区分空与满。
- 固定容量入队失败时，不能悄悄覆盖尚未出队的元素，除非设计的就是覆盖式环形缓冲区。
- C++ `std::deque` 支持随机访问，但不要把它误认为连续内存数组。

## 练习建议

1. 用两个栈实现队列，再用两个队列实现栈，分析均摊复杂度。
2. 用栈完成有效括号判断和逆波兰表达式求值。
3. 用单调双端队列求长度为 `k` 的滑动窗口最大值。
4. 为循环队列增加 `front`、`rear`、`isFull`，并实现覆盖式写入策略。
5. 用普通队列实现二叉树层序遍历，用栈实现非递归 DFS。
