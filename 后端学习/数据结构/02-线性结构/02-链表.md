# 链表：单链表、双链表与循环链表

## 学习目标

- 理解链表通过指针或引用连接离散节点的存储方式。
- 掌握单链表、双链表和循环链表的插入、删除与遍历。
- 能依据访问方向、操作位置和内存开销选择合适链表。

## 问题模型与直觉

链表解决的是“元素不必连续存放，但仍要保持一个确定的逻辑顺序”的问题。每个节点除了业务数据，还保存通向相邻节点的链接。访问者从 `head` 出发，沿链接依次到达后继节点。

可以把数组想成编号连续的一排房间，把链表想成散落各处、每个房间里都写着“下一间在哪里”的寻路卡。链表不需要整体搬家就能插入节点，但也无法根据第几个节点直接算出地址：要找第 `i` 个节点，必须沿链接走 `i` 步。

三种常见链表的差别在链接方向和尾部约定：

- **单链表**：节点保存 `next`，只能沿一个方向前进。尾节点满足 `tail.next == null`。
- **双链表**：节点同时保存 `prev` 与 `next`，可以双向遍历。头节点的 `prev` 和尾节点的 `next` 为空。
- **循环链表**：尾节点不指向空，而是重新连接到头节点。若维护尾指针，则非空表始终满足 `tail.next == head`。

链表所谓“插入、删除是 O(1)”有一个重要前提：**操作位置已经定位，并且拥有修改链接所需的节点引用**。如果先按下标或按值查找位置，定位本身仍需 O(n)。

## 结构不变量

### 单链表

1. 空表满足 `head == null`；若维护尾指针，还应满足 `tail == null`。
2. 非空表从 `head` 沿 `next` 最终恰好到达 `tail`，且 `tail.next == null`。
3. 除非特意构造环，否则任一节点不会被沿 `next` 重复访问。

### 双链表

1. 空表的 `head` 与 `tail` 同时为空。
2. 非空表满足 `head.prev == null`、`tail.next == null`。
3. 对任意相邻节点 `a` 和 `b`，若 `a.next == b`，则必须同时有 `b.prev == a`。

### 循环单链表

1. 空表通常用 `tail == null` 表示；头节点可由 `tail.next` 得到。
2. 非空表必须满足 `tail.next == head`。
3. 从头节点出发，走有限步会回到头节点；正常遍历不能等待 `null` 出现。

引入不存业务数据的哨兵节点，可以让真实头节点始终位于 `sentinel.next`，从而统一空表、头插和头删。本文示例直接维护头尾指针，以便观察每次更新。

## 单链表操作与指针更新顺序

### 头插

把新节点 `x` 插到原头节点之前：

1. 先令 `x.next = head`，让新节点记住原链表入口。
2. 再令 `head = x`。
3. 若插入前为空表，还要令 `tail = x`。

不能先覆盖 `head` 再读取原头引用，否则容易丢失整条旧链。

### 尾插

若维护 `tail`：

1. 空表时同时令 `head = tail = x`。
2. 非空表先令 `tail.next = x`，再令 `tail = x`。

顺序的含义是先把新节点接入可达链，再移动尾指针。若不维护 `tail`，必须从 `head` 遍历到末尾，尾插变为 O(n)。

### 在已知前驱 `prev` 后插入

要把 `x` 插入 `prev` 与 `nextNode = prev.next` 之间：

1. `x.next = prev.next`。
2. `prev.next = x`。
3. 如果 `prev` 原来是尾节点，更新 `tail = x`。

第一步必须先保存后半段入口；若先写 `prev.next = x`，再写 `x.next = prev.next`，`x.next` 会指向自己，旧后继链丢失。

### 删除已知前驱之后的节点

设待删节点为 `victim = prev.next`：

1. 确认 `victim` 存在。
2. 令 `prev.next = victim.next`，从主链绕过 `victim`。
3. 若 `victim == tail`，令 `tail = prev`。
4. 可选地令 `victim.next = null`，断开旧链接，便于调试与垃圾回收。

删除头节点没有前驱，需要单独令 `head = head.next`；删完唯一节点时还要把 `tail` 置空。

### 反转单链表

反转时维护三个角色：已经反转好的前缀 `prev`、当前节点 `current`、尚未处理的后继 `next`。每轮严格按以下顺序：

1. `next = current.next`，先保存后半链入口。
2. `current.next = prev`，把当前箭头反向。
3. `prev = current`，扩大已反转前缀。
4. `current = next`，继续处理原后继。

若未先保存 `next` 就改写 `current.next`，剩余链表会失去入口。

## 双链表操作与指针更新顺序

设要把新节点 `x` 插到相邻节点 `a` 与 `b` 之间，插入前有 `a.next == b` 且 `b.prev == a`：

1. 先让新节点保存两端：`x.prev = a`，`x.next = b`。
2. 再让左邻居指向新节点：`a.next = x`。
3. 最后让右邻居指向新节点：`b.prev = x`。

若使用首尾哨兵，`a` 和 `b` 始终存在；没有哨兵时，头部或尾部的一侧可能为空，需要分别更新 `head` 或 `tail`。

删除已知节点 `x` 时，先保存 `a = x.prev` 与 `b = x.next`，然后：

1. 若 `a` 存在，令 `a.next = b`；否则 `x` 是头节点，应令 `head = b`。
2. 若 `b` 存在，令 `b.prev = a`；否则 `x` 是尾节点，应令 `tail = a`。
3. 可选地清空 `x.prev` 与 `x.next`。

删除完成后，`a` 和 `b` 若都存在，两个方向必须重新互相对应。只改 `next` 而忘记 `prev`，正向遍历可能看似正常，反向遍历却会回到已删除节点。

## 循环链表操作与终止条件

维护尾指针的循环单链表可用 `head = tail.next` 得到头节点：

- **空表插入第一个节点**：令 `tail = x`，再令 `x.next = x`。唯一节点既是头也是尾，并指向自己。
- **头插**：先令 `x.next = tail.next`，再令 `tail.next = x`；`tail` 不变，新节点成为头。
- **尾插**：先令 `x.next = tail.next` 保存头，再令 `tail.next = x` 接入节点，最后令 `tail = x`。
- **删除头节点**：若只有一个节点，令 `tail = null`；否则令 `tail.next = head.next`。

循环遍历有两种可靠写法：已知节点数时恰好走 `size` 步；未知节点数时从 `head` 出发，并在下一次回到 `head` 时停止。使用后者时，空表必须预先处理，通常采用“先访问一次，再判断是否回到起点”的结构。

## 手工推演

### 单链表在中间插入

原链：`10 -> 30 -> 40 -> null`，要在节点 10 后插入 20。

| 步骤 | 更新 | 链接状态 |
| --- | --- | --- |
| 初始 | `prev = 10`，`prev.next = 30` | `10 -> 30 -> 40` |
| 1 | `20.next = prev.next` | `20 -> 30 -> 40`，原链仍完整 |
| 2 | `prev.next = 20` | `10 -> 20 -> 30 -> 40` |

### 双链表删除中间节点

原链：`4 <-> 5 <-> 6`，删除节点 5。先得到 `a = 4`、`b = 6`，再执行 `4.next = 6` 与 `6.prev = 4`，结果为 `4 <-> 6`。节点 5 的两个旧链接随后清空，不再属于链表。

### 循环链表尾插

原环为 `7 -> 8 -> 9 -> 7`，`tail = 9`，插入 10：

1. `10.next = tail.next`，所以 `10.next = 7`。
2. `tail.next = 10`，原尾 9 接到新节点。
3. `tail = 10`，新尾仍通过 `10.next` 回到头 7。

最终是 `7 -> 8 -> 9 -> 10 -> 7`，闭环没有在任何一步丢失入口。

## 为什么这些操作正确

链表操作的正确性可以归结为两点：**可达节点不丢失**和**结构不变量恢复**。

- 插入前先让新节点保存原后继，因此旧后半链始终可达；随后前驱改指向新节点，插入完成。
- 删除让前驱绕过待删节点并连接其后继，除待删节点外的相对顺序不变。
- 双链表对同一对相邻节点同时恢复 `next` 和 `prev`，因此正反两个方向一致。
- 循环链表每次插入都保留 `tail.next` 所指的头节点，最终重新满足 `tail.next == head`。
- 反转算法每轮开始时，`prev` 指向已经正确反转的前缀，`current` 指向未处理后缀的首节点；一轮后前缀增加一个节点，后缀减少一个节点，直到后缀为空。

## 复杂度

| 操作 | 单链表 | 双链表 | 循环链表 | 额外空间 |
| --- | --- | --- | --- | --- |
| 按下标访问/查找 | O(n) | O(n) | O(n) | O(1) |
| 头部插入/删除 | O(1) | O(1) | O(1)，需维护尾指针 | O(1) |
| 已知尾指针时尾插 | O(1) | O(1) | O(1) | O(1) |
| 已知节点时删除 | 需前驱，通常 O(n) | O(1) | 需前驱，通常 O(n) | O(1) |
| 存储 n 个元素 | O(n) | O(n) | O(n) | 每节点含链接域 |

### 复杂度如何推导

- 沿链接每前进一步只能越过一个节点，因此按下标或按值查找最坏访问 n 个节点，为 O(n)。
- 已知前驱或已知双链表目标节点后，插入删除只改常数个引用，为 O(1)。
- 维护尾指针可使尾插 O(1)，但单链表即使有尾指针，也不能 O(1) 删除尾节点，因为仍需找到它的前驱。
- 每个节点除数据外还存 1～2 个链接，因此总空间是 O(n)，常数开销通常高于连续数组。

## 适用场景

- 单链表：只需单向扫描、频繁头插或需要低链接开销。
- 双链表：需要双向遍历、已知节点后频繁插入删除，如 LRU 缓存的访问顺序链。
- 循环链表：循环调度、轮询队列、约瑟夫环、环形播放列表。
- 若核心需求是频繁随机访问，数组通常更合适；链表的节点分散，也不如数组利于 CPU 缓存。

## 不适用场景与实现边界

- 高频随机访问或数值计算通常不应使用链表；逐节点跳转会产生 O(n) 定位成本和较差的缓存局部性。
- 节点很小时，指针、对象头和分配器元数据可能比业务数据本身占用更多空间。
- “已知值”不等于“已知节点”；按值删除仍要先搜索。
- 不能在遍历时随意释放当前节点后再读取它的 `next`，应先保存后继。
- 如果链表允许外部长期持有节点引用，要明确删除后的引用是否失效，以及是否允许同一节点被重复插入。

## 三种语言的实现差异

- **C++**：示例使用裸指针以展示链接变化，因此必须显式释放节点，并遵循所有权约定。工程代码可用容器、智能指针或专用节点池，但循环结构不能直接依赖简单的引用计数打破环。
- **Java**：节点是对象引用，垃圾回收负责释放不可达对象；但意外形成的、仍可从根对象到达的环不会被回收。泛型节点能存任意引用类型，基本类型会装箱。
- **Go**：节点通常用指针连接，垃圾回收同样不替代结构正确性。返回局部节点地址是安全的，编译器会做逃逸分析；若大量创建小节点，分配与 GC 成本可能明显高于切片。

## C++17 完整示例

```cpp
#include <iostream>

class SinglyList {
    struct Node {
        int value;
        Node* next;
        explicit Node(int v) : value(v), next(nullptr) {}
    };
    Node* head = nullptr;
    Node* tail = nullptr;

public:
    ~SinglyList() {
        while (head != nullptr) {
            Node* next = head->next;
            delete head;
            head = next;
        }
    }

    void pushBack(int value) {
        Node* node = new Node(value);
        if (tail == nullptr) head = tail = node;
        else {
            tail->next = node;
            tail = node;
        }
    }

    bool remove(int value) {
        Node* previous = nullptr;
        Node* current = head;
        while (current != nullptr && current->value != value) {
            previous = current;
            current = current->next;
        }
        if (current == nullptr) return false;
        if (previous == nullptr) head = current->next;
        else previous->next = current->next;
        if (tail == current) tail = previous;
        delete current;
        return true;
    }

    void print() const {
        std::cout << "[";
        for (Node* node = head; node != nullptr; node = node->next) {
            if (node != head) std::cout << ", ";
            std::cout << node->value;
        }
        std::cout << "]\n";
    }
};

class DoublyList {
    struct Node {
        int value;
        Node* prev;
        Node* next;
        explicit Node(int v) : value(v), prev(nullptr), next(nullptr) {}
    };
    Node* head = nullptr;
    Node* tail = nullptr;

public:
    ~DoublyList() {
        while (head != nullptr) {
            Node* next = head->next;
            delete head;
            head = next;
        }
    }

    void pushBack(int value) {
        Node* node = new Node(value);
        node->prev = tail;
        if (tail == nullptr) head = node;
        else tail->next = node;
        tail = node;
    }

    void printForward() const {
        std::cout << "[";
        for (Node* node = head; node != nullptr; node = node->next) {
            if (node != head) std::cout << ", ";
            std::cout << node->value;
        }
        std::cout << "]\n";
    }

    void printBackward() const {
        std::cout << "[";
        for (Node* node = tail; node != nullptr; node = node->prev) {
            if (node != tail) std::cout << ", ";
            std::cout << node->value;
        }
        std::cout << "]\n";
    }
};

class CircularList {
    struct Node {
        int value;
        Node* next;
        explicit Node(int v) : value(v), next(nullptr) {}
    };
    Node* tail = nullptr;

public:
    ~CircularList() {
        if (tail == nullptr) return;
        Node* head = tail->next;
        tail->next = nullptr; // 先断环，再按普通链表释放
        while (head != nullptr) {
            Node* next = head->next;
            delete head;
            head = next;
        }
    }

    void pushBack(int value) {
        Node* node = new Node(value);
        if (tail == nullptr) {
            tail = node;
            node->next = node;
        } else {
            node->next = tail->next;
            tail->next = node;
            tail = node;
        }
    }

    void printSteps(int steps) const {
        std::cout << "[";
        if (tail != nullptr) {
            Node* node = tail->next;
            for (int i = 0; i < steps; ++i) {
                if (i > 0) std::cout << ", ";
                std::cout << node->value;
                node = node->next;
            }
        }
        std::cout << "]\n";
    }
};

int main() {
    SinglyList singly;
    for (int value : {1, 2, 3}) singly.pushBack(value);
    std::cout << "单链表: ";
    singly.print();
    singly.remove(2);
    std::cout << "删除 2: ";
    singly.print();

    DoublyList doubly;
    for (int value : {4, 5, 6}) doubly.pushBack(value);
    std::cout << "双链表正向: ";
    doubly.printForward();
    std::cout << "双链表反向: ";
    doubly.printBackward();

    CircularList circular;
    for (int value : {7, 8, 9}) circular.pushBack(value);
    std::cout << "循环链表两轮: ";
    circular.printSteps(6);
    return 0;
}
```

## Go 完整示例

```go
package main

import "fmt"

type singlyNode struct {
	value int
	next  *singlyNode
}

type singlyList struct {
	head *singlyNode
	tail *singlyNode
}

func (list *singlyList) pushBack(value int) {
	node := &singlyNode{value: value}
	if list.tail == nil {
		list.head, list.tail = node, node
	} else {
		list.tail.next = node
		list.tail = node
	}
}

func (list *singlyList) remove(value int) bool {
	var previous *singlyNode
	current := list.head
	for current != nil && current.value != value {
		previous, current = current, current.next
	}
	if current == nil {
		return false
	}
	if previous == nil {
		list.head = current.next
	} else {
		previous.next = current.next
	}
	if list.tail == current {
		list.tail = previous
	}
	return true
}

func (list *singlyList) print() {
	fmt.Print("[")
	for node := list.head; node != nil; node = node.next {
		if node != list.head {
			fmt.Print(", ")
		}
		fmt.Print(node.value)
	}
	fmt.Println("]")
}

type doublyNode struct {
	value      int
	prev, next *doublyNode
}

type doublyList struct {
	head *doublyNode
	tail *doublyNode
}

func (list *doublyList) pushBack(value int) {
	node := &doublyNode{value: value, prev: list.tail}
	if list.tail == nil {
		list.head = node
	} else {
		list.tail.next = node
	}
	list.tail = node
}

func (list *doublyList) printForward() {
	fmt.Print("[")
	for node := list.head; node != nil; node = node.next {
		if node != list.head {
			fmt.Print(", ")
		}
		fmt.Print(node.value)
	}
	fmt.Println("]")
}

func (list *doublyList) printBackward() {
	fmt.Print("[")
	for node := list.tail; node != nil; node = node.prev {
		if node != list.tail {
			fmt.Print(", ")
		}
		fmt.Print(node.value)
	}
	fmt.Println("]")
}

type circularNode struct {
	value int
	next  *circularNode
}

type circularList struct {
	tail *circularNode
}

func (list *circularList) pushBack(value int) {
	node := &circularNode{value: value}
	if list.tail == nil {
		list.tail = node
		node.next = node
	} else {
		node.next = list.tail.next
		list.tail.next = node
		list.tail = node
	}
}

func (list *circularList) printSteps(steps int) {
	fmt.Print("[")
	if list.tail != nil {
		node := list.tail.next
		for i := 0; i < steps; i++ {
			if i > 0 {
				fmt.Print(", ")
			}
			fmt.Print(node.value)
			node = node.next
		}
	}
	fmt.Println("]")
}

func main() {
	singly := &singlyList{}
	for _, value := range []int{1, 2, 3} {
		singly.pushBack(value)
	}
	fmt.Print("单链表: ")
	singly.print()
	singly.remove(2)
	fmt.Print("删除 2: ")
	singly.print()

	doubly := &doublyList{}
	for _, value := range []int{4, 5, 6} {
		doubly.pushBack(value)
	}
	fmt.Print("双链表正向: ")
	doubly.printForward()
	fmt.Print("双链表反向: ")
	doubly.printBackward()

	circular := &circularList{}
	for _, value := range []int{7, 8, 9} {
		circular.pushBack(value)
	}
	fmt.Print("循环链表两轮: ")
	circular.printSteps(6)
}
```

删除 2: [1, 3]
双链表正向: [4, 5, 6]
双链表反向: [6, 5, 4]

## 易错点

- 删除头节点、尾节点或唯一节点时，要同步维护 `head` 与 `tail`。
- 双链表修改连接时要成对更新 `prev` 和 `next`，漏掉一侧会破坏结构。
- 循环链表不能用 `node != null` 作为正常遍历的终止条件，否则会无限循环。
- C++ 手动管理节点时要防止内存泄漏、重复释放和悬空指针；示例析构函数会释放节点。
- 反转单链表时，必须先保存下一个节点，再改写当前节点的 `next`。
- Go/Java 有垃圾回收，但错误链接仍会造成逻辑环和对象长期不可回收。

## 练习建议

1. 迭代和递归两种方式反转单链表。
2. 用快慢指针判断链表是否有环，并找到环入口。
3. 合并两个有序单链表，再扩展到合并 k 个有序链表。
4. 实现带哨兵节点的双链表，并支持在指定节点前后 O(1) 插入。
5. 用循环链表模拟约瑟夫环，输出出列顺序。
