# 数组

## 学习目标

- 理解数组“连续存储、按下标访问”的本质。
- 掌握遍历、查找、插入和删除的实现方式与复杂度。
- 区分固定长度数组与 C++ `vector`、Java `ArrayList`、Go 切片等动态数组。

## 问题模型与直觉

数组解决的是“把一批同类型元素按顺序编号，并快速访问第 `i` 个元素”的问题。它可以想成一排连续的储物格：每格大小相同，第一格地址已知，因此无需从头数到目标位置，只要根据编号计算偏移量即可。

设数组首地址为 `base`，每个元素占 `elementSize` 字节，则下标 `i` 对应的地址为：

`address(i) = base + i * elementSize`

这个公式解释了数组最重要的性质：**按下标访问与数组长度无关**。访问第 2 个元素和第 200 万个元素都只需一次地址计算，因此都是 O(1)。代价是元素必须占据一段连续空间；在中间插入或删除时，为保持下标与顺序的对应关系，后续元素必须整体搬移。

数组还要区分两个概念：

- **长度 `length`**：当前真正存有多少个有效元素，下标范围是 `[0, length)`。
- **容量 `capacity`**：当前连续空间最多能容纳多少个元素，必须满足 `0 <= length <= capacity`。

固定数组创建后容量不变；动态数组会在容量不足时申请一块更大的连续空间，把旧元素复制或移动过去，再释放旧空间。C++ `vector`、Java `ArrayList` 和 Go 切片背后的可增长数组都采用类似思想。

## 存储不变量

只要数组处于合法状态，就应始终满足以下条件：

1. 有效元素恰好位于下标 `[0, length)`，中间没有“空洞”。
2. `length` 不超过 `capacity`，访问、修改和删除只能使用合法下标。
3. 元素的逻辑顺序与下标顺序一致；插入和删除后仍必须保持这一点。
4. 动态数组扩容后元素值与顺序不变，但底层首地址可能改变。

这些不变量决定了插入和删除的搬移方向。插入时目标位置即将被占用，必须从右向左搬移；删除后目标位置形成空洞，必须从左向右填补。

## 核心操作逐步过程

### 按下标访问与修改

1. 检查 `0 <= index < length`。
2. 用首地址和元素大小计算目标地址。
3. 读取或覆盖该位置的值。

整个过程没有随 `n` 增长的循环，所以是 O(1)。如果省略边界检查，C++ 原生数组可能产生未定义行为；Java 和 Go 会在运行时报告越界。

### 顺序查找

无序数组不能依据大小排除某一段，只能从下标 0 开始逐个比较。找到目标立即返回；若扫描完 `length` 个元素仍未命中，则目标不存在。最坏情况比较 `n` 次，因此为 O(n)。有序数组则可以使用二分查找，将复杂度降到 O(log n)。

### 在中间插入

假设要在下标 `index` 插入 `value`：

1. 检查 `0 <= index <= length`；尾插时允许 `index == length`。
2. 若 `length == capacity`，先扩容；固定数组则应明确返回失败。
3. 从原最后一个元素 `length - 1` 开始，到 `index` 为止，依次把元素复制到右侧一格。
4. 把 `value` 写入 `index`。
5. 最后令 `length++`。

必须从后向前移动。若从 `index` 向右移动，`a[index]` 会先覆盖 `a[index+1]`，使原来的 `a[index+1]` 永久丢失。

### 删除指定下标

1. 检查 `0 <= index < length`。
2. 保存被删值（若接口需要返回它）。
3. 从 `index + 1` 开始，把每个元素复制到左侧一格。
4. 令 `length--`；必要时清理原尾部槽位，避免对象引用滞留。

删除必须从前向后移动，因为左侧是已经腾出的槽位，不会覆盖尚未读取的元素。

## 手工推演：插入与删除

初始有效数组为 `[10, 20, 30, 40]`，`length = 4`，容量为 6。现在在下标 2 插入 25：

| 步骤 | 操作 | 有效区域附近的内容 |
| --- | --- | --- |
| 1 | 将下标 3 的 40 移到下标 4 | `[10, 20, 30, 40, 40, _]` |
| 2 | 将下标 2 的 30 移到下标 3 | `[10, 20, 30, 30, 40, _]` |
| 3 | 在下标 2 写入 25 | `[10, 20, 25, 30, 40, _]` |
| 4 | `length` 从 4 变为 5 | 有效数组为 `[10, 20, 25, 30, 40]` |

接着删除下标 1 的 20：

| 搬移 | 来源下标 | 目标下标 | 搬移后的前缀 |
| --- | ---: | ---: | --- |
| 25 左移 | 2 | 1 | `[10, 25, 25, 30, 40]` |
| 30 左移 | 3 | 2 | `[10, 25, 30, 30, 40]` |
| 40 左移 | 4 | 3 | `[10, 25, 30, 40, 40]` |

最后把 `length` 减为 4，逻辑数组是 `[10, 25, 30, 40]`。末尾残留的旧 40 已不属于有效区间；若存的是对象引用，通常应把该槽位清空。

## 为什么这些操作正确

- **访问正确性**：等长元素连续存储，地址公式唯一确定下标 `i` 的槽位。
- **插入正确性**：从后向前移动时，每次读取的源元素尚未被覆盖；移动结束后，原区间 `[index, length)` 整体右移一位且相对顺序不变，空出的 `index` 正好放新值。
- **删除正确性**：从前向后移动后，原区间 `(index, length)` 整体左移一位且相对顺序不变，被删元素不再位于有效区间。
- **动态扩容正确性**：只要把旧有效区间按原下标复制到新空间，新数组在值和顺序上与扩容前等价。

## 复杂度

| 操作 | 时间复杂度 | 额外空间 | 说明 |
| --- | --- | --- | --- |
| 按下标访问/修改 | O(1) | O(1) | 依赖合法下标 |
| 遍历 | O(n) | O(1) | 逐个访问 |
| 无序查找 | O(n) | O(1) | 最坏扫描全部元素 |
| 有序数组二分查找 | O(log n) | O(1) | 前提是保持有序 |
| 中间插入/删除 | O(n) | O(1) | 需要搬移元素 |
| 动态数组尾插 | 均摊 O(1) | 通常 O(n) 总存储 | 扩容时单次 O(n)，普通尾插 O(1) |

### 复杂度如何推导

- 访问只做地址计算，是 O(1)。
- 遍历和无序查找至多访问 `n` 个元素，是 O(n)。
- 在下标 `i` 插入要搬移 `n-i` 个元素；删除要搬移 `n-i-1` 个元素，因此中间位置最坏为 O(n)，尾部操作为 O(1)。
- 若动态数组容量按常数倍增长，例如 `1, 2, 4, 8, ...`，连续追加 `n` 个元素时，所有扩容复制量小于 `1+2+4+...+n < 2n`。把总 O(n) 搬移分摊到 n 次追加，每次均摊 O(1)。这不代表每一次尾插都是 O(1)：触发扩容的那一次仍是 O(n)。

## 适用场景

- 需要频繁按下标随机访问，如查表、矩阵、动态规划状态表。
- 元素数量相对稳定，或追加操作远多于中间插入、删除。
- 重视缓存局部性和遍历性能。

若业务频繁在中间插入、删除且不依赖随机访问，可考虑链表；若容量有上限并需要首尾循环复用，可考虑循环队列。

## 不适用场景与实现边界

- 元素数量巨大且无法找到足够大的连续内存时，分块结构或链式结构更合适。
- 频繁在头部或中间插入删除时，搬移成本会成为瓶颈。
- 需要按键而非按整数下标访问时，应考虑哈希表或搜索树。
- 二维数组在内存中通常仍是一维展开；必须明确行优先还是列优先，才能正确计算下标。
- 计算 `base + i * elementSize` 时，底层实现需要防止整数溢出；业务代码也不能用未经验证的外部输入作为下标。

## 三种语言的实现差异

- **C++**：原生数组长度固定且通常不做边界检查；`std::array` 封装固定长度，`std::vector` 管理动态容量。`vector` 扩容会使指针、引用和迭代器失效。
- **Java**：数组长度固定并带运行时越界检查；`ArrayList<Integer>` 可增长，但基本类型 `int` 会装箱为 `Integer`，存在额外对象或拆装箱成本。删除对象引用后，容器实现应清空尾槽。
- **Go**：数组类型包含长度，`[4]int` 与 `[5]int` 是不同类型；切片是描述底层数组一段区间的视图，包含指针、长度和容量。`append` 可能复用原数组，也可能分配新数组，因此追加后应使用返回的新切片。

## C++17 完整示例

下面用固定容量数组和逻辑长度演示插入、删除与查找。

```cpp
#include <iostream>
#include <stdexcept>

constexpr int CAPACITY = 8;

void printArray(const int data[], int length) {
    std::cout << "[";
    for (int i = 0; i < length; ++i) {
        if (i > 0) std::cout << ", ";
        std::cout << data[i];
    }
    std::cout << "]\n";
}

void insertAt(int data[], int& length, int index, int value) {
    if (length == CAPACITY) throw std::overflow_error("array is full");
    if (index < 0 || index > length) throw std::out_of_range("invalid index");
    for (int i = length; i > index; --i) data[i] = data[i - 1];
    data[index] = value;
    ++length;
}

void eraseAt(int data[], int& length, int index) {
    if (index < 0 || index >= length) throw std::out_of_range("invalid index");
    for (int i = index; i + 1 < length; ++i) data[i] = data[i + 1];
    --length;
}

int find(const int data[], int length, int target) {
    for (int i = 0; i < length; ++i) {
        if (data[i] == target) return i;
    }
    return -1;
}

int main() {
    int data[CAPACITY] = {10, 20, 30, 40};
    int length = 4;

    std::cout << "原数组: ";
    printArray(data, length);
    insertAt(data, length, 2, 25);
    std::cout << "插入 25: ";
    printArray(data, length);
    eraseAt(data, length, 1);
    std::cout << "删除下标 1: ";
    printArray(data, length);
    std::cout << "30 的下标: " << find(data, length, 30) << "\n";
    return 0;
}
```

## Java 17 完整示例

```java
public class Main {
    private static final int CAPACITY = 8;

    private static void printArray(int[] data, int length) {
        System.out.print("[");
        for (int i = 0; i < length; i++) {
            if (i > 0) System.out.print(", ");
            System.out.print(data[i]);
        }
        System.out.println("]");
    }

    private static int insertAt(int[] data, int length, int index, int value) {
        if (length == data.length) throw new IllegalStateException("array is full");
        if (index < 0 || index > length) throw new IndexOutOfBoundsException();
        for (int i = length; i > index; i--) data[i] = data[i - 1];
        data[index] = value;
        return length + 1;
    }

    private static int eraseAt(int[] data, int length, int index) {
        if (index < 0 || index >= length) throw new IndexOutOfBoundsException();
        for (int i = index; i + 1 < length; i++) data[i] = data[i + 1];
        return length - 1;
    }

    private static int find(int[] data, int length, int target) {
        for (int i = 0; i < length; i++) {
            if (data[i] == target) return i;
        }
        return -1;
    }

    public static void main(String[] args) {
        int[] data = new int[CAPACITY];
        int[] initial = {10, 20, 30, 40};
        System.arraycopy(initial, 0, data, 0, initial.length);
        int length = initial.length;

        System.out.print("原数组: ");
        printArray(data, length);
        length = insertAt(data, length, 2, 25);
        System.out.print("插入 25: ");
        printArray(data, length);
        length = eraseAt(data, length, 1);
        System.out.print("删除下标 1: ");
        printArray(data, length);
        System.out.println("30 的下标: " + find(data, length, 30));
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

const capacity = 8

func printArray(data *[capacity]int, length int) {
	fmt.Print("[")
	for i := 0; i < length; i++ {
		if i > 0 {
			fmt.Print(", ")
		}
		fmt.Print(data[i])
	}
	fmt.Println("]")
}

func insertAt(data *[capacity]int, length, index, value int) int {
	if length == len(data) {
		panic("array is full")
	}
	if index < 0 || index > length {
		panic("invalid index")
	}
	for i := length; i > index; i-- {
		data[i] = data[i-1]
	}
	data[index] = value
	return length + 1
}

func eraseAt(data *[capacity]int, length, index int) int {
	if index < 0 || index >= length {
		panic("invalid index")
	}
	for i := index; i+1 < length; i++ {
		data[i] = data[i+1]
	}
	return length - 1
}

func find(data *[capacity]int, length, target int) int {
	for i := 0; i < length; i++ {
		if data[i] == target {
			return i
		}
	}
	return -1
}

func main() {
	data := [capacity]int{10, 20, 30, 40}
	length := 4
	fmt.Print("原数组: ")
	printArray(&data, length)
	length = insertAt(&data, length, 2, 25)
	fmt.Print("插入 25: ")
	printArray(&data, length)
	length = eraseAt(&data, length, 1)
	fmt.Print("删除下标 1: ")
	printArray(&data, length)
	fmt.Println("30 的下标:", find(&data, length, 30))
}
```

## 易错点

- 下标范围是 `[0, length)`；插入位置允许等于 `length`，删除位置不允许。
- 固定数组的“容量”和当前有效元素的“长度”不是同一个概念。
- 从中间插入要从后向前搬移，否则尚未搬走的元素会被覆盖；删除则从前向后搬移。
- 动态数组扩容后，旧地址、迭代器或指针可能失效。
- 二分查找不能直接用于无序数组。

## 练习建议

1. 实现支持自动扩容的整数动态数组，提供 `get`、`set`、`append`、`insert`、`remove`。
2. 原地删除有序数组中的重复元素，并返回新长度。
3. 用二维数组完成矩阵转置与顺时针旋转 90°。
