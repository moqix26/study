# 并查集（Disjoint Set Union）

## 学习目标

- 理解并查集用“森林”维护互不相交集合的代表元。
- 掌握 `find`、`union`、路径压缩与按大小/按秩合并。
- 能说明近似常数复杂度的来源、并查集能回答什么以及不能回答什么。
- 能把并查集用于连通性、Kruskal、冗余边和等价关系问题。

## 1. 原理讲解：抽象与森林不变量

并查集维护一个随合并操作不断变化的集合划分。开始时通常每个元素各自成组；每接收一条“这两个元素属于同一组”的关系，就把两个集合合并。它关心的是等价类或无向连通分量，而不是集合内部的顺序和具体路径。

核心操作是：

- `find(x)`：返回元素 `x` 所在集合的代表元。
- `union(a,b)`：若两者属于不同集合，就合并两个集合。
- `connected(a,b)`：判断 `find(a) == find(b)`。

底层是一片有根树组成的森林。数组 `parent[x]` 指向父节点；根节点满足 `parent[root] == root`，根就是集合代表元。必须保持以下不变量：

1. 每个元素恰好属于一棵树。
2. 每棵树只有一个自指向的根。
3. 同一棵树中的所有节点经父指针最终到达同一根。
4. 若根维护 `size`，它等于该集合元素数；非根的 `size` 无需具有语义。

代表元本身通常没有业务含义，它可能随着合并策略变化。不要假设代表元一定是最小编号，除非实现显式维护该规则。

数组并不是原图的邻接关系。例如输入边有 `(0,1)` 和 `(1,2)`，按大小合并后完全可能出现 `parent[1]=0、parent[2]=0`。父指针只是在内存中编码“如何找到同一个代表元”，不是说原图真实存在边 `(0,2)`。

初始化 `n` 个元素时：

```text
parent = [0,1,2,...,n-1]
size   = [1,1,1,...,1]
components = n
```

每个元素都是一棵只有根的树。一次有效合并会让两棵树变成一棵，连通分量数减 1；若两元素本来已经同根，则集合划分没有变化，`union` 应返回失败或 `false`，分量数也不能再次减少。

### 1.1 `find` 为什么能代表集合归属

`find(x)` 沿 `parent` 指针直到遇到自指向节点。森林不变量保证路径不会进入环，也不会走到数组外；同一棵树的所有节点最终到达同一根。因此：

- `find(a) == find(b)` 当且仅当二者位于同一棵树，也就是同一集合；
- `find(a) != find(b)` 时两棵树互不相交，可以通过连接两个根完成合并。

如果错误地把任意内部节点挂接到另一个内部节点，可能丢失部分子树或制造环。`union` 的第一步必须分别 `find` 到根，随后只改变其中一个根的父指针。

## 2. 两个关键优化

### 2.1 路径压缩

普通 `find(x)` 沿父指针走到根。路径压缩在返回时让路径上的节点直接指向根：

```text
x -> a -> b -> root    变为    x -> root, a -> root, b -> root
```

它不改变集合归属，只缩短未来查找路径。递归模板 `parent[x] = find(parent[x])` 同时完成“找根”和“压缩路径”。

假设当前父数组局部关系为：

```text
6 -> 5 -> 3 -> 0，且 parent[0] = 0
```

调用 `find(6)` 时先递归找到根 0；返回阶段依次执行：

```text
parent[3] = 0
parent[5] = 0
parent[6] = 0
```

之后 3、5、6 都能一步到根。压缩不会把集合拆开，也不会把不同根连起来，只是把同一路径上的祖先关系改成更短的等价表示。

递归路径压缩代码简洁，但若实现没有按大小/秩优化，恶意初始结构可能形成极深链并导致调用栈溢出。可以用两次循环实现迭代压缩：第一遍找到根，第二遍沿原路径把节点全部指向根。也可采用“路径减半”，在向上走时让节点指向祖父。

### 2.2 按大小或按秩合并

合并时不能随意把一个根挂到另一个根下。按大小合并让较小树的根挂到较大树根下；按秩合并则比较树高的上界。本文使用按大小合并，并在新根上累加集合大小。

只使用按大小合并、不做路径压缩，任一节点每次深度增加时，其所在集合规模至少翻倍，因此深度最多 `O(log n)`。再配合路径压缩，长序列操作会更快。

为什么“小树挂大树”能限制高度？设某节点由于合并而深度增加 1，它所在的小树规模为 `s`，被挂到的树规模至少也是 `s`，合并后新集合规模至少 `2s`。同一节点深度每增加一次，其集合规模至少翻倍；规模最多为 `n`，所以深度最多增加 `log2 n` 次。

按秩合并中的 `rank` 通常表示高度上界，而不一定是路径压缩后的真实高度：

- 两根秩不同时，把低秩根挂到高秩根，较高树的秩不变；
- 两根秩相同时，任选一方为新根，并把新根秩加 1；
- 路径压缩会缩短真实路径，但通常不回头降低 `rank`。

按大小与按秩都可以得到近似常数的摊还复杂度，普通实现选择其中一种即可，不需要同时维护两套指标。

### 2.3 示例操作中的父数组变化

后面代码以 7 个元素和边 `(0,1)、(1,2)、(3,4)、(2,4)、(5,6)` 为例，使用按大小合并。初始状态：

```text
parent = [0,1,2,3,4,5,6]
size   = [1,1,1,1,1,1,1]
components = 7
```

逐步执行：

1. `union(0,1)`：两棵大小 1 的树合并，令 1 指向 0。
   `parent=[0,0,2,3,4,5,6]`，`size[0]=2`，分量数 6。
2. `union(1,2)`：先 `find(1)=0`，比较的是根 0 与根 2；令 2 指向 0。
   `parent=[0,0,0,3,4,5,6]`，`size[0]=3`，分量数 5。
3. `union(3,4)`：令 4 指向 3。
   `parent=[0,0,0,3,3,5,6]`，`size[3]=2`，分量数 4。
4. `union(2,4)`：根分别为 0 和 3，大小为 3 与 2；小树根 3 指向 0。
   `parent=[0,0,0,0,3,5,6]`，`size[0]=5`，分量数 3。此时 4 仍可暂时指向 3，但路径 `4->3->0` 已表明它属于根 0。
5. `union(5,6)`：令 6 指向 5，形成另一个大小 2 的集合，分量数 2。

随后调用 `connected(0,4)` 会执行 `find(4)`，把路径 `4->3->0` 压缩为 `4->0`。最后 `union(4,5)` 比较根 0 的大小 5 与根 5 的大小 2，令 5 指向 0，全部 7 个元素合为一个集合。6 可能仍暂时指向 5，下一次 `find(6)` 会将其压缩到 0。

这个轨迹说明两点：父数组不必在每次合并后全部直接指向根；只有根节点的 `size` 才可信，查询任意元素所在集合大小必须先执行 `find`。

## 3. 复杂度推导与含义

同时使用路径压缩和按秩/大小合并时，`m` 次操作的总时间为 `O(m α(n))`，单次摊还 `O(α(n))`。`α` 是反阿克曼函数，在任何现实规模下都不超过一个很小的常数，工程上常称“近似 `O(1)`”。

这不是说每一次 `find` 都严格常数时间；第一次可能沿较长路径，成本由后续被压缩后的廉价操作摊还。

摊还分析关注一长串操作的总成本，而不是单次峰值。一次较贵的 `find` 会主动压平路径，使路径上的多个节点未来都受益；按大小/秩又阻止树在合并时反复变深。两种优化互相配合后，能够证明总成本只多出反阿克曼函数 `α(n)` 这一极慢增长因子。

在现实可表示的数据规模中，`α(n)` 通常小于 5，因此工程上经常称为“近似常数”。但复杂度表述仍应写成摊还 `O(α(n))`，不能声称任意一次操作都严格 `O(1)`。

| 实现策略 | `find/union` 的典型上界 |
| --- | --- |
| 无优化，可能形成链 | 最坏 `O(n)` |
| 仅按大小/秩合并 | 最坏 `O(log n)` |
| 路径压缩 + 按大小/秩 | 摊还 `O(α(n))` |
| 空间 | `O(n)` |

一次 `union` 内部主要包含两次 `find` 和常数次数组更新，因此与 `find` 同阶。初始化必须写入 `n` 个父指针和大小，时间、空间都是 `O(n)`。

## 4. 适用场景、识别方法与限制

适用：

- 无向图增量加边过程中的连通性判断。
- Kruskal 最小生成树：加入边前判断两端是否已连通。
- 检测无向图冗余边/环。
- 省份数量、账户合并、等式变量归组、网格连通区域。

限制：

- 它回答“是否连通”，但不保存两点之间的具体路径。
- 基础并查集擅长只增加关系，不支持普通在线删边；删除会让一个集合拆开。
- 路径压缩会改变父指针形状，因此父指针不是原图的边，也不能用于还原合并历史。
- 有向图可达性不是普通并查集问题。
- 需要撤销操作时可使用“可回滚并查集”，通常放弃路径压缩并记录修改栈。

识别并查集问题时，可以检查是否同时满足以下特征：元素被划分为若干互不相交的组；操作主要是加入“同组关系”；查询只问两元素是否同组、组大小或组数量；不要求输出组内具体路径。满足这些条件时，并查集通常比每次重新 DFS/BFS 更合适。

以无向图增量连边为例，加入边 `(u,v)` 前：若两端根不同，连接两个分量且不会形成环；若根相同，说明已有路径连通 `u、v`，新边会形成环。这正是 Kruskal 跳过成环边和“冗余连接”问题的依据。

并查集不适合直接解决：有向图中从 `u` 是否可达 `v`；输出两点之间实际经过哪些边；边权最短路；在线删除任意边后重新判断连通。基础结构只保留集合划分，压缩后的父指针已经丢失原始路径信息。

实现边界包括：所有输入下标必须位于 `[0,n)`；重复 `union` 不改变大小和分量数；合并时只能更新根的大小；代表元不稳定，业务若要求每组最小编号应额外维护 `min[root]`；递归 `find` 的栈安全取决于树深；多线程并发 `find/union` 会读写同一父数组，不能无同步使用。

## 5. C++17 完整示例

```cpp
#include <iostream>
#include <numeric>
#include <stdexcept>
#include <utility>
#include <vector>
using namespace std;

class DSU {
private:
    vector<int> parent;
    vector<int> size;
    int components;

    void check(int x) const {
        if (x < 0 || x >= static_cast<int>(parent.size())) {
            throw out_of_range("element index out of range");
        }
    }

public:
    explicit DSU(int n) : components(n) {
        if (n < 0) throw invalid_argument("n must be non-negative");
        parent.resize(n);
        size.assign(n, 1);
        iota(parent.begin(), parent.end(), 0);
    }

    int find(int x) {
        check(x);
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }

    bool unite(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);
        if (rootA == rootB) return false;
        if (size[rootA] < size[rootB]) swap(rootA, rootB);
        parent[rootB] = rootA;
        size[rootA] += size[rootB];
        --components;
        return true;
    }

    bool connected(int a, int b) {
        return find(a) == find(b);
    }

    int componentSize(int x) {
        return size[find(x)];
    }

    int componentCount() const {
        return components;
    }
};

int main() {
    DSU dsu(7);
    for (auto [a, b] : vector<pair<int, int>>{{0, 1}, {1, 2}, {3, 4}, {2, 4}, {5, 6}}) {
        dsu.unite(a, b);
    }
    cout << boolalpha;
    cout << "connected(0,4): " << dsu.connected(0, 4) << '\n';
    cout << "connected(0,6): " << dsu.connected(0, 6) << '\n';
    cout << "size(0): " << dsu.componentSize(0) << '\n';
    cout << "components: " << dsu.componentCount() << '\n';
    cout << "union(4,5) changed: " << dsu.unite(4, 5) << '\n';
    cout << "components after union: " << dsu.componentCount() << '\n';
    return 0;
}
```

## 6. Java 17 完整示例

```java
public class Main {
    static class DSU {
        private final int[] parent;
        private final int[] size;
        private int components;

        DSU(int n) {
            if (n < 0) throw new IllegalArgumentException("n must be non-negative");
            parent = new int[n];
            size = new int[n];
            components = n;
            for (int i = 0; i < n; i++) {
                parent[i] = i;
                size[i] = 1;
            }
        }

        private void check(int x) {
            if (x < 0 || x >= parent.length) {
                throw new IndexOutOfBoundsException("element index out of range");
            }
        }

        int find(int x) {
            check(x);
            if (parent[x] != x) parent[x] = find(parent[x]);
            return parent[x];
        }

        boolean union(int a, int b) {
            int rootA = find(a);
            int rootB = find(b);
            if (rootA == rootB) return false;
            if (size[rootA] < size[rootB]) {
                int temp = rootA;
                rootA = rootB;
                rootB = temp;
            }
            parent[rootB] = rootA;
            size[rootA] += size[rootB];
            components--;
            return true;
        }

        boolean connected(int a, int b) {
            return find(a) == find(b);
        }

        int componentSize(int x) {
            return size[find(x)];
        }

        int componentCount() {
            return components;
        }
    }

    public static void main(String[] args) {
        DSU dsu = new DSU(7);
        int[][] edges = {{0, 1}, {1, 2}, {3, 4}, {2, 4}, {5, 6}};
        for (int[] edge : edges) dsu.union(edge[0], edge[1]);

        System.out.println("connected(0,4): " + dsu.connected(0, 4));
        System.out.println("connected(0,6): " + dsu.connected(0, 6));
        System.out.println("size(0): " + dsu.componentSize(0));
        System.out.println("components: " + dsu.componentCount());
        System.out.println("union(4,5) changed: " + dsu.union(4, 5));
        System.out.println("components after union: " + dsu.componentCount());
    }
}
```

## 7. Go 完整示例

```go
package main

import "fmt"

type DSU struct {
	parent     []int
	size       []int
	components int
}

func newDSU(n int) *DSU {
	if n < 0 {
		panic("n must be non-negative")
	}
	d := &DSU{
		parent:     make([]int, n),
		size:       make([]int, n),
		components: n,
	}
	for i := 0; i < n; i++ {
		d.parent[i] = i
		d.size[i] = 1
	}
	return d
}

func (d *DSU) check(x int) {
	if x < 0 || x >= len(d.parent) {
		panic("element index out of range")
	}
}

func (d *DSU) find(x int) int {
	d.check(x)
	if d.parent[x] != x {
		d.parent[x] = d.find(d.parent[x])
	}
	return d.parent[x]
}

func (d *DSU) union(a, b int) bool {
	rootA, rootB := d.find(a), d.find(b)
	if rootA == rootB {
		return false
	}
	if d.size[rootA] < d.size[rootB] {
		rootA, rootB = rootB, rootA
	}
	d.parent[rootB] = rootA
	d.size[rootA] += d.size[rootB]
	d.components--
	return true
}

func (d *DSU) connected(a, b int) bool {
	return d.find(a) == d.find(b)
}

func (d *DSU) componentSize(x int) int {
	return d.size[d.find(x)]
}

func main() {
	dsu := newDSU(7)
	edges := [][2]int{{0, 1}, {1, 2}, {3, 4}, {2, 4}, {5, 6}}
	for _, edge := range edges {
		dsu.union(edge[0], edge[1])
	}
	fmt.Println("connected(0,4):", dsu.connected(0, 4))
	fmt.Println("connected(0,6):", dsu.connected(0, 6))
	fmt.Println("size(0):", dsu.componentSize(0))
	fmt.Println("components:", dsu.components)
	fmt.Println("union(4,5) changed:", dsu.union(4, 5))
	fmt.Println("components after union:", dsu.components)
}
```

## 9. 易错点与要点总结

- `union` 必须连接两个**根**，不能直接写 `parent[b] = a`。
- 按大小合并后只更新新根的 `size`；查询大小前先 `find` 到根。
- 重复合并同一集合不应减少连通分量数。
- 路径压缩会改变树形，`parent` 不是输入图的真实边。
- “近似 `O(1)`”是摊还意义，不是每次严格常数时间。
- 初始化时每个节点自成集合，`parent[i]=i`、`size[i]=1`、分量数为 `n`。
- 网格坐标映射成一维编号时常用 `id = row*cols + col`，列数不能写错。
- Kruskal 只适用于无向图最小生成树；有向图不能直接套用。
