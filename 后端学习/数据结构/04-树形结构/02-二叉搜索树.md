# 二叉搜索树

## 学习目标

- 理解二叉搜索树（BST）的全局有序不变量，而不是只比较父子节点。
- 掌握查找、插入、删除、中序遍历以及前驱/后继的推导方法。
- 能说明 BST 为何平均较快、为何最坏会退化，以及何时应改用 AVL、红黑树或哈希表。

## 1. 原理讲解：核心不变量

二叉搜索树解决的是“数据持续变化，同时又要按键保持顺序”的问题。已排序数组能够二分查找，但在中间插入或删除通常要搬移大量元素；哈希表擅长等值查询，却不能直接按顺序遍历或回答前驱、后继和范围查询。BST 把比较结果编码进树的方向，使一次操作只需沿根到叶的一条路径前进。

对任一节点 `x`：

- `x` 左子树中的所有键都小于 `x.key`；
- `x` 右子树中的所有键都大于 `x.key`；
- 左右子树本身也分别是 BST。

这是**整棵子树范围的约束**。只检查“左孩子小于父节点、右孩子大于父节点”并不足够，例如根为 10、左孩子为 5、而 5 的右孩子为 12，局部父子关系成立，但 12 违反了根节点给整个左子树的上界。

更精确地说，搜索过程中每进入一棵子树，都同时继承了祖先施加的上下界。例如从根 10 进入左子树后，后续所有节点必须小于 10；若再从节点 5 进入右子树，则该位置允许的范围是 `(5,10)`，而不是简单的“大于 5”。这也是验证 BST 时传递上下界比只检查父子更可靠的原因。

本文采用“键唯一”策略：插入重复键时不改变树。工程中还可以选择：

- 在节点中维护 `count`，重复键只增加计数；
- 规定相等键始终进入同一侧，但这会使重复数据形成长链；
- 键由 `(主要字段, 唯一编号)` 组成，从比较器层面保证唯一。

比较规则本身也属于结构定义。自定义对象作为键时，比较器必须保持一致性：不能出现 `a < b`、`b < c`，却又得到 `c < a`；同一对键的比较结果也不能随时间变化。否则下降方向不再稳定，树中已有节点可能变得无法查到。

## 2. 操作原理

以下面的 BST 作为查找和删除推演样例，它由序列 `8, 3, 10, 1, 6, 14, 4, 7, 13` 依次插入得到：

```text
          8
        /   \
       3     10
      / \      \
     1   6      14
        / \    /
       4   7  13
```

只看任一节点，左侧整棵子树都更小，右侧整棵子树都更大。树的形状由插入顺序决定；同一组键换一个顺序，可能得到另一棵 BST，但中序结果仍相同。

### 2.1 查找与插入

从根开始比较：目标更小就进入左子树，更大就进入右子树。每次比较排除一整棵子树，因此耗时等于走过的路径长度，即 `O(h)`。

查找键 7 的轨迹是：

1. `7 < 8`，进入 8 的左子树；整个右子树都大于 8，不可能含 7。
2. `7 > 3`，进入 3 的右子树；节点 1 所在左子树全部小于 3，可排除。
3. `7 > 6`，进入 6 的右子树。
4. 到达键 7，查找成功。

查找键 2 时依次经过 `8 -> 3 -> 1`，由于 `2 > 1`，继续走向 1 的空右孩子。遇到空位置说明失败：如果 2 存在，按照所有比较结果，它只能位于这个位置，而该位置为空。

插入执行同样的下降过程，直到遇到空位置。新节点一定作为叶子加入，所以原有节点之间的大小关系没有变化；只要下降方向由比较结果决定，不变量自然保持。

例如向样例树插入 5：`5 < 8`、`5 > 3`、`5 < 6`、`5 > 4`，最终落在 4 的右孩子位置。沿途的比较同时确定了区间：进入 3 的右侧后下界是 3，进入 6 的左侧后上界是 6，进入 4 的右侧后允许范围为 `(4,6)`，5 正好满足。

插入正确性的直觉可以写成两部分：下降过程中，被排除的另一侧子树不可能容纳目标；到达空位置时，路径上的全部祖先约束构成一个允许区间，新键位于该区间内，因此接入为叶子不会破坏任何已有子树的顺序。

### 2.2 删除的三种情况

删除比插入更容易出错，因为必须重新连接子树：

1. **没有孩子**：直接删除，父节点对应指针置空。
2. **只有一个孩子**：让孩子接替被删节点的位置。
3. **有两个孩子**：找到右子树最小节点（中序后继），把它的键复制到当前节点，再从右子树删除该后继。

三类情况在样例树中可以逐一观察：

- 删除叶子 1：父节点 3 的左指针直接置空，其他连接不变。
- 删除只有左孩子 13 的节点 14：让 13 接替 14，成为 10 的右孩子。13 本来就在 10 的右子树中且大于 10，因此接替后仍合法。
- 删除有两个孩子的节点 3：它不能简单用左孩子或右孩子整体替代，否则另一棵子树可能无处连接。选择其右子树最小节点 4 作为中序后继，把 3 的键替换为 4，再删除原位置的 4。

删除 3 的结构变化如下：

```text
删除前：                后继键复制后：          删除原后继后：
      3                       4                       4
     / \                     / \                     / \
    1   6                   1   6                   1   6
       / \                     / \                       \
      4   7                   4   7                       7
```

后继替换为什么保持有序，可以从上下界逐项判断：

1. 后继来自被删节点的右子树，所以它大于被删节点左子树中的所有键。
2. 后继是右子树最小键，所以右子树中除它之外的所有键都大于它。
3. 后继沿左指针找到，因此它不可能有左孩子；删除原后继位置只需处理“无孩子或只有右孩子”的简单情况。
4. 后继仍处于被删节点原有的祖先区间内，因此不会违反更高层祖先施加的边界。

也可以对称地选择左子树最大节点，也就是中序前驱。两种方法都正确，但同一实现应固定一种策略，便于测试树形。

递归删除函数通常返回“删除后这棵子树的新根”。叶子删除会返回空，单孩子删除会返回孩子，两孩子删除通常仍返回原节点。调用者必须把返回值重新赋给自己的左/右指针；删除整棵树的根时，外部也必须接收新的根。

### 2.3 中序遍历、前驱与后继

BST 的中序遍历必然非递减，因为先完整访问更小的左子树，再访问根，最后访问更大的右子树。反过来，中序有序是验证 BST 的一种方法，但若允许极值或重复键，要仔细处理初始值和相等策略。

某节点若有右子树，其后继是右子树最左节点；若无右子树，则要向祖先回溯，找到第一个“当前节点位于其左子树”的祖先。维护父指针时可直接回溯；没有父指针时可从根重新搜索并记录候选答案。

样例中：

- 6 有右子树，所以后继是右子树最小节点 7。
- 7 没有右子树。向上回溯时，7 是 6 的右孩子，6 仍比 7 小；6 又位于 3 的右侧，3 也更小；继续到 8 时，当前路径位于 8 的左子树，故 8 是第一个更大的祖先，也是 7 的后继。
- 最小键是不断沿左指针到达的 1；最大键是不断沿右指针到达的 14。

范围查询 `[low, high]` 不必遍历全部节点：若当前键小于 `low`，可以跳过整棵左子树；若当前键大于 `high`，可以跳过整棵右子树；只有可能与范围相交的分支才继续。平衡时定位边界花费 `O(log n)`，再加输出 `k` 个结果，总计 `O(log n + k)`。

## 3. 高度与复杂度推导

所有核心操作都只沿一条根到叶路径运行，因此时间复杂度为 `O(h)`：

| 树形 | 高度 | 查找/插入/删除 |
| --- | --- | --- |
| 近似平衡 | `O(log n)` | `O(log n)` |
| 随机插入的期望情况 | 期望 `O(log n)` | 期望 `O(log n)` |
| 单调顺序插入导致退化 | `O(n)` | `O(n)` |

为什么平衡时是对数？若每下降一层，候选节点规模大致减半，经过 `h` 层后规模从 `n` 变为 1，即 `n / 2^h ≈ 1`，所以 `h ≈ log2(n)`。

这里真正决定成本的是高度 `h`，而不是“二叉”二字。依次插入 `1,2,3,4,5` 时，每个新键都进入右侧，树会退化成：

```text
1
 \
  2
   \
    3
     \
      4
       \
        5
```

查找 5 必须比较 5 次，与链表没有本质区别。普通 BST 的“平均 `O(log n)`”依赖输入顺序没有长期偏斜；若系统需要面对有序输入、对抗性输入或明确的最坏时延，必须使用 AVL、红黑树、Treap 等平衡结构。

空间复杂度：节点存储为 `O(n)`；递归实现的额外调用栈为 `O(h)`。迭代查找可做到 `O(1)` 额外空间。

一次删除即使包含“找后继并删除后继”也仍是 `O(h)`：查找目标沿一条路径，后继位于目标的右子树内，继续沿左链下降，整体不会超过常数倍树高。

## 4. 适用场景、限制与结构对比

BST 适合同时需要：动态插入删除、有序遍历、范围查询、前驱后继。它不适合直接承诺最坏 `O(log n)`，除非再增加平衡机制。

| 结构 | 单键查找 | 有序遍历/范围查询 | 最坏保证 | 特点 |
| --- | --- | --- | --- | --- |
| 普通 BST | 平均 `O(log n)` | `O(n)` / `O(h+k)`，平衡时期望 `O(log n+k)` | 无 | 实现简单，可能退化 |
| AVL | `O(log n)` | 支持 | 有 | 查找密集，旋转较严格 |
| 红黑树 | `O(log n)` | 支持 | 有 | 更新折中，标准库常用 |
| 哈希表 | 平均 `O(1)` | 不擅长 | 视实现而定 | 不维护键顺序 |

范围查询复杂度中的 `k` 表示实际输出的键数量；普通 BST 应写成 `O(h+k)`，因为定位范围起点仍受真实树高 `h` 影响。

数据库索引通常使用 B/B+ 树而不是内存 BST，因为磁盘和页存储更重视高分支因子，以减少 I/O 层数。

普通 BST 适合作为理解有序树操作的基础，也适用于数据规模可控、输入较随机、实现成本比严格最坏保证更重要的内部结构。以下情况不宜直接使用普通 BST：

- 需要稳定的最坏 `O(log n)` 延迟；
- 键大量按升序或降序到达；
- 只需要等值查找，不需要顺序、范围和前驱后继；
- 数据主要在磁盘或数据库页中；
- 需要线程安全但又没有明确的同步策略。

实现边界还包括：空树查找和删除应安全返回；删除不存在的键不应改变树；若节点保存键和值，后继替换必须复制整条键值记录；比较器与重复键策略一旦确定，不能只在某个操作中临时改变。

## 5. C++17 完整示例

```cpp
#include <iostream>
#include <vector>
using namespace std;

struct Node {
    int key;
    Node* left;
    Node* right;
    explicit Node(int value) : key(value), left(nullptr), right(nullptr) {}
};

Node* insert(Node* root, int key) {
    if (!root) return new Node(key);
    if (key < root->key) {
        root->left = insert(root->left, key);
    } else if (key > root->key) {
        root->right = insert(root->right, key);
    }
    return root; // 重复键忽略
}

bool contains(Node* root, int key) {
    while (root) {
        if (key == root->key) return true;
        root = key < root->key ? root->left : root->right;
    }
    return false;
}

Node* minimum(Node* root) {
    while (root && root->left) root = root->left;
    return root;
}

Node* erase(Node* root, int key) {
    if (!root) return nullptr;
    if (key < root->key) {
        root->left = erase(root->left, key);
    } else if (key > root->key) {
        root->right = erase(root->right, key);
    } else {
        if (!root->left) {
            Node* right = root->right;
            delete root;
            return right;
        }
        if (!root->right) {
            Node* left = root->left;
            delete root;
            return left;
        }
        Node* successor = minimum(root->right);
        root->key = successor->key;
        root->right = erase(root->right, successor->key);
    }
    return root;
}

void inorder(Node* root, vector<int>& out) {
    if (!root) return;
    inorder(root->left, out);
    out.push_back(root->key);
    inorder(root->right, out);
}

void destroy(Node* root) {
    if (!root) return;
    destroy(root->left);
    destroy(root->right);
    delete root;
}

void print(Node* root) {
    vector<int> values;
    inorder(root, values);
    for (int value : values) cout << value << ' ';
    cout << '\n';
}

int main() {
    Node* root = nullptr;
    for (int key : {8, 3, 10, 1, 6, 14, 4, 7, 13}) {
        root = insert(root, key);
    }
    cout << "inorder: ";
    print(root);
    cout << boolalpha << "contains 7: " << contains(root, 7) << '\n';
    cout << "contains 2: " << contains(root, 2) << '\n';

    root = erase(root, 3);  // 两个孩子
    root = erase(root, 1);  // 叶子
    root = erase(root, 14); // 一个孩子
    cout << "after erase: ";
    print(root);
    destroy(root);
    return 0;
}
```

## 6. Java 17 完整示例

```java
import java.util.ArrayList;
import java.util.List;

public class Main {
    static class Node {
        int key;
        Node left;
        Node right;

        Node(int key) {
            this.key = key;
        }
    }

    static Node insert(Node root, int key) {
        if (root == null) return new Node(key);
        if (key < root.key) {
            root.left = insert(root.left, key);
        } else if (key > root.key) {
            root.right = insert(root.right, key);
        }
        return root;
    }

    static boolean contains(Node root, int key) {
        while (root != null) {
            if (key == root.key) return true;
            root = key < root.key ? root.left : root.right;
        }
        return false;
    }

    static Node minimum(Node root) {
        while (root != null && root.left != null) root = root.left;
        return root;
    }

    static Node erase(Node root, int key) {
        if (root == null) return null;
        if (key < root.key) {
            root.left = erase(root.left, key);
        } else if (key > root.key) {
            root.right = erase(root.right, key);
        } else {
            if (root.left == null) return root.right;
            if (root.right == null) return root.left;
            Node successor = minimum(root.right);
            root.key = successor.key;
            root.right = erase(root.right, successor.key);
        }
        return root;
    }

    static void inorder(Node root, List<Integer> out) {
        if (root == null) return;
        inorder(root.left, out);
        out.add(root.key);
        inorder(root.right, out);
    }

    static void print(Node root) {
        List<Integer> values = new ArrayList<>();
        inorder(root, values);
        System.out.println(values);
    }

    public static void main(String[] args) {
        Node root = null;
        int[] keys = {8, 3, 10, 1, 6, 14, 4, 7, 13};
        for (int key : keys) root = insert(root, key);

        System.out.print("inorder: ");
        print(root);
        System.out.println("contains 7: " + contains(root, 7));
        System.out.println("contains 2: " + contains(root, 2));

        root = erase(root, 3);  // 两个孩子
        root = erase(root, 1);  // 叶子
        root = erase(root, 14); // 一个孩子
        System.out.print("after erase: ");
        print(root);
    }
}
```

## 7. Go 完整示例

```go
package main

import "fmt"

type Node struct {
	Key         int
	Left, Right *Node
}

func insert(root *Node, key int) *Node {
	if root == nil {
		return &Node{Key: key}
	}
	if key < root.Key {
		root.Left = insert(root.Left, key)
	} else if key > root.Key {
		root.Right = insert(root.Right, key)
	}
	return root
}

func contains(root *Node, key int) bool {
	for root != nil {
		if key == root.Key {
			return true
		}
		if key < root.Key {
			root = root.Left
		} else {
			root = root.Right
		}
	}
	return false
}

func minimum(root *Node) *Node {
	for root != nil && root.Left != nil {
		root = root.Left
	}
	return root
}

func erase(root *Node, key int) *Node {
	if root == nil {
		return nil
	}
	if key < root.Key {
		root.Left = erase(root.Left, key)
	} else if key > root.Key {
		root.Right = erase(root.Right, key)
	} else {
		if root.Left == nil {
			return root.Right
		}
		if root.Right == nil {
			return root.Left
		}
		successor := minimum(root.Right)
		root.Key = successor.Key
		root.Right = erase(root.Right, successor.Key)
	}
	return root
}

func inorder(root *Node, out *[]int) {
	if root == nil {
		return
	}
	inorder(root.Left, out)
	*out = append(*out, root.Key)
	inorder(root.Right, out)
}

func values(root *Node) []int {
	out := []int{}
	inorder(root, &out)
	return out
}

func main() {
	var root *Node
	for _, key := range []int{8, 3, 10, 1, 6, 14, 4, 7, 13} {
		root = insert(root, key)
	}
	fmt.Println("inorder:", values(root))
	fmt.Println("contains 7:", contains(root, 7))
	fmt.Println("contains 2:", contains(root, 2))

	root = erase(root, 3)  // 两个孩子
	root = erase(root, 1)  // 叶子
	root = erase(root, 14) // 一个孩子
	fmt.Println("after erase:", values(root))
}
```

## 9. 易错点与要点总结

- BST 是全局范围约束，验证时应传递上下界或检查完整中序序列。
- 必须先定义重复键策略，否则“左小右大”会在实现之间不一致。
- 删除有两个孩子的节点时，复制后继键后还要从右子树删除后继，不能留下重复节点。
- 递归插入和删除都要把返回的新子树根重新接回 `root.left` 或 `root.right`。
- 普通 BST 不保证平衡；已排序输入是最典型的退化来源。
- 中序遍历有序不等于树结构唯一；相同键集合可构成多棵不同 BST。
- 若节点还带有与键关联的完整记录，删除时要替换整条记录，而不只是整数键。
