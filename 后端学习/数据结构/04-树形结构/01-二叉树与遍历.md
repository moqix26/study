# 二叉树与遍历

## 学习目标

- 理解树、二叉树、满二叉树、完全二叉树、完美二叉树和平衡二叉树的区别。
- 掌握链式存储与顺序存储的适用边界。
- 能独立写出前序、中序、后序和层序遍历，并说明递归与迭代版本的关系。
- 能从“访问根节点的时机”推导遍历顺序，而不是死记模板。
- 能跟踪递归调用栈、显式栈和层序队列在每一步保存的状态。
- 能从节点数、层数和边数推导二叉树的常用性质，并正确分析退化树的空间复杂度。

## 1. 原理讲解：基本概念与性质

树用来表示具有层级和包含关系的数据。它由节点与连接节点的边组成，并满足：存在唯一根节点；除根外每个节点恰好有一个父节点；从根到任一节点只有一条简单路径。最后一条性质排除了环，也排除了“一个孩子同时属于两个父节点”的共享结构。

二叉树进一步限制每个节点至多有两个孩子，分别称为左孩子和右孩子。这里的“左、右”是结构的一部分，即使一个节点只有一个孩子，也必须区分它是左孩子还是右孩子。交换左右子树通常会得到另一棵二叉树。

二叉树算法频繁使用递归，是因为任一非空二叉树都可以拆成三个部分：根节点、左子树、右子树，而左右子树仍是二叉树。只要先处理空树这个最小情况，就可以用同一套逻辑处理任意规模的子树。

常用术语：

- **深度**：从根到该节点所经过的边数，根的深度为 0。
- **高度**：从该节点到最远叶子所经过的边数，叶子的高度为 0。树高等于根的高度。
- **叶子节点**：没有孩子的节点。
- **内部节点**：至少有一个孩子的节点。
- **层**：本文令根位于第 0 层，因此节点所在层数等于节点深度。
- **祖先与后代**：根到节点路径上位于该节点之前的节点都是其祖先；从该节点继续向下可达的节点都是其后代。
- **子树**：某节点以及它的全部后代构成的树。

“高度按边数计算”与代码里常见的“空树高度为 0、叶子高度为 1”并不冲突，后者实际上按节点数计算路径长度。使用哪种约定都可以，但同一份推导和代码必须保持一致。

几类容易混淆的二叉树：

- **严格二叉树/正则二叉树**：每个节点要么没有孩子，要么恰有两个孩子。它不要求所有叶子处于同一层。
- **满二叉树（国内常用定义）/完美二叉树**：除叶子外每个节点都有两个孩子，并且所有叶子在同一层。若根为第 0 层、树高为 `h`，节点数为 `2^(h+1)-1`，叶子数为 `2^h`。
- **完全二叉树**：除最后一层外全部填满，最后一层从左到右连续排列。完全二叉树适合用数组存储。
- **平衡二叉树（广义）**：树高保持在 `O(log n)` 量级；不同结构对“平衡”的精确定义不同，例如 AVL 要求每个节点左右子树高度差不超过 1。

### 1.1 节点数与高度的基本推导

若根位于第 0 层，则第 `d` 层最多有 `2^d` 个节点。原因是每个节点最多产生两个孩子：第 0 层最多 1 个，第 1 层最多 2 个，第 2 层最多 4 个，逐层翻倍。

因此，高度为 `h` 的二叉树最多有：

`1 + 2 + 4 + ... + 2^h = 2^(h+1) - 1`

个节点。等号只在每一层都填满时成立，也就是完美二叉树。反过来，容纳 `n` 个节点所需的最小高度约为 `log2(n)`；而最坏情况下每个节点都只有一个孩子，树退化为链，高度达到 `n-1`。

若一棵完全二叉树有 `n > 0` 个节点，并按层序从下标 0 开始存入数组，则最后一个节点下标为 `n-1`，其深度为 `floor(log2(n))`，所以树高也是 `floor(log2(n))`。这解释了为什么二叉堆的插入、删除只需沿 `O(log n)` 层移动。

### 1.2 叶子数公式 `n0 = n2 + 1`

令：

- `n0` 为没有孩子的节点数；
- `n1` 为恰有一个孩子的节点数；
- `n2` 为恰有两个孩子的节点数。

总节点数 `n = n0 + n1 + n2`。一棵非空树有 `n-1` 条边，因为除根外每个节点都恰好通过一条边连接父节点。另一方面，从父节点一侧计数，所有孩子指针贡献的边数是 `n1 + 2n2`。于是：

`n0 + n1 + n2 - 1 = n1 + 2n2`

消去 `n1` 后得到 `n0 = n2 + 1`。这个结论对任意非空二叉树成立，与树是否平衡、是否完全无关。

## 2. 存储方式

### 2.1 链式存储

每个节点保存值、左孩子指针和右孩子指针。它能自然表示任意形状的二叉树，也是算法题和普通业务代码的默认形式。代价是每个节点需要额外的指针空间，节点在内存中通常不连续。

链式节点的两个孩子指针本身就编码了结构，因此空孩子不会占用真实节点空间。它适合频繁修改树形、形状不规则、需要保存父子关系的场景。若算法需要从节点向上回溯，还可以增加父指针，但旋转、删除和换根时必须同步维护它。

### 2.2 顺序存储

完全二叉树可紧凑地放入数组。若下标从 0 开始，节点 `i` 的左孩子、右孩子和父节点下标分别为：

- 左孩子：`2*i + 1`
- 右孩子：`2*i + 2`
- 父节点：`(i - 1) / 2`（`i > 0`）

例如数组 `[A,B,C,D,E,F]` 对应：`A` 的孩子是 `B、C`，`B` 的孩子是 `D、E`，`C` 只有左孩子 `F`。数组中没有额外保存指针，父子关系完全由下标计算得到。

普通稀疏二叉树若机械使用这套编号会留下大量空位。例如一棵始终只有右孩子、深度为 4 的树，其节点下标依次为 `0、2、6、14、30`，只存 5 个节点却要把数组扩展到至少 31 个位置。堆由于始终保持完全二叉树形状，特别适合顺序存储。

## 3. 四种遍历及不变量

遍历的目标是让每个节点恰好被访问一次。深度优先遍历的区别只在于“何时访问根”：

- **前序：根 → 左 → 右**。进入子树时立即处理根，适合复制树、序列化结构。
- **中序：左 → 根 → 右**。对二叉搜索树会得到非递减序列。
- **后序：左 → 右 → 根**。离开子树前才处理根，适合计算高度、删除整棵树、目录大小汇总。
- **层序：逐层从左到右**。使用队列，适合最短层数、右视图、按层统计。

以下面的树作为完整推演样例：

```text
        1
      /   \
     2     3
    / \     \
   4   5     6
```

它的四种结果分别是：

- 前序：`1, 2, 4, 5, 3, 6`
- 中序：`4, 2, 5, 1, 3, 6`
- 后序：`4, 5, 2, 6, 3, 1`
- 层序：`1, 2, 3, 4, 5, 6`

### 3.1 递归遍历：访问时机决定顺序

对一个非空节点的一次递归调用，可以看成三个时间点：

1. **进入节点**：尚未递归左子树；
2. **左子树返回后**：即将递归右子树；
3. **右子树返回后**：即将结束当前调用。

前序在时间点 1 访问根，中序在时间点 2 访问根，后序在时间点 3 访问根。因此三种递归模板不是三套无关代码，而是把同一行“处理当前节点”放在三个不同位置。

以节点 2 为例：

```text
调用 visit(2)
  进入 2                 -> 前序此时输出 2
  调用 visit(4)
  visit(4) 返回
  左子树已经结束          -> 中序此时输出 2
  调用 visit(5)
  visit(5) 返回
  左右子树都已经结束      -> 后序此时输出 2
返回父节点 1
```

运行时调用栈保存“当前节点执行到哪里”和局部变量。以前序遍历为例，开始访问左链时栈会经历：

```text
visit(1)
visit(1) -> visit(2)
visit(1) -> visit(2) -> visit(4)
visit(4) 遇到空孩子并返回
visit(2) 转向 5
visit(2) 返回后，visit(1) 转向 3
```

每个调用只负责一棵子树：空树不输出；非空树输出根一次，并分别递归两棵互不重叠的子树。因此所有节点恰好访问一次，这是递归遍历正确性的核心。

### 3.2 迭代前序：显式栈保存待访问子树

递归版本由系统栈记住“处理完左子树后还要处理右子树”。迭代版本改用显式栈保存尚未访问的子树根。因为栈后进先出，要让左孩子先处理，就必须先压右、再压左。

样例树的完整状态如下，栈顶写在左侧：

| 步骤 | 弹出并访问 | 压栈动作 | 操作后栈 | 输出 |
| --- | --- | --- | --- | --- |
| 初始 | - | 压入 1 | `[1]` | `[]` |
| 1 | 1 | 先 3 后 2 | `[2,3]` | `[1]` |
| 2 | 2 | 先 5 后 4 | `[4,5,3]` | `[1,2]` |
| 3 | 4 | 无 | `[5,3]` | `[1,2,4]` |
| 4 | 5 | 无 | `[3]` | `[1,2,4,5]` |
| 5 | 3 | 压入 6 | `[6]` | `[1,2,4,5,3]` |
| 6 | 6 | 无 | `[]` | `[1,2,4,5,3,6]` |

不变量是：栈中节点都是已发现但尚未访问的子树根，且栈顶始终是按前序规则下一棵应处理的子树。

### 3.3 迭代中序：栈保存“左子树尚未处理完的祖先”

中序必须先走到最左节点，不能在节点第一次出现时就输出。算法重复执行两件事：

1. 当前节点非空时，把它压栈并继续向左；
2. 当前节点为空时，弹出最近祖先并访问，然后转向该祖先的右子树。

样例中的关键轨迹：

| 动作 | 当前节点 | 栈（栈顶在左） | 输出 |
| --- | --- | --- | --- |
| 压 1、向左 | 2 | `[1]` | `[]` |
| 压 2、向左 | 4 | `[2,1]` | `[]` |
| 压 4、向左到空 | 空 | `[4,2,1]` | `[]` |
| 弹 4，转右空 | 空 | `[2,1]` | `[4]` |
| 弹 2，转右 5 | 5 | `[1]` | `[4,2]` |
| 压 5 后弹出 | 空 | `[1]` | `[4,2,5]` |
| 弹 1，转右 3 | 3 | `[]` | `[4,2,5,1]` |

之后同样处理 `3、6`。压栈意味着“这个节点的根和右子树还没处理”；弹栈时左子树已经全部结束，因此正好可以访问根。

### 3.4 迭代后序：把一次节点处理拆成展开与访问

后序要求左右子树都完成后才能访问根。示例代码在栈中保存 `(节点, 是否已展开)`：

- 第一次弹到未展开节点时，不输出，而是安排未来任务；
- 先压入“稍后访问当前节点”的已展开任务，再压右孩子、再压左孩子；
- 因为栈后进先出，实际执行顺序成为左、右、根。

处理根 1 时，任务栈从 `[(1,未展开)]` 变为：

```text
[(2,未展开), (3,未展开), (1,已展开)]
```

继续展开 2 后变为：

```text
[(4,未展开), (5,未展开), (2,已展开), (3,未展开), (1,已展开)]
```

4 和 5 的已展开任务依次输出后，才轮到 `(2,已展开)` 输出 2。这个任务模型直接模拟递归调用“返回时处理根”的时机，也比容易混淆的双栈反转法更容易验证。

### 3.5 层序遍历：队列保持发现顺序

层序遍历使用先进先出队列。弹出一个节点时，按左、右顺序把孩子加入队尾。父节点总在孩子之前入队，同一层左侧节点也先于右侧节点入队，所以弹出顺序自然是逐层从左到右。

样例树的队列轨迹如下，队首写在左侧：

| 步骤 | 出队 | 入队 | 操作后队列 | 输出 |
| --- | --- | --- | --- | --- |
| 初始 | - | 1 | `[1]` | `[]` |
| 1 | 1 | 2、3 | `[2,3]` | `[1]` |
| 2 | 2 | 4、5 | `[3,4,5]` | `[1,2]` |
| 3 | 3 | 6 | `[4,5,6]` | `[1,2,3]` |
| 4 | 4 | 无 | `[5,6]` | `[1,2,3,4]` |
| 5 | 5 | 无 | `[6]` | `[1,2,3,4,5]` |
| 6 | 6 | 无 | `[]` | `[1,2,3,4,5,6]` |

若要按层分组，应在每轮开始时记录当前队列长度 `levelSize`。这批节点恰好属于同一层；处理它们时新加入的孩子属于下一层，不能计入当前轮。

树不是链表：若输入结构可能含环或同一节点被多个父节点引用，就已不再是普通树，遍历时必须额外维护 `visited` 集合。

## 4. 复杂度

设节点数为 `n`、树高为 `h`、最大层宽为 `w`：

| 操作 | 时间复杂度 | 额外空间 | 原因 |
| --- | --- | --- | --- |
| 递归 DFS | `O(n)` | `O(h)` | 每个节点一次，调用栈深度等于树高 |
| 迭代 DFS | `O(n)` | `O(h)`，最坏 `O(n)` | 显式栈保存待处理节点 |
| 层序 BFS | `O(n)` | `O(w)` | 队列最多容纳一层附近的节点 |

时间为什么都是 `O(n)`：每个真实节点恰好被取出或进入递归一次，每条父子边至多被检查常数次，因此总工作量与 `n` 成正比。即使遍历结果不保存到数组中，访问节点本身也无法少于 `n` 次。

递归 DFS 的栈深取决于当前根到节点的路径长度，而不是总节点数。平衡树中 `h = O(log n)`；退化成单链时 `h = O(n)`，递归可能栈溢出。复杂度中的 `O(h)` 不能无条件写成 `O(log n)`。

迭代 DFS 的精确峰值与写法和树形有关。前序栈通常保存当前路径旁尚未处理的右子树；在完美树中是 `O(h)`，某些通用任务栈写法最坏可保存 `O(n)` 个任务。层序队列的峰值由宽度决定：完美二叉树最后一层约有 `n/2` 个节点，因此最坏是 `O(n)`。

## 5. 适用场景、选择依据与实现边界

- 表达层级关系：文件目录、组织结构、语法树、DOM。
- 前序：导出树结构、先处理父任务再处理子任务。
- 中序：BST 有序输出、表达式树的中缀表达。
- 后序：依赖子结果计算父结果，如高度、余额汇总、递归释放资源。
- 层序：按层展示、无权树最短路径、寻找最浅满足条件的节点。

选择遍历顺序时应先确定依赖关系：若父节点必须先于孩子处理，选前序；若父节点依赖两个孩子的结果，选后序；若关心深度、最近层或同层统计，选层序；中序的特殊价值主要来自 BST 的有序性和表达式树的中缀结构。

二叉树并不适合所有层级数据。一个节点可拥有大量孩子时，强行转为普通二叉树会让结构含义变得间接，可以使用“孩子数组”、邻接表，或“左孩子—右兄弟”表示法。需要频繁按键等值查询时，普通二叉树也不会自动提供效率，必须额外维护 BST、堆等不变量。

实现时还应明确以下边界：

- 空树应直接返回空结果，不能把空指针入栈或入队。
- 极深且来源不可信的树应优先用迭代遍历，避免语言调用栈溢出。
- 节点值可能重复时，遍历序列通常不能唯一确定树结构；仅凭前序和中序重建也需要额外区分重复节点。
- 若树在遍历过程中会被并发修改，普通遍历无法保证一致视图，需要加锁、复制快照或定义弱一致语义。
- 若输入实际是图，必须使用 `visited` 防止环导致无限遍历；但加入 `visited` 后讨论的已是图遍历而非纯树遍历。

## 6. C++17 完整示例

```cpp
#include <iostream>
#include <queue>
#include <stack>
#include <string>
#include <utility>
#include <vector>
using namespace std;

struct Node {
    int value;
    Node* left;
    Node* right;
    explicit Node(int v) : value(v), left(nullptr), right(nullptr) {}
};

void preorderRecursive(Node* root, vector<int>& out) {
    if (!root) return;
    out.push_back(root->value);
    preorderRecursive(root->left, out);
    preorderRecursive(root->right, out);
}

void inorderRecursive(Node* root, vector<int>& out) {
    if (!root) return;
    inorderRecursive(root->left, out);
    out.push_back(root->value);
    inorderRecursive(root->right, out);
}

void postorderRecursive(Node* root, vector<int>& out) {
    if (!root) return;
    postorderRecursive(root->left, out);
    postorderRecursive(root->right, out);
    out.push_back(root->value);
}

vector<int> preorderIterative(Node* root) {
    vector<int> out;
    if (!root) return out;
    stack<Node*> st;
    st.push(root);
    while (!st.empty()) {
        Node* node = st.top();
        st.pop();
        out.push_back(node->value);
        if (node->right) st.push(node->right);
        if (node->left) st.push(node->left);
    }
    return out;
}

vector<int> inorderIterative(Node* root) {
    vector<int> out;
    stack<Node*> st;
    Node* current = root;
    while (current || !st.empty()) {
        while (current) {
            st.push(current);
            current = current->left;
        }
        current = st.top();
        st.pop();
        out.push_back(current->value);
        current = current->right;
    }
    return out;
}

vector<int> postorderIterative(Node* root) {
    vector<int> out;
    if (!root) return out;
    stack<pair<Node*, bool>> st;
    st.push({root, false});
    while (!st.empty()) {
        auto [node, expanded] = st.top();
        st.pop();
        if (expanded) {
            out.push_back(node->value);
            continue;
        }
        st.push({node, true});
        if (node->right) st.push({node->right, false});
        if (node->left) st.push({node->left, false});
    }
    return out;
}

vector<int> levelOrder(Node* root) {
    vector<int> out;
    if (!root) return out;
    queue<Node*> q;
    q.push(root);
    while (!q.empty()) {
        Node* node = q.front();
        q.pop();
        out.push_back(node->value);
        if (node->left) q.push(node->left);
        if (node->right) q.push(node->right);
    }
    return out;
}

void print(const string& name, const vector<int>& values) {
    cout << name << ":";
    for (int value : values) cout << ' ' << value;
    cout << '\n';
}

void destroy(Node* root) {
    if (!root) return;
    destroy(root->left);
    destroy(root->right);
    delete root;
}

int main() {
    //         1
    //       /   \  right
    //      2     3
    //     / \     \  right
    //    4   5     6
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);
    root->right->right = new Node(6);

    vector<int> pre, in, post;

    inorderRecursive(root, in);
    postorderRecursive(root, post);

    print("recursive inorder", in);
    print("recursive postorder", post);
    print("iterative preorder", preorderIterative(root));
    print("iterative inorder", inorderIterative(root));
    print("iterative postorder", postorderIterative(root));
    print("level order", levelOrder(root));

    destroy(root);
    return 0;
}
```

## 7. Java 17 完整示例

```java
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.Deque;
import java.util.List;
import java.util.Queue;

public class Main {
    static class Node {
        int value;
        Node left;
        Node right;

        Node(int value) {
            this.value = value;
        }
    }

    static class Task {
        Node node;
        boolean expanded;

        Task(Node node, boolean expanded) {
            this.node = node;
            this.expanded = expanded;
        }
    }

    static void preorderRecursive(Node root, List<Integer> out) {
        if (root == null) return;
        out.add(root.value);
        preorderRecursive(root.left, out);
        preorderRecursive(root.right, out);
    }

    static void inorderRecursive(Node root, List<Integer> out) {
        if (root == null) return;
        inorderRecursive(root.left, out);
        out.add(root.value);
        inorderRecursive(root.right, out);
    }

    static void postorderRecursive(Node root, List<Integer> out) {
        if (root == null) return;
        postorderRecursive(root.left, out);
        postorderRecursive(root.right, out);
        out.add(root.value);
    }

    static List<Integer> preorderIterative(Node root) {
        List<Integer> out = new ArrayList<>();
        if (root == null) return out;
        Deque<Node> stack = new ArrayDeque<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            Node node = stack.pop();
            out.add(node.value);
            if (node.right != null) stack.push(node.right);
            if (node.left != null) stack.push(node.left);
        }
        return out;
    }

    static List<Integer> inorderIterative(Node root) {
        List<Integer> out = new ArrayList<>();
        Deque<Node> stack = new ArrayDeque<>();
        Node current = root;
        while (current != null || !stack.isEmpty()) {
            while (current != null) {
                stack.push(current);
                current = current.left;
            }
            current = stack.pop();
            out.add(current.value);
            current = current.right;
        }
        return out;
    }

    static List<Integer> postorderIterative(Node root) {
        List<Integer> out = new ArrayList<>();
        if (root == null) return out;
        Deque<Task> stack = new ArrayDeque<>();
        stack.push(new Task(root, false));
        while (!stack.isEmpty()) {
            Task task = stack.pop();
            if (task.expanded) {
                out.add(task.node.value);
                continue;
            }
            stack.push(new Task(task.node, true));
            if (task.node.right != null) stack.push(new Task(task.node.right, false));
            if (task.node.left != null) stack.push(new Task(task.node.left, false));
        }
        return out;
    }

    static List<Integer> levelOrder(Node root) {
        List<Integer> out = new ArrayList<>();
        if (root == null) return out;
        Queue<Node> queue = new ArrayDeque<>();
        queue.offer(root);
        while (!queue.isEmpty()) {
            Node node = queue.poll();
            out.add(node.value);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        return out;
    }

    static void print(String name, List<Integer> values) {
        System.out.println(name + ": " + values);
    }

    public static void main(String[] args) {
        Node root = new Node(1);
        root.left = new Node(2);
        root.right = new Node(3);
        root.left.left = new Node(4);
        root.left.right = new Node(5);
        root.right.right = new Node(6);

        List<Integer> pre = new ArrayList<>();
        List<Integer> in = new ArrayList<>();
        List<Integer> post = new ArrayList<>();
        preorderRecursive(root, pre);
        inorderRecursive(root, in);
        postorderRecursive(root, post);
        print("recursive preorder", pre);
        print("recursive inorder", in);
        print("recursive postorder", post);
        print("iterative preorder", preorderIterative(root));
        print("iterative inorder", inorderIterative(root));
        print("iterative postorder", postorderIterative(root));
        print("level order", levelOrder(root));
    }
}
```

## 8. Go 完整示例

```go
package main

import "fmt"

type Node struct {
	Value       int
	Left, Right *Node
}

type Task struct {
	Node     *Node
	Expanded bool
}

func preorderRecursive(root *Node, out *[]int) {
	if root == nil {
		return
	}
	*out = append(*out, root.Value)
	preorderRecursive(root.Left, out)
	preorderRecursive(root.Right, out)
}

func inorderRecursive(root *Node, out *[]int) {
	if root == nil {
		return
	}
	inorderRecursive(root.Left, out)
	*out = append(*out, root.Value)
	inorderRecursive(root.Right, out)
}

func postorderRecursive(root *Node, out *[]int) {
	if root == nil {
		return
	}
	postorderRecursive(root.Left, out)
	postorderRecursive(root.Right, out)
	*out = append(*out, root.Value)
}

func preorderIterative(root *Node) []int {
	out := []int{}
	if root == nil {
		return out
	}
	stack := []*Node{root}
	for len(stack) > 0 {
		node := stack[len(stack)-1]
		stack = stack[:len(stack)-1]
		out = append(out, node.Value)
		if node.Right != nil {
			stack = append(stack, node.Right)
		}
		if node.Left != nil {
			stack = append(stack, node.Left)
		}
	}
	return out
}

func inorderIterative(root *Node) []int {
	out := []int{}
	stack := []*Node{}
	current := root
	for current != nil || len(stack) > 0 {
		for current != nil {
			stack = append(stack, current)
			current = current.Left
		}
		current = stack[len(stack)-1]
		stack = stack[:len(stack)-1]
		out = append(out, current.Value)
		current = current.Right
	}
	return out
}

func postorderIterative(root *Node) []int {
	out := []int{}
	if root == nil {
		return out
	}
	stack := []Task{{Node: root}}
	for len(stack) > 0 {
		task := stack[len(stack)-1]
		stack = stack[:len(stack)-1]
		if task.Expanded {
			out = append(out, task.Node.Value)
			continue
		}
		stack = append(stack, Task{Node: task.Node, Expanded: true})
		if task.Node.Right != nil {
			stack = append(stack, Task{Node: task.Node.Right})
		}
		if task.Node.Left != nil {
			stack = append(stack, Task{Node: task.Node.Left})
		}
	}
	return out
}

func levelOrder(root *Node) []int {
	out := []int{}
	if root == nil {
		return out
	}
	queue := []*Node{root}
	for len(queue) > 0 {
		node := queue[0]
		queue = queue[1:]
		out = append(out, node.Value)
		if node.Left != nil {
			queue = append(queue, node.Left)
		}
		if node.Right != nil {
			queue = append(queue, node.Right)
		}
	}
	return out
}

func main() {
	root := &Node{Value: 1}
	root.Left = &Node{Value: 2}
	root.Right = &Node{Value: 3}
	root.Left.Left = &Node{Value: 4}
	root.Left.Right = &Node{Value: 5}
	root.Right.Right = &Node{Value: 6}

	pre, in, post := []int{}, []int{}, []int{}
	preorderRecursive(root, &pre)
	inorderRecursive(root, &in)
	postorderRecursive(root, &post)
	fmt.Println("recursive preorder:", pre)
	fmt.Println("recursive inorder:", in)
	fmt.Println("recursive postorder:", post)
	fmt.Println("iterative preorder:", preorderIterative(root))
	fmt.Println("iterative inorder:", inorderIterative(root))
	fmt.Println("iterative postorder:", postorderIterative(root))
	fmt.Println("level order:", levelOrder(root))
}
```

## 10. 易错点与要点总结

- 左右孩子有顺序，交换后通常是另一棵树。
- 递归必须先写空节点出口，否则会访问空指针。
- 前序迭代应先压右再压左；栈是后进先出。
- 中序迭代在弹栈后必须转向该节点的右子树。
- 层序按层分组时，要在处理本层前保存当前队列长度，不能边遍历边用动态变化后的长度。
- “遍历时间 `O(n)`”不代表递归空间总是 `O(log n)`；退化树会达到 `O(n)`。
- 用数组存普通稀疏树时不要机械套堆下标公式，否则空间可能指数级浪费。
