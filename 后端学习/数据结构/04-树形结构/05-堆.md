# 堆（大顶堆与小顶堆）

## 学习目标

- 理解堆的“完全二叉树形状不变量”和“局部堆序不变量”。
- 掌握数组下标关系、上浮、下沉、插入、删除堆顶和原地建堆。
- 能解释 `heapify` 为什么是 `O(n)`，而不是简单相乘得到的 `O(n log n)`。
- 区分堆、二叉搜索树、优先队列和堆排序的职责。

## 1. 原理讲解：两个核心不变量

二叉堆是一种专门维护“当前最高优先级元素”的树形结构。它不追求任意键的快速查找，而是让最小值或最大值始终位于根，并且能在插入、移除堆顶后用一条根到叶路径恢复结构。

二叉堆同时满足：

1. **形状不变量**：它是一棵完全二叉树，最后一层从左到右连续填充。
2. **堆序不变量**：小顶堆中每个父节点不大于孩子；大顶堆中每个父节点不小于孩子。

堆序只约束父子，不保证左子树整体小于右子树。因此小顶堆只能保证根是全局最小值，不能像 BST 一样高效查找任意键或输出完整有序序列。

局部父子关系为什么足以保证根是全局极值？以小顶堆为例，根不大于它的孩子，孩子又不大于各自的孩子。沿根到任意节点的唯一路径连续使用传递性，可得根不大于任意后代，所以根是全局最小值。大顶堆完全对称。

但两个兄弟之间没有顺序要求。数组 `[1, 8, 2, 10, 9, 3]` 可以是小顶堆：每个父节点都不大于孩子，但左子树中的 8 大于右子树中的 2。这说明不能根据目标与根的比较排除一半子树，查找任意键最坏仍要扫描全部节点。

完全二叉树使堆可以紧凑存入数组。下标从 0 开始时：

- 父节点：`(i-1)/2`，要求 `i>0`
- 左孩子：`2*i+1`
- 右孩子：`2*i+2`
- 最后一个非叶子：`n/2-1`

这些公式来自完全二叉树的层序编号。下标 `0` 的孩子占 `1、2`，下标 `1` 的孩子占 `3、4`，下标 `2` 的孩子占 `5、6`，依次类推。下标从 1 开始时公式会变成父 `i/2`、左 `2i`、右 `2i+1`，实现中不能混用两套约定。

完全二叉树的最后一层只能从左到右填充，因此新节点一定追加在数组末尾，删除一个节点时也应优先移除数组末尾。堆的插入和删除堆顶都先维护这个**形状不变量**，再修复**堆序不变量**。

## 2. 核心操作

### 2.1 插入与上浮

新元素先追加到数组末尾，这样完全二叉树形状不变。它可能破坏与父节点之间的堆序，于是不断与父节点比较，优先级更高就交换，直到到达根或父子关系正确。路径长度不超过树高，所以是 `O(log n)`。

上浮不需要比较兄弟节点；在插入前，除新节点路径外的堆序都已经成立。

以已经建好的小顶堆数组 `[1,2,8,7,5,9]` 插入 0 为例：

1. 先追加到末尾：`[1,2,8,7,5,9,0]`。形状仍是完全二叉树。
2. 新元素下标 6，父下标 `(6-1)/2=2`，父值为 8。`0 < 8`，交换：`[1,2,0,7,5,9,8]`。
3. 0 的新下标为 2，父下标 0，父值为 1。`0 < 1`，交换：`[0,2,1,7,5,9,8]`。
4. 0 到达根，上浮结束。

每次交换后，原父节点下沉到新节点旧位置。它仍不大于该位置原有的孩子：插入前父节点已经不大于整条分支中的孩子，而新元素只沿祖先路径上移。因此需要继续检查的只有新元素与它的新父节点，路径外的堆序从未改变。

大顶堆上浮只需把比较方向反过来。例如最大堆 `[9,5,8,1,2,7]` 插入 10：

`[9,5,8,1,2,7,10] -> [9,5,10,1,2,7,8] -> [10,5,9,1,2,7,8]`

### 2.2 删除堆顶与下沉

堆顶是最高优先级元素。删除时用数组最后一个元素覆盖根，再缩短数组，保持形状紧凑。新根可能比孩子优先级低，因此每次选择两个孩子中优先级更高的那个交换，直到堆序恢复。

必须先在左右孩子中选出“更该上来”的孩子，再与父比较；只固定比较左孩子会得到错误结果。

从上面的小顶堆 `[0,2,1,7,5,9,8]` 删除堆顶：

1. 保存答案 0。
2. 用最后一个元素 8 覆盖根并缩短数组：`[8,2,1,7,5,9]`。形状保持完全，但根可能破坏堆序。
3. 根的孩子是 2 和 1，应选择更小的 1，而不是固定选择左孩子 2。交换后得到 `[1,2,8,7,5,9]`。
4. 8 的新下标为 2，只有左孩子 9；`8 <= 9`，下沉结束。

若错误地先与左孩子 2 交换，会得到根为 2、右孩子为 1 的非法小顶堆。下沉选择“两个孩子中优先级更高者”的原因是：交换后新的父节点必须同时满足与两个孩子的关系。

下沉正确性依赖一个局部前提：开始时左右孩子各自已经是合法堆，唯一可能出错的是当前节点与孩子的关系。把更高优先级孩子换上来后，新父节点同时压住两棵子树；旧父节点只可能在交换方向的子树中继续违规，因此算法沿一条路径向下即可。

空堆没有堆顶。大小为 1 的堆删除时直接移除唯一元素，不能覆盖根后再访问不存在的孩子。接口可以选择抛异常、返回可选值，或返回 `(value, ok)`，但行为必须明确。

### 2.3 自底向上建堆

给定无序数组时，从最后一个非叶子 `n/2-1` 开始向根执行下沉。叶子天然是合法堆，无需处理；当处理某节点时，它的两棵孩子子树都已经是堆，因此一次下沉就能修复该子树。

对数组 `[7,2,9,1,5,8]` 建小顶堆，`n=6`，最后一个非叶子下标为 `6/2-1=2`：

| 处理下标 | 当前数组 | 下沉过程 | 结果 |
| --- | --- | --- | --- |
| 2 | `[7,2,9,1,5,8]` | 9 与唯一孩子 8 交换 | `[7,2,8,1,5,9]` |
| 1 | `[7,2,8,1,5,9]` | 2 与更小孩子 1 交换 | `[7,1,8,2,5,9]` |
| 0 | `[7,1,8,2,5,9]` | 7 先与 1 交换，再与 2 交换 | `[1,2,8,7,5,9]` |

处理下标 1 时，它的孩子 3、4 都是叶子，天然是堆；处理根 0 时，下标 1、2 为根的两棵子树已经修复完成。因此从后往前的顺序保证了每次 `siftDown` 的前提。

对同一数组建大顶堆，只改变“谁的优先级更高”的比较函数：

`[7,2,9,1,5,8] -> [7,2,9,1,5,8] -> [7,5,9,1,2,8] -> [9,5,8,1,2,7]`

一个通用堆实现可把比较规则抽象为 `higherPriority(a,b)`。小顶堆在 `a < b` 时返回真，大顶堆在 `a > b` 时返回真；上浮、下沉和建堆的结构逻辑不需要复制。

## 3. `heapify` 为何是 `O(n)`

粗略说“`n` 个节点，每个下沉 `log n`”会给出宽松上界 `O(n log n)`，但绝大多数节点靠近叶子，下沉距离很短：

- 约 `n/2` 个节点高度为 0，不下沉；
- 约 `n/4` 个节点高度为 1；
- 约 `n/8` 个节点高度为 2；
- 依此类推。

总工作量近似为：

`n/4*1 + n/8*2 + n/16*3 + ... = n * Σ(k/2^(k+1)) = O(n)`。

注意：逐个调用 `push` 建堆仍是 `O(n log n)`；只有自底向上的 `heapify` 是 `O(n)`。

更精确地看，完全二叉树中高度至少为 `k` 的节点不超过 `n/2^(k+1)`。一个高度为 `k` 的节点最多下沉 `k` 层，所以总交换或比较次数可由下式控制：

`n/4*1 + n/8*2 + n/16*3 + ...`

无穷级数 `Σ(k/2^(k+1))` 收敛为常数，因此乘上 `n` 后仍是线性量级。`O(n log n)` 把每个节点都当成根节点那样可能下沉整棵树高，忽略了约一半节点是叶子、约四分之一节点最多走一层这一事实。

逐个 `push` 不同：后插入的大量元素会从数组末尾开始上浮，其中不少操作的路径长度接近 `log n`，总上界为 `O(n log n)`。两种建堆方式最终都得到合法堆，但构造过程和复杂度不同。

## 4. 复杂度

| 操作 | 时间复杂度 | 说明 |
| --- | --- | --- |
| 查看堆顶 | `O(1)` | 根位于数组下标 0 |
| 插入 | `O(log n)` | 最多上浮一个树高 |
| 删除堆顶 | `O(log n)` | 最多下沉一个树高 |
| 自底向上建堆 | `O(n)` | 大量节点离叶子很近 |
| 查找任意键 | `O(n)` | 堆没有全局左右有序性 |
| 堆排序 | `O(n log n)` | 建堆 `O(n)` 加 `n` 次删除堆顶 |

数组本身占 `O(n)`；原地堆操作除数组外只需 `O(1)` 辅助空间。

这里的 `O(log n)` 来自完全二叉树高度 `floor(log2 n)`。上浮和下沉每轮至少移动一层，且只走一条路径。查看堆顶只读取数组 0 号位置，不修改结构，因此是 `O(1)`。

## 5. 适用场景、限制与结构对比

- 任务调度：持续取最高优先级任务。
- Top K：维护大小为 `k` 的反向堆，时间 `O(n log k)`。
- 多路归并：堆中保存每一路当前最小元素。
- Dijkstra/Prim：反复取得当前最小距离或最小边。
- 数据流中位数：大顶堆保存较小一半，小顶堆保存较大一半。

| 需求 | 合适结构 |
| --- | --- |
| 反复取最小/最大 | 堆 / 优先队列 |
| 任意键查找、范围查询 | 平衡 BST |
| 等值查询且不关心顺序 | 哈希表 |
| 双端都要 `O(1)` 取值 | 双端队列（但不维护任意优先级） |

优先队列是抽象接口，描述“插入元素、查看/移除最高优先级元素”；二叉堆是实现优先队列的常见数据结构。堆排序则是利用大顶堆反复把最大值交换到数组末尾的一种算法，三者不能混为同一个概念。

堆适合优先级随数据到达不断变化、但每次只关心极值的场景。它不适合：频繁查找或删除任意键而又没有保存下标映射；需要完整有序遍历；需要高效前驱后继和范围查询；要求相同优先级自动保持插入顺序但节点中没有额外序号。

实现任意下标的优先级修改时，先更新值，再与父比较：若比父优先级高就上浮，否则可能需要下沉。删除任意元素也可与末尾交换后缩短数组，再根据新值与父子关系决定方向；若需要按任务 ID 找下标，还必须维护 `ID -> index` 映射，并在每次交换时同步更新。

边界与工程注意：下标公式可能发生整数溢出，计算孩子前应确认 `i` 在有效范围；只有左孩子时不能读取右孩子；自定义比较器必须形成稳定的严格顺序；堆默认不稳定，若同优先级要按到达顺序处理，可比较 `(priority, sequence)`；并发读写数组堆需要同步，单独保护 `top` 读取不足以覆盖同时发生的 `push/pop`。

## 6. C++17 完整示例

```cpp
#include <iostream>
#include <stdexcept>
#include <string>
#include <utility>
#include <vector>
using namespace std;

class BinaryHeap {
private:
    vector<int> data;
    bool minHeap;

    bool higherPriority(int a, int b) const {
        return minHeap ? a < b : a > b;
    }

    void siftUp(int index) {
        while (index > 0) {
            int parent = (index - 1) / 2;
            if (!higherPriority(data[index], data[parent])) break;
            swap(data[index], data[parent]);
            index = parent;
        }
    }

    void siftDown(int index) {
        int n = static_cast<int>(data.size());
        while (true) {
            int best = index;
            int left = index * 2 + 1;
            int right = index * 2 + 2;
            if (left < n && higherPriority(data[left], data[best])) best = left;
            if (right < n && higherPriority(data[right], data[best])) best = right;
            if (best == index) break;
            swap(data[index], data[best]);
            index = best;
        }
    }

public:
    BinaryHeap(const vector<int>& values, bool isMinHeap) : data(values), minHeap(isMinHeap) {
        for (int i = static_cast<int>(data.size()) / 2 - 1; i >= 0; --i) siftDown(i);
    }

    bool empty() const { return data.empty(); }

    int top() const {
        if (empty()) throw runtime_error("heap is empty");
        return data[0];
    }

    void push(int value) {
        data.push_back(value);
        siftUp(static_cast<int>(data.size()) - 1);
    }

    int pop() {
        int result = top();
        data[0] = data.back();
        data.pop_back();
        if (!data.empty()) siftDown(0);
        return result;
    }
};

void drain(const string& name, BinaryHeap& heap) {
    cout << name << ':';
    while (!heap.empty()) cout << ' ' << heap.pop();
    cout << '\n';
}

int main() {
    vector<int> values{7, 2, 9, 1, 5, 8};
    BinaryHeap minHeap(values, true);
    BinaryHeap maxHeap(values, false);
    minHeap.push(0);
    maxHeap.push(10);
    cout << "min top: " << minHeap.top() << '\n';
    cout << "max top: " << maxHeap.top() << '\n';
    drain("min pop order", minHeap);
    drain("max pop order", maxHeap);
    return 0;
}
```

## 7. Java 17 完整示例

```java
import java.util.ArrayList;
import java.util.List;

public class Main {
    static class BinaryHeap {
        private final List<Integer> data = new ArrayList<>();
        private final boolean minHeap;

        BinaryHeap(int[] values, boolean minHeap) {
            this.minHeap = minHeap;
            for (int value : values) data.add(value);
            for (int i = data.size() / 2 - 1; i >= 0; i--) siftDown(i);
        }

        private boolean higherPriority(int a, int b) {
            return minHeap ? a < b : a > b;
        }

        private void swap(int i, int j) {
            int temp = data.get(i);
            data.set(i, data.get(j));
            data.set(j, temp);
        }

        private void siftUp(int index) {
            while (index > 0) {
                int parent = (index - 1) / 2;
                if (!higherPriority(data.get(index), data.get(parent))) break;
                swap(index, parent);
                index = parent;
            }
        }

        private void siftDown(int index) {
            int n = data.size();
            while (true) {
                int best = index;
                int left = index * 2 + 1;
                int right = index * 2 + 2;
                if (left < n && higherPriority(data.get(left), data.get(best))) best = left;
                if (right < n && higherPriority(data.get(right), data.get(best))) best = right;
                if (best == index) break;
                swap(index, best);
                index = best;
            }
        }

        boolean isEmpty() {
            return data.isEmpty();
        }

        int top() {
            if (isEmpty()) throw new IllegalStateException("heap is empty");
            return data.get(0);
        }

        void push(int value) {
            data.add(value);
            siftUp(data.size() - 1);
        }

        int pop() {
            int result = top();
            int last = data.remove(data.size() - 1);
            if (!data.isEmpty()) {
                data.set(0, last);
                siftDown(0);
            }
            return result;
        }
    }

    static void drain(String name, BinaryHeap heap) {
        System.out.print(name + ":");
        while (!heap.isEmpty()) System.out.print(" " + heap.pop());
        System.out.println();
    }

    public static void main(String[] args) {
        int[] values = {7, 2, 9, 1, 5, 8};
        BinaryHeap minHeap = new BinaryHeap(values, true);
        BinaryHeap maxHeap = new BinaryHeap(values, false);
        minHeap.push(0);
        maxHeap.push(10);
        System.out.println("min top: " + minHeap.top());
        System.out.println("max top: " + maxHeap.top());
        drain("min pop order", minHeap);
        drain("max pop order", maxHeap);
    }
}
```

## 8. Go 完整示例

```go
package main

import "fmt"

type BinaryHeap struct {
	data    []int
	minHeap bool
}

func newBinaryHeap(values []int, minHeap bool) *BinaryHeap {
	data := append([]int(nil), values...)
	h := &BinaryHeap{data: data, minHeap: minHeap}
	for i := len(h.data)/2 - 1; i >= 0; i-- {
		h.siftDown(i)
	}
	return h
}

func (h *BinaryHeap) higherPriority(a, b int) bool {
	if h.minHeap {
		return a < b
	}
	return a > b
}

func (h *BinaryHeap) siftUp(index int) {
	for index > 0 {
		parent := (index - 1) / 2
		if !h.higherPriority(h.data[index], h.data[parent]) {
			break
		}
		h.data[index], h.data[parent] = h.data[parent], h.data[index]
		index = parent
	}
}

func (h *BinaryHeap) siftDown(index int) {
	for {
		best := index
		left, right := index*2+1, index*2+2
		if left < len(h.data) && h.higherPriority(h.data[left], h.data[best]) {
			best = left
		}
		if right < len(h.data) && h.higherPriority(h.data[right], h.data[best]) {
			best = right
		}
		if best == index {
			return
		}
		h.data[index], h.data[best] = h.data[best], h.data[index]
		index = best
	}
}

func (h *BinaryHeap) empty() bool {
	return len(h.data) == 0
}

func (h *BinaryHeap) top() int {
	if h.empty() {
		panic("heap is empty")
	}
	return h.data[0]
}

func (h *BinaryHeap) push(value int) {
	h.data = append(h.data, value)
	h.siftUp(len(h.data) - 1)
}

func (h *BinaryHeap) pop() int {
	result := h.top()
	last := h.data[len(h.data)-1]
	h.data = h.data[:len(h.data)-1]
	if !h.empty() {
		h.data[0] = last
		h.siftDown(0)
	}
	return result
}

func drain(name string, heap *BinaryHeap) {
	fmt.Print(name + ":")
	for !heap.empty() {
		fmt.Print(" ", heap.pop())
	}
	fmt.Println()
}

func main() {
	values := []int{7, 2, 9, 1, 5, 8}
	minHeap := newBinaryHeap(values, true)
	maxHeap := newBinaryHeap(values, false)
	minHeap.push(0)
	maxHeap.push(10)
	fmt.Println("min top:", minHeap.top())
	fmt.Println("max top:", maxHeap.top())
	drain("min pop order", minHeap)
	drain("max pop order", maxHeap)
}
```

## 10. 易错点与要点总结

- 堆只保证父子堆序，不保证左右子树之间有序。
- 插入先放数组末尾再上浮；删除堆顶先用末尾覆盖根再下沉。
- 下沉时必须在两个孩子中选择优先级更高者。
- 子节点下标可能越界，尤其只有左孩子时不能直接读取右孩子。
- `heapify` 的起点是 `n/2-1`，叶子无需处理。
- 空堆执行 `top/pop` 应明确报错或返回 `(值, 是否存在)`，不能静默读越界。
- Top K 中“求最大 K 个”通常维护大小为 K 的小顶堆，因为堆顶代表当前答案中最容易被淘汰的最小值。
- Go 标准库 `container/heap` 的 `Less` 定义决定堆顶优先级；方向写反是高频错误。
