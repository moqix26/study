## 3. 每篇文档的统一结构

核心文档按以下顺序组织：

1. 学习目标与前置知识
2. 核心定义、不变量与操作过程
3. 时间复杂度、空间复杂度与适用场景
4. C++17 完整程序
5. Java 17 完整程序
6. Go 完整程序
7. 示例输出、常见易错点与练习建议

代码围栏中的 `cpp`、`java`、`go` 示例均按“单个代码块可独立编译运行”编写，不依赖其他文档中的片段。

## 4. 推荐学习顺序

```text
复杂度
  → 数组与链表
  → 栈、队列、哈希表
  → 二叉树、BST、堆
  → 排序与二分
  → 图的 BFS/DFS、拓扑、最短路
  → AVL、红黑树、并查集、Trie
  → 滑动窗口、回溯、分治、动态规划
```

如果你有竞赛基础，可以快速阅读数组、栈、基础排序，但仍建议认真复习：

- 各结构的**不变量**与边界条件；
- AVL 与红黑树的旋转/修复；
- BFS 的入队标记时机；
- Dijkstra 的非负权前提；
- 二分区间的开闭约定；
- DP 状态定义与初始化。

## 5. 三种语言运行方式

```powershell
# C++17
g++ -std=c++17 main.cpp -o main.exe
.\main.exe

# Java 17（代码文件保存为 Main.java）
javac -encoding UTF-8 Main.java
java Main

# Go
go run main.go
```

批量验证方式见 [99-工具与验证](99-工具与验证/README.md)。

## 6. 学习验收标准

学完一个主题后，至少做到：

- 不看文档说出结构的不变量和主要操作；
- 能解释平均、最坏时间复杂度及额外空间；
- 能独立写出核心操作，或在完整实现中定位并修复边界错误；
- 能说清它与相邻结构的选型差异；
- 至少完成 2 道基础题和 1 道组合题。

不要把“看懂代码”当作“会写”。真正掌握的最低标准是：**能用自己的话解释一次，再独立完成一个小改动并通过测试。**
