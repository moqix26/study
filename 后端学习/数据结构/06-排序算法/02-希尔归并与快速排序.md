# 希尔排序、归并排序与快速排序

## 学习目标

- 理解希尔排序的分组插入、归并排序的分治合并和快速排序的分区思想。
- 掌握三种算法的完整递归/迭代实现及复杂度差异。
- 能解释归并排序为何稳定、快速排序为何可能退化。

## 三种算法的总体思路

希尔、归并和快速排序都比朴素 O(n²) 排序更适合较大数据，但它们突破瓶颈的方式完全不同：

- 希尔排序允许元素先进行远距离移动，快速减少逆序，再用插入排序收尾。
- 归并排序先把大问题分成独立小问题，再把两个有序结果线性合并。
- 快速排序先用枢轴把元素原地分区，让枢轴直接到达最终位置，再递归处理两侧。

理解它们时应分别抓住三个状态不变量：希尔排序的“按 gap 有序”、归并排序的“左右半区已有序”、快速排序的“分区边界两侧满足与枢轴的大小关系”。

## 希尔排序

### 问题直觉

插入排序对近乎有序数组很快，但逆序元素若相距很远，只能一格一格移动。例如最小值位于数组末尾时，需要连续右移前面所有元素。希尔排序先允许按较大间隔比较，使元素一次跨越多个位置，快速靠近最终区域；间隔逐渐缩小后，数组已经接近有序，最后一次 `gap = 1` 的插入排序成本就会较低。

给定间隔 `gap`，下标相差 `gap` 的元素属于同一组。例如长度 7、`gap = 3` 时，分组为：

- 下标 `0, 3, 6`
- 下标 `1, 4`
- 下标 `2, 5`

代码通常不真的创建这些组，而是让插入排序的下标每次减 `gap`。

### 状态不变量与步骤

完成某个间隔 `gap` 的整轮处理后，对所有合法下标 `i` 都有：同一组中前面的元素不大于后面的元素，即数组是 **gap-有序** 的。

每轮过程如下：

1. 选择当前 `gap`。
2. 从下标 `gap` 开始，把 `a[i]` 视为当前组的新元素。
3. 保存它，沿 `i-gap, i-2*gap, ...` 向前扫描。
4. 将组内所有比它大的元素向后移动 `gap` 个位置。
5. 把当前元素插入空位。
6. 缩小间隔，最终必须处理 `gap = 1`。

最后一轮 `gap = 1` 就是普通插入排序。因为 1-有序等价于整个数组相邻有序，所以算法结束时全局有序。

### 手工推演

输入 `[9, 4, 7, 3, 10, 5, 1]`，示例间隔序列为 `3, 1`：

| 间隔 | 分组及组内排序 | 整轮结束后的数组 |
| ---: | --- | --- |
| 3 | `[9,3,1] -> [1,3,9]`；`[4,10]` 不变；`[7,5] -> [5,7]` | `[1, 4, 5, 3, 10, 7, 9]` |
| 1 | 对整个近乎有序数组做插入排序 | `[1, 3, 4, 5, 7, 9, 10]` |

第一轮后数组并未全局有序，例如 5 仍在 3 前面；它只保证每隔 3 个位置观察时有序。这是正常的，中间轮次的目标是减少远距离逆序，而不是立刻完成排序。

### 为什么正确

每个固定间隔的处理本质上是若干互不混淆的插入排序，因此该轮结束后每个组都有序。间隔序列最终包含 1；当 `gap = 1` 处理完成时，整个数组作为唯一一组有序。因此不论前面使用什么正间隔，只要最终正确完成 1-间隔插入排序，最终结果一定正确。前面间隔影响的是效率而非最终正确性。

希尔排序不稳定：间隔大于 1 时，相等元素可能与其他元素进行跨距离移动，导致二者相对次序改变。

### 间隔序列与复杂度边界

希尔排序复杂度高度依赖间隔序列。示例使用 `n/2, n/4, ..., 1`，容易理解，但最坏时间仍可达到 O(n²)。更好的经验序列可显著改善实际表现，但不像归并排序那样有一个简单、统一的 O(n log n) 保证。它的优点是原地、实现较短、比直接插入排序更能处理远距离逆序。

## 归并排序

### 分治模型

归并排序使用“分而治之”：

1. **分**：把区间 `[left, right]` 从中点分成两半。
2. **治**：递归地把左半区和右半区分别排好。
3. **合**：利用“两半已经有序”这一前提，在 O(区间长度) 时间内合并。

单元素区间天然有序，是递归终点。真正完成排序的关键不只是拆分，而是合并阶段。

### 合并不变量与步骤

设左半区 `[left, middle]` 和右半区 `[middle+1, right]` 已分别有序。用指针 `i` 指向左半区首个未取元素，`j` 指向右半区首个未取元素，`k` 指向辅助数组下一写入位置。

循环不变量是：辅助数组 `[left, k)` 已经包含两半中最小的若干元素，并且有序；`i`、`j` 分别指向各自剩余部分的最小元素。

1. 比较 `a[i]` 与 `a[j]`，把较小者写入辅助数组并移动对应指针。
2. 某一半耗尽后，另一半剩余元素已经有序，可直接依次复制。
3. 把辅助数组的合并结果复制回原区间。

相等时先取左半区元素，即使用 `a[i] <= a[j]`，可以保持相等记录原来的先后关系，因此归并排序稳定。

### 手工推演

输入 `[7, 3, 5, 1]`：

1. 拆为 `[7, 3]` 与 `[5, 1]`，再拆成四个单元素区间。
2. 合并 `[7]` 与 `[3]` 得 `[3, 7]`。
3. 合并 `[5]` 与 `[1]` 得 `[1, 5]`。
4. 最终合并 `[3, 7]` 与 `[1, 5]`：

| 比较 | 取出 | 已合并部分 | 左侧剩余 | 右侧剩余 |
| --- | ---: | --- | --- | --- |
| 3 与 1 | 1 | `[1]` | `[3,7]` | `[5]` |
| 3 与 5 | 3 | `[1,3]` | `[7]` | `[5]` |
| 7 与 5 | 5 | `[1,3,5]` | `[7]` | 空 |
| 右侧耗尽 | 复制 7 | `[1,3,5,7]` | 空 | 空 |

### 为什么正确

在两个有序序列中，全局最小的未取元素必然是两个首元素之一；选择其中较小者不会漏掉更小值。每次取出一个元素后，同样结论对剩余部分继续成立，直到所有元素写完，因此合并结果有序且元素集合不变。递归保证左右半区先有序，合并再保证整个区间有序，由区间长度归纳可证明完整算法正确。

### 自顶向下与自底向上

本文代码使用递归的自顶向下归并。也可以从长度 1 的有序块开始，迭代合并成长度 2、4、8 的块，称为自底向上归并。二者时间与主要辅助空间相同；迭代版不需要递归栈，更适合某些严格控制栈深度的环境。

链表归并排序尤其自然：分割可用快慢指针，合并只需重接节点，不需要数组式 O(n) 临时元素缓冲。外部排序也常用归并，因为磁盘上的多个有序文件可以顺序读取并合并。

## 快速排序

### 分区模型

快速排序选择一个枢轴 `pivot`，通过一次线性扫描把区间重新排列，使枢轴左侧元素不大于它，右侧元素大于它。枢轴随后处于最终下标：以后不必再移动它，只需递归排序左右两侧。

本文示例使用 Lomuto 分区，并选择区间最后一个元素为枢轴。扫描过程中维护边界 `boundary`：

- `[left, boundary)` 中的元素都不大于枢轴。
- `[boundary, i)` 中的元素都大于枢轴。
- `[i, right)` 尚未检查。
- `a[right]` 保存枢轴。

### 分区步骤

1. 令 `pivot = a[right]`，`boundary = left`。
2. 让 `i` 从 `left` 扫描到 `right-1`。
3. 若 `a[i] <= pivot`，把它与 `a[boundary]` 交换，并令 `boundary++`；否则它留在“大于区”。
4. 扫描结束时，把枢轴与 `a[boundary]` 交换。
5. 返回 `boundary` 作为枢轴最终下标。

然后只递归 `[left, boundary-1]` 和 `[boundary+1, right]`，必须排除已经就位的枢轴。

### 手工推演 Lomuto 分区

输入区间 `[4, 2, 7, 3, 1, 5]`，枢轴为 5：

| 扫描值 | 动作 | `boundary` 之后 | 数组 |
| ---: | --- | ---: | --- |
| 4 | `4 <= 5`，放入小值区 | 1 | `[4, 2, 7, 3, 1, 5]` |
| 2 | 放入小值区 | 2 | `[4, 2, 7, 3, 1, 5]` |
| 7 | 大于枢轴，不动 | 2 | `[4, 2, 7, 3, 1, 5]` |
| 3 | 与边界处 7 交换 | 3 | `[4, 2, 3, 7, 1, 5]` |
| 1 | 与边界处 7 交换 | 4 | `[4, 2, 3, 1, 7, 5]` |
| 扫描结束 | 枢轴 5 与边界处 7 交换 | 枢轴下标 4 | `[4, 2, 3, 1, 5, 7]` |

此时 5 已在最终位置。左侧元素都不大于 5，右侧元素都大于 5；左侧内部尚未有序，递归会继续处理。

### 为什么正确

每次扫描后，若当前值不大于枢轴，就把它放入小值区并扩大边界；若大于枢轴，它自然归入边界与扫描指针之间的大值区。因此循环不变量始终成立。最后枢轴换到两区之间，其左侧不大于它、右侧大于它，所以枢轴下标不可能再改变。递归排序左右区间后，左区所有元素、枢轴、右区所有元素整体有序。

快速排序通常不稳定，因为分区中的远距离交换可能让相等记录跨越彼此。若稳定性是硬性要求，通常选择归并排序或标准库的稳定排序，而不是强行修改原地快排。

### 枢轴与退化

若每次分区近似对半，递归树高度为 O(log n)，每层总扫描 O(n)，所以时间 O(n log n)。若输入已有序且始终选择末尾枢轴，每次只缩小一个元素，递归深度变成 n，总时间 O(n²)。常见防护包括：

- 随机选择枢轴，降低持续遇到极端分割的概率。
- 三数取中，避免简单有序输入直接触发最差情况。
- 大量重复值时使用三路分区，单独形成 `== pivot` 区间。
- 只递归较小一侧、循环处理较大一侧，把栈深限制在 O(log n)。
- 深度超过阈值时切换到堆排序，这类混合策略称为 introsort。

## 复杂度

| 算法 | 最好时间 | 平均时间 | 最坏时间 | 额外空间 | 稳定性 |
| --- | --- | --- | --- | --- | --- |
| 希尔排序（折半间隔） | 依间隔而定 | 常优于 O(n²) | O(n²) | O(1) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | 平均 O(log n)，最坏 O(n) | 不稳定 |

### 复杂度逐项推导

- 希尔排序每个间隔都做若干插入排序，成本取决于间隔序列与输入分布；折半间隔的保守最坏上界为 O(n²)，额外只需临时变量，空间 O(1)。
- 归并排序的递推式为 `T(n) = 2T(n/2) + O(n)`。递归树约 log n 层，每层合计复制 n 个元素，所以任何输入都是 O(n log n)。数组辅助缓冲为 O(n)，递归栈为 O(log n)，总额外空间由 O(n) 主导。
- 快速排序的分区每层扫描当前区间一次。均衡时递推式近似 `T(n)=2T(n/2)+O(n)=O(n log n)`；极端时为 `T(n)=T(n-1)+O(n)=O(n²)`。原地分区本身 O(1) 额外空间，但递归栈平均 O(log n)、最坏 O(n)。

## 适用场景

- 希尔排序：中小规模、内存受限且希望实现简单的原地排序。
- 归并排序：要求稳定、需要可靠 O(n log n) 上界；也适合链表排序和外部排序。
- 快速排序：内存中数组的通用高性能排序，缓存局部性好，但应处理枢轴退化风险。

## 选型与不适用场景

| 需求 | 倾向选择 | 需要接受的代价 |
| --- | --- | --- |
| 中小规模、内存紧张、实现简单 | 希尔排序 | 理论界限依赖间隔，不稳定 |
| 稳定且要可靠 O(n log n) | 归并排序 | 数组版本需 O(n) 辅助空间 |
| 链表或外部文件排序 | 归并排序 | 需要合并流程 |
| 内存数组平均高性能 | 快速排序 | 不稳定，必须防退化与深递归 |
| 不能接受最坏 O(n²) 且要求原地 | 三者单独都非完美 | 可考虑堆排序或 introsort |

数据量很小或分区已很小时，直接插入排序往往更快；标准库通常会组合多种算法。若只是需要前 k 大而非完整有序，堆或选择算法可避免排序全部元素。

## 实现边界与三种语言差异

- 所有递归区间必须明确采用闭区间还是半开区间，分割点与终止条件应保持一致。
- 归并时复用一个全局辅助数组比每层反复申请临时数组更节省分配成本。
- 快排遇到大量相等键时，二路分区可能产生极不均衡递归，应考虑三路分区。
- **C++**：下标常混用无符号 `size_t` 与有符号整数，执行 `pivotIndex - 1` 或向左递减时要避免无符号下溢。标准 `std::sort` 通常不是简单快排，而是带退化保护的混合实现。
- **Java**：递归深度过大可能抛 `StackOverflowError`；对象数组稳定排序与基本类型数组排序的标准库策略可能不同。辅助数组若反复创建会增加 GC 压力，应尽量复用。
- **Go**：切片子区间仍共享底层数组，原地分区会修改原切片；标准 `sort`/`slices` 实现会使用工程化混合算法。递归参数用 `int`，空切片调用包装函数时应先避免构造右端点 `-1` 后误用。

## C++17 完整示例

```cpp
#include <iostream>
#include <utility>
#include <vector>

void shellSort(std::vector<int>& values) {
    for (std::size_t gap = values.size() / 2; gap > 0; gap /= 2) {
        for (std::size_t i = gap; i < values.size(); ++i) {
            int current = values[i];
            std::size_t j = i;
            while (j >= gap && values[j - gap] > current) {
                values[j] = values[j - gap];
                j -= gap;
            }
            values[j] = current;
        }
    }
}

void merge(std::vector<int>& values, std::vector<int>& buffer,
           int left, int middle, int right) {
    int i = left, j = middle + 1, k = left;
    while (i <= middle && j <= right) {
        if (values[i] <= values[j]) buffer[k++] = values[i++];
        else buffer[k++] = values[j++];
    }
    while (i <= middle) buffer[k++] = values[i++];
    while (j <= right) buffer[k++] = values[j++];
    for (int p = left; p <= right; ++p) values[p] = buffer[p];
}

void mergeSortRange(std::vector<int>& values, std::vector<int>& buffer,
                    int left, int right) {
    if (left >= right) return;
    int middle = left + (right - left) / 2;
    mergeSortRange(values, buffer, left, middle);
    mergeSortRange(values, buffer, middle + 1, right);
    merge(values, buffer, left, middle, right);
}

void mergeSort(std::vector<int>& values) {
    if (values.empty()) return;
    std::vector<int> buffer(values.size());
    mergeSortRange(values, buffer, 0, static_cast<int>(values.size()) - 1);
}

int partition(std::vector<int>& values, int left, int right) {
    int pivot = values[right];
    int boundary = left;
    for (int i = left; i < right; ++i) {
        if (values[i] <= pivot) std::swap(values[i], values[boundary++]);
    }
    std::swap(values[boundary], values[right]);
    return boundary;
}

void quickSort(std::vector<int>& values, int left, int right) {
    if (left >= right) return;
    int pivotIndex = partition(values, left, right);
    quickSort(values, left, pivotIndex - 1);
    quickSort(values, pivotIndex + 1, right);
}

void quickSort(std::vector<int>& values) {
    if (!values.empty()) quickSort(values, 0, static_cast<int>(values.size()) - 1);
}

void print(const std::string& name, const std::vector<int>& values) {
    std::cout << name << ": [";
    for (std::size_t i = 0; i < values.size(); ++i) {
        if (i > 0) std::cout << ", ";
        std::cout << values[i];
    }
    std::cout << "]\n";
}

int main() {
    const std::vector<int> source{9, 4, 7, 3, 10, 5, 1};
    auto shell = source;
    auto merged = source;
    auto quick = source;
    shellSort(shell);
    mergeSort(merged);
    quickSort(quick);
    print("希尔排序", shell);
    print("归并排序", merged);
    print("快速排序", quick);
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.Arrays;

public class Main {
    private static void shellSort(int[] values) {
        for (int gap = values.length / 2; gap > 0; gap /= 2) {
            for (int i = gap; i < values.length; i++) {
                int current = values[i];
                int j = i;
                while (j >= gap && values[j - gap] > current) {
                    values[j] = values[j - gap];
                    j -= gap;
                }
                values[j] = current;
            }
        }
    }

    private static void mergeSort(int[] values) {
        if (values.length < 2) return;
        mergeSort(values, new int[values.length], 0, values.length - 1);
    }

    private static void mergeSort(int[] values, int[] buffer, int left, int right) {
        if (left >= right) return;
        int middle = left + (right - left) / 2;
        mergeSort(values, buffer, left, middle);
        mergeSort(values, buffer, middle + 1, right);
        merge(values, buffer, left, middle, right);
    }

    private static void merge(int[] values, int[] buffer, int left, int middle, int right) {
        int i = left, j = middle + 1, k = left;
        while (i <= middle && j <= right) {
            buffer[k++] = values[i] <= values[j] ? values[i++] : values[j++];
        }
        while (i <= middle) buffer[k++] = values[i++];
        while (j <= right) buffer[k++] = values[j++];
        for (int p = left; p <= right; p++) values[p] = buffer[p];
    }

    private static void quickSort(int[] values) {
        if (values.length > 1) quickSort(values, 0, values.length - 1);
    }

    private static void quickSort(int[] values, int left, int right) {
        if (left >= right) return;
        int pivotIndex = partition(values, left, right);
        quickSort(values, left, pivotIndex - 1);
        quickSort(values, pivotIndex + 1, right);
    }

    private static int partition(int[] values, int left, int right) {
        int pivot = values[right];
        int boundary = left;
        for (int i = left; i < right; i++) {
            if (values[i] <= pivot) swap(values, i, boundary++);
        }
        swap(values, boundary, right);
        return boundary;
    }

    private static void swap(int[] values, int i, int j) {
        int temp = values[i];
        values[i] = values[j];
        values[j] = temp;
    }

    public static void main(String[] args) {
        int[] source = {9, 4, 7, 3, 10, 5, 1};
        int[] shell = source.clone();
        int[] merged = source.clone();
        int[] quick = source.clone();
        shellSort(shell);
        mergeSort(merged);
        quickSort(quick);
        System.out.println("希尔排序: " + Arrays.toString(shell));
        System.out.println("归并排序: " + Arrays.toString(merged));
```

## Go 完整示例

```go
package main

import "fmt"

func shellSort(values []int) {
	for gap := len(values) / 2; gap > 0; gap /= 2 {
		for i := gap; i < len(values); i++ {
			current := values[i]
			j := i
			for j >= gap && values[j-gap] > current {
				values[j] = values[j-gap]
				j -= gap
			}
			values[j] = current
		}
	}
}

func mergeSort(values []int) {
	if len(values) < 2 {
		return
	}
	buffer := make([]int, len(values))
	mergeSortRange(values, buffer, 0, len(values)-1)
}

func mergeSortRange(values, buffer []int, left, right int) {
	if left >= right {
		return
	}
	middle := left + (right-left)/2
	mergeSortRange(values, buffer, left, middle)
	mergeSortRange(values, buffer, middle+1, right)
	merge(values, buffer, left, middle, right)
}

func merge(values, buffer []int, left, middle, right int) {
	i, j, k := left, middle+1, left
	for i <= middle && j <= right {
		if values[i] <= values[j] {
			buffer[k] = values[i]
			i++
		} else {
			buffer[k] = values[j]
			j++
		}
		k++
	}
	for i <= middle {
		buffer[k] = values[i]
		i, k = i+1, k+1
	}
	for j <= right {
		buffer[k] = values[j]
		j, k = j+1, k+1
	}
	copy(values[left:right+1], buffer[left:right+1])
}

func quickSort(values []int) {
	if len(values) > 1 {
		quickSortRange(values, 0, len(values)-1)
	}
}

func quickSortRange(values []int, left, right int) {
	if left >= right {
		return
	}
	pivotIndex := partition(values, left, right)
	quickSortRange(values, left, pivotIndex-1)
	quickSortRange(values, pivotIndex+1, right)
}

func partition(values []int, left, right int) int {
	pivot := values[right]
	boundary := left
	for i := left; i < right; i++ {
		if values[i] <= pivot {
			values[i], values[boundary] = values[boundary], values[i]
			boundary++
		}
	}
	values[boundary], values[right] = values[right], values[boundary]
	return boundary
}

func clone(values []int) []int {
	result := make([]int, len(values))
	copy(result, values)
	return result
}

func main() {
	source := []int{9, 4, 7, 3, 10, 5, 1}
	shell := clone(source)
	merged := clone(source)
	quick := clone(source)
	shellSort(shell)
	mergeSort(merged)
	quickSort(quick)
	fmt.Println("希尔排序:", shell)
	fmt.Println("归并排序:", merged)
	fmt.Println("快速排序:", quick)
}
```

## 易错点

- 希尔排序的内层是“间隔为 gap 的插入排序”，下标移动量也是 `gap`。
- 归并时辅助数组的下标范围必须与原区间一致；相等时先取左侧才能保持稳定。
- 计算中点使用 `left + (right - left) / 2`，可避免某些语言中的整数加法溢出。
- 快速排序分区结束后，递归区间要排除已经就位的枢轴。
- Lomuto 分区在大量重复元素时也可能不均衡，可使用三路快排把元素分成 `< pivot`、`== pivot`、`> pivot`。
- 深度退化会导致递归栈过深；工程实现通常随机化或限制递归深度。

## 练习建议

1. 把归并排序改成自底向上的迭代版本。
2. 为快速排序加入随机枢轴和尾递归消除，并测试有序输入。
3. 实现三路快速排序，比较大量重复值时的递归次数。
4. 使用不同希尔间隔序列，统计比较和移动次数。
5. 用归并过程求数组中的逆序对数量。
