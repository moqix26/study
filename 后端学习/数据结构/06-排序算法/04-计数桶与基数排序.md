# 非比较排序：计数排序、桶排序与基数排序

## 学习目标

- 理解利用键值范围或数字位数绕过比较模型的排序方法。
- 掌握计数、桶和 LSD 基数排序的完整实现。
- 能判断值域、分布、负数和内存限制是否满足算法前提。

## 为什么能不比较就排序

基于比较的通用排序只能通过“a 是否小于 b”获取顺序信息，在常见决策树模型下需要至少 O(n log n) 次比较。计数、桶和基数排序利用了键的额外结构——有限整数值域、近似分布范围或可拆分的位——所以能够在特定条件下达到接近 O(n) 的时间。

这种加速不是免费的：它们不再适用于任意比较对象，通常要付出与值域、桶数或基数有关的额外空间。选型前必须先回答：键是否为离散整数、最大最小值差多大、分布是否均匀、位数有多少、是否包含负数，以及是否要求稳定地携带整条记录。

## 计数排序

### 问题模型与状态

计数排序适用于整数键落在有限范围 `[minimum, maximum]` 的情况。令值域大小：

`k = maximum - minimum + 1`

创建长度 k 的计数数组，值 `x` 映射到下标 `x - minimum`。减去最小值使负数也能映射为从 0 开始的非负下标。

统计完成后的不变量是：`counts[i]` 精确等于值 `i + minimum` 在输入中出现的次数。随后只要按下标从小到大输出相应次数，就自然得到有序数组。

### 纯数值回填版步骤

本文代码排序的是整数本身，因此采用直接回填：

1. 扫描输入求最小值与最大值，并安全计算值域大小。
2. 创建并清零 `counts`。
3. 对每个值 `x` 执行 `counts[x-minimum]++`。
4. 从 `counts[0]` 到 `counts[k-1]` 扫描。
5. 将值 `i+minimum` 写回数组 `counts[i]` 次。

这种写法会丢弃“相同键记录的原始身份”，所以只适合纯数值，不能据此宣称记录排序稳定。

### 稳定版为何需要前缀和

若输入记录含有 `(score, name)`，不能仅根据次数重新制造 score；必须把每条原记录放入输出数组。做法是把计数转换为前缀和：`counts[i]` 表示键不大于 i 的记录数，也就是该键在输出中的右边界位置。

从输入末尾向前扫描记录：对键 `i`，先把 `counts[i]` 减一，所得下标就是该记录的输出位置。反向扫描保证相同键中靠后的记录先占据靠后位置，原相对次序得以保持。

### 手工推演

输入 `[4, 2, 2, 8, 3, 3, 1]`，最小值 1、最大值 8：

| 值 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 出现次数 | 1 | 2 | 2 | 1 | 0 | 0 | 0 | 1 |

从小到大回填：输出 1 一次、2 两次、3 两次、4 一次、8 一次，得到 `[1, 2, 2, 3, 3, 4, 8]`。

若输入是 `[-2, 3, -2, 1]`，最小值为 -2，则映射下标分别为 `0, 5, 0, 3`。算法无需特殊分支就能处理负数；真正需要防范的是 `maximum - minimum + 1` 的溢出和过大内存申请。

### 为什么正确

统计阶段逐个处理所有输入元素，每个值只增加它唯一对应的计数槽，因此计数准确且总和为 n。回填阶段按数值下标递增输出，较小值一定先于较大值；每个值恰好输出其出现次数，所以既有序又保持多重集合完全一致。

## 桶排序

### 问题模型与桶不变量

桶排序先利用已知范围把元素粗分到多个有序区间，再分别排序桶内元素。若桶 0 的取值范围整体小于桶 1，桶 1 又整体小于桶 2，那么只要每个桶内部有序，按桶编号连接便得到全局有序结果。

本文采用固定桶宽 `width`，以最小值为起点：

`bucketIndex(x) = (x - minimum) / width`

因此桶 i 覆盖从 `minimum + i*width` 开始的一段范围。核心不变量是：若 `i < j`，则桶 i 中任何值都不大于桶 j 中任何值。

### 逐步过程

1. 扫描求最小值和最大值。
2. 根据范围和正桶宽计算桶数。
3. 每个元素按公式放入唯一桶。
4. 分别排序每个桶；可以用插入排序、快速排序或语言标准排序。
5. 按桶编号从小到大依次回写。

桶排序并不等于“桶内不用排序”。除非每个桶只对应一个离散值，否则同一桶中仍可能存在不同大小元素。

### 手工推演

输入 `[29, 25, 3, 49, 9, 37, 21, 43]`，`minimum = 3`，桶宽为 10：

| 桶 | 覆盖范围 | 分配结果 | 桶内排序后 |
| ---: | --- | --- | --- |
| 0 | 3～12 | `[3, 9]` | `[3, 9]` |
| 1 | 13～22 | `[21]` | `[21]` |
| 2 | 23～32 | `[29, 25]` | `[25, 29]` |
| 3 | 33～42 | `[37]` | `[37]` |
| 4 | 43～52 | `[49, 43]` | `[43, 49]` |

按桶顺序连接得到 `[3, 9, 21, 25, 29, 37, 43, 49]`。

### 为什么正确

映射公式把每个元素放入与其数值范围对应的唯一桶，不丢失元素。桶内排序后，每个桶自身有序；任意较小编号桶的上界不超过较大编号桶的下界，因此前一桶的任何元素不会大于后一桶元素。于是按编号连接后的整体序列有序。

### 分布为何决定性能

设有 b 个桶，第 i 个桶含 `n_i` 个元素。分配与连接总共 O(n+b)，桶内若使用 O(n_i log n_i) 排序，总成本为 `sum(n_i log n_i)`。分布均匀时每桶很小，这一项接近 O(n)；若所有元素落入同一桶，就退化成对 n 个元素做一次桶内排序，约 O(n log n)。大量空桶还会浪费 O(b) 空间。

所以桶宽不是随便选的：过宽会失去分散效果，过窄会产生过多空桶。数据范围极大或分布高度偏斜时，通常需要自适应边界、采样分桶，或直接选择比较排序。

## 基数排序

### 按位排序的模型

基数排序把一个键拆成多位。例如十进制整数 437 可拆成个位 7、十位 3、百位 4。LSD（Least Significant Digit）基数排序从最低位开始，每一轮按当前位做**稳定排序**，再向更高位推进。

本文以十进制为基数 `b = 10`，用计数过程稳定地按每一位分配。对于不足当前位数的非负整数，缺失高位视为 0。

### 每一位的步骤

1. 令位权 `exp` 依次为 1、10、100……
2. 当前数字位为 `(value / exp) % 10`。
3. 统计 0～9 每个数字出现次数。
4. 计算前缀和，得到各数字在输出中的结束位置。
5. 从输入末尾向前扫描，把元素稳定放入输出数组。
6. 将输出结果作为下一位的输入。
7. 最高有效位处理完后结束。

### 稳定性为什么不可缺少

处理十位时，只应重排“十位不同”的元素；十位相同的元素必须保留上一轮已经建立的个位顺序。若当前位排序不稳定，低位结果会被破坏，最终无法保证完整数字顺序。

### 手工推演

输入 `[29, 25, 3, 49, 9, 37, 21, 43]`：

| 轮次 | 稳定分组顺序 | 本轮结果 |
| --- | --- | --- |
| 个位 | 1:`21`；3:`3,43`；5:`25`；7:`37`；9:`29,49,9` | `[21, 3, 43, 25, 37, 29, 49, 9]` |
| 十位 | 0:`3,9`；2:`21,25,29`；3:`37`；4:`43,49` | `[3, 9, 21, 25, 29, 37, 43, 49]` |

观察十位为 2 的三个数：上一轮已经按个位形成 `21,25,29`，十位轮因为稳定而保留这一顺序。

### 为什么正确

第 1 轮结束后，数组按最低 1 位有序。假设第 t-1 轮后已经按低 t-1 位有序；第 t 轮按第 t 位稳定排序后，第 t 位较小的元素整体在前，第 t 位相同的元素仍保持低 t-1 位顺序，因此数组按低 t 位有序。处理完最高位后，按全部位有序，数学归纳成立。

### 负数和其他键

本文代码只接受非负整数。处理有符号整数有多种策略：把负数与非负数分开，对负数绝对值排序后逆序拼接；或对固定宽度二进制表示做符号位映射。前一种方法要特别小心最小负数的绝对值可能溢出。

基数排序也可用于定长字符串：从最右字符向左做稳定分配。若字符串长度不同，需要定义缺失字符的排序位置或先按长度分类。

## 复杂度

| 算法 | 时间复杂度 | 额外空间 | 主要前提 | 稳定性 |
| --- | --- | --- | --- | --- |
| 计数排序 | O(n + k) | O(k) 或 O(n+k) | 整数值域 k 不宜过大 | 可稳定 |
| 桶排序 | 平均 O(n + b)，最坏 O(n log n) | O(n + b) | 分布较均匀、桶策略合理 | 取决于桶内排序 |
| 基数排序 | O(d(n + b)) | O(n + b) | 位数 d 有限、基数 b 固定 | 稳定 |

### 复杂度逐项推导

- 计数排序扫描输入 O(n)、扫描值域 O(k)，总时间 O(n+k)。直接回填需 O(k) 计数空间；稳定记录版还需 O(n) 输出空间。
- 桶排序分配 O(n)、创建和遍历桶 O(b)、桶内排序成本取决于分布。均匀且桶数量合理时平均接近 O(n+b)，集中到单桶时使用标准比较排序约 O(n log n)。空间保存所有桶中元素与桶结构，为 O(n+b)。
- 基数排序每一位都扫描 n 个元素并处理 b 个数字桶，共 d 轮，所以 O(d(n+b))；辅助输出 O(n)，计数 O(b)。如果键位数 d 会随 n 明显增长，就不能简单把它当作线性时间。

## 适用场景

- 计数排序：年龄、分数、字符编码等小整数值域。
- 桶排序：已知数据分布范围且较均匀，例如归一化测量值。
- 基数排序：大量定长或位数有限的非负整数、编号、可按位处理的字符串。
- 当值域远大于元素数或键不是可分解的离散值时，比较排序通常更节省空间。

## 不适用场景与选型边界

- 值域 k 远大于 n 时不要用计数排序。例如只排序 100 个、范围却跨越 32 位整数的值，计数数组不可接受。
- 无法合理估计分布或存在极端离群值时，固定宽度桶排序可能创建大量空桶。
- 键无法拆成有限、可稳定排序的位时，不适合基数排序。
- 稳定排序带附属记录时，不能使用本文计数排序的纯数值回填方式；应实现前缀和输出版本。
- 需要通用比较器、数据量不大或空间紧张时，成熟的标准比较排序往往更可靠。

## 实现边界与三种语言差异

- `maximum - minimum + 1`、桶数和位权都可能溢出，必须在较宽类型中计算并在分配前检查。
- 空输入应直接返回；桶宽必须为正；基数排序必须明确负数约定。
- **C++**：整数提升和有符号溢出需要格外谨慎，容器 `max_size()` 也不代表一定有足够实际内存。`std::sort` 用于桶内排序不稳定，若桶排序对象并要求稳定，应选稳定桶内算法。
- **Java**：`int` 差值可能先溢出，应在相减前转换为 `long`；大型二维桶对象会产生较多对象开销。对记录做稳定计数排序时使用引用输出数组。
- **Go**：`int` 位宽取决于目标平台，范围计算可用 `int64` 并检查再转回 `int`；切片分配过大仍会失败。标准库整数排序是比较排序，桶内若要求稳定需自行选择稳定实现。

## C++17 完整示例

```cpp
#include <algorithm>
#include <iostream>
#include <stdexcept>
#include <vector>

void countingSort(std::vector<int>& values) {
    if (values.empty()) return;
    auto [minIt, maxIt] = std::minmax_element(values.begin(), values.end());
    int minimum = *minIt, maximum = *maxIt;
    long long range = static_cast<long long>(maximum) - minimum + 1;
    if (static_cast<unsigned long long>(range) > std::vector<int>().max_size()) {
        throw std::length_error("value range is too large");
    }
    std::vector<int> counts(static_cast<std::size_t>(range), 0);
    for (int value : values) {
        ++counts[static_cast<std::size_t>(static_cast<long long>(value) - minimum)];
    }
    std::size_t index = 0;
    for (std::size_t i = 0; i < counts.size(); ++i) {
        while (counts[i]-- > 0) {
            values[index++] = static_cast<int>(static_cast<long long>(i) + minimum);
        }
    }
}

void bucketSort(std::vector<int>& values, int bucketWidth) {
    if (values.empty()) return;
    if (bucketWidth <= 0) throw std::invalid_argument("bucket width must be positive");
    auto [minIt, maxIt] = std::minmax_element(values.begin(), values.end());
    int minimum = *minIt, maximum = *maxIt;
    long long bucketCount = (static_cast<long long>(maximum) - minimum) / bucketWidth + 1;
    if (static_cast<unsigned long long>(bucketCount) >
        std::vector<std::vector<int>>().max_size()) {
        throw std::length_error("too many buckets");
    }
    std::vector<std::vector<int>> buckets(static_cast<std::size_t>(bucketCount));
    for (int value : values) {
        auto bucketIndex = static_cast<std::size_t>(
            (static_cast<long long>(value) - minimum) / bucketWidth);
        buckets[bucketIndex].push_back(value);
    }
    std::size_t index = 0;
    for (auto& bucket : buckets) {
        std::sort(bucket.begin(), bucket.end());
        for (int value : bucket) values[index++] = value;
    }
}

void radixSort(std::vector<int>& values) {
    if (values.empty()) return;
    int maximum = 0;
    for (int value : values) {
        if (value < 0) throw std::invalid_argument("radix sort requires non-negative values");
        maximum = std::max(maximum, value);
    }
    std::vector<int> output(values.size());
    for (long long exp = 1; maximum / exp > 0; exp *= 10) {
        int counts[10] = {};
        for (int value : values) ++counts[(value / exp) % 10];
        for (int i = 1; i < 10; ++i) counts[i] += counts[i - 1];
        for (int i = static_cast<int>(values.size()) - 1; i >= 0; --i) {
            int digit = (values[i] / exp) % 10;
            output[--counts[digit]] = values[i];
        }
        values = output;
    }
}

void print(const std::string& name, const std::vector<int>& values) {
    std::cout << name << ": [";
    for (std::size_t i = 0; i < values.size(); ++i) {
        if (i > 0) std::cout << ", ";
        std::cout << values[i];
    }
    std::cout << "]\n";
}

int main() {
    const std::vector<int> source{29, 25, 3, 49, 9, 37, 21, 43};
    auto counted = source;
    auto bucketed = source;
    auto radix = source;
    countingSort(counted);
    bucketSort(bucketed, 10);
    radixSort(radix);
    print("计数排序", counted);
    print("桶排序", bucketed);
    print("基数排序", radix);
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.Collections;
import java.util.List;

public class Main {
    private static void countingSort(int[] values) {
        if (values.length == 0) return;
        int minimum = values[0], maximum = values[0];
        for (int value : values) {
            minimum = Math.min(minimum, value);
            maximum = Math.max(maximum, value);
        }
        long range = (long) maximum - minimum + 1;
        if (range > Integer.MAX_VALUE) {
            throw new IllegalArgumentException("value range is too large");
        }
        int[] counts = new int[(int) range];
        for (int value : values) counts[(int) ((long) value - minimum)]++;
        int index = 0;
        for (int i = 0; i < counts.length; i++) {
            while (counts[i]-- > 0) values[index++] = (int) ((long) i + minimum);
        }
    }

    private static void bucketSort(int[] values, int bucketWidth) {
        if (values.length == 0) return;
        if (bucketWidth <= 0) throw new IllegalArgumentException("bucket width must be positive");
        int minimum = values[0], maximum = values[0];
        for (int value : values) {
            minimum = Math.min(minimum, value);
            maximum = Math.max(maximum, value);
        }
        long bucketCountLong = ((long) maximum - minimum) / bucketWidth + 1;
        if (bucketCountLong > Integer.MAX_VALUE) {
            throw new IllegalArgumentException("too many buckets");
        }
        int bucketCount = (int) bucketCountLong;
        List<List<Integer>> buckets = new ArrayList<>();
        for (int i = 0; i < bucketCount; i++) buckets.add(new ArrayList<>());
        for (int value : values) {
            int bucketIndex = (int) (((long) value - minimum) / bucketWidth);
            buckets.get(bucketIndex).add(value);
        }
        int index = 0;
        for (List<Integer> bucket : buckets) {
            Collections.sort(bucket);
            for (int value : bucket) values[index++] = value;
        }
    }

    private static void radixSort(int[] values) {
        int maximum = 0;
        for (int value : values) {
            if (value < 0) throw new IllegalArgumentException("radix sort requires non-negative values");
            maximum = Math.max(maximum, value);
        }
        int[] output = new int[values.length];
        for (long exp = 1; maximum / exp > 0; exp *= 10) {
            int[] counts = new int[10];
            for (int value : values) counts[(int) (value / exp % 10)]++;
            for (int i = 1; i < 10; i++) counts[i] += counts[i - 1];
            for (int i = values.length - 1; i >= 0; i--) {
                int digit = (int) (values[i] / exp % 10);
                output[--counts[digit]] = values[i];
            }
            System.arraycopy(output, 0, values, 0, values.length);
        }
    }

    public static void main(String[] args) {
        int[] source = {29, 25, 3, 49, 9, 37, 21, 43};
        int[] counted = source.clone();
        int[] bucketed = source.clone();
        int[] radix = source.clone();
        countingSort(counted);
        bucketSort(bucketed, 10);
        radixSort(radix);
        System.out.println("计数排序: " + Arrays.toString(counted));
        System.out.println("桶排序: " + Arrays.toString(bucketed));
        System.out.println("基数排序: " + Arrays.toString(radix));
    }
}
```

## Go 完整示例

```go
package main

import (
	"fmt"
	"sort"
)

func countingSort(values []int) {
	if len(values) == 0 {
		return
	}
	minimum, maximum := values[0], values[0]
	for _, value := range values {
		if value < minimum {
			minimum = value
		}
		if value > maximum {
			maximum = value
		}
	}
	span := uint64(maximum) - uint64(minimum)
	maxInt := int(^uint(0) >> 1)
	if span >= uint64(maxInt) {
		panic("value range is too large")
	}
	counts := make([]int, int(span)+1)
	for _, value := range values {
		counts[int(uint64(value)-uint64(minimum))]++
	}
	index := 0
	for offset, count := range counts {
		for ; count > 0; count-- {
			values[index] = offset + minimum
			index++
		}
	}
}

func bucketSort(values []int, bucketWidth int) {
	if len(values) == 0 {
		return
	}
	if bucketWidth <= 0 {
		panic("bucket width must be positive")
	}
	minimum, maximum := values[0], values[0]
	for _, value := range values {
		if value < minimum {
			minimum = value
		}
		if value > maximum {
			maximum = value
		}
	}
	span := uint64(maximum) - uint64(minimum)
	maxInt := int(^uint(0) >> 1)
	bucketCountMinusOne := span / uint64(bucketWidth)
	if bucketCountMinusOne >= uint64(maxInt) {
		panic("too many buckets")
	}
	buckets := make([][]int, int(bucketCountMinusOne)+1)
	for _, value := range values {
		index := int((uint64(value) - uint64(minimum)) / uint64(bucketWidth))
		buckets[index] = append(buckets[index], value)
	}
	index := 0
	for _, bucket := range buckets {
		sort.Ints(bucket)
		index += copy(values[index:], bucket)
	}
}

func radixSort(values []int) {
	maximum := 0
	for _, value := range values {
		if value < 0 {
			panic("radix sort requires non-negative values")
		}
		if value > maximum {
			maximum = value
		}
	}
	output := make([]int, len(values))
	for exp := 1; maximum/exp > 0; {
		counts := [10]int{}
		for _, value := range values {
			counts[value/exp%10]++
		}
		for i := 1; i < len(counts); i++ {
			counts[i] += counts[i-1]
		}
		for i := len(values) - 1; i >= 0; i-- {
			digit := values[i] / exp % 10
			counts[digit]--
			output[counts[digit]] = values[i]
		}
		copy(values, output)
		if exp > maximum/10 {
			break
		}
		exp *= 10
	}
}

func clone(values []int) []int {
	result := make([]int, len(values))
	copy(result, values)
	return result
}

func main() {
	source := []int{29, 25, 3, 49, 9, 37, 21, 43}
	counted := clone(source)
	bucketed := clone(source)
	radix := clone(source)
	countingSort(counted)
	bucketSort(bucketed, 10)
	radixSort(radix)
	fmt.Println("计数排序:", counted)
	fmt.Println("桶排序:", bucketed)
	fmt.Println("基数排序:", radix)
}
```

桶排序: [3, 9, 21, 25, 29, 37, 43, 49]
基数排序: [3, 9, 21, 25, 29, 37, 43, 49]

## 易错点

- 计数数组大小取决于值域而非元素数；计算差值时要先提升到更宽或无符号类型，并拒绝无法表示的数组长度。
- 只按出现次数直接回填适合纯数值排序，但不是“携带原记录”的稳定实现。
- 桶宽过大可能让所有数据落入一个桶，过小则产生大量空桶和额外内存。
- 基数排序每一位必须稳定；回填时从后向前扫描可保持稳定性。
- 负数不能直接套用示例的十进制基数排序。
- 位权 `exp` 可能溢出；C++/Java 示例用更宽整数保存位权，Go 示例会在最后一个有效位处理完后停止乘 10。

## 练习建议

1. 把计数排序改为稳定版本，排序包含 `score` 与 `name` 的记录。
2. 为基数排序增加负数支持，并验证正负混合输入。
3. 根据样本分布自动选择桶宽，比较均匀分布与偏斜分布的性能。
4. 实现按 ASCII 字符逐位排序的定长字符串基数排序。
5. 设计阈值：值域过大时自动退回标准比较排序。
