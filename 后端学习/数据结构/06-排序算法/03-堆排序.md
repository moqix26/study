# 堆排序

## 学习目标

- 理解完全二叉树在数组中的下标映射和最大堆性质。
- 掌握自底向上建堆、下沉调整与原地堆排序。
- 能分析堆排序的 O(n log n) 时间上界及其稳定性。

## 问题模型与直觉

堆排序利用“最大堆能在 O(1) 时间给出当前最大元素”来反复确定数组尾部。它先把所有元素组织成最大堆，取出堆顶最大值放到数组最后，再在剩余元素中恢复最大堆，重复直到完成升序排列。

**最大堆不是完全有序的树，也不是二叉搜索树。** 它只保证每个父节点不小于自己的孩子，因此根节点一定是全局最大值；同一层的两个节点、左子树和右子树之间没有整体大小关系。这一弱约束足以快速找到最大值，也比维护全局有序更容易。

## 完全二叉树的数组表示

堆的形状是完全二叉树：除最后一层外每层填满，最后一层从左向右连续填充。正因为没有中间空位，所以无需存指针，可以直接放在连续数组中。

对从 0 开始的下标 `i`：

- 左孩子：`2*i + 1`
- 右孩子：`2*i + 2`
- 父节点：`(i - 1) / 2`，仅适用于 `i > 0`

若孩子下标不小于当前堆大小，说明该孩子不存在。长度为 `n` 时，下标 `[n/2, n)` 全是叶节点，最后一个非叶节点是 `n/2 - 1`。

## 状态不变量

堆排序执行期间，数组分成两个区域：

1. `[0, heapSize)` 是当前仍参与调整的最大堆。
2. `[heapSize, n)` 已按最终升序排列，并且其中每个元素都不小于堆区元素。

初始建堆后 `heapSize = n`、已排序后缀为空。每取出一次最大值，`heapSize` 减一，已排序后缀向左扩大一位。

## 下沉操作

下沉 `siftDown(root, heapSize)` 的前提是：以 `root` 的左右孩子为根的子树已经分别满足最大堆，只有 `root` 本身可能小于孩子。过程如下：

1. 根据下标计算左右孩子。
2. 在 `root`、左孩子、右孩子中找值最大的下标 `largest`。
3. 若 `largest == root`，根已不小于孩子，调整结束。
4. 否则交换 `root` 与较大孩子，并让 `root = largest` 继续向下检查。

必须先在两个孩子中选更大的一个。若父节点只与左孩子交换，而右孩子更大，交换后新父节点仍可能小于右孩子，堆性质没有恢复。

一次下沉最多从某节点走到叶子，路径长度不超过树高 `floor(log2 n)`，因此时间 O(log n)，额外空间 O(1)。

## 自底向上建堆

叶节点本身已经是合法堆，无需处理。从最后一个非叶节点 `n/2 - 1` 开始，逆序对每个节点执行下沉：

1. 处理某个节点时，它的孩子下标更大，孩子子树已经在之前的步骤中建成堆。
2. 对当前节点下沉后，以它为根的整棵子树成为堆。
3. 最终处理根节点后，整个数组成为最大堆。

这叫 Floyd 自底向上建堆。不要把它与“从空堆开始逐个插入”混淆：逐个插入需要 O(n log n)，而自底向上建堆是 O(n)。

## 排序阶段逐步过程

建好最大堆后：

1. 堆顶 `a[0]` 是堆区最大值。
2. 将它与堆区最后一个元素 `a[heapSize-1]` 交换。
3. 将 `heapSize` 减一。刚移到末尾的最大值进入已排序后缀，不再参与调整。
4. 交换到根的元素可能破坏最大堆，但它的左右子树仍是堆，因此从根执行一次下沉即可恢复。
5. 重复直到堆区只剩一个元素。

升序排序使用最大堆，因为每次取出的最大值应从数组右端向左就位；降序排序则可使用最小堆。

## 手工推演

输入 `[12, 11, 13, 5, 6, 7, 6]`。

### 建堆

最后一个非叶节点下标是 `7/2 - 1 = 2`。从下标 2、1、0 依次下沉：

- 下标 2 的值 13 已不小于孩子 7、6。
- 下标 1 的值 11 已不小于孩子 5、6。
- 下标 0 的值 12 小于右孩子 13，与 13 交换。

得到最大堆数组 `[13, 11, 12, 5, 6, 7, 6]`。这里只保证父节点不小于孩子，并不要求数组本身已经升序。

### 反复取堆顶

| 操作 | 交换后、尚未下沉 | 恢复堆后的堆区 | 已排序后缀 |
| --- | --- | --- | --- |
| 取 13 | `[6,11,12,5,6,7 | 13]` | `[12,11,7,5,6,6]` | `[13]` |
| 取 12 | `[6,11,7,5,6 | 12,13]` | `[11,6,7,5,6]` | `[12,13]` |
| 取 11 | `[6,6,7,5 | 11,12,13]` | `[7,6,6,5]` | `[11,12,13]` |
| 继续执行 | 逐步缩小堆区 | 最终只剩最小值 | `[6,6,7,11,12,13]` 等逐步形成 |

最终数组为 `[5, 6, 6, 7, 11, 12, 13]`。

## 为什么算法正确

建堆的正确性可按下标从后向前归纳：叶节点天然是堆；处理某非叶节点时，孩子子树已是堆，下沉会把该子树最大元素放到根，并在交换路径上持续恢复性质，所以该节点对应子树也成为堆。处理到根后整棵树是最大堆。

排序阶段每轮开始时，堆顶是堆区最大值，同时不大于已排序后缀中的元素。把它换到堆区末尾后，它就位于最终下标。缩小堆区不会再触碰该元素；下沉重新恢复剩余最大堆。循环不变量持续成立，最终所有元素按升序排列。

堆排序不稳定。例如两个键相等的记录可能在建堆或交换堆顶与堆尾时跨越很远距离，相对次序没有任何保护。

## 复杂度

| 阶段/操作 | 时间复杂度 | 额外空间 |
| --- | --- | --- |
| 单次下沉 | O(log n) | O(1) |
| 自底向上建堆 | O(n) | O(1) |
| n 次取堆顶并调整 | O(n log n) | O(1) |
| 完整堆排序（最好/平均/最坏） | O(n log n) | O(1) |

### 为什么建堆是 O(n)，不是 O(n log n)

把 n 个节点都粗略乘以 O(log n) 会得到过松的上界。实际有约一半节点是叶子，不下沉；约四分之一节点高度为 1；约八分之一节点高度为 2。总工作量可估计为：

`n/4 * 1 + n/8 * 2 + n/16 * 3 + ... = O(n)`

因此自底向上建堆是 O(n)。排序阶段要进行 n-1 次取顶，每次下沉最坏 O(log n)，所以是 O(n log n)。二者相加仍为 O(n log n)。输入即使已经有序，算法也不会像插入排序那样提前结束，最好、平均和最坏时间同阶。

循环下沉只使用几个下标和一个交换临时值，除输入数组外额外空间 O(1)。若写成递归下沉，会额外使用 O(log n) 调用栈。

## 适用与不适用场景

- 适合：要求原地排序、不能接受快速排序 O(n²) 最坏情况、又不需要稳定性的数组。
- 适合：内存预算严格，无法为归并排序提供 O(n) 辅助数组。
- 不太适合：要求稳定排序的记录。
- 不太适合：追求常见内存数据上的极致平均速度。堆排序访问位置跳跃，缓存局部性通常不如快速排序，常数也可能更大。
- 若只需前 k 大或持续动态取最大/最小值，应维护大小合适的堆或优先队列，而不是先对全部 n 个元素完整堆排序。

## 实现边界与三种语言差异

- 计算 `2*i+1` 时理论上可能溢出整数；安全实现可先判断 `i < heapSize/2`，或保证容器长度不会接近整数上限。
- `heapSize` 表示左闭右开的有效堆区，孩子比较必须排除已经进入有序后缀的元素。
- 相等孩子任选一个都不影响正确性，但交换条件通常应是严格变大，避免相等值之间做无意义移动。
- **C++**：`std::make_heap`、`std::push_heap`、`std::pop_heap` 提供堆操作，默认最大堆；要注意比较器语义和无符号下标向前递减时的下溢。
- **Java**：`PriorityQueue` 默认是最小堆，适合动态优先队列但不是原地堆排序；手写数组堆时所有下标为有符号 `int`，仍需注意乘法溢出和空数组。
- **Go**：标准库 `container/heap` 通过接口维护动态堆，默认顺序取决于 `Less` 定义；本文的原地堆排序直接在切片上操作，不需要接口装箱或额外容器。

## C++17 完整示例

```cpp
#include <iostream>
#include <utility>
#include <vector>

void siftDown(std::vector<int>& values, int root, int heapSize) {
    while (true) {
        int largest = root;
        int left = root * 2 + 1;
        int right = left + 1;
        if (left < heapSize && values[left] > values[largest]) largest = left;
        if (right < heapSize && values[right] > values[largest]) largest = right;
        if (largest == root) return;
        std::swap(values[root], values[largest]);
        root = largest;
    }
}

void heapSort(std::vector<int>& values) {
    int n = static_cast<int>(values.size());
    for (int root = n / 2 - 1; root >= 0; --root) {
        siftDown(values, root, n);
    }
    for (int end = n - 1; end > 0; --end) {
        std::swap(values[0], values[end]);
        siftDown(values, 0, end);
    }
}

int main() {
    std::vector<int> values{12, 11, 13, 5, 6, 7, 6};
    heapSort(values);
    std::cout << "堆排序: [";
    for (std::size_t i = 0; i < values.size(); ++i) {
        if (i > 0) std::cout << ", ";
        std::cout << values[i];
    }
    std::cout << "]\n";
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.Arrays;

public class Main {
    private static void siftDown(int[] values, int root, int heapSize) {
        while (true) {
            int largest = root;
            int left = root * 2 + 1;
            int right = left + 1;
            if (left < heapSize && values[left] > values[largest]) largest = left;
            if (right < heapSize && values[right] > values[largest]) largest = right;
            if (largest == root) return;
            swap(values, root, largest);
            root = largest;
        }
    }

    private static void heapSort(int[] values) {
        for (int root = values.length / 2 - 1; root >= 0; root--) {
            siftDown(values, root, values.length);
        }
        for (int end = values.length - 1; end > 0; end--) {
            swap(values, 0, end);
            siftDown(values, 0, end);
        }
    }

    private static void swap(int[] values, int i, int j) {
        int temp = values[i];
        values[i] = values[j];
        values[j] = temp;
    }

    public static void main(String[] args) {
        int[] values = {12, 11, 13, 5, 6, 7, 6};
        heapSort(values);
        System.out.println("堆排序: " + Arrays.toString(values));
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

func siftDown(values []int, root, heapSize int) {
	for {
		largest := root
		left := root*2 + 1
		right := left + 1
		if left < heapSize && values[left] > values[largest] {
			largest = left
		}
		if right < heapSize && values[right] > values[largest] {
			largest = right
		}
		if largest == root {
			return
		}
		values[root], values[largest] = values[largest], values[root]
		root = largest
	}
}

func heapSort(values []int) {
	for root := len(values)/2 - 1; root >= 0; root-- {
		siftDown(values, root, len(values))
	}
	for end := len(values) - 1; end > 0; end-- {
		values[0], values[end] = values[end], values[0]
		siftDown(values, 0, end)
	}
}

func main() {
	values := []int{12, 11, 13, 5, 6, 7, 6}
	heapSort(values)
	fmt.Println("堆排序:", values)
}
```

## 易错点

- 最后一个非叶节点是 `n/2 - 1`；叶节点无需下沉。
- 交换堆顶和堆尾后，恢复堆时的有效范围不包含已经就位的尾元素。
- 升序排序要建最大堆，降序排序要建最小堆。
- 下沉必须先比较两个孩子，再决定与哪个孩子交换。
- 建堆虽然调用了多次 O(log n) 下沉，但总复杂度是 O(n)，因为越靠近叶子的节点下沉高度越小。
- 示例使用循环下沉，避免了递归栈；空数组和单元素数组也能安全处理。
