# 基础排序：冒泡、选择与插入排序

## 学习目标

- 掌握三种 O(n²) 排序的循环不变量和手写实现。
- 理解稳定性、原地排序以及最好/最坏复杂度。
- 能根据数据规模和初始有序程度选择基础排序算法。

## 排序问题与三个基本概念

排序输入是一组可比较元素，输出必须同时满足两个条件：

1. 元素按指定键非递减或非递增排列。
2. 输出与输入包含完全相同的元素，只改变顺序，不丢失也不新增。

评价排序算法时，除了时间和空间，还要关注：

- **原地排序**：除少量临时变量外，不申请与输入规模同阶的额外空间。
- **稳定性**：两个排序键相等的记录，在排序后仍保持输入中的先后次序。例如学生按分数排序时，相同分数是否继续保留原报名顺序。
- **自适应性**：输入越接近有序，算法是否会明显更快。

冒泡、选择和插入排序最坏都为 O(n²)，但它们维护的不变量、数据移动方式和适用场景不同，不能只把它们记成三段相似的双重循环。

## 冒泡排序

### 直觉与循环不变量

冒泡排序反复比较相邻元素。如果左边大于右边，就交换二者。一次从左到右的扫描会使当前未排序区间的最大元素不断向右移动，最终停在该区间末尾。

第 `i` 轮开始前的不变量是：数组最右侧已经有 `i` 个元素处于最终位置，并且这部分元素不小于左侧未排序区间的任何元素。因此下一轮只需扫描到更靠左的位置。

### 逐步过程

对长度为 `n` 的数组升序排序：

1. 第一轮比较下标 `0/1、1/2、...、n-2/n-1`，逆序就交换。
2. 这一轮结束后，全局最大值位于下标 `n-1`。
3. 下一轮只扫描 `[0, n-2)` 的相邻对，因为最后一个元素已经就位。
4. 若某一整轮没有发生交换，说明所有相邻对都满足前者不大于后者，数组已经有序，可以提前结束。

### 手工推演

输入 `[5, 1, 4, 2, 8]`：

| 轮次 | 相邻比较与交换后的结果 | 本轮固定的尾部 |
| --- | --- | --- |
| 第 1 轮 | `5/1` 换、`5/4` 换、`5/2` 换、`5/8` 不换，得到 `[1, 4, 2, 5, 8]` | `[8]` |
| 第 2 轮 | `1/4` 不换、`4/2` 换、`4/5` 不换，得到 `[1, 2, 4, 5, 8]` | `[5, 8]` |
| 第 3 轮 | 所有相邻对已有序，没有交换 | 提前结束 |

### 为什么正确

一轮扫描中，只要较大的元素位于某个相邻对左侧，它就会被交换到右侧并继续参与下一次比较。因此未排序区间最大值最终一定到达最右端。每轮固定一个最大值，且不再触碰已固定尾部；重复至未排序区间只剩一个元素时，整个数组有序。

相等元素只有在使用严格条件 `left > right` 时才不交换，因此它们不会跨越彼此，冒泡排序是稳定的。若错误使用 `>=`，稳定性会被破坏。

## 选择排序

### 直觉与循环不变量

选择排序把数组分成“已排序前缀”和“未排序后缀”。每轮完整扫描后缀，找出其中最小元素，再把它与后缀第一个元素交换。

第 `i` 轮开始前的不变量是：区间 `[0, i)` 已经按序放置了全局最小的 `i` 个元素，并且不会再改变。当前任务是为下标 `i` 选择剩余元素中的最小值。

### 逐步过程

1. 令 `minIndex = i`。
2. 扫描 `i+1` 到 `n-1`；发现更小元素就更新 `minIndex`。
3. 扫描结束后，`minIndex` 指向整个未排序区间最小值。
4. 若 `minIndex != i`，交换 `a[i]` 与 `a[minIndex]`。
5. 已排序前缀扩大一位。

不能边扫描边随意把较小元素与 `a[i]` 交换；标准写法先记住最小值下标，整轮最多交换一次。这正是选择排序写入次数较少的原因。

### 手工推演

输入 `[5, 1, 4, 2, 8]`：

| 轮次 | 未排序区间 | 选中的最小值 | 交换后 |
| --- | --- | ---: | --- |
| `i=0` | `[5, 1, 4, 2, 8]` | 1 | `[1, 5, 4, 2, 8]` |
| `i=1` | `[5, 4, 2, 8]` | 2 | `[1, 2, 4, 5, 8]` |
| `i=2` | `[4, 5, 8]` | 4 | 不需交换 |
| `i=3` | `[5, 8]` | 5 | 不需交换 |

### 为什么正确

每轮扫描了整个未排序区间，因此选出的确是其中最小值。把它放到下标 `i` 后，前缀 `[0, i]` 恰好包含全局最小的 `i+1` 个元素且已排序，循环不变量得以维持。最后只剩一个元素时，它自然是最大值，数组完成排序。

标准选择排序通常不稳定。例如记录 `[2a, 2b, 1]` 按数值排序，第一轮把 1 与 2a 交换，得到 `[1, 2b, 2a]`，原本相同键的 2a、2b 次序被颠倒。若要稳定选择，可把最小元素取出并让中间区间整体右移，但会增加移动成本。

## 插入排序

### 直觉与循环不变量

插入排序类似整理手中的扑克牌：左侧牌已经排好，每次拿起一张新牌，从右向左寻找位置，把比它大的牌依次右移，再把新牌放入空位。

第 `i` 轮开始前的不变量是：前缀 `[0, i)` 已有序，并且包含输入前 `i` 个元素。当前元素 `key = a[i]` 要插入该前缀，完成后 `[0, i]` 有序。

### 逐步过程

1. 先保存 `key = a[i]`，因为后续右移会覆盖 `a[i]`。
2. 从 `j = i - 1` 向左扫描。
3. 只要 `a[j] > key`，就令 `a[j+1] = a[j]`，把较大元素右移。
4. 遇到 `a[j] <= key` 或越过数组左边界时停止。
5. 把 `key` 写到空出的 `j+1`。

使用右移而不是每步交换，通常能减少写操作。相等时停止并把新元素放在旧相等元素之后，可保持稳定性。

### 手工推演

输入 `[5, 2, 4, 6, 1, 3]`：

| 当前 `key` | 有序前缀的移动过程 | 插入后数组 |
| ---: | --- | --- |
| 2 | 5 右移，2 放到下标 0 | `[2, 5, 4, 6, 1, 3]` |
| 4 | 5 右移，遇到 2 停止 | `[2, 4, 5, 6, 1, 3]` |
| 6 | 6 不小于前缀尾 5，无需移动 | `[2, 4, 5, 6, 1, 3]` |
| 1 | 6、5、4、2 全部右移 | `[1, 2, 4, 5, 6, 3]` |
| 3 | 6、5、4 右移，遇到 2 停止 | `[1, 2, 3, 4, 5, 6]` |

### 为什么正确

每轮开始时前缀已经有序。右移所有严格大于 `key` 的元素后，空位左侧都不大于 `key`，右侧都大于 `key`，把 `key` 放入空位便得到更长的有序前缀。同时只是移动并重新放回原元素，没有丢失或新增元素。归纳到 `i = n-1` 后，整个数组有序。

## 复杂度

| 算法 | 最好时间 | 平均时间 | 最坏时间 | 额外空间 | 稳定性 |
| --- | --- | --- | --- | --- | --- |
| 冒泡排序（带提前结束） | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |

### 复杂度逐项推导

- **冒泡排序**：最坏要完成 `(n-1) + (n-2) + ... + 1 = n(n-1)/2` 次相邻比较，故为 O(n²)。带提前结束时，有序输入只扫描一轮 `n-1` 次，为 O(n)。交换次数等于被消除的相邻逆序数量。
- **选择排序**：无论输入是否有序，都要在剩余区间寻找最小值，比较次数固定为 `n(n-1)/2`，所以最好、平均、最坏都是 O(n²)。但每轮最多一次交换，总交换次数不超过 `n-1`。
- **插入排序**：移动次数与逆序对数量密切相关。有序输入每轮只比较一次，约 O(n)；逆序输入第 i 轮移动 i 个元素，总量为 `1+2+...+(n-1)=O(n²)`。近乎有序时逆序对少，实际性能接近线性。
- 三者都只使用若干下标和临时值，额外空间 O(1)。这里不计输入数组本身。

## 适用场景

- 教学、面试手写和极小规模数组。
- 插入排序适合近乎有序的数据，也常作为高级排序处理小区间的优化手段。
- 选择排序适合希望控制交换次数的简单场景。
- 冒泡排序直观但工程价值有限；大规模通用排序通常选择归并、快速、堆排序或标准库排序。

## 如何选择与何时不该使用

| 情况 | 更合适的选择 | 原因 |
| --- | --- | --- |
| 数据很少或几乎有序 | 插入排序 | 自适应，常数小，移动很快停止 |
| 元素写入非常昂贵，比较相对便宜 | 选择排序 | 最多 n-1 次交换 |
| 教学观察相邻逆序如何消除 | 冒泡排序 | 过程直观，可提前检测整体有序 |
| 要求稳定 | 冒泡或插入 | 严格比较条件下稳定 |
| 数据规模较大 | 通常都不选 | O(n²) 的增长过快，应使用 O(n log n) 算法 |

“n 很小”没有统一阈值，它取决于元素大小、比较成本、运行环境和标准库实现。工程中的快速排序、归并排序等经常在子区间缩小到十几个或几十个元素时切换到插入排序，而不是单独用基础排序处理全部大数组。

## 实现边界与三种语言差异

- 空数组和单元素数组应直接保持不变；循环边界不能产生负下标或访问 `i+1 == n`。
- 排序对象时，比较器必须具有一致、可传递的顺序关系，否则任何排序算法都可能得到不可预测结果。
- **C++**：交换大型对象可能触发移动构造；标准库 `std::sort` 通常使用混合策略，稳定需求应使用 `std::stable_sort`。手写模板时应通过比较器而不是写死整数比较。
- **Java**：对象数组可用 `Comparator`，基本类型数组避免装箱；移动对象变量移动的是引用，不是复制整个对象。比较器不应通过相减返回结果，因为整数溢出会破坏顺序。
- **Go**：示例直接处理 `[]int`；泛型实现可接受有序类型或显式比较函数。切片按引用语义共享底层数组，原地排序会让调用者看到元素顺序变化。

## C++17 完整示例

```cpp
#include <iostream>
#include <utility>
#include <vector>

void bubbleSort(std::vector<int>& values) {
    for (std::size_t end = values.size(); end > 1; --end) {
        bool swapped = false;
        for (std::size_t i = 1; i < end; ++i) {
            if (values[i - 1] > values[i]) {
                std::swap(values[i - 1], values[i]);
                swapped = true;
            }
        }
        if (!swapped) return;
    }
}

void selectionSort(std::vector<int>& values) {
    for (std::size_t i = 0; i < values.size(); ++i) {
        std::size_t minIndex = i;
        for (std::size_t j = i + 1; j < values.size(); ++j) {
            if (values[j] < values[minIndex]) minIndex = j;
        }
        if (minIndex != i) std::swap(values[i], values[minIndex]);
    }
}

void insertionSort(std::vector<int>& values) {
    for (std::size_t i = 1; i < values.size(); ++i) {
        int current = values[i];
        std::size_t j = i;
        while (j > 0 && values[j - 1] > current) {
            values[j] = values[j - 1];
            --j;
        }
        values[j] = current;
    }
}

void print(const std::string& name, const std::vector<int>& values) {
    std::cout << name << ": [";
    for (std::size_t i = 0; i < values.size(); ++i) {
        if (i > 0) std::cout << ", ";
        std::cout << values[i];
    }
    std::cout << "]\n";
}

int main() {
    const std::vector<int> source{5, 1, 4, 2, 8, 2};
    auto bubble = source;
    auto selection = source;
    auto insertion = source;
    bubbleSort(bubble);
    selectionSort(selection);
    insertionSort(insertion);
    print("冒泡排序", bubble);
    print("选择排序", selection);
    print("插入排序", insertion);
    return 0;
}
```

## Java 17 完整示例

```java
import java.util.Arrays;

public class Main {
    private static void bubbleSort(int[] values) {
        for (int end = values.length; end > 1; end--) {
            boolean swapped = false;
            for (int i = 1; i < end; i++) {
                if (values[i - 1] > values[i]) {
                    swap(values, i - 1, i);
                    swapped = true;
                }
            }
            if (!swapped) return;
        }
    }

    private static void selectionSort(int[] values) {
        for (int i = 0; i < values.length; i++) {
            int minIndex = i;
            for (int j = i + 1; j < values.length; j++) {
                if (values[j] < values[minIndex]) minIndex = j;
            }
            if (minIndex != i) swap(values, i, minIndex);
        }
    }

    private static void insertionSort(int[] values) {
        for (int i = 1; i < values.length; i++) {
            int current = values[i];
            int j = i;
            while (j > 0 && values[j - 1] > current) {
                values[j] = values[j - 1];
                j--;
            }
            values[j] = current;
        }
    }

    private static void swap(int[] values, int i, int j) {
        int temp = values[i];
        values[i] = values[j];
        values[j] = temp;
    }

    public static void main(String[] args) {
        int[] source = {5, 1, 4, 2, 8, 2};
        int[] bubble = source.clone();
        int[] selection = source.clone();
        int[] insertion = source.clone();
        bubbleSort(bubble);
        selectionSort(selection);
        insertionSort(insertion);
        System.out.println("冒泡排序: " + Arrays.toString(bubble));
        System.out.println("选择排序: " + Arrays.toString(selection));
        System.out.println("插入排序: " + Arrays.toString(insertion));
    }
}
```

## Go 完整示例

```go
package main

import "fmt"

func bubbleSort(values []int) {
	for end := len(values); end > 1; end-- {
		swapped := false
		for i := 1; i < end; i++ {
			if values[i-1] > values[i] {
				values[i-1], values[i] = values[i], values[i-1]
				swapped = true
			}
		}
		if !swapped {
			return
		}
	}
}

func selectionSort(values []int) {
	for i := 0; i < len(values); i++ {
		minIndex := i
		for j := i + 1; j < len(values); j++ {
			if values[j] < values[minIndex] {
				minIndex = j
			}
		}
		if minIndex != i {
			values[i], values[minIndex] = values[minIndex], values[i]
		}
	}
}

func insertionSort(values []int) {
	for i := 1; i < len(values); i++ {
		current := values[i]
		j := i
		for j > 0 && values[j-1] > current {
			values[j] = values[j-1]
			j--
		}
		values[j] = current
	}
}

func clone(values []int) []int {
	result := make([]int, len(values))
	copy(result, values)
	return result
}

func main() {
	source := []int{5, 1, 4, 2, 8, 2}
	bubble := clone(source)
	selection := clone(source)
	insertion := clone(source)
	bubbleSort(bubble)
	selectionSort(selection)
	insertionSort(insertion)
	fmt.Println("冒泡排序:", bubble)
	fmt.Println("选择排序:", selection)
	fmt.Println("插入排序:", insertion)
}
```

## 易错点

- 冒泡排序每轮的右边界应缩小；提前结束的标记必须在每轮开始时重置。
- 选择排序要记录最小值的下标，而不是只记录最小值，否则不便交换。
- 插入排序应先保存当前元素，再右移较大元素；直接覆盖会丢失待插入值。
- 稳定排序只在比较相等键且还关心原始次序时有意义；把 `>` 写成 `>=` 会破坏上述插入/冒泡实现的稳定性。
- 空数组和单元素数组不应触发越界。
