## 4. 统一 API 契约（与前端 08～11 对齐）

前端 Vue/React 08 章约定 **`code === 0` 表示成功**。Python 版保持一致，便于联调：

### 4.1 统一响应结构

```python
# app/schemas/common.py
from typing import Generic, TypeVar, Optional
from pydantic import BaseModel

T = TypeVar("T")

class Result(BaseModel, Generic[T]):
    code: int = 0
    message: str = "success"
    data: Optional[T] = None

    @classmethod
    def ok(cls, data: T = None, message: str = "success"):
        return cls(code=0, message=message, data=data)

    @classmethod
    def fail(cls, message: str, code: int = 1):
        return cls(code=code, message=message, data=None)
```

**逐行读 `Result` 统一响应**：

| 字段/方法 | 含义 | 改错会怎样 |
|-----------|------|------------|
| `code: int = 0` | 0 成功，非 0 业务失败 | 与前端 `=== 0` 不一致则联调崩 |
| `message` | 人类可读提示 | 前端 toast 直接展示 |
| `data: Optional[T]` | 泛型载荷 | 列表/对象/None 需前端分支 |
| `Result.ok(data=...)` | 工厂方法 | 避免各处手写 dict 字段漏写 |
| `Result.fail(code=401)` | 业务失败 | 与 HTTP 401 可同时存在，需约定以谁为准 |

## 5. 手把手：JWT 登录模块

### 5.1 依赖

```powershell
pip install "python-jose[cryptography]" passlib[bcrypt]
```

### 5.2 密码与 Token

```python
# app/core/security.py
from datetime import datetime, timedelta, timezone
from jose import jwt
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
SECRET_KEY = "change-me-in-production"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 60 * 24

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def create_access_token(subject: str) -> str:
    expire = datetime.now(timezone.utc) + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    return jwt.encode({"sub": subject, "exp": expire}, SECRET_KEY, algorithm=ALGORITHM)
```

### 5.3 登录路由

```python
# app/routers/auth.py
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session
from app.core.deps import get_db
from app.core.security import verify_password, create_access_token, hash_password
from app.schemas.common import Result
from app.schemas.auth import LoginRequest, RegisterRequest, TokenData

router = APIRouter(prefix="/api", tags=["auth"])

@router.post("/login")
def login(body: LoginRequest, db: Session = Depends(get_db)):
    user = user_service.get_by_username(db, body.username)
    if not user or not verify_password(body.password, user.password_hash):
        raise HTTPException(status_code=401, detail="用户名或密码错误")
    token = create_access_token(str(user.id))
    return Result.ok(TokenData(token=token, user=user_to_vo(user)))

@router.post("/register")
def register(body: RegisterRequest, db: Session = Depends(get_db)):
    if user_service.get_by_username(db, body.username):
        return Result.fail("用户名已存在")
    user_service.create(db, body.username, hash_password(body.password))
    return Result.ok(message="注册成功")
```

### 5.4 鉴权依赖

```python
# app/core/deps.py
from fastapi import Depends, HTTPException
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import jwt, JWTError
from app.core.security import SECRET_KEY, ALGORITHM

security = HTTPBearer()

def get_current_user_id(
    cred: HTTPAuthorizationCredentials = Depends(security),
) -> int:
    try:
        payload = jwt.decode(cred.credentials, SECRET_KEY, algorithms=[ALGORITHM])
        return int(payload["sub"])
    except (JWTError, ValueError):
        raise HTTPException(status_code=401, detail="Token 无效或已过期")
```

## 6. 订单模块核心逻辑

### 6.1 创建订单流程

```mermaid
sequenceDiagram
    participant FE as 前端
    participant API as FastAPI
    participant SVC as OrderService
    participant DB as MySQL
    participant RD as Redis
    participant MQ as Celery

    FE->>API: POST /api/orders + Bearer Token
    API->>SVC: create_order(user_id, dto)
    SVC->>RD: SETNX 幂等键
    alt 重复提交
        SVC-->>API: 409
    end
    SVC->>DB: BEGIN 事务
    SVC->>DB: INSERT order
    SVC->>DB: UPDATE stock WHERE stock >= qty
    alt 库存不足
        SVC->>DB: ROLLBACK
    end
    SVC->>DB: COMMIT
    SVC->>MQ: send_order_created.delay(order_id)
    SVC-->>API: OrderVO
    API-->>FE: Result.ok
```

### 6.2 扣库存 SQL（防超卖）

```sql
UPDATE product
SET stock = stock - :qty
WHERE id = :product_id AND stock >= :qty;
-- affected_rows == 0 表示库存不足，业务层抛异常并回滚
```

### 6.3 Service 伪代码

```python
def create_order(db: Session, user_id: int, dto: CreateOrderDTO) -> OrderVO:
    if not redis.set(f"idempotent:{dto.idempotency_key}", "1", nx=True, ex=300):
        raise BusinessException("请勿重复提交", code=409)

    with db.begin():
        product = product_repo.get_for_update(db, dto.product_id)
        if product.stock < dto.quantity:
            raise BusinessException("库存不足")
        order = order_repo.insert(db, user_id, dto, product)
        rows = product_repo.deduct_stock(db, dto.product_id, dto.quantity)
        if rows == 0:
            raise BusinessException("库存不足")
    send_order_created.delay(order.id)
    return order_to_vo(order)
```

## 12. 简历怎么写

### 不要这样写

- 熟悉 Kubernetes 微服务治理
- 精通分布式事务
- 海量高并发架构经验

### 推荐写法

```text
【项目名称】简易电商后端 API | FastAPI + SQLAlchemy + MySQL + Redis
【时间】2024.xx - 2024.xx
【描述】
- 实现 JWT 登录注册、商品分页/详情、订单创建与查询等 10+ REST 接口
- 商品详情 Redis 缓存（Cache Aside），降低 MySQL 读压力
- 下单 SQLAlchemy 事务 + 条件 UPDATE 扣库存，Redis 幂等防重复提交
- Celery + RabbitMQ 实现下单异步通知；docker-compose 部署全栈
【技术栈】Python 3.12, FastAPI, SQLAlchemy, MySQL 8, Redis, Celery, Docker
```

## 13. 常见报错与排查

| 报错/现象 | 原因 | 解决 |
|-----------|------|------|
| 前端 CORS 错误 | 未配 CORSMiddleware 或 origin 不对 | 加 5173 到 allow_origins |
| 401 但 token 刚登录 | 拦截器未带 Bearer 前缀 | `Authorization: Bearer ${token}` |
| `code` 解析失败 | 后端未统一 Result 结构 | 对齐 §4.1 格式 |
| 下单成功但库存未减 | 未 commit / 事务边界错误 | `with db.begin()` 或显式 commit |
| Celery 任务不执行 | worker 未启动 / broker 地址错 | `celery -A app.tasks worker -l info` |
| Redis 缓存不生效 | key 不一致或 TTL 为 0 | 打日志对比 key |
| 并发测试超卖 | 无行锁/条件 UPDATE | `SELECT FOR UPDATE` 或 §6.2 SQL |
| JWT 过期太快 | exp 配置过短 | 调整 ACCESS_TOKEN_EXPIRE_MINUTES |
| Swagger 401 | 未点 Authorize | docs 页填入 Bearer token |

## 16. FAQ

**Q1：项目要做到什么程度才够面试？**  
**能演示 + 能讲 15 分钟 + 每技术点能落到代码**；不必真实上线，但接口与 README 要完整。

**Q2：JWT 存在哪？Redis Session 呢？**  
MVP 常用 **无状态 JWT**（前端 localStorage）；Session 存 Redis 适合要服务端踢人场景——能说清选型即可。

**Q3：统一 Result `{code,message,data}` 有必要吗？**  
与 [Vue 08](../../前端学习/Vue/08-Axios网络请求与前后端联调.md) 拦截器对齐，减少前端分支判断；面试加分项。

**Q4：Week 1～4 必须严格按周吗？**  
表格是**建议里程碑**；全职可压缩，业余可拉长，关键是**交付物勾选**（§0.3）。

**Q5：没有前端怎么办？**  
Swagger + curl/Postman 走通全流程；联调可后补，但 API 契约先按 §4.1 设计好。

**Q6：防超卖用哪种方案？**  
MVP 推荐 **条件 UPDATE** `WHERE stock >= qty`；高并发再谈 Redis 预扣（12 章）。

**Q7：Celery 在项目中至少用在哪？**  
下单后通知、延迟关单（countdown）、异步写日志——选 **1～2 个**做透即可。

**Q8：简历能写「高并发秒杀」吗？**  
学习项目**不要夸大**；写「脚本模拟并发验证库存扣减」更可信。

**Q9：和 Java 10 项目实战怎么对照？**  
能力目标一致：JWT + CRUD + 缓存 + 事务 + MQ + Docker；栈换 FastAPI/SQLAlchemy。

**Q10：Git 仓库要公开吗？**  
面试前建议 **GitHub 公开 + README 架构图**；敏感信息用 `.env.example`。

**Q11：项目最难的部分怎么选故事？**  
选你**真踩过坑**的：缓存不一致、事务未 commit、CORS、Celery broker 地址错等。

**Q12：需要单元测试吗？**  
MVP 不强制；有时间可为 `order_service` 写 pytest，面试可提「下一步补测试」。

## 17. 闭卷自测

1. **概念**：demo-api 推荐目录分层（routers/services/models/schemas）各层职责？
2. **概念**：JWT 登录全流程涉及哪些接口与 Header？
3. **概念**：商品详情 Cache Aside 读写顺序？
4. **概念**：下单事务应包含哪些步骤？commit 边界在哪？
5. **概念**：Celery 在项目中解决什么问题？为什么不放 HTTP 里同步做？
6. **概念**：docker-compose 里应有哪些与 demo-api 相关的服务？
7. **动手**：写出统一 Result 成功时 JSON 结构示例。
8. **动手**：列出简历项目描述中应避免的 3 种夸大写法。
9. **综合**：用 STAR 模板 5 句话讲「缓存与 DB 不一致」的排查与修复。
10. **综合**：15 分钟项目讲解的 7 步结构是什么？（见 §11）

### 自测参考答案

1. routers 接 HTTP；services 业务；models ORM；schemas Pydantic 入参出参。
2. 注册/登录发 token；后续请求 `Authorization: Bearer <token>`；Depends 解析 user。
3. 读：先 Redis miss 再 DB 并回写；写：先更 DB 再 DEL 缓存。
4. 校验库存/用户 → INSERT order → 条件 UPDATE stock → 同一事务 commit。
5. 异步通知/关单；附属流程延迟可接受，解耦主链路 RT。
6. mysql、redis、rabbitmq、api、worker（+ nginx 可选）。
7. `{"code":0,"message":"ok","data":{...}}`
8. 精通 K8s、海量高并发、分布式事务（未做却写）。
9. Situation 缓存旧价；Task 保证更新后读新；Action 先更 DB 再删 key；Result 二次 GET 正确。（开放题）
10. 背景 → 技术栈 → 职责 → 核心流程 → 难点1 → 难点2 → 不足与迭代（§11）。
