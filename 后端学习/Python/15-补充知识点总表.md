## 21. 总表使用 FAQ

**Q1：总表能代替 01～14 章吗？**  
**不能**。总表只有索引；学习必须看分章手把手与代码。

**Q2：⬜ 🔶 ✅ 怎么标？**  
⬜ 听说过；🔶 能写 demo；✅ 能面试讲 3 分钟。

**Q3：Java 和 Python 总表都要背吗？**  
**通用知识**（MySQL/Redis/HTTP）背一份；框架对照看 §16。

**Q4：第一优先级没学完能面试吗？**  
校招实习：**01～07 + 10** 扎实比 11～14 概念堆砌更重要。

**Q5：04～09 在总表里为何重要？**  
工程主线：接口 → ORM → DB → 缓存 → MQ → 部署 → 项目。

**Q6：算法 40 题和项目哪个优先？**  
**项目能讲**优先；算法每日 1 题维持手感（13 章）。

**Q7：Django 在哪？**  
本路线 FastAPI；Django 见 §20 延伸，00 FAQ 也有说明。

**Q8：前端要学到哪才能联调？**  
[Vue 08](../../前端学习/Vue/08-Axios网络请求与前后端联调.md) 或 React 08 + 10 章 API 契约。

**Q9：总表更新频率？**  
学完一章就更新自评；15 章本身随路线扩充而更新。

**Q10：和 Java 15 总表怎么配合？**  
MySQL/Redis/场景题 **交叉复习**；Web 层各看各栈 §16 对照。

**Q11：面试前 30 分钟怎么用？**  
§17 自评扫 ⬜ → 对应 §2～§13 回链 → 14 章 STAR 过一遍。

**Q12：C++ / 数据结构总表要吗？**  
算法原理见 [数据结构 12](../数据结构/09-综合复习/README.md)；系统底层见 C++ 路线。

**Q13：只会 Python 不会 Linux 部署？**  
09 章 + [Linux 12 Docker](../Linux/07-Docker与Compose.md) 最低限度。

**Q14：微服务总表项不会怎么办？**  
诚实说 MVP 单体；11 章概念能画架构即可。

**Q15：如何知道「学完了」？**  
§17 自评 ≥80% ✅ + 10 章项目可演示 + 14 章 4 场景 STAR + 13 章 40 B 题。
