# Go AI Agent 学习库

> **定位**：面向 Go 后端开发者的 AI 应用工程路线，重点是模型 API、流式对话、Tool Calling、RAG、Agent 编排、评估、安全与生产化。  
> **当前主线关系**：先完成 Go HTTP/Gin、MySQL、Redis 与普通后端项目，再把本目录作为应用层扩展。  
> **最后系统核验**：2026-07-15。

本目录不再以 Java、Spring AI 或 LangChain4j 为前置。核心概念先用语言无关方式解释，工程实现统一采用 Go；OpenAI 特定示例优先使用 Responses API，其他厂商必须按 Chat、Streaming、Tool、Structured Output、Embedding 等能力逐项验证，不能只因为“OpenAI 兼容”就假设全部端点可用。

## 1. 从哪里开始

第一次进入时，按下面顺序：

1. 先读 [00 学习路线图](./00-学习路线图与说明.md)，确认当前是否满足前置。
2. 按 01～08 完成应用工程核心。
3. 09～10 属于协议、部署与原理选修，不必阻塞项目。
4. 进入 [11 Go 端到端项目](./11-Go端到端Agent项目实战.md)，把前面的能力串起来。
5. 用 [12 面试专题](./12-面试专题与知识点总表.md) 做证据化复盘。
6. 所有可运行代码统一从 [`Go工程/agentgo`](./Go工程/agentgo/README.md) 进入。

如果你当前仍在学习 Gin、MySQL 或 Redis，不建议现在顺序通读本目录。可以先读 01 的概念部分，然后回到 Go 主线。

## 4. 可运行工程

[`Go工程/agentgo`](./Go工程/agentgo/README.md) 不是文档里的“伪完整项目”，而是本目录的验证基线：

- Go 1.26 module；
- Gin HTTP API；
- 无 API Key 时使用可预测 Mock Provider；
- 配置 Key 后可调用 OpenAI Responses API；
- POST + SSE 流式响应；
- 有界 Tool Calling 循环；
- 带租户过滤的内存 RAG；
- 可选 PostgreSQL + pgvector 存储；
- 会话窗口、请求取消、超时与错误分类；
- 单元测试；真实模型与数据库集成需要凭据或容器另行验证。

当前状态必须分开理解：V1 的 Mock/内存路径可离线运行并由单测覆盖；V2 已提供 OpenAI Responses 与 pgvector adapter，但真实账号、真实模型和数据库容器不是默认单测的一部分；V3 仍是生产增强设计，不是已完成功能。

最小验证：

```powershell
cd F:\study\后端学习\AIAgent\Go工程\agentgo
go test ./...
go run ./cmd/server
Invoke-RestMethod http://localhost:8080/health
```

默认 Mock 模式不需要密钥，也不会访问外网。

## 6. 本资料坚持的工程原则

1. **先普通后端，后 AI 层**：Agent 不能替代 HTTP、数据库、缓存、鉴权与测试。
2. **先确定性流程，后自主编排**：能用普通代码解决的步骤，不强行交给模型决定。
3. **模型输出一律不可信**：结构化输出仍要做业务校验；Tool 参数仍要鉴权。
4. **不记录原始 Chain-of-Thought**：只记录请求、工具、结果摘要、状态、时延和错误。
5. **引用不等于忠实**：返回了 citation，不代表答案真的被证据支持，必须评估。
6. **供应商能力逐项核验**：Chat 可用不代表 Embedding、Tool 或 Responses API 可用。
7. **不编造性能和简历数字**：没有复现实验就写“尚未测量”。
8. **复杂组件由指标驱动**：多 Agent、Milvus、微调、MCP 都不是默认必选。
9. **每次升级都保留回归集**：版本、Prompt、模型、检索参数变化后重新跑同一组评测。
10. **文档与代码互相校验**：重要实现以工程源码和测试为准，Markdown 负责解释。

## 7. 版本与时效说明

AI API、模型名和框架版本变化很快。本目录采用以下策略：

- 稳定原理写在核心正文；
- 易变模型、价格、端点和依赖放到附录 A；
- 示例模型通过环境变量配置，不把“最新模型”写死在业务代码；
- OpenAI 特定实现以官方开发者文档为准；
- 其他供应商必须核验官方能力表；
- 每次修改工程后运行 `99-工具与验证/verify_aiagent.ps1 -Run`。

## 9. 当前推荐结论

对正在学习 Go 后端的读者，本目录是后置扩展，不是现在替换 Gin/MySQL/Redis 的新主线。先把一个普通后端项目做正确，再学习如何把模型作为一个有配额、有延迟、会失败、输出不可信的外部依赖接进系统。
