### 0.1 用一句话弄懂本系列

**LLM Infra** = 让大模型 **算得快、装得下、批得起、分布式跑起来** 的那一层代码——不是 Spring 调 API，而是 **vLLM、TensorRT-LLM、DeepSpeed、FlashAttention、NCCL** 所在的世界。

## 11. 闭卷自测

1. LLM Infra 和 AIAgent 的分工？
2. PagedAttention 解决什么问题？
3. 为什么推理岗要学 CUDA 而不只学 Python？
4. DP、TP、PP 各是什么？
5. INT8 量化 calibration 做什么？
6. Continuous batching 与 static batching 区别？
7. pybind11 典型用途？
8. FlashAttention 优化的是 CPU 还是 GPU 瓶颈？
9. 读 vLLM 源码建议从哪几个目录开始？
10. 本系列前置 C++ 最少要到哪一章？

<details>
<summary>参考答案</summary>

1. Infra=C++/CUDA 推理训练系统；Agent=Java 应用层 RAG/Tool。
2. KV Cache 显存碎片与浪费；按 block 分页分配。
3. 算子、调度、内存管理在 C++/CUDA；Python 是胶水。
4. 数据并行、张量并行、流水线并行。
5. 用校准集统计激活范围，定 scale/zero point。
6. Continuous 动态插入完成序列；static 固定 batch 等 padding。
7. 把 C++ 算子/export 给 PyTorch Python 层调用。
8. GPU HBM 带宽（减少 materialize 大 attention 矩阵）。
9. `vllm/core/scheduler.py`、`vllm/attention/`、`csrc/`。
10. 至少 01～09、10～12；建议 18～20 完成后再 13。

</details>
