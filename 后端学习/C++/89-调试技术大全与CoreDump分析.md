## §1 GDB 命令速查表

| 命令 | 用途 | 示例 | 备注 |
|------|------|------|------|
| `break` | 断点 | `break file.cpp:42` | 条件：break 42 if pid==0 |
| `watch` | 写观察点 | `watch var` | 变量被写时停 |
| `rwatch` | 读观察点 | `rwatch buf[0]` | 检测非法读附近 |
| `catch throw` | 捕获 C++ 异常 | `catch throw` | 异常抛出时停 |
| `bt / backtrace` | 调用栈 | `bt full` | 含局部变量 |
| `frame` | 切换栈帧 | `frame 3` | 看上层调用 |
| `info threads` | 线程列表 | `info threads` | 多线程必备 |
| `thread apply all bt` | 全线程栈 | `thread apply all bt` | 死锁定位 |
| `x/Nxf addr` | 检查内存 | `x/16xb ptr` | 十六进制字节 |
| `p *ptr` | 打印解引用 | `p *std_ptr` | 注意 pretty printer |
| `set pagination off` | 关分页 | `脚本化调试` | 自动化 |
| `record` | 反向调试 | `record ; reverse-step` | 需 record 目标支持 |

## §2 断点进阶

```gdb
break HttpServer::handle if fd < 0
commands 1
  silent
  bt 3
  continue
end
```
**条件断点**减少噪音；**commands** 自动打栈继续。

## §3 Watchpoint 与内存越界

```gdb
watch *(int*)0x7fff...
rwatch buffer[128]   # 读监控
```
硬件 watchpoint 数量有限（通常 4）；大范围用 ASan（12 章）。

## §4 Core Dump 生成与分析

```bash
ulimit -c unlimited
./app   # crash 后产生 core 或 core.pid
gdb ./app core
(gdb) bt
(gdb) info registers
(gdb) list
```

**生产环境**：`gcore PID` 不杀进程；或 systemd `coredumpctl`。

## §5 内存图与对象布局

```gdb
p sizeof(MyClass)
p &obj
x/32xb &obj
info vtbl obj   # 虚表（需调试符号）
```
对照 [29 章 vtable](29-对象模型与虚函数表深入.md)。

## §6 多线程与死锁

```gdb
info threads
thread 2
bt
thread apply all bt
```
两线程互持锁：栈上常见 `pthread_mutex_lock` 等待。

## §11 FAQ

1. **core 文件在哪？** 当前目录或 `/var/lib/systemd/coredump/`；`coredumpctl list`。
2. **Release 看不到行号？** 编译加 `-g`；RelWithDebInfo；保留符号。
3. **watchpoint 不触发？** 变量被优化到寄存器；`set var` 或 -O0 调试版。
4. **GDB 与 LLDB？** Linux C++ 常用 GDB；macOS 部分用 LLDB。
5. **反向调试慢？** record 开销大；仅短片段使用。
6. **和 61 章区别？** 61 线上分层；89 调试器深度。
