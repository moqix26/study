## 1. 运算符优先级与结合性

C++ 表达式不是「从左到右随便算」，而是按**优先级**（谁先算）和**结合性**（同级从左还是右）决定求值顺序。

### 1.1 优先级速查表（常用层）

| 优先级（高→低） | 运算符示例 | 记忆口诀 |
|-----------------|------------|----------|
| 1 | `()` `[]` `->` `.` `++` `--`（后缀） | 先找「东西本身」 |
| 2 | `++` `--`（前缀） `+` `-` `!` `~` `*` `&` | 一元修饰 |
| 3 | `*` `/` `%` | 乘除模 |
| 4 | `+` `-` | 加减 |
| 5 | `<<` `>>` | 移位 |
| 6 | `<` `<=` `>` `>=` | 关系 |
| 7 | `==` `!=` | 相等 |
| 8 | `&` | 按位与 |
| 9 | `^` | 按位异或 |
| 10 | `\|` | 按位或 |
| 11 | `&&` | 逻辑与（短路） |
| 12 | `\|\|` | 逻辑或（短路） |
| 13 | `?:` | 三元 |
| 14 | `=` `+=` `-=` … | 赋值 |
| 15 | `,` | 逗号（最低） |

> **C++17/20 提示**：不要依赖未定义的顺序如 `i = i++`；C++17 起多数内置运算符求值顺序更明确，但**仍应写清晰代码**。


### 术语三件套：运算符优先级（Operator Precedence）

| 维度 | 内容 |
|------|------|
| **定义** | 决定表达式中多个运算符**谁先被计算**的规则。 |
| **生活类比** | 像数学里「先乘除后加减」，但 C++ 层数更多。 |
| **为什么重要** | 写错优先级会导致逻辑 bug 且难调试；面试常考 `*p++` 类题。 |
| **本章用到** | §1、§1.2 小例子 |


### 1.2 小例子：优先级陷阱

```cpp
#include <iostream>

int main() {
    int a = 2, b = 3;
    std::cout << a + b * 4 << '
';       // 14，不是 20
    std::cout << (a + b) * 4 << '
';     // 20
    bool ok = a == b - 1;                 // a==2，true
    std::cout << std::boolalpha << ok << '
';
    return 0;
}
```

**预期输出**：
```text
14
20
true
```

### 1.3 小例子：逻辑短路

```cpp
#include <iostream>

int main() {
    int x = 0;
    // 短路：右侧不执行
    if (x != 0 && 10 / x > 1) {
        std::cout << "不会到这里
";
    } else {
        std::cout << "安全分支
";
    }
    return 0;
}
```

若写成 `10 / x` 且无短路保护，**除零未定义行为**。


### 逐行读代码：指针与递增的优先级

```cpp
int arr[] = {10, 20, 30};
int* p = arr;
int v1 = *p++;   // 先取 *p，再 p 指向下一元素
int v2 = (*p)++; // 先取 *p，再对当前元素自增
std::cout << v1 << ' ' << v2 << ' ' << arr[0] << ' ' << arr[1] << '\n';
```

| 行/片段 | 含义 | 改错会怎样 |
|---------|------|------------|
| `int* p = arr` | p 指向 arr[0] | 类型错则解引用崩溃 |
| `*p++` | 后缀++ 优先级高于* | 写成 `(*p)++` 语义不同 |
| `(*p)++` | 对 p 所指 int 自增 | 对指针自增用 `++p` |
| `arr[0]` | 第一个元素可能被改 | — |


## 2. 整型提升 Integer Promotion


### 术语三件套：整型提升（整型提升 Integer Promotion）

| 维度 | 内容 |
|------|------|
| **定义** | 较小整型（如 `char`、`short`）在运算前**提升为 int**（若 int 能装下所有值）。 |
| **生活类比** | 把小杯子里的水倒进标准杯再运算。 |
| **为什么重要** | 避免 CPU 用非原生宽度寄存器做算术；也影响重载决议。 |
| **本章用到** | §2 |

```cpp
#include <iostream>
#include <type_traits>

int main() {
    char c = 'A';
    short s = 100;
    auto r = c + s;  // int + int
    std::cout << typeid(r).name() << ' ' << r << '\n';
    std::cout << std::is_same_v<decltype(c + s), int> << '\n'; // 1
    return 0;
}
```

## 3. 寻常算术转换 Usual Arithmetic Conversions


### 术语三件套：寻常算术转换（寻常算术转换 Usual Arithmetic Conversions）

| 维度 | 内容 |
|------|------|
| **定义** | 二元运算两侧类型不同时，转换为**公共类型**（通常向更大范围或浮点）。 |

## 4. 截断与窄化 Narrowing


### 术语三件套：截断与窄化（截断与窄化 Narrowing）

| 维度 | 内容 |
|------|------|
| **定义** | 把大范围/高精度值赋给小类型时**丢失信息**；列表初始化 `{}` 禁止窄化。 |
| **生活类比** | 把 3.99 升强制记录为 3。 |
| **为什么重要** | 静默截断是经典 bug；C++11 `{}` 帮你编译期拦截。 |

```cpp
    std::cout << approx << ' ' << good << '\n';
    return 0;
}
```

## 8. auto 类型推导


### 术语三件套：auto（auto 类型推导）

| 维度 | 内容 |
|------|------|
| **定义** | 编译器**从初始化式**推导类型；引用需 `auto&`，指针 `auto*`。 |
| **生活类比** | 让编译器根据右边猜左边盒子大小。 |
| **为什么重要** | 少写冗长迭代器类型；但别在没初始化时用 auto。 |
| **本章用到** | §8 |

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v{1, 2, 3};
    auto it = v.begin();       // vector<int>::iterator
    auto& r = v[0];            // int&
    const auto& cr = v[1];     // const int&
    r = 99;
    for (auto x : v) std::cout << x << ' ';  // 拷贝
    std::cout << '\n';
    for (auto& x : v) x *= 2;  // 修改原容器
    return 0;
}
```

## 9. decltype 与 decltype(auto)

|------|------|

```cpp
    std::cout << f(x) << '\n';
    return 0;
}
```

## 10. 表达式求值顺序（C++17 要点）

| 表达式形式 | C++17 顺序保证 |
|------------|----------------|
| `a[b]` | `b` 先于 `a` |
| `a << b` | `a` 先于 `b` |
| 函数调用 `f(a,b)` | 参数求值顺序**未规定**（避免依赖） |
| 赋值 `a = b` | **右先于左** |

```cpp
#include <iostream>

int side(int& x, int v) {
    x = v;
    return v;
}

int main() {
    int a = 0;
```

## FAQ


**Q：`unsigned` 和 `int` 比较谁转成谁？**  
A：有符号转无符号；若 int 为负，会变成很大 unsigned，比较结果常出乎意料。

**Q：`auto` 能推导数组吗？**  
A：会退化为指针，除非绑定引用 `auto&` 到数组。

**Q：`decltype((x))` 为什么有时是引用？**  
A：若表达式是左值且带括号形成 lvalue 表达式，decltype 得引用类型。

**Q：字面量 `2` 和 `2.0` 在 `/` 中区别？**  
A：`2/3` 整数除法；`2.0/3` 浮点除法。

**Q：列表初始化 `{}` 和 `()` 区别？**  
A：`{}` 禁止窄化，且与 initializer_list 有关（41 章）。

**Q：C++20 的 `consteval` 和 `constexpr`？**  
A：`consteval` 必须编译期求值，否则错误；`constexpr` 可运行时 fallback。

**Q：为什么推荐 `std::size_t` 做下标？**  
A：与 `vector::size()` 同类型，避免 signed/unsigned 警告。

**Q：`*p++` 先解引用还是先 ++？**  
A：后缀 `++` 优先级高：等价于 `*(p++)`，先取值再移动指针。
