## 1. 三种「未定义」层次

在深入案例前，先分清标准用语：

| 类别 | 程序是否仍「有效」 | 例子 |
|------|-------------------|------|
| **UB** | 否；可崩溃、可假装成功 | 有符号溢出、data race |
| **未指定（unspecified）** | 是；多种结果 | 函数实参求值顺序（C++14 前部分） |
| **实现定义（implementation-defined）** | 是；实现须文档化 | `char` 是否有符号 |

**陷阱**：把 UB 当成「结果随机」——错。一旦 UB，整个程序语义消失，不存在「随机但合法」的输出。

## 2. 序点、求值顺序与表达式陷阱

### 2.1 sequenced-before 规则

C++17 起用 **sequenced-before / unsequenced / indeterminately sequenced** 描述求值顺序。

```cpp
int i = 0;
int x = i++ + i++;  // UB：对 i 的两次修改 unsequenced
int y = ++i + i++;  // UB
```

**标准立场**：若两个副作用对同一内存位置 unsequenced，且至少一个是写 → **UB**。

### 2.2 函数实参求值顺序

```cpp
void f(int a, int b, int c);
f(i++, i++, i++);  // UB
```

C++17 起赋值与复合赋值的左右操作数**左到右** sequenced；普通函数调用实参仍**无规定顺序**。

### 2.3 短路求值保证

```cpp
if (p != nullptr && *p == 42) { /* OK */ }
// && 左侧 sequenced-before 右侧
```

### 2.4 printf 经典坑

```cpp
int n = 0;
printf("%d %d\n", n++, n++);  // UB，不是实现定义
```

面试结论：**没有标准答案的输出**。

## 3. 数组与指针越界

### 3.1 vector 越界

```cpp
#include <vector>
void oob() {
    std::vector<int> v{1, 2, 3};
    int x = v[99];   // UB
    // int y = v.at(99);  // 抛 out_of_range，定义良好
}
```

`operator[]` 为性能不检查；见 [04 章](04-STL标准库容器与算法.md)、[44 章](44-vector-deque-string容器原理与实务.md)。

### 3.2 指针算术 one-past-end

```cpp
int arr[3] = {1, 2, 3};
int* end = arr + 3;  // OK：尾后指针
// int z = *end;     // UB：解引用尾后
```

### 3.3 负下标与 ptrdiff_t

```cpp
int a[5];
int* p = a + 5;
// p[-1] 合法（若 p 指向尾后）；p[-6] UB
```

## 4. 悬垂引用与 use-after-free

### 4.1 返回局部引用

```cpp
const std::string& bad() {
    std::string s = "tmp";
    return s;  // UB：悬垂引用
}
```

### 4.2 迭代器失效

```cpp
std::vector<int> v{1, 2, 3};
int& r = v[0];

```

### 4.3 string_view 悬垂

```cpp

```

## 5. 违反 ODR

### 5.1 规则回顾

每个非 inline 实体在整个程序中**恰好一个定义**；类定义在头文件可多次但须**逐字相同**。详见 [48 章](48-编译预处理与链接原理.md)。

### 5.2 典型违反

```cpp
// a.cpp
int compute() { return 1; }
// b.cpp
int compute() { return 2; }  // ODR 违反，未定义行为
```

链接器可能报 duplicate symbol，也可能静默链接某一版本。

### 5.3 inline 与模板

- 函数模板定义通常放头文件（ODR 要求相同）
- `inline` 变量 C++17 起也需跨 TU 一致

## 6. Strict Aliasing

### 6.1 规则

通过非 `char`/`unsigned char`/`std::byte` 的左值访问对象，若动态类型不符 → UB。

```cpp
float f = 1.0f;
int* p = reinterpret_cast<int*>(&f);
int bits = *p;  // UB（严格别名）
```

### 6.2 合法路径

```cpp
#include <bit>
#include <cstring>
uint32_t u = std::bit_cast<uint32_t>(f);  // C++20
// 或 memcpy(&u, &f, 4);
```

### 6.3 编译器优化

编译器假设 `float*` 与 `int*` 不别名，可合并加载；错误 punning 导致 Release 与 Debug 行为分裂。

## 7. 有符号溢出与移位 UB

```cpp
int a = INT_MAX;
int b = a + 1;           // UB
unsigned u = UINT_MAX;
unsigned v = u + 1;      // 模 2^n，定义良好

int x = 1 << 31;         // UB（32 位 int 常如此）
int y = -1 >> 1;         // 实现定义（算术右移）
```

GCC `-fwrapv` 强制有符号环绕（性能损失）。

## 8. 未初始化、空指针、双重释放

```cpp
int x;
if (x == 0) {}           // UB：indeterminate value

// *p = 1;               // UB

// delete ???;           // 双重释放 UB
```

## 9. 并发 UB（衔接 25 章）

```cpp
bool flag = false;       // 非 atomic
// 线程1: flag = true;
// 线程2: while (!flag) {}
// → data race → UB
```

**volatile 不能用于线程同步**。见 [25 章](25-无锁编程与内存序.md)、[08 章](08-多线程与并发编程.md)。

## 10. UB 驱动的编译器优化

```cpp
void f(int* p) {
    if (p == nullptr) return;
    *p = 42;
    if (p == nullptr) {
        do_critical_cleanup();  // 可能被优化掉！
    }
}
```

解引用后编译器假定 `p` 非空，第二个检查恒 false。

**Lemon Market**：代码「在 Windows 上能跑」不代表正确；可能是 UB 碰巧未触发。

## 12. 陷阱案例库

### 12.1 vector<bool> 代理

`vector<bool>` 返回代理对象，非真引用；`auto&`、取地址可能失败。

### 12.2 lambda 引用捕获悬垂

```cpp
auto make() {
    int x = 0;
    return [&]() { return x; };  // x 析构后 UB
}
```

### 12.3 moved-from unique_ptr

```cpp
auto p = std::make_unique<int>(1);
auto q = std::move(p);
// *p;  // UB
```

### 12.4 虚调用 use-after-free

对象已析构仍通过基类指针虚调用 → UB；见 [29 章](29-对象模型与虚函数表深入.md)。

### 12.5 枚举 switch 未覆盖

```cpp
enum class E { A, B };
E e = static_cast<E>(99);
switch (e) { case E::A: break; case E::B: break; }
// 若落到 default 外 → UB（C++）
```

### 12.6 线程 detach + 引用捕获

```cpp
void bad() {
    int local = 0;
    std::thread([&]{ use(local); }).detach();
}  // local 销毁，线程仍可能运行
```

### 12.7 memset 非平凡类型

对非平凡可复制对象 `memset` → UB；应用默认成员初始化或赋值。

### 12.8 递归无限深度

栈溢出 → UB。

## 17. FAQ

**Q：UB 和未指定行为区别？**  
未指定：多种结果，程序仍有效。UB：无意义。

**Q：assert 能防 UB 吗？**  
仅 Debug；Release 关掉后仍 UB。

**Q：与 [07 异常](07-异常处理与RAII.md) 关系？**  
异常是定义良好的控制流；UB 无法 catch。

**Q：智能指针能消灭 UB 吗？**  
减少 UAF/双删；不防悬垂、迭代器失效、data race。

## §21 placement new 与显式析构

在预分配缓冲区构造对象：

```cpp
#include <new>
alignas(T) std::byte buf[sizeof(T)];
T* p = new (buf) T(args...);
p->~T();  // 必须显式析构；delete p 错误（非堆分配）
```

忘记显式析构 → 析构函数未调用（资源泄漏）；对非平凡析构类型双重析构 → UB。


## §22 std::launder 与对象生命周期

C++17 `std::launder` 解决「同一存储反复构造不同类型」的指针可达性：

```cpp
#include <new>
struct A { int x; };
struct B { int y; };
std::byte storage[sizeof(A)];
auto* a = new (storage) A{1};
a->~A();
auto* b = new (storage) B{2};
// 通过原 A* 读 B → UB；需 launder
auto* bp = std::launder(reinterpret_cast<B*>(storage));
```

与 [41 章](41-构造析构与三五法则大全.md) 对象生命周期衔接。


## §23 goto 跨越初始化

```cpp
void f(bool skip) {
    if (skip) goto end;
    int x = 42;  // 若 skip 为 true，goto 跳过初始化 → UB
end:
    return;
}
```

用 `{}` 块或 `std::optional` 替代。


## §24 联合体非活跃成员

读取非当前活跃 union 成员 → UB（C++17 起有更严规则）：

```cpp
union U { int i; float f; };
U u{.i = 1};
// float x = u.f;  // UB（除非通过 char* 逐字节检查）
```


## §25 信号处理与 async-signal-safe

信号处理函数只能调用 async-signal-safe 函数；`malloc`、`printf` 不安全。见 [11 章](11-Linux与系统编程入门.md)。


## §26 setjmp/longjmp 与析构

`longjmp` 跳过栈展开 → 局部 RAII 析构不执行 → 资源泄漏或 UB。C++ 代码避免与异常混用。


## §27 常量表达式中的 UB

`constexpr` 函数若在编译期求值路径触发 UB → 编译失败；运行时路径仍可能 UB。


## §28 浮点 NaN 与比较

`NaN == NaN` 为 false；某些浮点环境违反 IEC 559 假设时操作可能 UB。


## §29 offsetof 限制

`offsetof` 仅用于标准布局类型；非标准布局用 `offsetof` → UB。


## §30 位域可移植性

位域布局实现定义；跨平台序列化位域 → 坑。
