## 2. Concepts（概念）

### 2.1 动机

```cpp
// C++17：错误信息可能几十行 SFINAE
template<typename T>
typename T::value_type get_val(T& c) { return *c.begin(); }

// C++20：意图清晰
#include <concepts>
#include <iterator>

template<std::ranges::range R>
auto get_first(R& r) {
    return *std::ranges::begin(r);
}
```

### 2.2 定义与 requires

```cpp
#include <concepts>
#include <type_traits>

template<typename T>
concept Integral = std::is_integral_v<T>;

template<Integral T>
T double_it(T x) { return x * 2; }

template<typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> std::convertible_to<T>;
};

template<Addable T>
T sum2(T a, T b) { return a + b; }
```

### 2.3 标准 concepts 速查

| concept | 含义 |
|---------|------|
| `std::same_as<T,U>` | 类型相同 |
| `std::derived_from<D,B>` | 继承关系 |
| `std::convertible_to<F,T>` | 隐式转换 |
| `std::regular` | 可复制、可比较等 |
| `std::ranges::range` | 有 begin/end |
| `std::invocable<F,Args...>` | 可调用 |

### 2.4 abbreviated function template

```cpp
void print(std::integral auto x) {
    std::cout << x << '\n';
}
```

**Infra 实践**：公共头文件模板 API 加 concept，减少误用 `TensorView` / `Buffer` 类型。

## 3. Ranges

### 3.1 核心思想

**Range** = 能拿到 `begin`/`end` 的东西；**views** 惰性变换，不立刻分配。

```cpp
#include <ranges>
#include <vector>
#include <iostream>

namespace rv = std::ranges::views;

int main() {
    std::vector<int> v{1, 2, 3, 4, 5, 6};
    auto even_squares = v
        | rv::filter([](int x){ return x % 2 == 0; })
        | rv::transform([](int x){ return x * x; });

    for (int x : even_squares)
        std::cout << x << ' ';  // 4 16 36
}
```

### 3.2 常用 views

| view | 作用 |
|------|------|
| `filter` | 谓词筛选 |
| `transform` | 映射 |
| `take` / `drop` | 头尾切片 |
| `reverse` | 逆序 |
| `keys` / `values` | 对 map 适配（C++23 增强） |
| `zip` | 多 range 并行（C++23） |

### 3.3 ranges 算法

```cpp
#include <algorithm>
#include <ranges>

std::vector<int> data{3, 1, 4, 1, 5};
std::ranges::sort(data);
bool ok = std::ranges::all_of(data, [](int x){ return x > 0; });
```

**注意**：views 引用原容器；容器销毁后 **悬垂**——与 [04 章](04-STL标准库容器与算法.md) 迭代器失效同理。

## 4. std::format（C++20）

### 4.1 基本用法

```cpp
#include <format>
#include <string>
#include <chrono>

int main() {
    std::string s = std::format("req_id={} latency={:.2f}ms", 42, 3.14159);
    // req_id=42 latency=3.14ms

    std::string hex = std::format("{:#x}", 255);  // 0xff
    std::string pad = std::format("{:>8}", "ok"); // 右对齐
}
```

### 4.2 自定义类型

```cpp
struct RequestId { int id; };
template<> struct std::formatter<RequestId> : std::formatter<int> {
    auto format(RequestId r, std::format_context& ctx) const {
        return std::formatter<int>::format(r.id, ctx);
    }
};
```

## 5. std::expected（C++23）

### 5.1 动机

异常适合 **罕见不可恢复** 错误；加载配置、解析参数等 **预期失败** 用返回值更清晰。

```cpp
#include <expected>
#include <string>
#include <fstream>

enum class LoadErr { NotFound, ParseFail };

std::expected<std::string, LoadErr> load_file(const std::string& path) {
    std::ifstream in(path);
    if (!in) return std::unexpected(LoadErr::NotFound);
    std::string content((std::istreambuf_iterator<char>(in)),
                         std::istreambuf_iterator<char>());
    if (content.empty()) return std::unexpected(LoadErr::ParseFail);
    return content;
}

void demo() {
    if (auto r = load_file("cfg.toml"); r)
        use(*r);
    else if (r.error() == LoadErr::NotFound)
        ; // 降级默认配置
}
```

### 5.2 与 optional / 错误码

`optional` 仅有/无值；`expected<T,E>` 失败时带 **结构化错误 E**；异常用于 invariant 破坏，expected 用于 **可恢复** API 边界。

## 6. mdspan（C++23）

### 6.1 多维视图

```cpp
#include <mdspan>
#include <vector>

void demo() {
    std::vector<float> buf(2 * 3 * 4);
    std::mdspan<float, std::extents<size_t, 2, 3, 4>> tensor(buf.data());
    tensor[1, 2, 3] = 1.0f;

    // 动态 extent
    std::mdspan<float, std::dextents<size_t, 3>> dyn(
        buf.data(), std::dextents<size_t, 3>{2, 3, 4});
}
```

### 6.2 与 Infra 张量

mdspan **不拥有** 内存；shape 用 `extents`，配合 [18 章](18-高性能C++与内存对齐.md) 对齐 buffer；GPU 侧仍用 CUDA API。

## 7. C++20 Modules 简述

### 8.1 语法骨架

```cpp
// math.cppm
export module math;
export int add(int a, int b) { return a + b; }

// main.cpp
import math;
```

### 8.2 相对 `#include` 的优劣

| 维度 | `#include` | **modules** |
|------|------------|-------------|
| 编译隔离 | 宏/声明泄漏 | 更强边界 |
| 编译速度 | 重复解析 | 可显著加速（理想） |
| 工具链 | 成熟 | **CMake/编译器版本敏感** |
| 与第三方 | 简单 | 库需 module 支持 |

**2026 建议**：新项目 **头文件仍主流**；modules 在大型 monorepo 试点——关注编译器 release note。

<details>
<summary>自测参考答案</summary>

1. **约束模板参数**，编译期可读错误，替代 SFINAE 晦涩失败。
2. **惰性**；在迭代/物化时（如 for 循环、`ranges::to`）才计算。
3. **`3.1`**（一位小数）。
4. expected 失败时带 **错误类型 E**；optional 仅表示无值。
5. **否**；非 owning 视图，只包装指针 + shape/strides。
6. modules **不传播** include 宏（接口更干净）；include 宏可泄漏。
7. 析构时 **自动 request_stop + join**（配合 stop_token）。
8. span **一维**连续；mdspan **多维** + layout。

#include <thread>

void worker(std::stop_token st) {
