## §2 异步日志 Q&A

| 编号 | 面试问题 | 标准答法（口述版） | 追问/坑点 |
|------|----------|-------------------|-----------|
| Q1 | 异步原理？ | format+入队；后台写 | 队列满 |
| Q2 | spdlog async？ | thread_pool+overflow | block/overrun |
| Q3 | 批量刷盘？ | 64KB 或 10ms | fdatasync |
| Q4 | 背压？ | 丢 debug/采样/阻塞 | error 不 silent drop |
| Q5 | 结构化？ | JSON+trace_id | [32 §JSON](32-fmt-spdlog与可观测性工程.md) |
| Q6 | 多 sink？ | console+file+remote | filter |
| Q7 | 与 64 定时器？ | 定时 flush | 优雅退出 |
| Q8 | Agent？ | Filebeat/Vector | K8s stdout |
| Q9 | 吞吐？ | 百万条/s 级 | benchmark |
| Q10 | 编译期级别？ | SPDLOG_ACTIVE_LEVEL | debug 剥离 |

### spdlog 异步

```cpp
#include <spdlog/async.h>
#include <spdlog/sinks/rotating_file_sink.h>
spdlog::init_thread_pool(8192, 1);
auto sink = std::make_shared<spdlog::sinks::rotating_file_sink_mt>(
    "logs/app.log", 100*1024*1024, 3);
auto logger = std::make_shared<spdlog::async_logger>(
    "async", sink, spdlog::thread_pool(),
    spdlog::async_overflow_policy::overrun_oldest);
```
