## 1. 构造函数家族

```cpp
#include <iostream>
#include <string>
#include <vector>

class Widget {
public:
    Widget() : id_(0), name_("default") {}
    Widget(int id, std::string name) : id_(id), name_(std::move(name)) {}
    Widget(const Widget& other) : id_(other.id_), name_(other.name_) {
        std::cout << "copy ctor\n";
    }

    int id_;
    std::string name_;

```

## 2. 委托构造 Delegating Constructor


### 术语三件套：委托构造（委托构造 Delegating Constructor）

| 维度 | 内容 |
|------|------|
| **定义** | 一个 ctor 调用同类另一个 ctor，减少重复。 |
| **生活类比** | 装修先走总包再走分包。 |
| **为什么重要** | DRY；C++11 起。 |
| **本章用到** | §2 |


```cpp
#include <iostream>
#include <string>

class Demo {
public:
    Demo() : Demo(0) {}  // 委托示例
    explicit Demo(int x) : x_(x) {}
    Demo(std::initializer_list<int> il) {
        for (int v : il) sum_ += v;
    }
    ~Demo() { std::cout << "~Demo\n"; }
    Demo(const Demo&) = default;
    Demo& operator=(const Demo&) = delete;
private:
    int x_{0};
    int sum_{0};
};
```

### 逐行读代码：initializer_list 与 explicit

```cpp
class Vec {
public:
    Vec(std::initializer_list<int> il) : data_(il) {}
    explicit Vec(std::size_t n) : data_(n, 0) {}
private:
    std::vector<int> data_;
};
Vec a{1, 2, 3};      // initializer_list
Vec b = {4, 5};      // 可能禁止若 explicit 拷贝 ctor
Vec c(10);           // explicit size
```

| 行/片段 | 含义 | 改错会怎样 |
|---------|------|------------|
| `initializer_list` ctor |  brace 初始化列表 | 与 `(10)` 区分 |
| `explicit Vec(size_t)` | 禁止 `Vec x = 10` | 防隐式转换 |
| `Vec{1,2,3}` | 调 il ctor | — |
| Rule of Zero | 成员会自管则不必手写特殊成员 | string/vector/unique_ptr |


## 8. 三五法则案例 §8

### 8.1 需要手写析构的类（Rule of Three）

```cpp
#include <cstring>
#include <iostream>

class Buffer {
    char* p_;
    std::size_t n_;
public:
    explicit Buffer(std::size_t n) : p_(new char[n]{}), n_(n) {}
    ~Buffer() { delete[] p_; }
    Buffer(const Buffer& o) : p_(new char[o.n_]{}), n_(o.n_) {
        std::memcpy(p_, o.p_, n_);
    }
    Buffer& operator=(const Buffer& o) {
        if (this != &o) {
            Buffer tmp(o);
            swap(tmp);
        }
        return *this;
    }
    void swap(Buffer& o) noexcept {
        std::swap(p_, o.p_); std::swap(n_, o.n_);
    }
};
```

### 8.2 Rule of Zero 现代写法

#include <vector>
class BufferModern {

public:
    explicit BufferModern(std::size_t n) : data_(n) {}
    // 编译器生成的三五成员足够

## FAQ


**Q：拷贝 ctor 与拷贝赋值区别？**  
A：构造新对象 vs 已存在对象赋新值。

**Q：移动 ctor 何时触发？**  
A：返回局部对象、std::move、右值引用绑定。

**Q：=default 与空函数体 ctor？**  
A：default 是特殊成员；可能 trivial，语义不同。

**Q：explicit 对 initializer_list？**  
A：C++20 可对 converting ctor 更细控制。

**Q：析构函数 virtual？**  
A：多态基类必须 virtual，否则 delete 基指针泄漏。

**Q：三五法则还适用吗？**  
A：适用；Rule of Zero 是首选，手写是管理 raw 资源时。

**Q：委托构造能委托到另一个委托链吗？**  
A：可以，不能循环。

**Q：成员初始化顺序？**  
A：按**声明顺序**，非初始化列表顺序。
