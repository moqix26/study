## 1. 树形关联容器

### 1.1 map 与 set

```cpp
std::map<std::string, int> freq;
freq["apple"]++;
std::set<int> uniq = {3,1,4,1,5};  // {1,3,4,5}
```


底层 **红黑树**；key 唯一（map）或仅 key（set）。

### 1.2 multimap / multiset

允许重复 key；`equal_range(k)` 返回区间。

## 3. 自定义比较

```cpp
struct ByLen {
    bool operator()(const std::string& a, const std::string& b) const {
        return a.size() < b.size() || (a.size() == b.size() && a < b);
    }
};
std::set<std::string, ByLen> s;
```

必须 **严格弱序**，与 [46 章](46-迭代器分类与算法库完全指南.md) `sort` 一致。

## 4. 自定义哈希

```cpp
struct Point { int x, y; bool operator==(const Point& o) const { return x==o.x && y==o.y; } };
namespace std {
template<> struct hash<Point> {
    size_t operator()(const Point& p) const noexcept {
        return (static_cast<size_t>(p.x) << 16) ^ static_cast<size_t>(p.y);
    }
};
}
```

## 5. 性能选型

| 需求 | 推荐 | 原因 |
|---|---|---|
| 有序遍历 | map/set | O(log n) 稳定 |
| 最快查找 | unordered_* | 均摊 O(1) |
| 范围查询 | map | `lower_bound` |
| 内存紧凑 | map | 哈希 load factor 有开销 |

## 6. 与 [03 章](03-面向对象与类设计.md) 对象作 key

需 consistent `operator<` 或 `hash+==`；**勿用裸指针** 作 key（地址不稳定）。


## 7.1 应用：词频 / 索引 1

```cpp
#include <unordered_map>
#include <string>
#include <vector>
#include <iostream>

void word_freq_1() {
    std::vector<std::string> words{"a","b","a","c","b","a"};
    std::unordered_map<std::string, int> m;
    for (const auto& w : words) ++m[w];
    for (const auto& [k, v] : m)
        std::cout << k << ':' << v << '\n';
}
```

**multimap 变体 1**：同一 key 多 value 用 `equal_range`；[04 章](04-STL标准库容器与算法.md) 已有示例。

**负载因子 1**：`max_load_factor()` / `rehash(n)` 控制 unordered 性能。

## 附录 A：高频面试问答

### A.1 map[] 副作用？

不存在则 **插入默认**；用 `at` 或 `find` 只读。

### A.2 unordered 最坏复杂度？

O(n) 哈希冲突；良好 hash 均摊 O(1)。

### A.3 map 迭代器有序？

中序遍历 key 升序。

### A.4 透明比较 C++14？

`less<>` 允许 `find(string_view)` 无临时 string。

### A.5 节点 extract C++17？

 splice 节点无需拷贝 key/value。
