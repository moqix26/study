## 1. 存储持续性概述

### 术语三件套：存储持续性

| 维度 | 内容 |
| --- | --- |
| **定义** | 对象在程序生命周期中**存在多久**、占哪块存储。 |
| **生活类比** | 租期：日租（自动）、年租（静态）、自己 new（动态）、员工卡（thread_local）。 |
| **为什么重要** | 决定指针/引用是否有效；理解 UB 与线程安全直觉。 |
| **本章用到** | §1 |

| 类型 | 关键词 | 生命 | 典型存储区 |
| --- | --- | --- | --- |
| 自动（automatic） | 局部变量、形参 | 进入块创建，离开销毁 | 栈（stack） |
| 静态（static） | static、全局、字面量 | 程序启动到结束 | 静态存储区（BSS/Data） |
| 动态（dynamic） | new / delete | 手动控制 | 堆（heap） |
| 线程（thread） | thread_local | 线程存活期 | 依实现，常近 TLS |

> **Primer Plus 第9章主线**：先弄清「变量住哪、活多久」，再谈「名字谁能看见」。

#### 本节小结

- 四种持续性：自动、静态、动态、线程
- 栈快、小、自动回收；堆灵活、需手动或智能指针
- 全局/静态零初始化；局部自动默认不初始化

## 2. 自动变量（栈）

### 术语三件套：自动变量

| 维度 | 内容 |
| --- | --- |
| **定义** | 在块或函数内定义、无 static 的变量；进入作用域分配，离开销毁。 |
| **生活类比** | 酒店退房：人走房清。 |
| **为什么重要** | 占栈空间；递归过深可能栈溢出。 |
| **本章用到** | §2 |

```cpp

#include <iostream>
void foo(int n) {
    int x = 10;           // 自动变量
    for (int i = 0; i < n; ++i) {
        int y = i * 2;    // 每轮新建，轮末销毁
        std::cout << y << ' ';
    }
    std::cout << "\n";
}  // x 在此销毁
int main() {
    foo(3);
    return 0;
}
```

**内存直觉**：每次调用 `foo`，栈帧压入保存 `n`、`x`、`i`、`y`；返回时弹出。

#### 常见错误

1. **返回局部变量地址或引用 — 悬垂，UB**
2. **未初始化就读局部 int — 值不确定**
3. **在栈上分配超大数组 — 可能 stack overflow**

#### 本节小结

- 自动变量在栈上
- 作用域结束即销毁
- 形参也是自动变量

## 3. 静态变量

### 3.1 全局变量

```cpp

#include <iostream>
int g_count = 0;  // 静态存储，零初始化后程序全程存活
void inc() { ++g_count; }
int main() {
    inc(); inc();
    std::cout << g_count << '\n';  // 2
    return 0;
}
```

### 3.2 局部 static

```cpp

#include <iostream>
void counter() {
    static int n = 0;  // 只初始化一次，生命贯穿程序
    ++n;
    std::cout << n << '\n';
}
int main() {
    counter(); counter(); counter();  // 1 2 3
    return 0;
}
```

C++11 起，局部 static 的**首次初始化线程安全**（magic static）。

### 3.3 类静态成员（预告）

```cpp

struct Counter {
    static int total;  // 声明
    int id;
};
int Counter::total = 0;  // 定义须在类外一处
```

### 术语三件套：静态变量

| 维度 | 内容 |
| --- | --- |
| **定义** | 存储持续性为整个程序；作用域仍受限于块/类/文件。 |
| **生活类比** | 公司公告栏：人人可见，离职不撤。 |
| **为什么重要** | 计数器、单例、缓存共享状态。 |
| **本章用到** | §3 |

#### 常见错误

1. **在头文件定义非 inline 全局变量 — 多重定义链接错误**
2. **以为 static 局部每次调用都初始化 — 实际只一次**
3. **类静态成员只声明不定义 — 链接 undefined reference**

#### 本节小结

- 全局与 static 都在静态区
- 局部 static 保留上次值
- 类静态全类一份

## 4. 外部链接与 extern

### 术语三件套：extern

| 维度 | 内容 |
| --- | --- |
| **定义** | 声明**在其他翻译单元已定义**的全局对象或函数；不分配存储。 |
| **生活类比** | 通讯录写「电话在总机登记」。 |
| **为什么重要** | 多 .cpp 共享全局状态或跨文件调用。 |
| **本章用到** | §4 |

```cpp

// shared.h
#pragma once
extern int shared_value;
void print_shared();

// file1.cpp
#include "shared.h"
#include <iostream>
void print_shared() {
    std::cout << shared_value << '\n';
}

// file2.cpp
#include "shared.h"
int shared_value = 42;  // 唯一定义
```

**链接属性速记**：`extern` 全局声明 → 外部链接；`static` 全局 / 匿名 namespace → 内部链接（仅本 TU）。

#### 常见错误

1. **多个 .cpp 定义同名全局 — ODR 违反**
2. **extern 与定义类型不一致 — UB**
3. **在 .h 写 `int x;` 而非 `extern int x;` — 每个包含者都定义一份**

#### 本节小结

- extern 只声明不定义
- 定义只能有一处
- 头文件用 extern 声明

## 5. 名称空间 namespace

### 术语三件套：名称空间

| 维度 | 内容 |
| --- | --- |
| **定义** | 把名字包在命名域，用 `::` 限定，避免冲突。 |
| **生活类比** | 姓+名：张::伟 vs 李::伟。 |
| **为什么重要** | 大型项目、标准库 `std::`、模块边界。 |
| **本章用到** | §5 |

```cpp

namespace utils {
    int add(int a, int b) { return a + b; }
    namespace detail {
        void log(const char* msg) { /* ... */ }
    }
}
int main() {
    return utils::add(1, 2);
}
```

### 5.1 嵌套与 C++17 嵌套写法

```cpp
namespace company::project { void init(); }
```

#### 本节小结

- namespace 防名字冲突
- 嵌套可分层组织
- C++17 支持 `a::b` 合并声明

## 6. using 声明 vs using 编译指令

| 形式 | 作用 | 范围 | 头文件建议 |
| --- | --- | --- | --- |
| `using std::cout;` | 引入**一个**名字 | 当前作用域 | 一般避免 |
| `using namespace std;` | 引入**整个**命名空间 | 当前作用域及子块 | **禁止**在头文件 |

> **Primer Plus 建议**：.cpp 实现文件可局部 using；头文件保持 `std::` 前缀或精确 using 声明。

```cpp

#include <iostream>
#include <vector>
void demo() {
    using std::cout;           // 仅 cout
    cout << "ok\n";
}
// using namespace std;  // 若写在这里，只影响 demo 之后同文件后续
```

#### 常见错误

1. **头文件 `using namespace std` — 污染所有包含者**
2. **混用 using 声明与同名局部变量 — 遮蔽规则复杂**
3. **在命名空间内 using 另一整个 namespace — 间接污染**

#### 本节小结

- using 声明 = 一个名字
- using directive = 整个空间
- 头文件忌 directive

## 7. 未命名名称空间

### 术语三件套：未命名名称空间

| 维度 | 内容 |
| --- | --- |
| **定义** | 匿名 namespace，其中名字具有**内部链接**，仅本翻译单元可见。 |
| **生活类比** | 办公室内部便签，不外传。 |
| **为什么重要** | 替代 static 全局的现代写法。 |
| **本章用到** | §7 |

```cpp

namespace {
    int secret = 0;
    void bump() { ++secret; }
}
void api() { bump(); }  // 本文件可调用 bump
```

C++ 推荐用**未命名 namespace** 限本文件符号；C 风格 `static` 全局函数仍合法但较旧。

#### 本节小结

- 匿名 namespace = 内部链接
- 替代 static 全局
- 每 TU 独立副本

## 8. 作用域解析与 Primer Plus 综合

```cpp

int count = 0;  // 全局
namespace app {
    int count = 10;
    void show() {
        std::cout << count << '\n';       // 10
        std::cout << ::count << '\n';     // 0
    }
}
```

### 8.1 存储 + 名称空间对照表

| 代码 | 持续性 | 链接 | 可见性 |
| --- | --- | --- | --- |
| `void f(){ int x; }` | 自动 | 无 | 块内 |
| `static int s;` 在函数内 | 静态 | 无 | 块内 |
| `int g;` 全局 | 静态 | 外部 | 全程序（若 extern 声明） |
| `namespace{ int h; }` | 静态 | 内部 | 本 TU |
| `new int` | 动态 | 无 | 指针传播 |

## 9. 综合示例（Primer Plus 风格）

### 例 97.1：跨文件计数器

> 两个源文件共享 `extern int hits`。

**为什么**：extern 声明 + 单一定义

```cpp
extern int hits;
void touch(){ ++hits; }
```

**要点**：hits 在另一 cpp 定义

### 例 97.2：单例式局部 static

> 懒加载配置。

**为什么**：局部 static 只构造一次

```cpp
Config& cfg(){ static Config c; return c; }
```

**要点**：线程安全初始化

### 例 97.3：math 命名空间

> 避免与 std 冲突。

**为什么**：namespace 封装

```cpp
namespace math { const double pi=3.14; }
```

**要点**：调用 math::pi

### 例 97.4：头文件污染实验

> 理解为何忌 using directive。

**为什么**：包含链扩散

```cpp
// bad.h: using namespace std;
```

**要点**：所有包含者受影响

### 例 97.5：匿名 helper

> 隐藏实现细节。

**为什么**：未命名 namespace

```cpp
namespace { void helper(){} }
```

**要点**：符号不出现在链接表

## §10 栈帧与自动变量再探

结合 96 章：调用 `g()` 时，形参和局部变量占栈帧；返回时帧销毁，其中自动变量地址失效。

## §11 静态初始化顺序

不同翻译单元间全局 static 初始化顺序未指定；用函数内 static 或 `constexpr` 减轻风险。

## §12 inline 变量与头文件

C++17 `inline int x=0;` 可在头文件定义，ODR 安全。

## §13 thread_local 入门

`thread_local int tid;` 每线程一份，持续在线程整个生命周期。

## §14 链接与名称修饰

C++ 名字修饰（name mangling）；`extern "C"` 供 C 互操作。

## §15 作用域嵌套规则

内层遮蔽外层同名；`::` 回全局。

## FAQ

**Q：static 局部变量线程安全吗？**  
A：C++11 起局部 static 初始化线程安全。

**Q：extern 和 extern "C" 一样吗？**  
A：不同；后者指定 C 链接，用于 C 互操作。

**Q：using namespace std 在 .h 为何不好？**  
A：污染所有包含该头文件的翻译单元。

**Q：未命名 namespace 与 static 区别？**  
A：C++ 推荐未命名 namespace；static 全局仍可用但较旧。

**Q：自动变量未初始化？**  
A：局部变量默认不初始化，值不确定；应显式初始化。

**Q：两个 .cpp 能定义同名全局函数吗？**  
A：不能，除非 inline 或 static/匿名 namespace 内部链接。

**Q：堆和栈变量寿命？**  
A：栈随作用域；堆直到 delete 或智能指针释放。

**Q：全局变量默认初始化？**  
A：零初始化（指针为 nullptr，数值为 0）。
