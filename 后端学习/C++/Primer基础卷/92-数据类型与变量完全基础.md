## §1 变量：定义、初始化与赋值

> 生活类比：带标签的储物盒：标签是名字，盒子装一种类型的值。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | 变量是具名内存位置；定义创建；初始化给初值；赋值改值。 |
| **生活类比** | 带标签的储物盒：标签是名字，盒子装一种类型的值。 |
| **为什么重要** | 一切计算都作用在具名数据上。 |
| **本章用到** | §1 |

### 最小可运行例子

```cpp
#include <iostream>
int main() {
    int age = 18;
    age = 19;
    std::cout << age << "\n";
    return 0;
}
```

**逐行读代码**：
1. `int age = 18` 定义并初始化。
2. `age = 19` 赋值替换原值。

#### 常见错误

1. `int x; std::cout << x;` 未定义行为——值不确定。
2. `int a, b = 2;` 只有 b 被初始化为 2。

#### 循序渐进练习

定义三个不同类型的变量并打印；每个都初始化。

#### 本节小结

- 定义时可初始化；先初始化再使用。

## §2 整数族

> 生活类比：不同容量的杯子装不同体积的水。

| 类型 | 最小宽度 | 典型用途 | 常见字节 |
| --- | --- | --- | --- |
| `short` | 至少16位 | 极少单独用 | 2 |
| `int` | 至少16位，常32位 | 计数默认 | 4 |
| `long` | 至少32位 | 平台相关 | 4或8 |
| `long long` | 至少64位 | 大整数 | 8 |

```cpp
#include <iostream>
int main(){
  std::cout<<sizeof(short)<<" "<<sizeof(int)
         <<" "<<sizeof(long long)<<"\n";
  return 0;
}
```

`<cstdint>` 提供 `int32_t`、`uint64_t` 固定位宽。

## §3 unsigned 与溢出

> 生活类比：无符号像汽车里程表——只增不减，加满后回零。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | unsigned 表示 0 及正数；同样位数可表更大正数，但无负数。 |
| **生活类比** | 无符号像汽车里程表——只增不减，加满后回零。 |
| **为什么重要** | 混用 signed/unsigned 易出逻辑 bug。 |
| **本章用到** | §3 |

### 最小可运行例子

```cpp
#include <iostream>
int main() {
    unsigned char c = 255;
    c = c + 1;
    std::cout << (int)c << "\n";
    return 0;
}
```

**逐行读代码**：
1. 255+1 模 256 得 0。
2. 有符号溢出是未定义行为；无符号按模运算。

#### 常见错误

1. `unsigned a=5,b=10; if(a-b>0)` 永远真。
2. 用 unsigned 表示负数语义错误。

#### 循序渐进练习

验证 unsigned 0 减 1 的输出；思考为何。

#### 本节小结

- 需要负数勿用 unsigned；比较时统一类型。

## §4 浮点与精度陷阱

> 生活类比：二进制小数像十进制里 1/3 不能有限表示——0.1 在 double 里也不精确。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | float/double/long double 表示近似实数；财务用整数「分」。 |
| **生活类比** | 二进制小数像十进制里 1/3 不能有限表示——0.1 在 double 里也不精确。 |
| **为什么重要** | 比较浮点要用误差范围，勿用 ==。 |
| **本章用到** | §4 |

### 最小可运行例子

```cpp
#include <iostream>
#include <cmath>
int main() {
    double a = 0.1 + 0.2;
    std::cout << (a == 0.3) << "\n";
    std::cout << std::fabs(a - 0.3) < 1e-9 << "\n";
    return 0;
}
```

**逐行读代码**：
1. `0.1+0.2==0.3` 常为 false。
2. `fabs` 差值小于 epsilon 判等。

#### 常见错误

1. 累加大量小数用 float 误差放大。
2. 把 float 与 int 混算未注意提升规则。

#### 循序渐进练习

打印 `0.1+0.2` 用 `setprecision(17)` 看真实存储。

#### 本节小结

- 默认字面量 `3.14` 是 double；`3.14f` 是 float。

## §5 char 与 ASCII

> 生活类比：字符是字母表符号，底层是小整数编码。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | char 占 1 字节；`'A'` 与 65 可互换上下文。 |
| **生活类比** | 字符是字母表符号，底层是小整数编码。 |
| **为什么重要** | 为字符串与 IO 打基础（第95章）。 |
| **本章用到** | §5 |

### 最小可运行例子

```cpp
#include <iostream>
int main() {
    char ch = 'A';
    std::cout << ch << " " << (int)ch << "\n";
    std::cout << "\t newline\n";
    return 0;
}
```

**逐行读代码**：
1. 单引号字符，双引号字符串。
2. 转义：\n \t \\ \' \" \0

#### 常见错误

1. `char` 是否 signed 由实现定义。
2. 把字符串用单引号。

#### 循序渐进练习

打印 A-Z 及对应 ASCII 码。

#### 本节小结

- 需要无符号字节用 `unsigned char`。

## §6 bool 类型

> 生活类比：开关只有开/关两态。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | bool 值 true/false；可隐式与整数转换。 |
| **生活类比** | 开关只有开/关两态。 |
| **为什么重要** | 条件表达式与循环条件的基础。 |
| **本章用到** | §6 |

### 最小可运行例子

```cpp
#include <iostream>
int main() {
    bool ok = true;
    std::cout << ok << "\n";
    std::cout << std::boolalpha << ok << "\n";
    return 0;
}
```

**逐行读代码**：
1. 默认输出 0/1。
2. `boolalpha` 输出 true/false。

#### 常见错误

1. 用 bool 存大量数据浪费（虽通常1字节）。
2. 与 int 混用不加显式转换可读性差。

#### 循序渐进练习

写 `bool from_int = 42;` 观察输出。

#### 本节小结

- 条件判断产生 bool；非零转 true。

## §7 字面量大全

> 生活类比：源码里的「裸常数」自带类型。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | 十进制、0x十六进制、0八进制、0b二进制（C++14）、后缀 u/L/f。 |
| **生活类比** | 源码里的「裸常数」自带类型。 |
| **为什么重要** | 写对字面量避免意外类型推导。 |
| **本章用到** | §7 |

#### 常见错误

1. `052` 是八进制 42 不是十进制 52。
2. 混淆 L 与 LL 后缀。

#### 循序渐进练习

列表：`42, 42u, 42LL, 0x2A, 0b101010, 3.14f` 各是什么类型？

#### 本节小结

- 默认整数字面量 int；过大才提升 long/long long。

## §8 auto 类型推导

> 生活类比：让编译器根据初始化式猜类型——必须初始化。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | C++11 `auto i=42;` 推导为 int。 |
| **生活类比** | 让编译器根据初始化式猜类型——必须初始化。 |
| **为什么重要** | 简化冗长类型名；类型仍在编译期确定。 |
| **本章用到** | §8 |

### 最小可运行例子

```cpp
auto i = 42;
auto d = 3.14;
auto b = false;
```

**逐行读代码**：
1. 三者分别为 int、double、bool。
2. `auto x;` 非法。

#### 常见错误

1. auto Everything 降低可读性。
2. 忘记 auto 变量仍强类型。

#### 循序渐进练习

用 auto 定义至少三个变量并 `typeid` 或习惯推断类型。

#### 本节小结

- auto 不是动态类型；只是省打字。

## §9 typedef 与 using

> 生活类比：给复杂类型起别名，像给长文件名建快捷方式。

### 术语三件套

| 维度 | 内容 |
| --- | --- |
| **定义** | `typedef unsigned long ulong;` `using ulong2 = unsigned long;` |
| **生活类比** | 给复杂类型起别名，像给长文件名建快捷方式。 |
| **为什么重要** | 提升可读性；using 支持模板别名。 |
| **本章用到** | §9 |

### 最小可运行例子

```cpp
using byte = unsigned char;
byte b = 255;
```

**逐行读代码**：
1. typedef 与 using 不创建新类型。
2. 头文件中 typedef 注意作用域。

#### 常见错误

1. #define 伪类型无类型检查。
2. 别名过长仍难读——适度。

#### 循序渐进练习

用 using 定义 `using cents = long long;` 表示金额分。

#### 本节小结

- 现代 C++ 优先 using =。
