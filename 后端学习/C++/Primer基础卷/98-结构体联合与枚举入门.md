## 1. struct 基础

### 术语三件套：struct

| 维度 | 内容 |
| --- | --- |
| **定义** | 聚合类型，成员默认 **public**；把异构字段合成一条记录。 |
| **生活类比** | 表格一行：学号、姓名、分数。 |
| **为什么重要** | POD 互操作、序列化、数据库行映射。 |
| **本章用到** | §1 |

```cpp

#include <iostream>
#include <string>
struct Student {
    int id;
    std::string name;
    double score;
};
int main() {
    Student s{1001, "小明", 95.5};
    std::cout << s.name << ' ' << s.score << '\n';
    return 0;
}
```

### 1.1 聚合初始化

```cpp
Student s1{1, "A", 90.0};
Student s2 = {2, "B", 85.0};
Student s3{};  // 值初始化
```

### 1.2 成员访问

```cpp
s.id = 1002;
Student* p = &s;
p->score = 88.0;
```

#### 常见错误

1. **未初始化成员就读 — string 空 OK，数值可能为 0 或垃圾（视初始化方式）**
2. **memcpy 非平凡类型 struct — UB**
3. **结构体对齐填充导致 sizeof 大于成员之和 — 正常**

#### 本节小结

- struct 默认 public
- 聚合初始化用 {}
- `.` 与 `->` 访问成员

## 2. 嵌套结构与成员数组

```cpp

struct Date { int y, m, d; };
struct Event {
    std::string title;
    Date when;
    int attendees[4];
};
Event e{"Meeting", {2026, 7, 3}, {1, 2, 3, 4}};
std::cout << e.when.m << '\n';
```

### 术语三件套：嵌套结构

| 维度 | 内容 |
| --- | --- |
| **定义** | 成员类型是另一 struct；形成层次数据模型。 |
| **生活类比** | 信封里再套便签：地址块在联系人里。 |
| **为什么重要** | 配置、协议、图形点线面。 |
| **本章用到** | §2 |

#### 本节小结

- 嵌套用成员类型
- 初始化可嵌套 {}
- 数组成员下标访问

## 3. 位域

### 术语三件套：位域

| 维度 | 内容 |
| --- | --- |
| **定义** | 成员后加 `: 位数`，压缩存储标志位。 |
| **生活类比** | 行李箱侧袋只放钥匙，不占大格。 |
| **为什么重要** | 协议头、硬件寄存器模拟。 |
| **本章用到** | §3 |

```cpp

struct Flags {
    unsigned ready : 1;
    unsigned mode  : 3;
    unsigned       : 0;  // 对齐填充到下一单元（实现定义）
};
```

**注意**：位域布局实现定义；不宜跨平台二进制序列化；通常不能取地址。

#### 常见错误

1. **假设位域跨平台布局一致**
2. **对位域成员取地址**
3. **位域类型带符号溢出行为复杂**

#### 本节小结

- 位域省空间
- 布局依实现
- 常用于标志位

## 4. union 共用体

### 术语三件套：union

| 维度 | 内容 |
| --- | --- |
| **定义** | 所有成员共享同一起始地址；同一时刻只应 активно 使用一个成员。 |
| **生活类比** | 一个抽屉换放不同物品，不能同时。 |
| **为什么重要** | 协议解析、节省内存；误用 strict aliasing UB。 |
| **本章用到** | §4 |

```cpp

union Value {
    int i;
    float f;
    char bytes[4];
};
Value v;
v.i = 42;
// 读 v.f 若未写入 f — 传统写法 UB；C++20 可用 std::bit_cast
```

### 4.1 典型用途与风险

| 用途 | 说明 | 风险 |
| --- | --- | --- |
| 网络协议 | 同一槽位不同消息类型解释 | 必须跟踪活跃成员 |
| Variant 前身 | 手动判别 tag | 非平凡类型需 placement new |
| 类型双关 | 读另一类型比特 | 用 bit_cast，勿随意 punning |

#### 常见错误

1. **写入 i 后读 f 当 float — strict aliasing UB**
2. **union 含 string 等非平凡成员却不手动析构 — 资源泄漏**
3. **多线程无同步读写同一 union — 数据竞争**

#### 本节小结

- union 共享内存
- 同时只活一个成员语义
- 现代 C++ 优先 variant/bit_cast

## 5. enum 与 enum class

```cpp

enum Color { Red, Green, Blue };       // 旧式，枚举元泄漏到外层
enum class Status { Ok, Fail, Pending }; // 强类型，需 Status::Ok
Status s = Status::Ok;
// int x = Status::Ok;  // 错误：无隐式转 int
```

|  | enum | enum class |
| --- | --- | --- |
| 作用域 | 枚举元注入外层 | 枚举域内 |
| 隐式转 int | 可以 | 不可以（需 static_cast） |
| 底层类型 | 实现定义 | 可指定 `: uint8_t` |
| 推荐 | 遗留/C 互操作 | **现代 C++ 首选** |

### 术语三件套：enum class

| 维度 | 内容 |
| --- | --- |
| **定义** | 强类型枚举，不污染外层命名，不隐式转整型。 |
| **生活类比** | 带门禁的楼层号，不能当门牌乱用。 |
| **为什么重要** | 类型安全、重载决议清晰。 |
| **本章用到** | §5 |

#### 本节小结

- enum class 强类型
- 指定底层类型可省空间
- 旧 enum 易名字冲突

## 6. typedef 与 using 别名

```cpp

typedef unsigned long ulong;
using ulong2 = unsigned long;
using ScoreMap = std::map<int, double>;
template<typename T>
using Vec = std::vector<T>;  // 模板别名只能 using
```

**陷阱**：`typedef int* Ptr, Val;` 中 `Ptr` 是 `int*`，`Val` 是 `int` 而非 `int*`。

### 术语三件套：类型别名

| 维度 | 内容 |
| --- | --- |
| **定义** | 为复杂类型起短名；不改变类型本质。 |
| **生活类比** | 昵称：本质还是同一人。 |
| **为什么重要** | 提升可读性；模板别名只能 using。 |
| **本章用到** | §6 |

#### 本节小结

- typedef 与 using 等效（简单别名）
- 模板别名用 using
- typedef 多变量声明有陷阱

## FAQ

**Q：struct 和 class 区别？**  
A：默认访问：struct public，class private；其余几乎相同。

**Q：C++20 指定初始化器？**  
A：指定成员：`Point{.x=1,.y=2};`

**Q：union 大小？**  
A：至少能容纳最大成员，常含对齐填充。

**Q：enum class 底层类型？**  
A：可指定 `enum class E : uint8_t`。

**Q：typedef 和 using 选哪个？**  
A：模板别名只能 using；简单别名二者皆可。

**Q：位域能取地址吗？**  
A：通常不能，非完整对象。

**Q：struct 能继承吗？**  
A：可以，public 继承常见。

**Q：聚合初始化条件？**  
A：无用户声明构造、无私有/保护非静态成员等（细则见标准）。
