## 1. 为什么需要动态内存

### 1.1 栈 vs 堆：生活类比

**栈**像酒店标准间：大小固定，退房自动打扫（函数结束自动销毁）。  
**堆**像长租仓库：你要多少申请多少，**必须自己退租**（`delete`），否则泄漏。

```cpp
#include <iostream>

void demo_stack() {
    int arr[100];          // 栈上，大小编译期固定
    // int arr[n];         // C++ 标准中 n 非常量时不可 VLA（与 C 不同）
}

void demo_heap() {
    int* p = new int;      // 堆上单个 int
    *p = 42;
    delete p;              // 必须配对
    p = nullptr;           // 良好习惯：避免悬垂指针

    int* arr = new int[10]; // 堆上数组
    arr[0] = 1;
    delete[] arr;           // 数组用 delete[]
}
```


### 术语三件套：动态分配（Dynamic Allocation）

| 维度 | 内容 |
|------|------|
| **定义** | 在程序运行时从**堆**申请内存，返回指针；生命周期由程序员控制。 |
| **生活类比** | 临时租仓库放货，用完要还钥匙。 |
| **为什么重要** | 大小可在运行时决定；对象可跨函数存活。 |
| **本章用到** | §1～§3 |


### 1.2 new 与 delete 配对规则

| 表达式 | 释放方式 | 常见错误 |
|--------|----------|----------|
| `new T` | `delete p` | 用 `delete[]` |
| `new T[n]` | `delete[] p` | 用 `delete` |
| `new T(...)` | `delete p` | 忘记置 `nullptr` |

```cpp
// 最小完整示例
#include <iostream>

int main() {
    double* pd = new double(3.14);
    std::cout << *pd << '
';
    delete pd;

    int* pi = new int[5]{1, 2, 3, 4, 5};
    for (int i = 0; i < 5; ++i)
        std::cout << pi[i] << ' ';
    std::cout << '
';
    delete[] pi;
    return 0;
}
```

#### 常见错误：new[] 却 delete

**错误写法：**

```cpp
int* p = new int[10];
delete p;  // 未定义行为
```

**正确写法：**

```cpp
int* p = new int[10];
delete[] p;
```

**原因**：`delete` 与 `delete[]` 内部机制不同；混用是 UB，可能崩溃或内存破坏。


## 2. 类成员含指针：灾难是如何发生的

### 2.1 浅拷贝：复印门牌号

假设类里有一个 `char*` 指向堆上字符串：

```cpp
#include <cstring>
#include <iostream>

class BadString {
    char* str;
    int len;
public:
    BadString(const char* s) {
        len = std::strlen(s);
        str = new char[len + 1];
        std::strcpy(str, s);
    }
    ~BadString() {
        delete[] str;
        std::cout << "释放: " << (str ? "(已无效)" : "null") << '\n';
    }
    // 编译器生成的拷贝构造 = 浅拷贝：只复制指针值！
};

void oops() {
    BadString a("hi");
    BadString b = a;   // 浅拷贝：a.str 与 b.str 指向同一块内存
}   // 析构两次 → double free → 崩溃
```


### 术语三件套：浅拷贝（Shallow Copy）

| 维度 | 内容 |
|------|------|
| **定义** | 只复制指针成员的值（地址），两个对象共享同一块堆内存。 |
| **生活类比** | 两人共用一把钥匙，退租时各交一次钥匙给房东。 |
| **为什么重要** | 默认拷贝对含 raw 指针的类是危险的。 |
| **本章用到** | §2 |


### 2.2 深拷贝：各自买房

**深拷贝**：新对象自己 `new` 一份副本，互不影响。

```cpp
#include <cstring>
#include <iostream>

class GoodString {
    char* str;
    int len;
public:
    GoodString(const char* s = "") {
        len = std::strlen(s);
        str = new char[len + 1];
        std::strcpy(str, s);
    }
    ~GoodString() { delete[] str; }

    // 拷贝构造：深拷贝
    GoodString(const GoodString& other) {
        len = other.len;
        str = new char[len + 1];
        std::strcpy(str, other.str);
        std::cout << "深拷贝构造\n";
    }

    // 拷贝赋值
    GoodString& operator=(const GoodString& other) {
        if (this == &other) return *this;
        delete[] str;
        len = other.len;
        str = new char[len + 1];
        std::strcpy(str, other.str);
        return *this;
    }

    const char* c_str() const { return str; }
};
```

### 2.3 赋值运算符的安全写法：copy-and-swap

先构造临时副本，再交换——异常更安全：

```cpp
#include <utility>
// 在 GoodString 中添加：
void swap(GoodString& other) noexcept {
    std::swap(str, other.str);
    std::swap(len, other.len);
}
GoodString& operator=(GoodString other) {  // 传值
    swap(other);
    return *this;
}
```

## 3. 三五法则入门（Rule of Three）

### 3.1 何时需要手写三个函数

若类**直接管理**堆资源（raw `new`/`delete`），通常需要同时定义：

1. **析构函数** `~T()`
2. **拷贝构造函数** `T(const T&)`
3. **拷贝赋值** `operator=(const T&)`

缺任何一个，编译器生成的默认版本都可能错误。

| 场景 | 建议 |
|------|------|
| 只有 `int`、`double`、`string`、`vector` 成员 | **Rule of Zero**：不写，用编译器默认 |
| 有 `char*`、`FILE*`、socket 等 raw 资源 | **Rule of Three**（C++11 后扩展为 Five 含移动） |

### 3.2 最小 Rule of Three 模板

```cpp
class Resource {
    int* data;
    std::size_t n;
public:
    explicit Resource(std::size_t size) : data(new int[size]{}), n(size) {}
    ~Resource() { delete[] data; }
    Resource(const Resource& o) : data(new int[o.n]{}), n(o.n) {
        for (std::size_t i = 0; i < n; ++i) data[i] = o.data[i];
    }
    Resource& operator=(const Resource& o) {
        if (this != &o) {
            Resource tmp(o);
            std::swap(data, tmp.data);
            std::swap(n, tmp.n);
        }
        return *this;
    }
};
```


### 术语三件套：三五法则（Rule of Three / Five / Zero）

| 维度 | 内容 |
|------|------|
| **定义** | 管理资源的类应显式定义析构、拷贝、移动相关成员，或干脆用 RAII 容器实现 Rule of Zero。 |
| **生活类比** | 要么亲自当管家（手写），要么雇专业保洁（`vector`/`string`）。 |
| **为什么重要** | 面试与工程代码质量的核心考点。 |
| **本章用到** | §3～§4 |


## 4. 教学版 String 类完整实现

> 以下实现接近 Primer Plus 风格，便于理解；工程里请用 `std::string`。

### 4.1 头文件式声明（单文件教学版）

```cpp
#include <iostream>
#include <cstring>
#include <cstddef>

class String {
    char* str;
    int len;
    static int num_strings;  // 跟踪对象数（可选教学）

    static int strlen(const char* s) { return s ? static_cast<int>(std::strlen(s)) : 0; }

public:
    String(const char* s = "");
    String(const String& st);
    ~String();
    String& operator=(const String& st);
    char& operator[](int i);
    const char& operator[](int i) const;
    friend std::ostream& operator<<(std::ostream& os, const String& st);
    friend bool operator<(const String& st1, const String& st2);
    int length() const { return len; }
};

int String::num_strings = 0;

String::String(const char* s) {
    len = strlen(s);
    str = new char[len + 1];
    std::strcpy(str, s);
    ++num_strings;
}

String::String(const String& st) {
    len = st.len;
    str = new char[len + 1];
    std::strcpy(str, st.str);
    ++num_strings;
}

String::~String() {
    delete[] str;
    --num_strings;
}

String& String::operator=(const String& st) {
    if (this == &st) return *this;
    delete[] str;
    len = st.len;
    str = new char[len + 1];
    std::strcpy(str, st.str);
    return *this;
}

char& String::operator[](int i) {
    if (i < 0 || i >= len) throw std::out_of_range("String::operator[]");
    return str[i];
}

const char& String::operator[](int i) const {
    if (i < 0 || i >= len) throw std::out_of_range("String::operator[]");
    return str[i];
}

std::ostream& operator<<(std::ostream& os, const String& st) {
    os << st.str;
    return os;
}

bool operator<(const String& st1, const String& st2) {
    return std::strcmp(st1.str, st2.str) < 0;
}
```

### 4.2 使用示例

```cpp
int main() {
    String s1("Hello");
    String s2 = s1;          // 深拷贝
    s2[0] = 'h';
    std::cout << s1 << ", " << s2 << '\n';  // Hello, hello
    String s3;
    s3 = s1;                 // 赋值
    return 0;
}   // 三个析构自动调用
```

## 5. 成员含动态数组

### 5.1 指针成员表示可变长数组

```cpp
class StudentScores {
    std::string name;
    double* scores;
    std::size_t count;
public:
    StudentScores(const std::string& n, const double* src, std::size_t c)
        : name(n), scores(new double[c]), count(c) {
        for (std::size_t i = 0; i < c; ++i) scores[i] = src[i];
    }
    ~StudentScores() { delete[] scores; }
    StudentScores(const StudentScores& o)
        : name(o.name), scores(new double[o.count]), count(o.count) {
        for (std::size_t i = 0; i < count; ++i) scores[i] = o.scores[i];
    }
    // operator= 同理深拷贝
    double average() const {
        double sum = 0;
        for (std::size_t i = 0; i < count; ++i) sum += scores[i];
        return count ? sum / count : 0;
    }
};
```

### 5.2 现代写法：vector 替代 raw 数组

```cpp
#include <vector>
#include <string>

class StudentScoresModern {
    std::string name;
    std::vector<double> scores;
public:
    StudentScoresModern(std::string n, std::vector<double> s)
        : name(std::move(n)), scores(std::move(s)) {}
    // Rule of Zero：不写析构/拷贝/赋值
    double average() const {
        if (scores.empty()) return 0;
        double sum = 0;
        for (double v : scores) sum += v;
        return sum / scores.size();
    }
};
```


#### 常见错误：自赋值未检查

**错误写法：**

```cpp
String& operator=(const String& st) {
    delete[] str;
    str = new char[st.len + 1];
    // 若 this==&st，str 已被 delete
}
```

**正确写法：**

```cpp
if (this == &st) return *this;
delete[] str; ...
```

**原因**：自赋值时先释放自身资源会导致读取已释放内存。

## 7. 逐步构建：从 int* 到 String

### 7.1 第 1 步：仅构造与析构

```cpp
class Step1 {
    int* p;
public:
    Step1(int v) : p(new int(v)) {}
    ~Step1() { delete p; }
    int get() const { return *p; }
};
// 尚不可拷贝——编译器生成的拷贝会浅拷贝
```

### 7.2 第 2 步：加上拷贝构造

```cpp
class Step2 {
    int* p;
public:
    Step2(int v) : p(new int(v)) {}
    ~Step2() { delete p; }
    Step2(const Step2& o) : p(new int(*o.p)) {}
};
// 可拷贝构造，但赋值仍有问题
```

### 7.3 第 3 步：补全赋值

```cpp
class Step3 {
    int* p;
public:
    Step3(int v) : p(new int(v)) {}
    ~Step3() { delete p; }
    Step3(const Step3& o) : p(new int(*o.p)) {}
    Step3& operator=(const Step3& o) {
        if (this != &o) {
            delete p;
            p = new int(*o.p);
        }
        return *this;
    }
};
```

## FAQ

**Q：`new` 失败会怎样？**  
A：默认抛 `std::bad_alloc`；可 `new (nothrow) T` 返回 nullptr。

**Q：能否重载 `new`/`delete`？**  
A：可以，用于内存池；初学阶段不必。

**Q：String 为什么要 `len+1`？**  
A：为 `\0`  terminator 留位，兼容 C 字符串函数。

**Q：友元 `operator<<` 为何不是成员？**  
A：左操作数是 `ostream`，类外友元更自然。

**Q：Rule of Zero 与 Three 矛盾吗？**  
A：不矛盾；Zero 是「用容器不写」，Three 是「raw 资源必须写」。

**Q：移动 ctor 能省深拷贝吗？**  
A：能；C++11 起对临时对象可「偷」资源，见 05/41 章。

**Q：`delete p` 后 p 还有值吗？**  
A：有，为悬垂指针；应 `p = nullptr`。

**Q：类数组 `new T[n]` 默认构造吗？**  
A：对 class 类型会调用 n 次默认构造。
