## 1. 为什么需要继承

### 1.1 生活类比：物种与品种

「狗」是 general 概念，「柯基」「金毛」在共有行为上**扩展**。「柯基 **是一种** 狗」——这就是 **is-a**。

没有继承时，你只能复制粘贴代码：

```cpp
class Dog {
public:
    void eat() { /* ... */ }
    void sleep() { /* ... */ }
};
class Corgi {
public:
    void eat() { /* 重复 */ }
    void sleep() { /* 重复 */ }
    void shortLegRun() { /* 特有 */ }
};
```

用继承：

```cpp
class Dog {
public:
    void eat() { std::cout << "dog eats\n"; }
    void sleep() { std::cout << "dog sleeps\n"; }
};
class Corgi : public Dog {
public:
    void shortLegRun() { std::cout << "corgi runs\n"; }
};
```


### 术语三件套：继承（Inheritance）

| 维度 | 内容 |
|------|------|
| **定义** | 派生类（derived）获得基类（base）的成员，并可新增或改写行为。 |
| **生活类比** | 子品种继承父母的能力，再学新技能。 |
| **为什么重要** | 代码重用 + 统一接口，是多态的基础。 |
| **本章用到** | §1～§2 |


## 2. is-a 关系与 public 继承

### 2.1 只在对的关系上用 public 继承

| 关系 | 手段 | 例子 |
|------|------|------|
| is-a | public 继承 | `Corgi : public Dog` |
| has-a | 成员组合 | `Car` 有 `Engine` |
| implements | public 继承纯虚接口 | `ConsoleLogger : ILogger` |

```cpp
class Employee {
    std::string name;
protected:
    double salary;  // 派生类可访问
public:
    Employee(std::string n, double s) : name(std::move(n)), salary(s) {}
    virtual void show() const {
        std::cout << name << ", salary: " << salary << '\n';
    }
    virtual ~Employee() = default;
};

class Manager : public Employee {
    int team_size;
public:
    Manager(std::string n, double s, int t)
        : Employee(std::move(n), s), team_size(t) {}
    void show() const override {
        Employee::show();
        std::cout << "  team: " << team_size << '\n';
    }
};
```

### 2.2 protected 访问控制

| 访问 | 类内 | 派生类 | 外部 |
|------|------|--------|------|
| public | ✓ | ✓ | ✓ |
| protected | ✓ | ✓ | ✗ |
| private | ✓ | ✗ | ✗ |

**protected** 像「家庭内部可见」：外人不能碰，子女可以。

## 3. 虚函数入门

### 3.1 没有 virtual 的问题

```cpp
class Base {
public:
    void greet() { std::cout << "Base\n"; }
};
class Derived : public Base {
public:
    void greet() { std::cout << "Derived\n"; }
};

Derived d;
Base* pb = &d;
pb->greet();  // 输出 Base — 静态绑定，看指针类型
```

### 3.2 加上 virtual

```cpp
class Base {
public:
    virtual void greet() { std::cout << "Base\n"; }
    virtual ~Base() = default;
};
class Derived : public Base {
public:
    void greet() override { std::cout << "Derived\n"; }
};

Derived d;
Base* pb = &d;
pb->greet();  // Derived — 动态绑定，看对象实际类型
```


### 术语三件套：虚函数（virtual function）

| 维度 | 内容 |
|------|------|
| **定义** | 通过基类指针/引用调用时，根据**对象实际类型**选择函数版本。 |
| **生活类比** | 遥控器选台：按同一按钮，电视/音响各自响应。 |
| **为什么重要** | 多态核心；与 [43 章](../43-继承与多态模型完全指南.md) 衔接。 |
| **本章用到** | §3 |


#### 常见错误：基类析构非 virtual

**错误写法：**

```cpp
class Base { public: ~Base() {} };
```

**正确写法：**

```cpp
class Base { public: virtual ~Base() = default; };
```

**原因**：`delete basePtr` 若指向派生对象，非虚析构只调 Base 析构，派生部分泄漏。


## 4. 派生类构造与析构顺序

### 4.1 构造：先基类，后派生

```cpp
class A {
public:
    A() { std::cout << "A ctor\n"; }
    ~A() { std::cout << "A dtor\n"; }
};
class B : public A {
public:
    B() { std::cout << "B ctor\n"; }
    ~B() { std::cout << "B dtor\n"; }
};
// B b; 输出：A ctor → B ctor → B dtor → A dtor
```

### 4.2 初始化列表必须调用基类 ctor

```cpp
class B : public A {
    int x;
public:
    B(int v) : A(), x(v) {}  // 先 A() 再初始化 x
};
```

若基类无默认构造，派生类**必须**在初始化列表显式调用基类构造函数。

## 5. 纯虚函数与抽象类

```cpp
class Shape {
public:
    virtual double area() const = 0;  // 纯虚：= 0
    virtual ~Shape() = default;
};
// Shape s;  // 错误：抽象类不可实例化

class Circle : public Shape {
    double r;
public:
    explicit Circle(double radius) : r(radius) {}
    double area() const override { return 3.14159 * r * r; }
};
```


### 术语三件套：抽象类（Abstract Class）

| 维度 | 内容 |
|------|------|
| **定义** | 含纯虚函数（`= 0`）的类，不能创建对象，用于定义接口。 |
| **生活类比** | 动物图鉴里的「抽象动物」——不能直接养，只能养具体品种。 |
| **为什么重要** | 框架与插件接口的常见模式。 |

## 6. Brass 与 BrassPlus 示例（Primer Plus 风格）

### 6.1 AcctABC 抽象基类

```cpp
#include <iostream>
#include <string>

class AcctABC {
protected:
    std::string fullName;
    long acctNum;
    double balance;
public:
    AcctABC(const std::string& name, long an, double bal);
    virtual ~AcctABC() = default;
    void ViewAcct() const;
    double Balance() const { return balance; }
    virtual void ViewTrades() const = 0;
    virtual bool Withdraw(double amount) = 0;
};

AcctABC::AcctABC(const std::string& name, long an, double bal)
    : fullName(name), acctNum(an), balance(bal) {}

void AcctABC::ViewAcct() const {
    std::cout << "Client: " << fullName << ", #" << acctNum
              << ", Balance: " << balance << '\n';
}
```

### 6.2 Brass 派生类

```cpp
class Brass : public AcctABC {
public:
    Brass(const std::string& name, long an, double bal);
    virtual void ViewTrades() const override;
    virtual bool Withdraw(double amount) override;
};

Brass::Brass(const std::string& name, long an, double bal)
    : AcctABC(name, an, bal) {}

bool Brass::Withdraw(double amount) {
    if (amount <= balance) {
        balance -= amount;
        return true;
    }
    return false;
}

void Brass::ViewTrades() const {
    std::cout << "Brass: standard account\n";
}
```

### 6.3 BrassPlus 再派生

```cpp
class BrassPlus : public Brass {
    double maxLoan;
    double rate;
    double owes;
public:
    BrassPlus(const std::string& name, long an, double bal,
              double ml = 500.0, double r = 0.11125);
    virtual bool Withdraw(double amount) override;
    virtual void ViewTrades() const override;
    void ViewAcct() const;  // 非虚：隐藏基类同名非虚函数
};

bool BrassPlus::Withdraw(double amount) {
    double total = balance + maxLoan - owes;
    if (amount <= total) {
        if (amount <= balance) balance -= amount;
        else {
            owes += amount - balance;
            balance = 0;
        }
        return true;
    }
    return false;
}
```

### 6.4 多态使用

```cpp
#include <vector>
#include <memory>

void test_accounts() {
    std::vector<std::unique_ptr<AcctABC>> accounts;
    accounts.push_back(std::make_unique<Brass>("Jim", 123, 1000));
    accounts.push_back(std::make_unique<BrassPlus>("Bob", 456, 2000));
    for (const auto& p : accounts) {
        p->ViewAcct();
        p->ViewTrades();
    }
}
```

## 7. 函数重写 vs 重载 vs 隐藏

| 概念 | 条件 | 关键字 |
|------|------|--------|
| 重写 override | 基类 virtual，签名相同 | `override` |
| 重载 overload | 同作用域，签名不同 | 无 |
| 隐藏 hide | 派生同名非 virtual | 小心 `using` |

```cpp
class Base {
public:
    virtual void f(int) {}
    void g() {}
};
class Derived : public Base {
public:
    void f(int) override {}
    void g() {}  // 隐藏 Base::g，非重写
};
```

## FAQ

**Q：继承与组合如何选？**  
A：is-a 用继承；has-a 用组合；能组合优先组合。

**Q：virtual 构造函数？**  
A：不存在；构造时对象类型已确定。

**Q：纯虚类能有 ctor？**  
A：可以；派生类构造时仍调基类 ctor。

**Q：override 写错签名？**  
A：编译报错，比裸 virtual 安全。

**Q：BrassPlus::ViewAcct 为何不是 override？**  
A：基类 `ViewAcct` 非 virtual，这是隐藏而非多态。

**Q：能用 `final` 吗？**  
A：C++11 `final` 禁止进一步 override。

**Q：多重继承？**  
A：下一章 [103](103-代码重用包含与多重继承入门.md) 展开。

**Q：智能指针存基类？**  
A：`unique_ptr<Base>` 需虚析构。
