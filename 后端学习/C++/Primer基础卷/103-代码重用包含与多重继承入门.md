## 1. 包含：has-a 关系

### 1.1 生活类比

汽车 **有** 引擎，不是 **是一种** 引擎——用成员对象组合：

```cpp
class Engine {
public:
    void start() { std::cout << "engine on\n"; }
};
class Car {
    Engine eng;  // has-a
    std::string model;
public:
    void drive() {
        eng.start();
        std::cout << model << " moving\n";
    }
};
```


### 术语三件套：组合（Composition）

| 维度 | 内容 |
|------|------|
| **定义** | 类 A 持有类 B 的对象成员，通过 B 的能力完成 A 的行为。 |
| **生活类比** | 电脑包含 CPU，不是 CPU 的一种。 |
| **为什么重要** | 松耦合、易维护；优先于继承。 |
| **本章用到** | §1 |

## 3. 私有继承与保护继承

### 3.1 语法与语义

```cpp
class Student : private std::string {  // 实现继承
    // string 接口在 Student 外部不可见
public:
    Student(const char* name) : std::string(name) {}
    using std::string::c_str;  // 选择性暴露
};
```

| 继承方式 | 对外 is-a？ | 典型用途 |
|----------|-------------|----------|
| public | 是 | 接口扩展 |
| protected | 否（对外像 private） | 少见 |
| private | 否 | **实现重用**，替代 has-a 的另一种写法 |

**private 继承** ≈ 「按实现重用基类代码」，工程上更常用 **has-a + 转发**。

## 4. 多重继承（MI）基础

```cpp
class Report {
public:
    virtual void report() const = 0;
    virtual ~Report() = default;
};
class Saveable {
public:
    virtual void save() const = 0;
    virtual ~Saveable() = default;
};
class Document : public Report, public Saveable {
public:
    void report() const override { std::cout << "doc report\n"; }
    void save() const override { std::cout << "doc save\n"; }
};
```

## 5. 菱形继承与虚继承

### 5.1 问题

```cpp
class Top { int x; };
class Left : public Top {};
class Right : public Top {};
class Bottom : public Left, public Right {};  // 两份 Top
```

`Bottom` 中有 **两个** `Top` 子对象 → 歧义。

### 5.2 虚继承解决

```cpp
class Top { public: int x; };
class Left : virtual public Top {};
class Right : virtual public Top {};
class Bottom : public Left, public Right {};  // 仅一份 Top
```


### 术语三件套：虚继承（virtual inheritance）

| 维度 | 内容 |
|------|------|
| **定义** | 在 MI 中共享虚基类子对象的唯一实例。 |
| **生活类比** | 家族族谱合并到同一个曾祖父。 |
| **为什么重要** | 解决菱形重复基类；有运行时成本。 |
| **本章用到** | §5 |


#### 常见错误：MI 歧义未解决

**错误写法：**

```cpp
class D : public B, public C {};
// B 与 C 都有 void f();
d.f(); // 歧义
```

**正确写法：**

```cpp
d.B::f();  // 或 virtual 继承 + 唯一实现
```

**原因**：编译器不知选哪条继承路径的 `f`。


## 6. MI 陷阱

1. **歧义**：两基类同名成员 → 作用域解析 `D::B::f()`
2. **函数隐藏**：比单继承更绕，善用 `using`
3. **对象布局复杂**：调试困难，见 [29 章](../29-对象模型与虚函数表深入.md)
4. **优先组合或接口类**：Java 式单继承 + interface 在 C++ 常用纯虚类模拟

## 7. 模板复用预告

```cpp
template<typename T>
class Stack {
    std::vector<T> data;
public:
    void push(const T& v) { data.push_back(v); }
    T pop() {
        T t = data.back();
        data.pop_back();
        return t;
    }
};
```

**继承**复用「接口与行为层次」；**模板**复用「类型无关算法与容器」。详见 [06 章](../06-模板与泛型编程.md)。

| 手段 | 关系 | 例子 |

## 9. 包含对象的程序设计（Primer Plus 风格）

### 9.1 Student 与 String 组合

```cpp
#include <string>
#include <iostream>

class Student {
    std::string name;
    std::vector<int> scores;
public:
    Student(std::string n, std::vector<int> s)
        : name(std::move(n)), scores(std::move(s)) {}
    double average() const {
        if (scores.empty()) return 0;
        long sum = 0;
        for (int v : scores) sum += v;
        return static_cast<double>(sum) / scores.size();
    }
    const std::string& getName() const { return name; }
};
```

组合使 `Student` **不必**继承 `string` 或 `vector`，只暴露业务接口。

### 9.2 为何 has-a 优于 illogical is-a

```cpp
// 错误设计：Stack 不是一种 vector，只是用 vector 实现
// class Stack : public std::vector<int> { };  // 暴露过多 vector 接口

class Stack {
    std::vector<int> data;  // has-a
public:
    void push(int v) { data.push_back(v); }
    int pop() {
        int t = data.back();
        data.pop_back();
        return t;
    }
    bool empty() const { return data.empty(); }
};
```

**封装**：用户看不到 `push_back`/`insert`，栈语义清晰。

## 10. 私有继承详解

```cpp
class Person {
    std::string name;
public:
    Person(const std::string& n) : name(n) {}
    const std::string& getName() const { return name; }
};

class StudentPriv : private Person {
public:
    StudentPriv(const std::string& n) : Person(n) {}
    using Person::getName;  // 恢复 name 访问
};
```

| 对比 | private 继承 | has-a 成员 |
|------|--------------|------------|
| 空基类优化 | 可能（EBO） | 成员必有大小 |
| 访问基 protected | 可以 | 通过接口 |
| 可读性 | 较差 | **更好** |

## 11. 多重继承实战：Worker 与 Singer

```cpp
class Worker {
    std::string name;
public:
    Worker(const std::string& n) : name(n) {}
    virtual void Show() const { std::cout << name << '\n'; }
    virtual ~Worker() = default;
};

class Singer {
    int vocal_range;
public:
    Singer(int v = 0) : vocal_range(v) {}
    virtual void Show() const { std::cout << "range: " << vocal_range << '\n'; }
    virtual ~Singer() = default;
};

class SingerWorker : public Worker, public Singer {
public:
    SingerWorker(const std::string& n, int v)
        : Worker(n), Singer(v) {}
    void Show() const override {
        Worker::Show();
        Singer::Show();
    }
};
```

**歧义**：若两基类都有同名非 virtual 函数，必须 `Worker::Show()` 限定。

## 12. 虚继承完整示例

```cpp
#include <iostream>

class Person {
public:
    std::string name;
    Person(const std::string& n) : name(n) {}
};

class Student : virtual public Person {
public:
    Student(const std::string& n, int g) : Person(n), grade(g) {}
    int grade;
};

class Employee : virtual public Person {
public:
    Employee(const std::string& n, int id) : Person(n), empId(id) {}
    int empId;
};

class StudentEmployee : public Student, public Employee {
public:
    StudentEmployee(const std::string& n, int g, int id)
        : Person(n), Student(n, g), Employee(n, id) {}
};

int main() {
    StudentEmployee se("Li", 90, 1001);
    std::cout << se.name << '\n';  // 无歧义：唯一 Person
}
```

**最派生类**负责初始化虚基 `Person(n)`。

## 15. 保护继承 protected inheritance

```cpp
class Device {
public:
    void on() {}
protected:
    int voltage_{5};
};

class Phone : protected Device {  // 对外：on/voltage 都变 protected
public:
    void call() { on(); }
};

// Phone p; p.on();  // 错误：protected 继承后基类 public 成员对外为 protected
```

| 继承 | 基类 public 在派生类外 | 典型场景 |
|------|------------------------|----------|
| public | public | is-a |
| protected | protected | 几乎不用 |
| private | private | 实现继承 |

## 16. 组合实现 Poker 牌局（生活类比）

一副牌 **有** 52 张牌（has-a `vector<Card>`），不是「是一种 vector」：

```cpp
struct Card { int suit; int rank; };

class Deck {
    std::vector<Card> cards;
public:
    Deck() {
        for (int s = 0; s < 4; ++s)
            for (int r = 1; r <= 13; ++r)
                cards.push_back({s, r});
    }
    void shuffle() { /* std::shuffle */ }
    Card deal() {
        Card c = cards.back();
        cards.pop_back();
        return c;
    }
};
```

## 17. 多重继承与接口分离

```cpp
class Printable {
public:
    virtual void print() const = 0;
    virtual ~Printable() = default;
};
class Serializable {
public:
    virtual std::string serialize() const = 0;
    virtual ~Serializable() = default;
};
class Report : public Printable, public Serializable {
public:
    void print() const override { std::cout << serialize(); }
    std::string serialize() const override { return "report data"; }
};
```

两个纯虚接口 → MI 合理；若共用一个 `Object` 基类则考虑虚继承。

## 18. iostream 与 MI（标准库案例）

`std::iostream` 同时继承 `istream` 与 `ostream`，共享 `ios_base` 状态——标准库在底层使用虚继承避免菱形重复。初学知道即可，详见 [49 IO 完全指南](../49-IO流与文件操作完全指南.md)。

## 19. 常见错误汇总

#### 错误 1：继承仅为复用代码

应优先组合；继承会绑定 public 接口，破坏封装。

#### 错误 2：忘记虚析构

MI 下通过基类指针 delete 任一路径都需要虚析构链。

#### 错误 3：虚继承构造顺序

最派生类直接初始化虚基，中间类构造函数中对虚基的初始化被忽略。

## FAQ

**Q：组合 vs 私有继承？**  
A：优先组合；private 继承极少，多用于 STL 适配器内部。

**Q：虚继承性能？**  
A：有额外 indirection；只在菱形场景需要。

**Q：一个类几个虚基？**  
A：可能多个，布局由编译器决定。

**Q：模板与继承？**  
A：模板编译期多态，继承运行时多态。

**Q：iostream 为何 MI？**  
A：输入输出多接口组合，历史设计。

**Q：还能用 mixin 吗？**  
A：CRTP 等模式见 06/77 章。

**Q：has-a 能访问 private 吗？**  
A：不能，需 public 接口或 friend。

**Q：MI 几个基类合适？**  
A：能少就少；超过 2 要审视设计。
