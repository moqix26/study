## §1 为什么要读开源

- **简历差异化**：「读过 vLLM 调度模块」比「了解 C++11」更有说服力。
- **工程直觉**：看 abseil 如何处理异常安全，比教科书具体。
- **面试素材**：STAR 故事来自真实 PR，不是虚构项目。
- **选库标准**：fmt 为何比 iostream 快——读源码得答案。

## §2 阅读大型 C++ 项目的通用流程

- **Step 1 编译跑通**：没有 green build 不要深入读；记录 CMake 选项。
- **Step 2 读 README/CONTRIBUTING**：构建矩阵、代码风格、测试命令。
- **Step 3 目录地图**：列出 src/include/tests/examples 职责，画 mermaid。
- **Step 4 跟调用链**：从 main 或公开 API 入口，depth≤5 层。
- **Step 5 做笔记**：每个模块一句话 + 关键类名 + 链接行号。
- **Step 6 提 issue/PR**：文档、测试、小 bug 优先。

## §3 工具链：不止用 IDE

- `rg` / `grep -r`：全库搜符号，比 Ctrl+F 快。
- `cscope` / `clangd`：跳转定义、找引用。
- `git log -p -- path`：看文件历史，理解演进。
- `cmake --build . --target help`：列出所有 target。
- `ctest -R pattern`：只跑相关测试。
- GitHub Code Search：跨 repo 搜 API 用法。

## §4 fmt：最小可读库（入门首选）

**仓库**：https://github.com/fmtlib/fmt

**目录结构**：
```text
include/fmt/
src/format.cc
test/
CMakeLists.txt
```

**推荐阅读路径**：从 `fmt::format` → `detail::vformat` → `buffer` 增长策略

**编译命令**：
```bash
cmake -DFMT_TEST=ON -B build && cmake --build build && ctest
```

**贡献切入点**：补充 benchmark、文档示例、typo fix

## §5 spdlog：日志架构范本

**仓库**：https://github.com/gabime/spdlog

**目录结构**：
```text
include/spdlog/
src/
example/
tests/
```

**推荐阅读路径**：logger → sinks → pattern_formatter → async thread pool

**编译命令**：
```bash
cmake -DSPDLOG_BUILD_EXAMPLE=ON -B build
```

**贡献切入点**：新 sink、formatter 测试、文档

## §6 abseil：Google C++ 基础库

**仓库**：https://github.com/abseil/abseil-cpp

**目录结构**：
```text
absl/strings/
absl/container/
absl/synchronization/
```

**推荐阅读路径**：string_view、InlinedVector、Mutex 实现取舍

**编译命令**：
```bash
cmake -DABSL_BUILD_TESTING=ON -B build
```

**贡献切入点**：需 CLA；从小测试/文档开始

## §7 llama.cpp：单机推理引擎

**仓库**：https://github.com/ggerganov/llama.cpp

**目录结构**：
```text
src/
ggml/
examples/
tests/
```

**推荐阅读路径**：main → llama_decode → ggml_compute_forward

**编译命令**：
```bash
cmake -B build && cmake --build build -j
```

**贡献切入点**：backend 适配、量化、example

## §8 vLLM：Serving 级调度（进阶）

**仓库**：https://github.com/vllm-project/vllm

**目录结构**：
```text
csrc/
vllm/
CMakeLists.txt
setup.py
```

**推荐阅读路径**：Python 调度 + C++ custom op；PagedAttention 论文对照

**编译命令**：
```bash
需 CUDA；先读文档再 pip install -e .
```

**贡献切入点**：C++ kernel、benchmark、文档；门槛高

## §9 从 Issue 到 PR 的完整流程

### 9.1 选 issue

标签 `good first issue` / `help wanted`；避免「重构全库」

### 9.2 Fork + 分支

`git checkout -b fix/issue-123-typo`

### 9.3 本地复现

issue 描述的操作必须能复现或写最小测试

### 9.4 改代码

一次 PR 只做一件事；跑全量测试

### 9.5 写 PR 描述

Fixes #123 + 根因 + 测试截图/命令

### 9.6 Code Review

礼貌回复；改 requested changes 后再 push

### 9.7 合并后

写博客/笔记；更新简历 STAR

## §10 给 C++ 简历加分的写法

| 弱写法 | 强写法 |
|--------|--------|
| 了解开源 | 向 fmt 提交 PR #xxxx：修复 xxx 边界条件，补充 3 个单元测试 |
| 读过 llama | 阅读 llama.cpp ggml 后端，本地编译 Q4 量化推理，输出 tokens/s 数据 |
| 会用 spdlog | 对比 spdlog async 与 sync 在 mini-http 压测下 P99 差异（链 12 章） |

**STAR 模板**：
```text
S：mini-http 需要结构化日志，选型 spdlog
T：理解 async sink 是否引入队列延迟
A：阅读 spdlog 源码 + 压测 + 提文档 PR 澄清 buffer 行为
R：P99 降 15%；PR merged；简历附 GitHub 链接
```

## §12 常见编译问题速查

| 现象 | 排查 |
|------|------|
| 找不到头文件 | 检查 `target_include_directories` 与 install 前缀 |
| undefined reference | 漏链库或未 `target_link_libraries` |
| C++ 标准不一致 | 统一 `CMAKE_CXX_STANDARD 17` |
| CUDA 版本不匹配 | vLLM 需查 README 对应 CUDA 版本 |
| 子模块未拉 | `git submodule update --init --recursive` |
| 权限/CLA | abseil/Google 项目需签 CLA |
| 测试超时 | ctest `-j` 并行或 `-R` 过滤 |
| MSVC vs GCC | WSL 编译与 Windows 行为差异，以 CI 为准 |

## §14 分级练习

### 基础

1. 克隆 fmt，编译并运行 `test/format-test`，截图 ctest 通过。
2. 用 `rg 'class.*sink' spdlog` 列出所有 sink 类名。
3. 读 CONTRIBUTING.md，总结 5 条代码风格要求。
4. 在本地 fork fmt，改 README 一处 typo，提 PR（可不 merge）。
5. 画 llama.cpp 目录 mermaid（至少 6 个节点）。

### 进阶

1. 为 fmt 某函数写额外单元测试（边界：空串、超大 width）。
2. 用 perf 对比 fmt vs sprintf 吞吐（链 12 章）。
3. 跟读 llama.cpp 一次 complete 请求的栈（GDB break main）。
4. 对比 abseil InlinedVector 与 std::vector 内存布局（sizeof）。
5. 写 800 字博客：spdlog async 队列满时的行为。

### 挑战

1. 向 spdlog 或 fmt 提交被 merge 的 PR（文档/测试均可）。
2. 本地跑通 vLLM 最小 example（有 GPU 时）。
3. 给 llama.cpp issue 写可复现最小 repo（即使未 fix）。

### 练习参考答案（要点）

基础 1：`git clone` → `cmake -DFMT_TEST=ON -B build` → `ctest --test-dir build`。
基础 2：常见 sink：stdout_color_sink、rotating_file_sink、udp_sink 等。
进阶 1：在 test/ 仿照现有 TEST 宏添加 CASE。
挑战 1：PR 描述含 Fixes、测试命令、before/after。

## §15 常见问题 FAQ

1. **没 GPU 能读 vLLM 吗？** 能读 Python 调度与文档；C++ kernel 需 CUDA 环境或读论文+源码注释。
2. **必须先读哪个项目？** fmt → spdlog → abseil → llama.cpp → vLLM，难度递增。
3. **读源码要逐行吗？** 不要。跟一条链 + 记模块边界；细节用时再查。
4. **英文看不懂怎么办？** 先看代码与测试；术语对照 01～50 章 + 词典。
5. **PR 被拒怎么办？** 看 review 意见改；礼貌问清；小步重提。
6. **要签 CLA 吗？** abseil、部分 Google 项目要；fmt/spdlog 通常不要。
7. **如何找 good first issue？** GitHub Issues 标签过滤 + 看评论是否有人指导。
8. **读开源算项目经历吗？** 算，但必须能 STAR + 链接 + 可验证产出（PR/博客/压测）。
9. **cscope 和 clangd 选哪个？** clangd 日常；cscope 大仓库离线也稳。
10. **需要读设计模式吗？** 读 abseil/fmt 时对照 [21 章](21-设计模式与Infra工程实践.md)。
11. **和 32 章关系？** 32 讲怎么用 fmt/spdlog；本章讲怎么读其源码并贡献。
12. **WSL 磁盘不够？** 浅 clone `--depth 1`；只 build 需要的 target。
13. **如何证明读过？** 笔记+调用链图+口述+可选 PR；面试时讲 trade-off。
14. **llama.cpp 和 vLLM 面试怎么选？** 推理引擎岗讲 llama+量化；Serving 岗讲 vLLM 调度+PagedAttention。
15. **贡献算实习吗？** 不算正式实习，但是强加分项；配合 36 章写进简历。

15. S 情境 T 任务 A 行动 R 结果。

## §17 费曼检验

3 分钟向非程序员解释：**为什么「能编译跑通」是读开源的第一步？**
