## 1. 函数原型与定义

### 1.1 声明 vs 定义

```cpp
// math_utils.h
#pragma once
int add(int a, int b);           // 原型 / 声明
double average(int a, int b);    // 可只声明

// math_utils.cpp
#include "math_utils.h"
int add(int a, int b) { return a + b; }  // 定义
double average(int a, int b) {
    return static_cast<double>(a + b) / 2.0;
}
```


### 术语三件套：函数原型（Function Prototype）

| 维度 | 内容 |
|------|------|
| **定义** | 告诉编译器**函数名、返回类型、参数列表**；调用前必须可见。 |
| **生活类比** | 餐厅菜单：菜名和配料，后厨再做（定义在 .cpp）。 |
| **为什么重要** | 头文件放声明、源文件放定义，是 C++ 工程化基础（09 章 CMake）。 |
| **本章用到** | §1 |

### 逐行读代码：重载与默认参数

```cpp
void log(int level, const std::string& msg = "info");
void log(const std::string& msg);  // 可能与默认参数冲突

int add(int a, int b) { return a + b; }
double add(double a, double b) { return a + b; }
```

| 行/片段 | 含义 | 改错会怎样 |
|---------|------|------------|
| `log(int, string=...)` | 两参，第二参有默认 | 默认参数只在一处声明 |
| `log(string)` | 单参重载 | 与 `log(1)` 可能歧义 |
| `add(int,int)` | 整型相加 | 调用 `add(1,2)` 匹配 |
| `add(double,double)` | 浮点相加 | `add(1,2.0)` 需转换规则 |

## FAQ


**Q：重载为什么不能用仅返回类型区分？**  
A：调用时无法仅从实参决定用哪个。

**Q：默认参数放 .h 还是 .cpp？**  
A：通常放**声明**（头文件），且只写一次。

**Q：inline 一定更快吗？**  
A：不一定；过度 inline 增大代码体积；编译器可忽略 inline 建议。

**Q：constexpr 函数必须很简单吗？**  
A：C++14 起可有多语句、局部变量、循环。

**Q：noexcept(false) 意义？**  
A：明确可能抛异常；移动操作无 noexcept 时 vector 重分配可能拷贝。

**Q：递归深度限制？**  
A：受栈大小限制；大 n 用循环或显式栈。

**Q：`std::function` 与函数指针？**  
A：`std::function` 可存 lambda、成员函数，更灵活，有类型擦除开销。

**Q：extern "C" 做什么？**  
A：C 链接名，供 C 或其他语言调用；不参与 C++ 重载。
