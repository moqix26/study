## 2. memory_order 六种

| 序 | 含义 | 典型 |
|----|------|------|
| relaxed | 原子性无顺序 | 统计计数 |
| consume | 依赖链 | 少用，常等同 acquire |
| acquire | 读：后续不重排到前 | 消费数据前读 flag |
| release | 写：之前不重排到后 | 发布数据后写 flag |
| acq_rel | 读改写 | CAS 成功 |
| seq_cst | 全局单序 | 默认，最易推理 |


## 3. happens-before

若 A happens-before B，则 A 的副作用对 B 可见。

```cpp
std::atomic<bool> ready{false};
int data = 0;

void producer() {
    data = 42;
    ready.store(true, std::memory_order_release);
}
void consumer() {
    while (!ready.load(std::memory_order_acquire)) {}
    assert(data == 42);
}
```


## 4. release-acquire 配对

**发布-订阅**：release 写之前的普通写，对 acquire 读之后的普通读可见。

[25 章 SPSC 队列](25-无锁编程与内存序.md) head/tail 更新常用此模式。


## 5. seq_cst 与降级

默认 `atomic` 为 seq_cst。热路径在证明无额外依赖后可降级 relaxed/acquire/release。

错误降级 → 看不见新写入 → 极难复现 bug。


## 6. 硬件 TSO 直觉

x86 **Total Store Order**：Store→Store、Store→Load 有序；Load→Load、Load→Store 可能重排。

C++ 原子映射为带 `lock` 前缀或 fence 的指令。ARM 更弱 → 跨平台代码更依赖 memory_order 而非假设 x86。

见 [22 章](22-计算机体系结构导读.md)、[18 章 false sharing](18-高性能C++与内存对齐.md)。
