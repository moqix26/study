| Q2 | load average 含义？ | 可运行+不可中断睡眠队列均值；IO 高也会 load 高。 | [53 OS](53-操作系统面试八股与口述模板.md) |

| Q5 | user vs sys CPU？ | sys 高可能 syscall/锁/内核路径多。 | 上下文切换 |
| Q6 | C++ 热点常见？ | 锁竞争、内存分配、日志格式化、JSON 解析。 | [24 分配器](24-内存分配器与对象池.md) |

| Q10 | 误优化案例？ | 盲目 -O3 或 inline 反而变大；需 profile 驱动。 | [18 高性能](18-高性能C++与内存对齐.md) |

## §2 内存泄漏：ASan 与 Valgrind

| 编号 | 面试问题 | 标准答法（口述版） | 追问/坑点 |
|------|----------|-------------------|-----------|
| Q1 | 内存泄漏排查路径？ | 现象 RSS 涨→ASan/Valgrind 测试环境→massif/heap profile→修复。 | 生产先限流 |
| Q2 | ASan 用法？ | -fsanitize=address -g；检测 UAF/heap overflow；约 2x 内存。 | CI 集成 |
| Q3 | Valgrind memcheck？ | 慢但全面；--leak-check=full；生产不适用。 | 虚机测试 |
| Q4 | RSS vs VSZ？ | RSS 物理驻留；VSZ 虚拟地址空间。 | mmap |
| Q5 | C++ 常见泄漏？ | 裸 new 无 delete、循环 shared_ptr、全局容器只增不减。 | [02 内存](02-指针引用与内存管理.md) |
| Q6 | tcmalloc/jemalloc profile？ | HEAPPROFILE 环境变量；pprof 看分配热点。 | [24 章](24-内存分配器与对象池.md) |
| Q7 | OOM Killer？ | dmesg 看谁被杀；调 limits 或修泄漏。 | 容器 OOMKilled |
| Q8 | 内存碎片？ | 长期分配释放导致；arena/tcmalloc；重启缓解。 | 大块分配 |
| Q9 | Sanitizer 组合？ | ASan+UBSan；不能全开 TSan（与 ASan 冲突）。 | [08 线程](08-多线程与并发编程.md) |
| Q10 | 生产 heap dump？ | gcore+gdb heap；或 jeprof 触发；谨慎性能。 | 审批流程 |

## §7 STAR 案例库

### §7.1 STAR 案例 1

**S（情境）**：凌晨 QPS 跌零，监控 CPU 100% 单核 C++ 网关。

**T（任务）**：15 分钟恢复服务。

**A（行动）**：top 定位线程；perf 火焰图指向 JSON parse；热点加 cache+simdjson；限流兜底。

**R（结果）**：5 分钟恢复，CPU 降至 40%，复盘文档进 [36 章](36-面试STAR表达与简历手册.md)。

**连环追问**：
- 为何单核？
- 如何验证修复？
- perf 采样频率？

### §7.2 STAR 案例 2

**S（情境）**：RSS 每周涨 20GB 最终 OOMKilled。

**T（任务）**：找到泄漏并发版。

**A（行动）**：ASan 复现 UAF；修复智能指针循环；CI 开 ASan nightly。

**R（结果）**：RSS 平稳；K8s 不再重启。

**连环追问**：
- ASan 生产能用吗？
- shared_ptr 陷阱？
- 与 [24 分配器](24-内存分配器与对象池.md) 关系？

### §7.3 STAR 案例 3

**S（情境）**：MySQL 死锁告警频发。

**T（任务）**：降低死锁率 90%。

**A（行动）**：SHOW ENGINE；固定多表更新顺序；索引减少 gap lock；短事务。

**R（结果）**：死锁从 200/天到 20/天。

**连环追问**：
- Next-Key Lock？
- Gap lock 触发？
- 见 [51 章](51-MySQL原理与索引事务面试专章.md)
