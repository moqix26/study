## 1. 项目目标与成功标准

### 1.1 核心目标

1. 构建 **可复现** 的领域 SFT 流水线。
2. 验证集 perplexity 或任务指标 **优于 base 模型**。
3. 提供 **可交互 Demo** 与 **实验记录**。
4. 文档达到「他人 clone 后能跑」标准。

### 1.2 非目标（scope 控制）

- 不做全参预训练
- 不追求 SOTA benchmark
- 不必须 RLHF（可选加分 [16 章](16-RLHF-DPO与GRPO入门.md)）

## 4. Phase 1：数据工程

数据格式见 §4.1 messages jsonl；流程：raw → QA 抽取 → 去重 → train:val:test = 8:1:1（**document-level** 划分）。train ≥ 500 条；交付 `data/processed/*.jsonl` + `DATA_CARD.md`。

```json
{"messages": [{"role":"user","content":"..."},{"role":"assistant","content":"..."}], "source":"doc.md", "split":"train"}
```

参考 [18 章](18-大模型数据工程与预处理.md)；与 [AIAgent 06 ingest](../AIAgent/06-RAG检索增强生成基础.md) 可共用 `kb-docs`。

## 5. Phase 2：LoRA 微调

### 5.1 超参起点（0.5B）

```yaml
model_name: Qwen/Qwen2.5-0.5B-Instruct
max_seq_len: 2048
train:
  lr: 2.0e-4
  batch_size: 4
  gradient_accumulation_steps: 8
  epochs: 3
  warmup_ratio: 0.03
lora:
  r: 16
  alpha: 32
  target_modules: [q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj]
  dropout: 0.05
```

详见 [15 章](15-微调SFT与LoRA-PEFT.md)；多卡见 [17 章](17-分布式训练DDP-FSDP与DeepSpeed.md)。

### 5.2 训练命令

```bash
export WANDB_PROJECT=domain-llm-sft
python src/train/train_lora.py --config configs/lora_qwen_0.5b.yaml
```

### 5.3 监控要点

- train/val loss 曲线应下降；val 过早上升 → 过拟合，减 epoch 或加数据
- GPU 显存：0.5B LoRA bf16 约 6～8G
- 保存 `load_best_model_at_end` 的 checkpoint

### 5.4 交付物

- `outputs/checkpoint-best/`
- wandb run URL
- `configs/lora_qwen_0.5b.yaml` 与 git commit 绑定

## 7. 风险与排错

| 问题 | 处理 |
|------|------|
| OOM | 减 batch、增 grad_accum、QLoRA |
| 过拟合 | 增数据、减 epoch、增 lora dropout |
| 复读机 | 检查数据重复、调高 temperature 仅 infer |
| 通用能力崩 | 混入 10% 通用 alpaca 子集 |
| 评估造假 | 保留 base 样例文件，面试可 live compare |

## 9. FAQ

**Q1：没有领域数据怎么办？**  
用公开 Huatuo-Lite、COIG 子集，或从 Wikipedia 构造 QA；DATA_CARD 写清来源。

**Q2：0.5B 够简历吗？**  
够；强调 **工程完整度** 与 **指标对比**，非模型规模。

**Q3：要和 AIAgent 项目合并吗？**  
可：Java 调 Python FastAPI 推理；或同一 kb-docs，Java RAG + Python SFT 对比。

**Q4：必须 merge LoRA 吗？**  
Demo 可 PeftModel；部署 vLLM 通常 merge，见 [23 章](23-模型导出ONNX与TorchScript.md)。

**Q5：与 LLMInfra 19 如何双项目写简历？**  
「Python 负责模型微调与评估；C++ 负责同一架构推理引擎实现，理解 train-deploy 全栈。」
