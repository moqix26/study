## 1. 基础类型与变量



### 1.1 预声明类型


| 类别    | 类型                             | 零值      | 备注                |
| ----- | ------------------------------ | ------- | ----------------- |
| 布尔    | `bool`                         | `false` |                   |
| 字符串   | `string`                       | `""`    | **不可变** UTF-8     |
| 整型    | `int`, `int8`…`int64`, `uint`… | `0`     | `int` 随平台 32/64 位 |
| 浮点    | `float32`, `float64`           | `0`     |                   |
| 字节/符文 | `byte`(=uint8), `rune`(=int32) | `0`     | 处理 Unicode        |


**术语（rune）**：Go 中表示 **一个 Unicode 码点**；遍历 string 用 `range` 得到 rune。

### 1.2 变量声明

```go
var age int = 20
var name = "alice"      // 推断 string
country := "CN"         // 短声明，函数内常用

const Pi = 3.14159
const (
	Sunday = iota // 0
	Monday        // 1
)
```


| 方式               | 场景     |
| ---------------- | ------ |
| `var x T`        | 包级、需零值 |
| `:=`             | 函数内短声明 |
| `const` / `iota` | 枚举常量   |




### 1.3 流程控制

Go **只有** `for`，无 while：

```go
for i := 0; i < 10; i++ { /* ... */ }

sum := 0
for sum < 1000 {
	sum += sum
}

for k, v := range m { /* map/slice/string */ }

switch day {
case Sunday, Saturday:
	// 自动 break
default:
}
```



### 1.4 `string`、`byte` 与 `rune`：长度为什么不等于字符数 ⭐

Go 的 `string` 是一段**只读字节序列**，通常装 UTF-8 文本，但语言本身不保证内容一定是合法 UTF-8。

```go
s := "Go语言"

fmt.Println(len(s))                    // 8：字节数，不是字符数
fmt.Println(utf8.RuneCountInString(s)) // 4：Unicode 码点数

fmt.Println([]byte(s))       // UTF-8 字节数值
fmt.Printf("%c\n", []rune(s)) // [G o 语 言]，%c 按码点显示
```


| 写法                    | 得到什么                 | 典型用途                  |
| --------------------- | -------------------- | --------------------- |
| `len(s)`              | 字节数                  | 网络协议长度、存储大小           |
| `s[i]`                | 第 `i` 个字节，类型为 `byte` | ASCII、二进制协议           |
| `for _, r := range s` | 逐个解码出的 `rune`        | 中文、Emoji 等 Unicode 文本 |
| `[]byte(s)`           | 可修改的字节切片，会发生转换/拷贝    | IO、哈希、协议处理            |
| `[]rune(s)`           | 码点切片                 | 按“字符”反转或截取            |


`range` 返回的索引仍是**该 rune 在原字符串中的字节偏移**：

```go
for i, r := range "A中B" {
	fmt.Printf("i=%d r=%c\n", i, r)
}
// i=0 A；i=1 中；i=4 B
```

注意：“一个用户看到的字符”不一定等于一个 rune，例如某些 Emoji 由多个码点组合。普通后端业务掌握 UTF-8、byte、rune 的区别即可；需要严格处理用户可见字符时再使用 Unicode 文本库。

## 2. array 与 slice



### 2.1 array：值类型、定长

```go
var a [3]int = [3]int{1, 2, 3}
b := [...]int{4, 5, 6} // 编译器数长度
```

**array 赋值 = 整块拷贝**，很少直接使用；多用 **slice**。

### 2.2 slice 底层结构 ⭐

**术语（slice）**：对底层 **array 的引用视图**，含指针、长度、容量。

```text
type slice struct {
    ptr *T    // 指向底层数组某元素
    len int   // 当前元素个数
    cap int   // 从 ptr 起底层数组可用容量
}
```

```mermaid
    end
    subgraph underlying_array
        A0["[0]"]
        A1["[1]"]

```





### 2.3 创建与切片

```go
s := make([]int, 0, 10) // len=0 cap=10
s = append(s, 1, 2, 3)
sub := s[1:3]   // [2 3], len=2
sub2 := s[1:3:3] // cap 也被截断
copy(dst, src)
```



### 2.4 扩容规则（面试必背）

当 `len+新增 > cap` 时：

1. 如果一次 `append` 所需容量已经超过旧容量的 2 倍，运行时会优先以“至少装得下”为目标，而不是机械翻倍。
2. 对较小 slice，容量通常接近 **2 倍增长**；旧容量达到约 256 后，公式会平滑过渡，增长率逐渐趋近 **1.25 倍**。
3. 运行时还会按元素大小和内存分配器的 size class **向上取整**，因此最终 `cap` 不保证正好是 2 倍或 1.25 倍。
4. 扩容会分配新数组并复制旧元素；扩容前后两个 slice 之后可能指向不同数组。

因此，面试不要把某个固定阈值和倍率说成语言规范。更准确的表述是：**具体策略属于 runtime 实现细节，核心语义是容量不足时分配更大的底层数组并复制；可预估规模时应提前分配容量。**

**动手验证**：

```go
s := make([]int, 0, 1)
for i := 0; i < 20; i++ {
	oldCap := cap(s)
	s = append(s, i)
	if cap(s) != oldCap {
		fmt.Printf("len=%d cap: %d -> %d\n", len(s), oldCap, cap(s))
	}
}
```

预估写入 10 万个元素时优先：`make([]T, 0, 100_000)`，可以减少扩容次数、内存复制和 GC 压力。

### 2.5 常见坑


| 坑         | 示例                         | 后果                     |
| --------- | -------------------------- | ---------------------- |
| 共享底层      | `sub := s[:2]` 后改 `sub[0]` | `s[0]` 也变              |
| append 误判 | `append(sub, 99)` cap 够    | 可能改原 slice             |
| nil slice | `var s []int`              | `len=0 cap=0`，可 append |




### 2.6 小切片引用大数组：隐蔽的内存保留

slice 只保存指针、长度、容量。即使只留下很小的一段，只要它仍引用原数组，整个大数组通常都不能被 GC 回收。

```go
func firstKB(data []byte) []byte {
	return data[:1024] // 若 data 是 100MB，返回值仍可能让 100MB 数组存活
}

func firstKBCopy(data []byte) []byte {
	out := make([]byte, 1024)
	copy(out, data[:1024])
	return out // 只保留新的 1KB 数组
}
```

Go 1.21+ 也可以写 `out := slices.Clone(data[:1024])`。这类问题常见于读取大文件、解析大 HTTP body 后把很小字段长期缓存。

## 3. map



### 3.1 创建与使用

```go
m := make(map[string]int)
m["go"] = 1
m["rust"] = 2

v, ok := m["java"]
if !ok {
	// 不存在
}
delete(m, "go")

for k, v := range m {
	_ = k
	_ = v
}
```

**术语（map）**：哈希表实现，**遍历顺序随机**，**非线程安全**。

### 3.2 零值与 make


| 状态                  | 读    | 写         |
| ------------------- | ---- | --------- |
| `var m map[K]V` nil | 返回零值 | **panic** |
| `make(map[K]V)`     | 正常   | 正常        |




### 3.3 并发安全

并发写 map **直接 panic**；用 `sync.Mutex` 或 `sync.Map`（见 04 章）。

### 3.4 map 的底层思路与扩容边界 ⭐

可以把 map 理解为“哈希值定位桶，再在桶内寻找 key”。当前 runtime 的具体字段和桶组织可能随版本变化，面试重点应放在稳定结论：

1. key 先经过哈希函数得到位置；冲突时需要在同一组候选位置继续比较。
2. 元素增多或冲突严重时，map 会渐进式迁移到更大的存储结构；一次写操作可能顺带搬迁一部分旧数据。
3. 因为扩容和元素移动，**map 元素不可取地址**：`&m[k]` 不合法。
4. 遍历顺序不稳定，不能把 map range 顺序用于业务排序或测试断言。
5. 删除元素不意味着进程内存立即下降；runtime 可能保留已申请的桶供后续复用。

```go
type User struct{ Age int }
m := map[string]User{"alice": {Age: 18}}

// m["alice"].Age++ // 编译错误：map 索引结果不可寻址
u := m["alice"]
u.Age++
m["alice"] = u
```

**哪些类型能做 key？** 必须可比较：整数、字符串、指针、channel、interface（动态值也必须可比较）、仅含可比较字段的 array/struct 可以；slice、map、func 不可以。浮点数虽然可比较，但 `NaN != NaN`，通常不适合作 key。

**预分配**：已知大概元素数时使用 `make(map[K]V, n)`，hint 不是硬容量上限，但能减少扩容成本。

## 4. struct



### 4.1 定义与字面量

```go
type User struct {
	ID       int64  `json:"id"`
	Username string `json:"username"`
	Active   bool   `json:"-"`
}

u := User{ID: 1, Username: "alice"}
p := &User{Username: "bob"} // 指针字面量
```

**术语（struct tag）**：字段后的反引号字符串，供 `encoding/json`、`gorm` 等反射读取。

### 4.2 嵌入（组合）

```go
type Account struct {
	User
	Balance float64
}

a := Account{User: User{Username: "alice"}, Balance: 100}
fmt.Println(a.Username) //  promoted field
```

Go **无 class 继承**，用 **嵌入实现组合**。

### 4.3 struct 内存对齐：字段顺序为什么会影响大小

CPU 通常更高效地访问按自然边界对齐的数据，因此编译器可能在字段之间插入 padding（填充字节）。

```go
type A struct {
	Flag bool  // 1 byte + padding
	ID   int64 // 8 bytes
	Code int32 // 4 bytes + tail padding
}

type B struct {
	ID   int64
	Code int32
	Flag bool
}

fmt.Println(unsafe.Sizeof(A{}), unsafe.Sizeof(B{}))
```

在常见 64 位平台上，`B` 往往比 `A` 更紧凑。实际大小应以目标平台的 `unsafe.Sizeof` 为准。

工程原则：

- 普通业务 struct 优先可读性，不要为了省几个字节打乱领域含义。
- 数百万对象、缓存条目或性能敏感结构才值得测量并调整字段顺序。
- `unsafe.Sizeof` 不包含指针指向的数据，例如 string 内容、slice 底层数组和 map 元素。

## 5. 指针基础

```go
x := 42
p := &x   // 取地址
fmt.Println(*p) // 解引用

```


| 对比 C++      | Go                |
| ----------- | ----------------- |
| 指针运算        | **禁止**            |

```go
func increment(n int) {
	n++ // 不影响外部
}
func incrementPtr(n *int) {
	*n++
}
```

## 6. for 与 range

```go
nums := []int{10, 20, 30}
for i, v := range nums {
	_ = i
	_ = v
}

// 只要值
for _, v := range nums { _ = v }

// string → rune
for i, r := range "Go语言" {
	_, _ = i, r
}
```

**注意**：`range` 的 `v` 是 **拷贝**；改 `v` 不改 slice 元素，需 `nums[i] = x`。

## 9. 动手：词频统计（完整）

```go
package main

import (
	"bufio"
	"fmt"
	"os"
	"strings"
)

func main() {
	if len(os.Args) != 2 {
		fmt.Fprintln(os.Stderr, "usage: wordcount <file>")
		os.Exit(1)
	}
	f, err := os.Open(os.Args[1])
	if err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	defer f.Close()

	freq := make(map[string]int)
	sc := bufio.NewScanner(f)
	for sc.Scan() {
		line := sc.Text()
		for _, w := range strings.Fields(line) {
			w = strings.ToLower(strings.Trim(w, ".,!?\"'"))
			if w == "" {
			    continue
			}
			freq[w]++
		}
	}
	if err := sc.Err(); err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	for word, count := range freq {
		fmt.Printf("%s: %d\n", word, count)
	}
}
```

**test.txt**：

```text
Go is great
go is fun
```

**预期输出**（顺序不定）：

```text
go: 2
is: 2
great: 1
fun: 1
```

## 10. 常见报错与排查（≥8 条）


| #   | 现象                               | 原因                | 解决                                  |
| --- | -------------------------------- | ----------------- | ----------------------------------- |
| 1   | `index out of range`             | slice 下标 ≥ len    | 检查 len；循环 `< len`                   |
| 2   | `assignment to entry in nil map` | 未 make 就写         | `m := make(map[...]...)`            |

## 11. FAQ（≥10）



### Q1：slice 和 array 面试怎么说？

array 定长值类型；slice 是描述符 `{ptr,len,cap}` 引用底层数组，动态扩容。

### Q2：make 和 new 区别？

`make` 用于 **slice、map、chan**，返回已初始化引用；`new(T)` 分配零值，返回 `*T`。

### Q3：map  key 能是 slice 吗？

不能；key 必须 **可比较**（无 slice/map/func）。

### Q4：struct 能比较吗？

字段都可比较时才能 `==`；含 slice/map 不可。

### Q5：string 底层？

只读字节序列；改 string 需转 `[]byte` 或拼接。

### Q6：为什么 Go 无 while？

语法极简，`for` 三种形式覆盖 while。

### Q7：iota 干什么用？

const 块内自增枚举，常用于状态码、位掩码。

### Q8：json tag 里 omitempty？

零值字段序列化时 **省略**。

### Q9：指针 nil 解引用？

panic；使用前判 nil。

### Q10：词频题 map 遍历无序怎么办？

要排序：把 key 放进 slice，`sort.Strings` 再输出。

### Q11：和 [数据结构 02](../数据结构/02-线性结构/01-数组.md) 关系？

理论看数据结构；本章是 **Go API 与底层**。

### Q12：LeetCode 用 Go 注意什么？

`append` 预分配 cap；map 存在 O(1)；注意 **slice 共享底层** 陷阱。

## 13. 费曼检验

**3 分钟解释 slice 和 map 给非程序员。**

**提纲**：

1. slice = **动态数组视图**，能变长，多个 slice 可能共用一块数组。
2. map = **字典**，查单词快，但 **不能多人同时改**（要加锁）。
3. struct = **一条记录**，字段可贴标签给 JSON 用。
