## 0. 读前导读（零基础也能跟上）

### 0.1 用一句话弄懂本章

**一句话**：Ubuntu 像「应用商店 + 仓库管理员」——**`apt update` 刷新商品目录**，**`apt install` 一键装 JDK/MySQL/Redis**，不用自己编译源码。

**生活类比**：

| 概念 | 类比 |
|------|------|
| **APT（Advanced Package Tool）** | 超市收银系统：选商品、算总价、自动找零钱（依赖） |
| **dpkg** | 仓库搬运工：把 `.deb` 盒子搬到指定货架 |
| **sources.list** | 超市进货渠道清单（官方仓 vs 清华镜像仓） |
| **依赖（dependency）** | 买咖啡机必须配滤纸——apt 自动一起装 |
| **~/study/linux-practice** | 你的实验台：装完环境在这里跑 Spring Boot 联调 |

**术语（Package / 软件包）**：Ubuntu 把软件打成 `.deb` 标准盒，带版本号和依赖清单。  
**为什么重要**：后端开发机 80% 工具（JDK、MySQL、Git）都靠 apt 装，比手动下载 tar 包省心。  
**本章用到的地方**：§1～§8 全部安装流程。

## 1. APT 与 dpkg 的关系

### 1.1 两层结构

| 层级 | 工具 | 作用 |
|------|------|------|
| 高层 | **apt** / **apt-get** | 自动解决依赖、从源下载 |
| 底层 | **dpkg** | 安装本地 `.deb` 包 |

日常开发 **99% 用 apt**；只有离线 deb 或排查损坏包时才直接 `dpkg`。

### 1.2 apt 核心命令

**术语（apt update vs apt upgrade）**：`update` 只刷新「商品目录」；`upgrade` 把已装软件升到目录里的新版本。  
**生活类比**：update=刷新外卖 App 菜单；upgrade=把已点的旧套餐换成新套餐。

| 步骤 | 你的动作 | 预期看到什么 | 若不对 |
|------|----------|--------------|--------|
| 1 | `sudo apt update` | `Reading package lists... Done` | 见 §12 网络/DNS 报错 |
| 2 | `apt search openjdk-17 \| head -5` | 列出 openjdk 相关包 | 先 update；检查 universe 源 |
| 3 | `sudo apt install -y curl` | `Setting up curl ...` | 包名拼写；`apt search curl` |
| 4 | `curl --version \| head -1` | `curl 7.x.x` | install 失败查 §12 |
| 5 | `sudo apt list --upgradable \| wc -l` | 数字 ≥ 0 | 正常；0 表示无需升级 |

```bash
sudo apt update              # 更新包索引（不升级软件）
sudo apt upgrade -y          # 升级已安装包
sudo apt install -y 包名      # 安装
sudo apt remove -y 包名       # 卸载（保留配置）
sudo apt purge -y 包名        # 卸载并删配置
sudo apt autoremove -y        # 删除不再需要的依赖
sudo apt search 关键词        # 搜索包
apt show 包名                 # 查看包详情
```

**`apt update` 预期输出（节选）**：

```text
Hit:1 http://archive.ubuntu.com/ubuntu jammy InRelease
Get:2 http://archive.ubuntu.com/ubuntu jammy-updates InRelease [128 kB]
...
Reading package lists... Done
Building dependency tree... Done
Reading state information... Done
All packages are up to date.
```

**`apt install -y curl` 预期（已安装则不同）**：

```text
Reading package lists... Done
Building dependency tree... Done
The following NEW packages will be installed:
  curl
...
Setting up curl (7.81.0-1ubuntu1.20) ...
```

### 1.3 dpkg 常用（排查用）

```bash
dpkg -l | grep openjdk
dpkg -L openjdk-17-jdk-headless    # 列出包安装的文件
dpkg -S /usr/bin/java              # 查文件属于哪个包
sudo dpkg -i xxx.deb               # 安装本地 deb
sudo dpkg --configure -a           # 修复中断的安装
```

**`dpkg -l | grep openjdk` 预期（未装时无输出；已装）**：

```text
ii  openjdk-17-jdk-headless:amd64  17.0.x+xx  amd64  ...
ii  openjdk-17-jre-headless:amd64  17.0.x+xx  amd64  ...
```

## 2. 软件源 sources.list 与国内镜像

### 2.1 源文件位置

```bash
cat /etc/apt/sources.list
ls /etc/apt/sources.list.d/
```

Ubuntu 22.04 默认使用 **archive.ubuntu.com**；国内 VM 建议换 **清华 / 阿里 / 中科大** 镜像加速。

### 2.2 备份并替换源（清华镜像示例）

| 步骤 | 你的动作 | 预期看到什么 | 若不对 |
|------|----------|--------------|--------|
| 1 | `sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak` | 无输出 | 检查 sudo 权限 |
| 2 | 写入清华源（见下方 tee 命令） | 文件写入成功 | 22.04 用 jammy；24.04 用 noble |
| 3 | `sudo apt update` | `Hit:1 https://mirrors.tuna...` | GPG 错误见 §12 |
| 4 | `apt policy curl \| head -3` | 显示 Candidate 版本 | 源未生效则重读 sources.list |

```bash
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak

sudo tee /etc/apt/sources.list << 'EOF'
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-security main restricted universe multiverse
EOF

sudo apt update
```

**Ubuntu 24.04** 把 `jammy` 改为 `noble`。

**预期 update**：

```text
Hit:1 https://mirrors.tuna.tsinghua.edu.cn/ubuntu jammy InRelease
...
Reading package lists... Done
```

### 2.3 深入解释：main / universe / multiverse

| 组件 | 含义 |
|------|------|
| **main** | 官方支持的开源软件 |
| **restricted** | 受限驱动（如部分显卡） |
| **universe** | 社区维护开源软件 |
| **multiverse** | 版权受限软件 |

OpenJDK、MySQL、Redis 多在 **main** 或 **universe**；MySQL 官方也提供 **独立 apt 源**（下文安装节）。

### 2.4 在 ~/study/linux-practice 记录装机进度（推荐）

| 步骤 | 你的动作 | 预期看到什么 | 若不对 |
|------|----------|--------------|--------|
| 1 | `mkdir -p ~/study/linux-practice/notes` | 目录存在 | 权限 |
| 2 | 换源成功后 `date >> ~/study/linux-practice/notes/apt.log` | 文件有新行 | 路径 |
| 3 | 每装完一项 append 版本 | 如 `java -version >> notes/apt.log` | 重定向 2>&1 |
| 4 | 装完运行 check-env.sh | 全部通过 | 对照 §12 |
| 5 | VMware 打快照「env-08-done」 | 快照列表可见 | 08 章练习建议 |

```bash
mkdir -p ~/study/linux-practice/notes
echo "=== $(date) apt mirror switched ===" >> ~/study/linux-practice/notes/apt.log
# 装完 JDK 后：
java -version >> ~/study/linux-practice/notes/apt.log 2>&1
```

**为什么重要**：重装 VM 或帮同学装机时，有一份「我做过什么」比凭记忆可靠；09 章可把上述写成函数 `log_step()`。

## 3. 安装 OpenJDK 17

### 3.1 搜索与安装

| 步骤 | 你的动作 | 预期看到什么 | 若不对 |
|------|----------|--------------|--------|
| 1 | `apt search openjdk-17-jdk \| head -3` | 列出 openjdk-17-jdk | `apt update` 后再搜 |
| 2 | `sudo apt install -y openjdk-17-jdk` | `Setting up openjdk-17-jdk` | 磁盘满 `df -h` |
| 3 | `java -version` | `openjdk version "17.0.x"` | `command not found` 见 §12 |
| 4 | `javac -version` | `javac 17.0.x` | 只装 JRE 未装 JDK |
| 5 | `readlink -f $(which java)` | 路径含 `java-17-openjdk` | 多版本用 update-alternatives |

```bash
apt search openjdk-17
sudo apt install -y openjdk-17-jdk
```

**安装过程预期（节选）**：

```text
The following additional packages will be installed:
  openjdk-17-jdk-headless openjdk-17-jre openjdk-17-jre-headless ...
...
Setting up openjdk-17-jdk:amd64 (17.0.x+xx) ...
update-alternatives: using /usr/lib/jvm/java-17-openjdk-amd64/bin/java
```

### 3.2 验证

```bash
java -version
javac -version
echo $JAVA_HOME
```

**预期 `java -version`**：

```text
openjdk version "17.0.x" 2024-xx-xx
OpenJDK Runtime Environment (build 17.0.x+xx-Ubuntu-xx)
OpenJDK 64-Bit Server VM (build 17.0.x+xx-Ubuntu-xx, mixed mode, sharing)
```

### 3.3 配置 JAVA_HOME（推荐）

```bash
sudo tee /etc/profile.d/jdk17.sh << 'EOF'
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
EOF

source /etc/profile.d/jdk17.sh
echo $JAVA_HOME
```

**预期**：

```text
/usr/lib/jvm/java-17-openjdk-amd64
```

### 3.4 多版本切换（了解）

```bash
sudo update-alternatives --config java
```


```bash

# MySQL 内创建开发用户
CREATE USER 'dev'@'localhost' IDENTIFIED BY 'Dev@123456';
CREATE USER 'dev'@'%' IDENTIFIED BY 'Dev@123456';
GRANT ALL PRIVILEGES ON *.* TO 'dev'@'localhost';
GRANT ALL PRIVILEGES ON *.* TO 'dev'@'%';
FLUSH PRIVILEGES;
EXIT;
```

```bash
mysql -u dev -p
# 输入 Dev@123456
```

**预期登录后**：

```text

+-----------+
| VERSION() |
+-----------+
| 8.0.xx    |
+-----------+
```

## 10. 深入解释：为什么先 apt update 再 install

`apt update` 只更新 **/var/lib/apt/lists/** 里的包索引，不改动已装软件。索引过期时 `apt install` 可能：

- 找不到新版本
- 报 `Unable to locate package`
- 依赖解析失败

**最佳实践**：装机第一天 `update + upgrade`；之后每周一次；换源后 **必须** `apt update`。

## 12. 常见报错与排查

| 报错信息（关键词） | 可能原因 | 解决方案 |
|-------------------|---------|---------|
| `Unable to locate package` | 包名错或未 update | `apt search`；`sudo apt update` |
| `E: Could not get lock /var/lib/dpkg` | 另一个 apt 正在跑 | 等完成或 `sudo kill` 占用进程；`sudo dpkg --configure -a` |
| `dpkg: error processing package` | 依赖断裂 | `sudo apt -f install`；`sudo dpkg --configure -a` |
| `GPG error` / 签名失败 | 源密钥问题 | 换官方/清华源；`sudo apt-key` 或改用 signed-by |
| `Temporary failure resolving` | DNS/网络 | 07 章查网络；换镜像 IP |
| `mysql: Access denied` | 用户密码错 | 重置密码；`sudo mysql` auth_socket 进 |
| `ERROR 1698: Access denied for user 'root'@'localhost'` | Ubuntu auth_socket | `sudo mysql` 后改认证方式 |
| `Job for mysql.service failed` | 配置错误/port 占用 | `journalctl -u mysql -n 50`；`ss -tlnp \| grep 3306` |
| `redis-cli: Could not connect` | 服务未起 | `sudo systemctl start redis-server` |
| `npm ERR! code EACCES` | 全局安装权限 | 用 nvm；勿 sudo npm |
| `curl: (6) Could not resolve host` | 下载脚本时 DNS 失败 | 检查网络；手动配 sources |
| `The following packages have unmet dependencies` | 依赖冲突 | `sudo apt -f install`；勿混用多个 MySQL 源 |
| `java: command not found` | JDK 未装或 PATH | 重装 openjdk-17-jdk；配置 JAVA_HOME |
| `Package 'mysql-server' has no installation candidate` | 源不全 | 检查 sources.list 含 universe |

## 17. 常见问题 FAQ

**Q1：`apt update` 和 `apt upgrade` 有什么区别？**  
update 只更新 `/var/lib/apt/lists/` 里的**包索引**；upgrade 根据新索引**升级已安装软件**。换源后必须先 update。

**Q2：为什么 `apt install xxx` 报 `Unable to locate package`？**  
包名拼错、未 `apt update`、sources.list 缺 **universe** 组件、或该包不在当前 Ubuntu 版本仓库。

**Q3：`E: Could not get lock /var/lib/dpkg/lock-frontend` 怎么办？**  
另一个 apt 进程在跑（含自动更新）；等完成，或 `ps aux | grep apt` 查占用；勿随意 kill，必要时 `sudo dpkg --configure -a`。

**Q4：Ubuntu 22.04 装 MySQL 后 root 无法 `mysql -u root -p`？**  
默认可能用 **auth_socket**；先用 `sudo mysql` 进，再改认证或创建 dev 用户（§4.3）。

**Q5：Redis 装完 `redis-cli ping` 不通？**  
`sudo systemctl start redis-server`；`systemctl status redis-server` 看是否 active。

**Q6：NodeSource 脚本 curl 失败？**  
07 章查 DNS/网络；可改用 nvm（§6.3）或离线 deb。

**Q7：`java: command not found` 但明明装过 JDK？**  
PATH 未含 `/usr/lib/jvm/.../bin`；配置 §3.3 的 `JAVA_HOME` 并 `source /etc/profile.d/jdk17.sh`。

**Q8：MySQL 3306 宿主机连不上 VM？**  
改 `bind-address=0.0.0.0`、创建 `dev@'%'`、07 章 ufw allow 3306（仅开发 VM）。

**Q9：`npm ERR! code EACCES` 全局安装失败？**  
不要用 `sudo npm`；用 nvm 或改 npm 全局 prefix 到用户目录。

**Q10：装坏环境如何快速恢复？**  
VMware **快照回滚**（装机前必打）；或 `apt purge` 后重装，比手工删文件干净。

**Q11：`The following packages have unmet dependencies`？**  
`sudo apt -f install` 修复；勿混用 MySQL 官方源与 Ubuntu 源两套。

**Q12：`~/study/linux-practice/check-env.sh` 某行失败怎么定位？**  
单独运行失败项（如 `redis-cli ping`）；对照 §11 速查表与 §12 报错表。

### 动手题（2 道）

### 综合题（2 道）

9. 按顺序口述：全新 VM 从零到 Spring Boot 能连 MySQL 的 6 步（含 apt、JDK、MySQL 用户、端口）。
10. `apt update` 成功但 `install openjdk-17-jdk` 仍找不到包——列 3 条排查。

### 自测参考答案

1. apt 高层管理依赖与下载；dpkg 底层装 deb；日常 **apt**。
2. remove 保留配置；purge 删配置。
3. main=官方支持；universe=社区维护（很多开发工具在此）。
4. 索引缓存仍是旧源地址，不 update 找不到新源里的包。
5. root 仅允许本地 socket 认证，不能密码登录；需 sudo mysql 或改 plugin。
6. 部分工具（Maven、IDE）找不到 java；`java -version` 可能仍可用若已在 PATH。
7. 示例：`mysql -u dev -p -e "CREATE DATABASE shop;"` 等（需先创建 dev 用户）。
8. `java -version`；`mysql --version`；`redis-cli ping`；`node -v`；`git --version`（或 ss 查 3306/6379）。
9. update/upgrade → 装 openjdk-17-jdk + JAVA_HOME → 装 mysql-server → 创建 dev 用户 → bind-address/ufw（如需远程）→ curl/health 或 mysql 登录验证。
10. 检查包名 `apt search`；sources 含 universe；是否 22.04 对应 jammy；是否拼写 openjdk-17-jdk。
