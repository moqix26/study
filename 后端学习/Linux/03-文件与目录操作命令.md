## 1. ls：列出目录内容

| 步骤 | 命令 | 预期 | 若不对 |
|------|------|------|--------|
| 1 | `cd ~/study/linux-practice` | 无报错 | 目录不存在 → `mkdir -p ~/study/linux-practice` |
| 2 | `ls` | 列出文件名 | 空目录正常 |
| 3 | `ls -l` | 权限、大小、时间列 | |
| 4 | `ls -la` | 含 `.` `..` 隐藏项 | |
| 5 | `ls -lh` | 大小带 K/M | |
| 6 | `ls -lt` | 新文件在上 | |

### 1.1 基本用法

```bash
cd ~/study/linux-practice
ls
```

**预期输出**：当前目录下的文件和文件夹名（可能为空或含前几章笔记）。

### 1.2 ls -l 长格式

```bash
ls -l
```

**预期输出示例**：

```text
total 12
drwxrwxr-x 2 study study 4096 Jun 23 14:00 demo
-rw-rw-r-- 1 study study  156 Jun 23 13:00 fhs-notes.md
-rw-rw-r-- 1 study study   45 Jun 23 12:00 day1.txt
```

| 字段 | 含义 |
|------|------|
| 第 1 列 | 类型与权限（`d` 目录，`-` 普通文件） |
| 第 2 列 | 硬链接数 |
| 第 3～4 列 | 所有者、组 |
| 第 5 列 | 字节大小 |
| 后三列 | 修改时间、文件名 |

### 1.3 常用组合

```bash
ls -la
```

**预期**：含 `.` `..` 和隐藏文件（以 `.` 开头）。

```bash
ls -lh
```

**预期**：大小列显示 `K`、`M`（human-readable）。

```bash
ls -lt
```

**预期**：按修改时间 **新→旧** 排序。

```bash
ls -R
```

**预期**：递归子目录（输出可能很长）。

```bash
ls -d */
```

**预期**：只列出当前层 **目录名**（尾斜杠表示目录 glob）。

## 2. cd、pushd、popd：切换目录

### 2.1 cd  essentials

```bash
cd ~
cd /var/log
cd -
cd ..
cd study/linux-practice
```

| 命令 | 作用 |
|------|------|
| `cd 路径` | 进入目录 |
| `cd` 或 `cd ~` | 回家目录 |
| `cd ..` | 上一级 |
| `cd -` | 回到上一次目录 |

```bash
cd /tmp
pwd
cd -
pwd
```

**预期**：第二次 `pwd` 回到 `/tmp` 之前的目录。

### 2.2 pushd / popd 目录栈

适合在多个目录间来回（编译、看日志、改配置）：

```bash
cd ~/study/linux-practice
pushd /var/log
pushd /etc
popd
pwd
popd
pwd
```

**预期输出流程**：

```text
/etc /var/log ~/study/linux-practice
/var/log ~/study/linux-practice
/home/study/study/linux-practice
```

`pushd` = `cd` + 把旧目录压栈；`popd` = 弹出并 `cd` 回去。

```mermaid
flowchart LR
  A[practice 目录] -->|pushd /var/log| B[/var/log]
  B -->|pushd /etc| C[/etc]
  C -->|popd| B
  B -->|popd| A
```

## 4. cp 与 mv

### 4.1 cp 复制文件

```bash
cd ~/study/linux-practice/project
echo "server.port=8080" > config/application.properties
cp config/application.properties config/application.properties.bak
ls -l config/
```

**预期**：两个文件大小相近。

### 4.2 cp -r 复制目录

```bash
cp -r config config-backup
tree -L 1
```

**预期**：`config` 与 `config-backup` 并列。

### 4.3 mv 移动或重命名

```bash
mv config/application.properties.bak config/app.properties.bak
ls config/
```

**预期**：文件名已变，无 `.bak` 在原名。

```bash
mv config-backup backup-config
ls -d */
```

**预期**：目录名变为 `backup-config`。

**注意**：`mv` 到已存在同名 **文件** 会覆盖（部分系统提示）；`mv` 到已存在 **目录** 则移入该目录下。

## 5. rm 与 rm -rf：删除与安全

### 5.1 删除文件与空目录

```bash
cd ~/study/linux-practice/project
touch logs/empty.log
rm logs/empty.log
rmdir logs 2>/dev/null || mkdir -p logs
```

`rmdir` 只能删 **空目录**。

### 5.2 rm -r 删目录树

```bash
rm -r backup-config
ls
```

**预期**：`backup-config` 消失。

### 5.3 rm -rf 危险警告（必读）

| 命令 | 后果 |
|------|------|
| `rm -rf /` | 摧毁系统（现代 Ubuntu 有保护仍 **绝对禁止**） |
| `rm -rf /*` | 同上 |
| `rm -rf ~/project` | 删掉整个项目——**无回收站** |
| `rm -rf ./*` 在错误目录 | 误删当前目录全部 |

**安全守则**：

1. 删前 **`pwd` + `ls`**
2. 慎用 `sudo rm -rf`
3. 重要代码 **`git commit`** 后再删（[Git 02](../../前端学习/Git/02-本地版本控制核心操作.md)）
4. 用 `trash-cli` 代替 rm（可选：`sudo apt install trash-cli`）

```bash
# 练习用：只删明确路径
rm -rf ~/study/linux-practice/tmp-lab
```

### 5.4 交互式删除

```bash
rm -ri config/app.properties.bak
```

**预期**：提示 `remove ...?` → 输入 `y` 才删。

## 6. find：强大查找

### 6.1 按名称

```bash
cd ~/study/linux-practice
find . -name "*.md"
```

**预期**：列出 `fhs-notes.md` 等。

```bash
find /etc -name "nginx.conf" 2>/dev/null
```

**预期**：有路径或空（未装 nginx 则空）。

### 6.2 按类型

```bash
find project -type d
find project -type f
```

**预期**：分别只有目录、只有文件。

### 6.3 按时间（查最近改动的日志）

```bash
find project -type f -mtime -1
```

**预期**：24 小时内修改过的文件。

### 6.4 组合与 exec（入门）

```bash
find project -name "*.properties" -exec cat {} \;
```

**预期**：打印找到的 properties 内容。

**后端场景**：找大于 100MB 的日志：

```bash
find /var/log -type f -size +100M 2>/dev/null
```

（可能需 `sudo`。）

## 7. locate：索引快查

`locate` 查预建数据库，比 `find /` 快，但 **不是实时**。

```bash
sudo apt install -y mlocate
sudo updatedb
locate passwd
```

**预期输出**（多行路径）：

```text
/etc/passwd
/usr/share/locale/...
...
```

```bash
locate nginx.conf | head
```

**注意**：新建文件后要 `sudo updatedb` 才能 locate 到。

| 对比 | find | locate |
|------|------|--------|
| 速度 | 实时遍历，慢 | 查库，快 |
| 实时性 | 实时 | 依赖 updatedb |
| 权限 | 受目录权限影响 | 同样受库中路径影响 |

## 8. tar、gzip、zip：打包与压缩

### 8.1 tar 归档（常配合 gzip）

```bash
cd ~/study/linux-practice
tar -cvf project-backup.tar project/
```

**预期**：生成 `project-backup.tar`，终端列出文件名。

```bash
tar -czvf project-backup.tar.gz project/
ls -lh project-backup.*
```

**预期**：

```text
-rw-rw-r-- 1 study study  xxx Jun 23 15:00 project-backup.tar
-rw-rw-r-- 1 study study  xxx Jun 23 15:00 project-backup.tar.gz
```

`.tar.gz` 通常更小。

### 8.2 解包

```bash
mkdir restore-test && cd restore-test
tar -xzvf ../project-backup.tar.gz
tree -L 2
```

**预期**：还原 `project/` 结构。

### 8.3 zip（与 Windows 互传方便）

```bash
sudo apt install -y zip unzip
cd ~/study/linux-practice
zip -r project.zip project/
unzip -l project.zip | head
```

**预期**：列出 zip 内文件列表。

**Java 09 场景**：上传 jar 或配置到服务器常先 `tar czvf app.tar.gz` 再 scp。

## 9. 通配符 Globbing

Shell 展开通配符 **before** 命令执行。

| 模式 | 含义 | 示例 |
|------|------|------|
| `*` | 任意长度字符 | `*.md` |
| `?` | 单字符 | `file?.txt` |
| `[abc]` | 选一 | `[Cc]ase` |
| `[0-9]` | 数字一位 | `log.[0-9]` |

```bash
cd ~/study/linux-practice
ls *.md 2>/dev/null
ls project/config/*.properties 2>/dev/null
echo project/src/main/{java,resources}
```

**预期 brace 展开**：

```text
project/src/main/java project/src/main/resources
```

**危险**：`rm *.log` 若 glob 无匹配，bash 可能保留字面 `*.log` 导致误删——删前 **`echo rm *.log`** 预览。

## 12. 深入解释：两个「为什么」

### 12.1 为什么 `rm -rf` 没有回收站却仍在生产环境使用？

Linux 服务器 **无图形回收站**；`rm` 直接 unlink 目录项，数据块稍后回收。生产删日志、清临时目录用 `rm` 是因为 **可脚本化、确定性高**。

**正确姿势**：

- 自动化脚本里 **写绝对路径变量**，如 `LOG_DIR=/var/log/myapp`
- 删前 `test -d "$LOG_DIR"` 或 `[ -d "$LOG_DIR" ] || exit 1`
- 人手工删：**Tab 补全** + 看提示符路径

**小案例**：有人本想在 `project/logs` 执行 `rm -rf *.log`，却在 `project` 下执行了 `rm -rf logs`——整个日志目录没了。Git 只跟踪 `.gitkeep` 时损失小；若是未备份的生产 log，无法恢复。

### 12.2 为什么后端发布常用 tar.gz 而不是 zip？

- Linux 服务器 **默认有 tar**；`tar czvf` 保留 **权限、符号链接**（zip 需额外处理）
- 管道友好：`tar czf - app/ | ssh user@host 'cd /opt && tar xzf -'`
- zip 适合 **与 Windows 同事互传**；上线 Linux 以 tar.gz 为主

[Java 09](../../后端学习/Java/09-LinuxDockerNginx部署基础.md) 传 jar 时常：

```bash
scp app.jar user@server:/opt/app/
# 或整包
tar czvf release.tar.gz app.jar config/ && scp release.tar.gz user@server:/opt/
```

## 13. 常见报错与排查

| 报错信息（关键词） | 可能原因 | 解决方案 |
|-------------------|---------|---------|
| `No such file or directory`（cd） | 路径不存在 | `mkdir -p` 或检查拼写 |
| `cp: cannot stat` | 源不存在 | `ls` 源路径 |
| `cp: -r not specified` | 复制目录缺 `-r` | `cp -r dir dest` |
| `mv: cannot overwrite directory` | 目标目录已有同名 | 换名或先删/合并 |
| `rm: cannot remove`: Permission denied | 无写权限 | `ls -l`；`sudo` 谨慎使用 |
| `find: paths must precede expression` | 选项顺序错 | `find . -name "*.md"` |
| `locate: command not found` | 未装 mlocate | `sudo apt install mlocate` |
| `tar: Error is not recoverable` | 包损坏或路径错 | 重打 tar；检查磁盘空间 `df -h` |
| `unzip: cannot find zipfile` | 文件非 zip 或下载不全 | 重新下载 |
| `bash: *.md: No such file or directory` | glob 无匹配 | 正常；勿对 rm 盲目依赖 glob |
| `fatal: not a git repository` | 未 git init | 在 project 根 `git init` |
| `File exists`（mkdir） | 无 `-p` 且父目录缺失 | `mkdir -p` |

## 19. 常见问题 FAQ

**Q：`cp` 目录为什么报错？**  
复制目录必须 **`cp -r`**；只 `cp dir dest` 会提示 `-r not specified`。

**Q：`rm -rf` 和 Windows 删除一样吗？**  
不一样。Linux **无回收站**（除非装 `trash-cli`）；删错只能靠 Git/备份。

**Q：`find .` 和 `locate` 选哪个？**  
练习目录用 `find`（实时）；全盘快查用 `locate`（需 `updatedb`）。

**Q：glob `*.md` 无匹配时 `rm *.md` 危险吗？**  
bash 可能保留字面 `*.md` 或行为因 `nullglob` 而异；删前 **`echo rm *.md`** 预览。

**Q：能在 `/mnt/hgfs/study` 里 `git init` 吗？**  
可以读 Markdown；**大量小文件/编译** 放 `~/study/linux-practice` 更快。

**Q：notehub-api 和真实 Spring Boot 目录一致吗？**  
骨架一致；真实项目还有 `pom.xml`、`target/`（已在 .gitignore）。

**Q：Windows 练过的命令在 VMware 一样吗？**  
一样；路径用 `/`，区分大小写，练习环境以 **Ubuntu VM** 为准。

## 20. 闭卷自测

### 概念题（6 道）

1. `ls -l` 第一列 `d` 和 `-` 分别表示什么？
2. `cd -` 做什么？`pushd`/`popd` 解决什么问题？
3. `mkdir -p a/b/c` 中 `-p` 的作用？
4. `cp` 与 `mv` 在 **inode** 层面有何不同（同文件系统内）？
5. 为何说 `rm -rf /` 绝对禁止？现代 Ubuntu 有何保护仍不能试？
6. `find . -name "*.yml"` 中 `.` 表示什么？

### 动手题（2 道）

7. 写一条命令：在 `notehub-api` 下递归查找所有 **目录**（仅目录）。
8. 把 `project/` 打成 `project-backup.tar.gz` 并解到 `restore-test/` 的两条核心命令？

### 综合题（2 道）

9. 在 `project/logs` 误执行 `rm -rf logs` 前，应用 §5 哪三条安全守则？
10. [todo.md](../../todo.md) 要求 Ubuntu 里 `git clone`——若 skeleton 在 VM 本地，如何与 Windows GitHub 协作（一句流程）？

### 自测参考答案

1. `d`=目录；`-`=普通文件。
2. `cd -` 回到上次目录；pushd/popd 在多个工作目录间切换像栈。
3. 父目录不存在则一并创建；已存在不报错。
4. `cp` 新建副本新 inode；`mv` 同 FS 内常仅改目录项（重命名/移动）。
5. 递归删根下一切；有保护也不应在 VM 试，Practice 目录误删同样惨。
6. 当前目录为搜索起点。
7. `find . -type d`（在 notehub-api 内）。
8. `tar -czvf project-backup.tar.gz project/`；`mkdir restore-test && cd restore-test && tar -xzvf ../project-backup.tar.gz`
9. 删前 `pwd`+`ls`；慎用 `sudo rm`；重要数据先 `git commit`。
10. Windows push GitHub；VM `git clone` 同一仓库，或共享文件夹只读同步（生产开发推荐 clone 到 `~/projects`）。
