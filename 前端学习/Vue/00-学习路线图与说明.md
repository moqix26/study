## 1. 这套资料适合谁

- 已学完或正在学 [HTML/CSS/JS](../HTML%20CSS%20JS/00-学习路线图与说明.md)（至少 06～09 章：DOM、事件、fetch、模块化）的同学
- 想从原生 JS 升级到 **Vue 3** 框架开发的学习者
- 计划与后端项目做**前后端联调**、投递全栈/前端岗位的同学（[Java / Spring Boot](../../后端学习/Java/00-学习路线图与说明.md) 或 [Python / FastAPI](../../后端学习/Python/00-学习路线图与说明.md)，**选一条后端主线**）
- 目标：能独立搭建 Vue 3 + Vite 项目、写组件、走路由、管状态、调接口、做一个能演示的**商城前台 MVP**

**不适合**：

- 已熟练 Vue 2 Options API 多年且做过大型 SPA 的资深开发者（可直接看 12、13、14 查漏补缺）
- 完全零基础、连 HTML 标签和 JS 变量都不熟（请先学 HTML CSS JS 01～05）

**前置要求（自检）**：

| 能力 | 对应 HTML CSS JS 章节 | 自检方式 |
|------|----------------------|----------|
| HTML 语义化结构 | 01～02 | 能写 header/nav/main/footer |
| Flex 布局 | CSS 03 | 能居中、两栏布局 |
| ES6+ 箭头函数、解构、模板字符串 | JS 06 | 能改写旧式 function |
| Promise / async-await | JS 08 | 能写 `await fetch(...)` |
| 模块化 import/export | JS 09 | 能拆 js 文件 |
| HTTP 基础、JSON | JS 08 + 后端 04 概念 | 知道 GET/POST、状态码 200/401 |
| TypeScript 基础（建议） | [TS 01～06](../TypeScript/00-学习路线图与说明.md) | 08 章联调前完成更佳 |

## 2. 技术栈主线（本资料默认路线）

```text
Vue 3 入门与环境（create-vue + Vite）
  → 模板语法与响应式（ref / reactive / v-for / v-if）
  → 计算属性、侦听器与表单绑定（computed / watch / v-model）
  → 组件与通信（props / emit / 插槽入门）
  → 组合式 API（script setup / composables）
  → Vue Router 单页路由（动态路由 / 导航守卫）
  → Pinia 状态管理（userStore / cartStore）
  → Axios 网络请求与前后端联调（拦截器 / token）
  → Element Plus UI 工程化（Form / Table / Layout）
  → Vite 构建与项目部署（env / Nginx / Docker）
  → 项目实战 + 进阶特性 + 面试专题
```

与后端的关系（全栈视角）：

| Vue（本资料） | 后端（对应） | 联调要点 |
|---------------|--------------|----------|
| Axios 调 `/api/users` | 04 Spring Boot `@GetMapping` | 统一 JSON 结构 |
| 登录存 token | JWT + 拦截器 | Header: `Authorization: Bearer xxx` |
| 商品列表分页 | 05 MyBatis + PageHelper | `pageNum` / `pageSize`  query |
| Pinia 存用户信息 | Redis Session（可选） | 前端持久化 localStorage |
| 路由守卫鉴权 | 后端 401 处理 | 双端都要拦未登录 |
| Vite proxy / Nginx 反代 | 09 部署 + CORS | 生产同域 `/api` |
| npm run build → dist | jar + docker-compose | Nginx 同时托管前后端 |

## 4. 必备环境与工具

### 4.1 Node.js

- 推荐 **Node.js 18 LTS** 或 **20 LTS**（与 Vite 5、Vue 3.4+ 兼容良好）
- 不推荐 Node 16 以下（部分依赖可能报错）

验证：

```bash
node -v
# 预期输出：v18.x.x 或 v20.x.x

npm -v
# 预期输出：9.x.x 或 10.x.x
```

若 `node` 命令不存在：从 [nodejs.org](https://nodejs.org/) 下载 LTS 安装包，安装时勾选 **Add to PATH**。

### 4.2 包管理器

| 工具 | 说明 | 本资料默认 |
|------|------|------------|
| npm | Node 自带，`create-vue` 默认 | ✅ 默认 |
| pnpm | 更快、省磁盘 | 可选 `npm i -g pnpm` |
| yarn | 老牌替代 | 可选 |

```bash
# 可选：全局安装 pnpm
npm install -g pnpm
pnpm -v
```

### 4.3 编辑器与插件

- **VS Code** 或 **Cursor**（本资料示例以 VS Code 为准）
- **必装插件**：

| 插件 | 作用 | 注意 |
|------|------|------|
| Vue - Official（原 Volar） | Vue 3 语法高亮、类型提示 | 禁用 Vetur（Vue 2 旧插件） |
| ESLint | 代码规范 | 可选 |
| Prettier | 格式化 | 可选 |
| Error Lens | 行内报错 | 可选 |

### 4.4 浏览器与调试

- Chrome / Edge 最新版
- 安装 **Vue DevTools** 扩展（Chrome 网上应用店搜索 Vue.js devtools）
- 打开 DevTools → **Vue** 面板，可查看组件树、Pinia 状态、路由

### 4.5 后端联调环境（08 章起需要）

| 组件 | 用途 | 启动方式 |

### 方案 C：待办 / 笔记 SPA（轻量）

- 可不依赖后端

**推荐目录结构（shop-vue）**：

```text
shop-vue/
├── public/
├── src/
│   ├── api/              # request.js, auth.js, product.js, order.js
│   ├── assets/
│   ├── components/       # AppHeader, ProductCard, EmptyState
```

## 9. 常见问题 FAQ

### Q1：要先学完前端 HTML/CSS/JS 再学 Vue 吗？

**建议**至少完成 HTML CSS JS 的 JS 06～09（ES6、异步、模块化）再开始 Vue 01。HTML/CSS 基础可在 Vue 项目中边做边补。

### Q2：Vue 2 还要学吗？

新项目和本资料均以 **Vue 3** 为准。若老项目维护需要 Vue 2，了解 Options API 与 Vue 3 差异即可（见 12、13 章对比表）。

### Q3：Vue 和 React 选哪个？

国内中小厂、外包、部分大厂业务线 **Vue 岗位仍很多**；React 生态同样大。本资料走 Vue 线；学通一个框架后迁移另一个不难。React 平行路线见 [React 学习资料](../React/00-学习路线图与说明.md)（shop-react）。

### Q4：Pinia 和 Vuex？

Vue 3 新项目 **只用 Pinia**（官方推荐）。面试可能问 Vuex 区别，见 13 章。

### Q5：跨域怎么办？

- **开发**：Vite `server.proxy` 把 `/api` 转到 8080
- **生产**：Nginx 同域反代 `/api`，或后端 CORS
- 详见 08 章、10 章

### Q6：build 后白屏？

常见原因：`base` 路径错误、资源 404、路由 history 模式未配 `try_files`。见 10 章排错表。

### Q7：只做前端够吗？

初级 **前端岗** 有 Vue 项目即可；**全栈/实习** 建议前后端都能 demo。shop-vue + 后端 demo 是最佳组合。

### Q8：TypeScript 要学吗？

create-vue 可选 TypeScript。初学可先用 **JavaScript**，熟练后再迁 TS；面试初级不强制 TS。

### Q9：和 UniApp / 小程序？

本资料是 **Web SPA**。小程序是另一条线；Vue 语法相通，组件 API 不同。

### Q10：Volar 和 Vetur 冲突？

Vue 3 项目 **只装 Vue - Official（Volar）**，禁用或卸载 Vetur，否则报错、无提示。

## 15. 常见环境报错（入门阶段）

| 报错 / 现象 | 原因 | 处理 |
|-------------|------|------|
| `'npm' 不是内部或外部命令` | Node 未装或未加 Path | 重装 Node LTS，勾选 PATH |
| `npm ERR! ERESOLVE` | 依赖版本冲突 | `npm install --legacy-peer-deps` 或换 Node 20 |
| 端口 5173 被占用 | 上次 dev 未关 | 换端口 `npm run dev -- --port 5174` 或杀进程 |
| Volar 报红但可运行 | tsconfig/jsconfig 缺失 | 加 jsconfig paths 或忽略 |
| 页面空白无报错 | main.js 未 mount | 检查 `#app` 与 `createApp` |
| 中文乱码 | 文件非 UTF-8 | VS Code 右下角改 UTF-8 |
| create-vue 卡住 | 网络 | 换 npm 镜像 `npm config set registry https://registry.npmmirror.com` |
