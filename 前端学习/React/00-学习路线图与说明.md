## 0. 读前导读（零基础也能跟上）

> **读者假设**：你已会 HTML/CSS/JS 基础（至少 DOM、事件、fetch），或正在学 [HTML CSS JS](../HTML%20CSS%20JS/00-学习路线图与说明.md)；本文件是 **React 全路线导航**，不教具体 API，但会告诉你「先学什么、后学什么、和 Vue/Java 怎么对齐」。

### 0.1 用一句话弄懂本文件夹

**一句话**：React 是用 **乐高式组件** 拼出网页界面的 JS 库；本资料从 `npm create vite` 搭 `shop-react` 商城前台，一路学到路由、Zustand 全局状态、Axios 联调 [Java 04 Spring Boot](../../后端学习/Java/04-SpringBoot核心开发.md)，最后能部署、能面试。

**生活类比——React 像「乐高组件盒」**：

| 概念 | 生活类比 | 代码里是什么 |
|------|----------|--------------|
| **React 本身** | 乐高官方标准接口（凸点尺寸统一） | 组件化 + 单向数据流规则 |
| **函数组件** | 一块标准积木（ProductCard） | `function ProductCard() { return JSX }` |
| **JSX** | 说明书上的「拼装示意图」（HTML+JS 混合） | `return <div>{name}</div>` |
| **Hook** | 给积木加功能插件（计数、联网） | `useState`、`useEffect` |
| **Zustand** | 全店共用的「全局抽屉」（登录态、购物车） | `useUserStore()` |
| **React Router** | 商场各楼层指示牌（URL 换页不换大楼） | `/products`、`/cart` |
| **Vite** | 装修队 + 电力（dev 服务器、打包） | `npm run dev` / `build` |

**为什么重要**：React 在全球与大厂业务线主流；与 [Vue 路线](../Vue/00-学习路线图与说明.md) 二选一或对照学，都能对接同一套 Spring Boot demo；**08 章联调窗口**与 Java 04 章 CRUD 接口对齐。

**本文件用到的地方**：§2 技术栈、§3 学习顺序、§10 与后端对齐、§22 Vue 映射表。

## 2. 技术栈主线（本资料默认路线）

```text
React 18 入门与环境（Vite + React 模板）
  → JSX 语法与函数组件（props / children / 条件与列表渲染）
  → State 与事件处理（useState / 受控表单 / 状态提升）
  → 组件通信与组件设计（单向数据流 / 组合 vs 继承）
  → Hooks 深入（useEffect / useRef / 自定义 Hook）
  → React Router 单页路由（动态路由 / 导航守卫）
  → Zustand 状态管理（userStore / cartStore）
  → Axios 网络请求与前后端联调（拦截器 / token）
  → Ant Design UI 工程化（Form / Table / Layout）
  → Vite 构建与项目部署（env / Nginx / Docker）
  → 项目实战 + 进阶特性 + 面试专题
```

与后端的关系（全栈视角）：

| React（本资料） | 后端（对应） | 联调要点 |
|-----------------|--------------|----------|
| Axios 调 `/api/users` | 04 Spring Boot `@GetMapping` | 统一 JSON 结构 |
| 登录存 token | JWT + 拦截器 | Header: `Authorization: Bearer xxx` |
| 商品列表分页 | 05 MyBatis + PageHelper | `pageNum` / `pageSize` query |
| Zustand 存用户信息 | Redis Session（可选） | 前端持久化 localStorage |
| 路由守卫鉴权 | 后端 401 处理 | 双端都要拦未登录 |
| Vite proxy / Nginx 反代 | 09 部署 + CORS | 生产同域 `/api` |
| npm run build → dist | jar + docker-compose | Nginx 同时托管前后端 |

与 Vue 路线的平行对照（便于横向迁移）：

| 能力 | Vue 资料 | React 资料 |
|------|----------|------------|
| 脚手架 | `npm create vue@latest` | `npm create vite@latest -- --template react` |
| 视图语法 | 模板 + 指令 | JSX |
| 响应式 | ref / reactive | useState / useReducer |
| 副作用 | watch / onMounted | useEffect |
| 路由 | Vue Router | React Router v6 |
| 全局状态 | Pinia | Zustand |
| UI 库 | Element Plus | Ant Design |
| 练手项目 | shop-vue | shop-react |

- 推荐 **Node.js 18 LTS** 或 **20 LTS**（与 Vite 5、React 18 兼容良好）
- 不推荐 Node 16 以下（部分依赖可能报错）

验证：

```bash
node -v
# 预期输出：v18.x.x 或 v20.x.x

npm -v
# 预期输出：9.x.x 或 10.x.x
```

若 `node` 命令不存在：从 [nodejs.org](https://nodejs.org/) 下载 LTS 安装包，安装时勾选 **Add to PATH**。

## 9. 常见问题 FAQ

### Q1：要先学完前端 HTML/CSS/JS 再学 React 吗？

**建议**至少完成 HTML CSS JS 的 JS 06～09（ES6、异步、模块化）再开始 React 01。HTML/CSS 基础可在 React 项目中边做边补。

### Q2：React 18 和 React 19 怎么选？

本资料以 **React 18** 为主线（生态最稳、资料最多）。React 19 新增 Actions、use 等 API，01 章会简要说明差异；新项目用 18 或 19 均可，Vite 模板默认较新版本。

### Q3：Vue 和 React 选哪个？

国内 **Vue 岗位仍很多**；React 在全球与大厂业务线同样主流。本资料走 React 线；若你已学 [Vue](../Vue/00-学习路线图与说明.md)，对照表可加速迁移。

### Q4：Redux 还要学吗？

中小型项目 **Zustand 足够**（本资料默认）。面试可能问 Redux，见 13 章对比；大型老项目可能仍用 Redux Toolkit。

### Q5：跨域怎么办？

- **开发**：Vite `server.proxy` 把 `/api` 转到 8080
- **生产**：Nginx 同域反代 `/api`，或后端 CORS
- 详见 08 章、10 章

### Q6：build 后白屏？

常见原因：`base` 路径错误、资源 404、路由 BrowserRouter 未配 `try_files`。见 10 章排错表。

### Q7：只做前端够吗？

初级 **前端岗** 有 React 项目即可；**全栈/实习** 建议前后端都能 demo。shop-react + 后端 demo 是最佳组合。

### Q8：TypeScript 要学吗？

Vite 可选 TypeScript 模板。初学可先用 **JavaScript**，熟练后再迁 TS；面试初级不强制 TS。

### Q9：和 Next.js？

本资料是 **Vite SPA**。Next.js 是 React 全栈框架（SSR/SSG），就业加分但概念更多；学完 11 章后可自学 Next。

### Q10：Class 组件还要学吗？

新项目 **只用函数组件 + Hooks**。面试可能问 Class 与 Hooks 区别，见 12、13 章。

### Q11：React 和「乐高」类比怎么理解？

**页面 = 底板**（App），**组件 = 标准积木块**（ProductCard、SearchBar），**props = 包装盒规格**（价格、库存），**state = 积木上可变的数字**（计数、搜索词）。拼法统一（函数组件 + JSX），所以大项目也能协作——见 §0.6、[04 章](04-组件通信与组合.md)。

### Q12：为什么要和 Vue 平行学？

不是必须，但 **同周做同一功能**（如商品列表 + 搜索）再对照写法，迁移效率远高于交替跳章。映射表见 §22、[Vue 00](../Vue/00-学习路线图与说明.md)。

### Q13：08 章一定要等 Java 04 吗？

**建议对齐**：Java 04 有 `@RestController` 和 JSON 返回后，React 08 的 Axios 才有真实目标。可先 Mock，但联调窗口应在后端 04～05 与前端 08 同一阶段（§10）。

### Q14：Zustand 为什么叫「全局抽屉」？

登录态、购物车多个页面都要用——像店里共用的抽屉，任意柜台（组件）打开都能拿；比 props 一层层传省事。对应 [07 章](07-Zustand状态管理.md)，Vue 侧见 [Pinia](../Vue/07-Pinia状态管理.md)。

### Q15：每天学多久合适？

每天 **1.5～2 小时**（§6）：30 分钟读 + 60 分钟敲 + 30 分钟练习 + 15 分钟笔记/commit。赶进度可压缩读，**不可跳过手敲 demo**。

### 9.2 自测参考答案

1. **库**（只管 UI）；路由 **React Router**（06）；状态 **Zustand**（07）。
2. 例：01 脚手架；02 商品列表；03 搜索/登录；06 多 URL；07 全局 user/cart store。
3. React 在 **JS 里写 JSX**（className、`{}`）；Vue 用 **template 指令**（v-if、v-for）。
4. **插件**；例：`useState`（计数）、`useEffect`（副作用/拉数据）。
5. **跨页面共享状态**（登录、购物车）；任意组件订阅，不必 props drilling。
6. 开发：**Vite proxy** 转 `/api`；生产：**Nginx 同域反代** 或后端 CORS。
7. 常见 **localStorage / Zustand persist**；Header **`Authorization: Bearer <token>`**（08 章拦截器）。
8. `npm create vite@latest shop-react -- --template react` → `cd shop-react` → `npm install` → `npm run dev`。
9. 开放题；例：列表 Vue `v-for` vs React `map+key`；全局 Pinia vs Zustand。
10. **06 Router**（/cart、/login?redirect=）+ **07 userStore**（isLoggedIn）+ **08 登录 API**（Java 04）。

## 16. 常见环境报错（入门阶段）

| 报错 / 现象 | 原因 | 处理 |
|-------------|------|------|
| `'npm' 不是内部或外部命令` | Node 未装或未加 Path | 重装 Node LTS，勾选 PATH |
| `npm ERR! ERESOLVE` | 依赖版本冲突 | `npm install --legacy-peer-deps` 或换 Node 20 |
| 端口 5173 被占用 | 上次 dev 未关 | 换端口 `npm run dev -- --port 5174` 或杀进程 |
| JSX 报红但可运行 | 缺 eslint 或文件扩展名 | 确保组件文件为 `.jsx` |
| 页面空白无报错 | main.jsx 未 render | 检查 `#root` 与 `createRoot` |
| 中文乱码 | 文件非 UTF-8 | VS Code 右下角改 UTF-8 |
| create-vite 卡住 | 网络 | 换 npm 镜像 `npm config set registry https://registry.npmmirror.com` |
| `React is not defined` | 旧 JSX 转换需显式 import | Vite 默认自动 JSX，检查 vite.config |

## 22. React 与 Vue 核心概念映射表

学完 React 01～03 后，可用下表与 [Vue 01～03](../Vue/00-学习路线图与说明.md) 对照复习：

| 概念 | Vue 3 | React 18 |
|------|-------|----------|
| 项目脚手架 | create-vue | create vite --template react |
| 根挂载 | `#app` + createApp | `#root` + createRoot |
| 视图层 | template 模板 | JSX return |
| 局部状态 | ref / reactive | useState / useReducer |
| 派生数据 | computed | useMemo |
| 副作用 | watch / onMounted | useEffect |
| 双向绑定 | v-model | 受控 value + onChange |
| 条件渲染 | v-if / v-show | && / 三元 / className |
| 列表 | v-for :key | map + key |
| 父子通信 | props + emit | props + 回调函数 |
| 插槽 | slot | children |
| 全局状态 | Pinia | Zustand |
| 路由 | Vue Router | React Router |
| UI 库 | Element Plus | Ant Design |
| 样式隔离 | scoped | CSS Modules / 命名 |

**迁移建议**：先在同一周完成「同一业务能力」（如商品列表 + 搜索），再对比写法差异，比交替跳章更高效。

## 24. 常见学习路径误区

| 误区 | 为什么有害 | 正确做法 |
