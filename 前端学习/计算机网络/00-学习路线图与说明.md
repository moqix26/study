# 计算机网络学习路线图

> 面向第一次系统学习计算机网络的 Go 后端学习者。
>
> 这套笔记不要求你先背 OSI 七层，也不要求一次读完。我们只围绕一件真实的事学习：**客户端怎样访问一个 Go API，失败时怎样知道卡在哪一步。**

---

## 1. 先看懂这一条主线

当你执行：

```powershell
curl.exe -v http://localhost:8080/health
```

看起来只是一条命令，背后实际发生了这些事：

```text
解析 URL
  ↓
确定主机 localhost、端口 8080、路径 /health
  ↓
把 localhost 解析为本机 IP
  ↓
与 8080 端口上的 Go 程序建立 TCP 连接
  ↓
发送 HTTP 请求
  ↓
Go Handler 处理请求
  ↓
返回 HTTP 状态码、响应头和 JSON
  ↓
关闭连接或留着复用
```

这就是本系列的骨架。以后遇到“接口访问不了”，不要只说“网络有问题”，而是判断：

1. 域名有没有解析成 IP？
2. IP 能不能到达？
3. 对方端口有没有程序监听？
4. TCP 是否建立成功？
5. HTTP 请求是否正确？
6. 服务端业务是否返回错误？

## 3. 学计网最容易犯的三个错误

### 错误一：一开始背 OSI 七层

七层模型有助于考试分类，但对第一次理解网络不友好。本系列先使用更贴近互联网实现的 TCP/IP 四层：

| 层 | 先理解成什么 | 常见内容 |
|---|---|---|
| 应用层 | 程序之间怎样组织消息 | HTTP、DNS |
| 传输层 | 怎样找到程序并传得可靠 | TCP、UDP、端口 |
| 网际层 | 怎样跨网络找到主机 | IP、路由 |
| 网络接口层 | 数据怎样经过当前这段网络 | Wi-Fi、以太网、MAC |

先能解释真实请求，再补七层名称，理解会容易很多。

### 错误二：术语看懂了，却没见过真实报文

`GET`、`200`、`Content-Type` 不能只在表格里背。使用：

```powershell
curl.exe -v https://httpbin.org/get
```

在输出中：

- `>` 开头的是客户端发出的 HTTP 请求；
- `<` 开头的是服务器返回的 HTTP 响应；
- `Connected to` 表示 TCP 已连接；
- `Could not resolve host` 表示 DNS 阶段失败；
- `Connection refused` 通常表示目标端口没有服务监听。

### 错误三：把不同层的问题混在一起

下面三种结果含义完全不同：

| 现象 | 已经成功的部分 | 失败位置 |
|---|---|---|
| `Could not resolve host` | 几乎还没开始连接 | DNS |
| `Connection refused` | 已得到 IP，并到达目标主机 | TCP 端口 |
| HTTP `404 Not Found` | DNS、TCP、HTTP 都已成功 | 路由或业务资源 |

看到 404 时，网络连接其实通常是正常的。

## 4. 每章采用同一种学习方法

### 第一步：只记本章的三个结论

每章开头都会给出“先记住”。第一次读时，不追求掌握全部细节。

### 第二步：顺着一次请求看过程

所有抽象概念都回到下面这个例子：

```text
客户端 curl → Go 服务 localhost:8080 → /health → JSON 响应
```

等你理解本地请求后，再换成：

```text
浏览器 → DNS → 公网服务器 443 → HTTPS → Go API
```

### 第三步：亲手执行命令

Windows PowerShell 下优先使用：

```powershell
curl.exe -v http://localhost:8080/health
Test-NetConnection localhost -Port 8080
Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue
nslookup www.baidu.com
ipconfig
```

这里写 `curl.exe`，是为了避免旧版 PowerShell 把 `curl` 解释成其他命令。

### 第四步：用自己的话复述

不要逐字背答案。尝试回答：

> 当我访问 `http://localhost:8080/health` 时，从输入命令到得到 JSON，发生了什么？

说不清的那一步，就是下一次要回看的位置。

## 5. 需要掌握到什么程度

### 当前学习阶段必须会

- 区分 IP、端口、域名、URL；
- 知道 HTTP 在 TCP 连接上发送请求和响应；
- 看懂常见方法、状态码、请求头和 JSON body；
- 用 `curl.exe -v` 测试 Go API；
- 区分 DNS 失败、端口失败和 HTTP 业务错误；
- 知道生产环境为什么使用 HTTPS；
- 理解 Cookie、Session、JWT 各自保存什么；
- 知道 CORS 是浏览器规则，不是服务器之间通信规则。

### 项目阶段再深入

- TCP 重传、滑动窗口、拥塞控制；
- DNS 缓存与负载均衡；
- HTTP/2、HTTP/3；
- TLS 证书链和握手细节；
- Cache-Control、ETag、CDN；
- 反向代理、超时、连接池和可信代理。

### 暂时不必钻研

- 手算复杂子网题；
- 每一个 TCP 状态转换；
- 加密算法数学证明；
- HTTP/2 帧的每个二进制字段；
- 路由协议内部算法。

这些并不是没用，而是现在投入产出比低。

## 6. 准备一个最小 Go 实验服务

后续命令可以配合这个服务练习。代码先读懂，再自己敲到练习目录：

```go
package main

import (
	"encoding/json"
	"log"
	"net/http"
)

func main() {
	http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Content-Type", "application/json")
		json.NewEncoder(w).Encode(map[string]string{"status": "ok"})
	})

	log.Println("listening on http://localhost:8080")
	log.Fatal(http.ListenAndServe(":8080", nil))
}
```

启动后运行：

```powershell
curl.exe -v http://localhost:8080/health
```

你应该看到类似：

```text
> GET /health HTTP/1.1
> Host: localhost:8080

< HTTP/1.1 200 OK
< Content-Type: application/json

{"status":"ok"}
```

如果能把这几行分别解释清楚，你已经迈过了计网最重要的一道门槛。
