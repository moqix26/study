## 2. 如何实现一个登录页

### 思路清单

| 步骤 | 内容 |
|------|------|
| HTML | `form` + `label` + `input[type=text/password]` + `button[type=submit]` |
| CSS | 居中卡片、输入框统一样式、错误提示红色 |
| JS | 阻止默认提交 → 校验非空 → `fetch` POST → 存 token → 跳转 |
| 状态 | 提交中禁用按钮、错误提示、成功跳转 |

### 面试怎么说

「登录页核心是表单语义 + 前端校验 + 异步请求 + 登录态存储。我会用 `preventDefault` 拦截默认提交，校验通过后再发请求，根据状态码处理 401 等错误，成功后将 token 存 localStorage 并跳转首页。」

## 3. 如何实现搜索建议框

1. 监听 `input` 事件
2. **防抖** 300~500ms，减少请求次数
3. `fetch` 带关键词参数
4. 渲染下拉列表；无结果显示「无匹配」
5. 点击项填入输入框；点击外部关闭下拉

## 5. 如何实现轮播图

- 数据：图片 URL 数组 + `currentIndex`
- 切换：`index = (index + 1) % length`
- 自动播放：`setInterval`，`mouseenter` 时 `clearInterval`
- 指示点：根据 index 切换 active class
- CSS：`transform: translateX(-${index * 100}%)` 或切换显示隐藏

2. CSS 变量定义两套颜色，或 `.dark-mode .card { background: #222; }`

## 8. 排错专题：样式不生效

按顺序查：

1. CSS 文件路径、`link` 是否 404（Network 面板）
2. 选择器是否命中（Elements → Styles）
3. 是否被覆盖（删除线）
4. 优先级 / `!important`
5. 拼写：`background-color` vs `backgroundColor`
6. 强缓存：Ctrl+F5

---

## 9. 排错专题：JS 获取不到元素

1. 选择器拼写、`#` / `.` 是否用对
2. `<script>` 是否在 DOM 之前执行 → 用 `DOMContentLoaded` 或 script 放底部
3. 元素是否 JS 执行后才动态创建 → 用事件委托或回调里再获取

## 10. 排错专题：接口失败

| 现象 | 可能原因 |
|------|----------|
| Network 红色 | URL 错、服务未启动、跨域 |
| 404 | 路径或方法不对 |

## 11. 面试高频简答题要点

### HTML

- **语义化**：用合适标签表达含义，利于 SEO、无障碍、可维护
- **`src` vs `href`**：`src` 嵌入资源并替换元素内容；`href` 建立关联/跳转
- **块级 vs 行内**：布局习惯不同；行内默认不能设宽高

### CSS

- **盒模型**：content + padding + border + margin；`border-box` 让 width 含 padding/border
- **BFC**：独立布局区域，可清除浮动、阻止 margin 合并
- **Flex vs Grid**：Flex 一维；Grid 二维
- **优先级**：行内 > id > class > 元素；同权后写覆盖先写

### JavaScript

- **数据类型**：7 种基本 + object；`typeof null === 'object'` 历史遗留
- **`==` vs `===`**：前者隐式转换，推荐后者
- **闭包**：函数能访问定义时的词法环境；常用于私有变量、防抖
- **事件冒泡/捕获**：先捕获后冒泡；`stopPropagation` 阻止传播
- **Promise**：pending/fulfilled/rejected；链式 `then/catch`
- **跨域**：浏览器同源策略；JSONP/CORS/代理

## 15. 手写代码题合集

### 15.1 数组去重

```js
// 方法 1：Set（最简洁）
const unique = arr => [...new Set(arr)];

// 方法 2：filter + indexOf
const unique2 = arr => arr.filter((item, index) => arr.indexOf(item) === index);

// 方法 3：reduce
const unique3 = arr => arr.reduce((acc, item) =>
  acc.includes(item) ? acc : [...acc, item], []);
```

### 15.2 深拷贝

```js
// 简易版（JSON 法，无法拷贝函数、undefined、Date、循环引用）
function deepCloneSimple(obj) {
  return JSON.parse(JSON.stringify(obj));
}

// 完整版（递归，处理数组、对象、Date、循环引用）
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== "object") return obj;
  if (obj instanceof Date) return new Date(obj);
  if (map.has(obj)) return map.get(obj); // 循环引用

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);

  for (const key of Object.keys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```

### 15.3 防抖（立即执行选项）

```js
function debounce(fn, delay = 300, immediate = false) {
  let timer = null;
  return function (...args) {
    const callNow = immediate && !timer;
    clearTimeout(timer);
    timer = setTimeout(() => {
      timer = null;
      if (!immediate) fn.apply(this, args);
    }, delay);
    if (callNow) fn.apply(this, args);
  };
}
```

### 15.4 节流

```js
function throttle(fn, interval = 200) {
  let last = 0;
  return function (...args) {
    const now = Date.now();
    if (now - last >= interval) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```

### 15.5 数组扁平化

```js
// 方法 1：flat
const flatten = arr => arr.flat(Infinity);

// 方法 2：reduce 递归
function flatten2(arr) {
  return arr.reduce((acc, item) =>
    acc.concat(Array.isArray(item) ? flatten2(item) : item), []);
}
```

### 15.6 手动实现 Promise.all

```js
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const results = [];
    let count = 0;
    promises.forEach((p, i) => {
      Promise.resolve(p).then(val => {
        results[i] = val;
        count++;
        if (count === promises.length) resolve(results);
      }).catch(reject);
    });
  });
}
```

### 15.7 函数柯里化

```js
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return (...args2) => curried.apply(this, [...args, ...args2]);
  };
}

// 示例
const add = (a, b, c) => a + b + c;
const curriedAdd = curry(add);
curriedAdd(1)(2)(3); // 6
curriedAdd(1, 2)(3); // 6
```

### 15.8 实现 `instanceof`

```js
function myInstanceof(obj, constructor) {
  let proto = Object.getPrototypeOf(obj);
  while (proto) {
    if (proto === constructor.prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
  return false;
}
```

## 16. CSS 布局题

### 16.1 居中的 N 种方法

```css
/* 1. Flex（最推荐）*/
.parent { display: flex; justify-content: center; align-items: center; }

/* 2. Grid */
.parent { display: grid; place-items: center; }

/* 3. 绝对定位 + transform（未知宽高） */
.child {
  position: absolute; top: 50%; left: 50%;
  transform: translate(-50%, -50%);
}

/* 4. 绝对定位 + margin（已知宽高） */
.child {
  position: absolute; top: 0; left: 0; right: 0; bottom: 0;
  margin: auto; width: 200px; height: 100px;
}

/* 5. text-align + line-height（单行文本） */
.parent { text-align: center; line-height: 200px; }
```

### 16.2 等高列

```css
/* Flex 天然等高 */
.columns { display: flex; } /* align-items: stretch 默认 */

/* Grid 也天然等高 */
.columns { display: grid; grid-template-columns: repeat(3, 1fr); }
```

### 16.3 三栏布局（左右固定，中间自适应）

```css
/* Flex */
.layout { display: flex; }
.left { width: 200px; }
.main { flex: 1; min-width: 0; }
.right { width: 200px; }

/* Grid */
.layout { display: grid; grid-template-columns: 200px 1fr 200px; }
```

## 17. 综合场景题

### 17.1 如何设计一个按钮组件？

面试说："从 props（样式变体、尺寸、状态）、插槽（内容）、事件绑定三个维度考虑：
- 样式变体：primary / secondary / ghost / danger，通过 class 或 CSS 变量控制
- 尺寸：small / medium / large，控制 padding 和字号
- 状态：hover / active / disabled / loading，不同视觉 + 行为
- 图标支持：可前置/后置图标
- 可访问性：button 原生标签（不用 div），focus 样式可见"

### 17.2 前端如何做权限控制？

"前端权限分三层：
1. **路由权限**：根据角色控制哪些页面可访问（路由守卫）
2. **组件/按钮权限**：根据权限点控制按钮是否显示（v-if / 条件渲染）
3. **请求拦截**：后端也会校验，前端 401/403 后跳转登录或提示

权限数据通常登录后返回，存 store 中，也可以存字典快速查找。"

### 17.3 列表页大数据量渲染优化？

"三种方案：
1. **分页**：最简单，前端请求时带 page/pageSize
2. **虚拟滚动**：只渲染可视区的 DOM 节点（如 react-window、vue-virtual-scroller）
3. **无限滚动**：自动加载下一页（Intersection Observer + 拼接数据）

还要注意：列表项不用 index 当 key；图片懒加载；复杂计算用 useMemo/computed。"

## 18. 面试简答题完整参考答案

### HTML 相关
- **语义化的价值**：利于 SEO（搜索引擎理解页面结构）、无障碍（屏幕阅读器）、代码可维护性（`nav` 比 `<div class="nav">` 直观）
- **src vs href**：`src` 用于嵌入资源（img/script/iframe），会暂停 HTML 解析去加载；`href` 建立文档关联（a/link），不阻塞解析

### CSS 相关
- **盒模型**：每个元素是可嵌套的四层盒子（content + padding + border + margin）。`box-sizing: border-box` 让 width 包含 padding 和 border，布局计算更直观
- **BFC**：Block Formatting Context，独立渲染区域。能解决：高度塌陷（包浮动子元素）、margin 合并、浮动覆盖。触发方式：`display: flow-root`（推荐）、`overflow: hidden`、`float`、`position: absolute`
- **Flex vs Grid**：Flex 是一维布局（沿主轴排列），适合导航栏、卡片行、表单字段；Grid 是二维布局（同时控制行和列），适合页面骨架、图片墙、仪表盘
- **rem vs em vs px**：rem 相对根元素（`html`）字号，适合全局间距和字号（响应式只需改根字号）；em 相对自身或父元素字号，嵌套容易失控；px 固定像素，适合 1px 边框

### JavaScript 相关问题
- **闭包**：函数能记住并访问其定义时的作用域，即使函数在其他地方执行。应用：防抖节流、回调、模块私有变量、React 的 useState
- **Event Loop**：JS 单线程的异步机制。同步代码先执行，遇到异步任务交给 Web API，回调按类型进微任务队列（Promise.then）或宏任务队列（setTimeout）。事件循环每次：清空微任务 → 取一个宏任务 → 清空微任务 → 可能渲染
- **Promise 状态**：pending（等待）→ fulfilled（成功）或 rejected（失败），状态一旦改变就不可逆
- **箭头函数的 this**：箭头函数没有自己的 this，沿用定义时外层作用域的 this。不能用作构造函数，没有 arguments 对象

### Web 相关
- **跨域原因与解决**：浏览器同源策略限制（协议+域名+端口三者相同）。解决：后端设 CORS 头（`Access-Control-Allow-Origin`）、开发环境代理转发、JSONP（仅 GET 且过时）
- **HTTP 状态码分类**：2xx 成功（200 OK, 201 Created）、3xx 重定向（301 永久, 302 临时, 304 缓存）、4xx 客户端错（400 参数, 401 未认证, 403 禁止, 404 不存在）、5xx 服务端错（500 内部, 502 网关, 503 不可用）
- **Cookie vs Token**：Cookie 浏览器自动带到同源请求，可设 HttpOnly 防 XSS 读取；Token（JWT）存 localStorage 或内存，手动放 Authorization 头，前后端分离常用
