# Codex 与 Cursor 共享对话协议

> 位置：`F:\study\.memory`  
> 编码：UTF-8  
> 作用：让 Codex 与 Cursor 通过简短、可追溯的对话摘要接力，避免依赖某一端的聊天上下文。

## 1. 基本模式

Codex 和 Cursor 不假设能直接读取对方的聊天记录，而是通过本目录异步交接：

1. 开始处理学习任务前，先读本文件。
2. 再按文件名倒序读取最新的对话摘要；涉及学习进度时同时读取 `F:\study\learn.md`。
3. 每次回复用户前，在完成实际工作后生成一份新的对话摘要；即使只是确认或纠正，也简要记录。
4. 一份摘要对应一次用户—助手交互，不覆盖旧摘要，不把多次交互伪装成一次。
5. Codex 与 Cursor 都使用同一格式，并在摘要中明确标记记录者。

## 2. 文件命名

```text
YYYY-MM-DD_HH-mm-ss_记录者.md
```

记录者使用：

- `codex`
- `cursor`

如果同一秒出现重名，在末尾增加 `_02`、`_03`。

示例：

```text
2026-07-14_15-38-04_codex.md
```

## 3. 每份摘要必须包含

```markdown
# 对话摘要

- 时间：
- 记录者：Codex / Cursor
- 用户本轮要求：

## 已完成

## 关键结论与用户偏好

## 文件或环境变化

## 当前状态

## 下一步

## 未解决事项
```

没有内容的栏目写“无”，不要删除栏目。

## 4. 写作要求

- 只写接手所需的信息，不复制整段聊天，不堆砌代码和终端输出。
- 明确区分：用户说过的事实、助手的判断、已经实际完成的操作。
- 文件变化必须写绝对路径，并注明“已修改 / 已创建 / 已删除 / 仅建议”。
- 学习进度以 `F:\study\learn.md` 为当前状态板；摘要记录本轮变化，不替代状态板。
- 不记录密码、令牌、Cookie、私钥及其他敏感凭据。
- 使用简体中文，时间使用 Asia/Shanghai（UTC+8）。

## 5. 冲突处理

信息冲突时按以下优先级判断：

1. 用户最新的明确口述
2. 工作区文件和程序的实际状态
3. 时间更新的对话摘要
4. 更早的对话摘要

发现旧摘要有误时，不静默篡改旧文件；新建一份摘要说明纠正内容。

## 6. 操作边界

- 摘要只用于交接，不自动扩大用户授权范围。
- 删除文件、提交 Git、推送远端或修改用户练习代码，仍需以用户当前明确要求为准。
- 接手者应继续当前任务，避免因为换助手就重新询问已经记录清楚的信息。

## 7. 快速接手流程

```text
读取 readme.md
  → 按名称倒序读取最新摘要
  → 涉及学习时读取 learn.md
  → 核对工作区实际状态
  → 从“下一步 / 未解决事项”继续
  → 回复前写入本轮新摘要
```
